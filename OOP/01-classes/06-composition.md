# 06 · Композиция и агрегация 🖼️⭐

> 🎯 **Цель блока:** понять, как объекты **составляются** из других объектов — композиция и
> агрегация — главный способ строить сложные системы из простых частей.

---

## 📖 Объекты состоят из объектов

Реальные сущности составные: машина состоит из двигателя и колёс, заказ — из товаров. В ООП это
выражают, **вкладывая** объекты в объекты.

🖼️
```
   ┌─ Заказ ──────────────────┐
   │ клиент: Клиент            │ ← объект внутри объекта
   │ товары: [Товар, Товар]    │ ← список объектов
   │ итого()                   │
   └───────────────────────────┘
```

```python
class Заказ:
    def __init__(self, клиент):
        self.клиент = клиент          # Заказ ИМЕЕТ клиента
        self.товары = []              # Заказ ИМЕЕТ товары
    def добавить(self, товар):
        self.товары.append(товар)
    def итого(self):
        return sum(т.цена for т in self.товары)
```

💡 Это отношение «**имеет** / has-a»: Заказ **имеет** товары. Строить большое из малого через
вложение — основной, самый гибкий приём ООП (важнее наследования, см. модуль 16).

## ⭐ Композиция vs агрегация

Оба — «объект содержит другие», но различаются по **времени жизни**:

```
   КОМПОЗИЦИЯ — часть НЕ живёт без целого (сильная связь, «владение»)
        Дом и Комнаты: снёс дом — комнаты исчезли
        Заказ и его Позиции: удалил заказ — позиции не нужны

   АГРЕГАЦИЯ — часть живёт САМА по себе (слабая связь, «использование»)
        Команда и Игроки: распустили команду — игроки остались (перейдут в другую)
        Плейлист и Песни: удалил плейлист — песни на месте
```

🖼️
```
   композиция:  Целое ◆──── Часть   (часть умирает с целым)
   агрегация:   Целое ◇──── Часть   (часть переживает целое)
```

💡 Различие важно для понимания **владения** и времени жизни (перекликается с владением памятью
в [Rust](../../Rust/02-memory/08-ownership.md) и [C++](../../Cpp/02-memory/11-smart-pointers.md)).
На практике: композиция — когда «часть принадлежит только мне», агрегация — когда «я лишь
ссылаюсь на общий объект».

---

## ⭐ «Имеет» vs «является» (has-a vs is-a)

Главный вопрос проектирования отношений:

```
   has-a (имеет) → КОМПОЗИЦИЯ:  Машина ИМЕЕТ Двигатель
   is-a (является) → НАСЛЕДОВАНИЕ: Спорткар ЯВЛЯЕТСЯ Машиной (модуль 10)
```

💡 ⭐ Запомни этот тест. Путаница «имеет/является» — источник плохих иерархий наследования.
Если тянет написать наследование — спроси: «X **является** Y или X **имеет** Y?». Чаще ответ —
«имеет», и тогда нужна **композиция**, а не наследование (модуль 16 — почему это важно).
