# 03 · Конструкторы и состояние 🖼️⭐

> 🎯 **Цель блока:** понять, как объект «рождается» с правильным начальным состоянием — через
> конструктор — и почему это важно для надёжности.

---

## 📖 Конструктор — рождение объекта в корректном состоянии

**Конструктор** — особый метод, который вызывается при создании объекта и **настраивает** его
начальное состояние.

```python
class Счёт:
    def __init__(self, владелец, баланс=0):  # конструктор
        self.владелец = владелец
        self.баланс = баланс
```

```python
a = Счёт("Аня", 100)   # объект рождается СРАЗУ корректным: есть владелец и баланс
```

💡 ⭐ Главная идея: объект должен быть **валидным с момента создания**. Не «создал пустой, потом
не забудь заполнить» — а «создал → уже готов к работе». Это убирает целый класс багов («забыл
проинициализировать»).

---

## ⭐ Инварианты: правила, которые всегда верны

**Инвариант** — условие, которое должно выполняться **всё время жизни** объекта. Конструктор —
первое место, где его устанавливают и проверяют:

```python
class Счёт:
    def __init__(self, владелец, баланс=0):
        if баланс < 0:
            raise ValueError("баланс не может быть отрицательным")  # защита инварианта
        self.владелец = владелец
        self.баланс = баланс
```

💡 Инвариант счёта: «баланс ≥ 0». Конструктор не даёт создать «сломанный» объект. Дальше методы
(пополнить/снять) должны **поддерживать** инвариант. Объект, который сам следит за своей
корректностью, — основа надёжного дизайна (связь с инкапсуляцией, модуль 05).

---

## 📖 Имена конструкторов в разных языках

```
   Python   — __init__(self, ...)
   Java/C#  — имя класса:  public Счёт(...) { ... }
   C++      — имя класса:  Счёт(...) { ... }   (+ деструктор ~Счёт())
   JS       — constructor(...) { ... }
```

💡 Синтаксис разный, идея одна: метод, который запускается при рождении объекта и приводит его в
корректное состояние. В C++ есть и **деструктор** (вызывается при уничтожении — вспомни
[RAII](../../Cpp/02-memory/10-raii.md)).

---

## ⭐ Несколько способов создать объект

Иногда объект можно собрать по-разному. Подходы:

```
   параметры по умолчанию  — Счёт(владелец, баланс=0)
   перегрузка конструкторов (Java/C++) — несколько конструкторов с разными параметрами
   фабричные методы        — Счёт.из_строки("Аня:100")  (именованные «конструкторы»)
```

💡 Если способов создания много или они сложные — лучше **фабричный метод** или **Builder**
(модуль 19), чем гигантский конструктор с десятком параметров. Но это уже про дизайн —
сейчас важно: конструктор отвечает за корректное начальное состояние.

## ⚠️ Ловушки

- ❌ Создавать «пустой» объект и заполнять потом — легко забыть и получить невалидное состояние.
- ❌ Не проверять аргументы в конструкторе — объект родится «сломанным».
- ❌ Пихать в конструктор тяжёлую работу (запросы к сети/БД) — он должен **настраивать**, а не
  «делать всё».
- ❌ Конструктор с 10 параметрами — признак, что классу нужен Builder или разбиение.
