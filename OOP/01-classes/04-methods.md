# 04 · Методы и поведение 🖼️

> 🎯 **Цель блока:** понять методы как поведение объекта и идею «обмена сообщениями» — объекты
> не лезут в чужие данные, а **просят** друг друга через методы.

---

## 📖 Метод — действие объекта над своим состоянием

**Метод** — функция, живущая внутри класса и работающая с состоянием объекта (через `self`/`this`).

```python
class Счёт:
    def __init__(self, баланс=0):
        self.баланс = баланс
    def пополнить(self, сумма):     # метод меняет состояние
        self.баланс += сумма
    def снять(self, сумма):
        if сумма > self.баланс:
            raise ValueError("недостаточно средств")
        self.баланс -= сумма
    def показать(self):              # метод читает состояние
        return self.баланс
```

💡 Методы делятся на: **меняющие** состояние (пополнить/снять) и **читающие** (показать). Хорошая
практика — чётко различать их (команды vs запросы): запрос не должен незаметно менять объект.

## ⭐ Обмен сообщениями: объекты просят, а не лезут

Ключевая идея ООП: объект **не трогает** чужие поля напрямую — он **просит** другой объект
через его метод.

🖼️
```
   ❌ плохо:  заказ.клиент.баланс -= цена      (залезли внутрь клиента)
   ✅ хорошо: заказ.клиент.снять(цена)          (попросили клиента, он сам решит)
```

💡 ⭐ Это называют «**обменом сообщениями**»: ты посылаешь объекту сообщение (вызов метода), а
**он сам** решает, как его обработать (и может отказать — «недостаточно средств»). Объект —
хозяин своих данных. Это основа инкапсуляции (модуль 05) и слабой связанности (модуль 17).

---

## 📖 Метод знает «свой» объект

```python
a = Счёт(100)
b = Счёт(500)
a.пополнить(50)   # self = a → меняется баланс a (стал 150)
b.пополнить(50)   # self = b → меняется баланс b (стал 550)
```

💡 Один и тот же метод `пополнить` работает с **разными** объектами — через `self`/`this`. Код
метода один (в классе), но действует на состояние того объекта, у которого вызван.

## ⭐ Команды и запросы (CQS)

Полезный принцип — **разделение команд и запросов** (Command-Query Separation):

```
   КОМАНДА — что-то меняет, ничего осмысленного не возвращает (пополнить, снять, удалить)
   ЗАПРОС  — что-то возвращает, НИЧЕГО не меняет (баланс, размер, найти)

   правило: «задавая вопрос, не меняй ответ» — запрос не должен иметь побочных эффектов
```

💡 Когда запрос тихо меняет состояние — это источник коварных багов («прочитал значение, а оно
изменилось»). Чёткое разделение делает код предсказуемым. Это перекликается с «чистыми
функциями» функционального стиля.

## ⚠️ Ловушки

- ❌ Лезть в чужие поля напрямую вместо вызова методов — ломает инкапсуляцию.
- ❌ Запрос с побочными эффектами (`получить_следующий()` тихо двигает указатель и меняет состояние).
- ❌ «Анемичные» объекты — только поля, всё поведение снаружи (это процедурный код в маске ООП).
- ❌ Метод-«бог», делающий десять дел сразу. Метод — одно понятное действие.
