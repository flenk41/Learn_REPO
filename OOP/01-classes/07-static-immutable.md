# 07 · Статические члены и неизменяемость 🖼️

> 🎯 **Цель блока:** понять разницу между «принадлежит объекту» и «принадлежит классу»
> (статические члены), и зачем нужны неизменяемые объекты.

---

## ⭐ Член объекта vs член класса (статический)

```
   ЧЛЕН ОБЪЕКТА (экземпляра) — у каждого объекта СВОЙ:  баланс каждого счёта свой
   ЧЛЕН КЛАССА (статический) — ОБЩИЙ для всех объектов класса
```

🖼️
```
   class Счёт:
       ставка = 0.05        ← СТАТИЧЕСКОЕ поле: одно на все счета
       def __init__(self, b):
           self.баланс = b  ← поле ОБЪЕКТА: у каждого своё

   a = Счёт(100); b = Счёт(500)
   a.баланс ≠ b.баланс        (своё у каждого)
   Счёт.ставка — одно на всех (общее)
```

💡 Статические члены принадлежат **классу**, а не объекту: общие константы (ставка, лимит),
счётчики (сколько объектов создано), вспомогательные функции, не зависящие от конкретного объекта
(`Math.sqrt`).

```python
class Счёт:
    ставка = 0.05            # статическое поле
    всего_счетов = 0
    def __init__(self, b):
        self.баланс = b
        Счёт.всего_счетов += 1   # общий счётчик
```

## 📖 Статические методы

```
   обычный метод   — работает с состоянием объекта (нужен self/this)
   статический метод — НЕ нужен объект; утилита, связанная с классом по смыслу
```

```python
class Деньги:
    @staticmethod
    def из_копеек(копейки):     # фабрика, не нужен объект
        return Деньги(копейки / 100)
```

💡 Статический метод — когда логика **относится к классу**, но не к конкретному объекту. Часто
это фабричные методы (модуль 03) или утилиты. ⚠️ Если статических методов много и они держат
общее изменяемое состояние — это маскировка процедурного кода (или скрытый Singleton, модуль 19).

## ⭐ Неизменяемые объекты (immutable)

**Неизменяемый объект** — после создания его состояние **не меняется**. Хочешь другое значение —
создаёшь новый объект.

```python
# вместо изменения — создаём новый
class Точка:
    def __init__(self, x, y):
        self._x, self._y = x, y
    def сдвинуть(self, dx, dy):
        return Точка(self._x + dx, self._y + dy)   # НОВЫЙ объект, старый цел
```

💡 ⭐ Зачем неизменяемость:

```
   ✅ безопасность в потоках — нечего «портить» одновременно (нет гонок, см. трек ОС)
   ✅ предсказуемость — объект не изменится «за спиной»
   ✅ можно безопасно делиться и кэшировать (никто не испортит)
   ✅ проще рассуждать о коде
```

Многие ценности так и делают неизменяемыми: деньги, даты, координаты, строки (в большинстве
языков). Связь с [mutable/immutable в Python](../../Python/02-memory/10-mutable-immutable.md) и
неизменяемостью в функциональном стиле.

## ⚠️ Ловушки

- ❌ Делать изменяемым то, что логически должно быть значением (деньги, дата) — источник багов.
- ❌ Класс целиком из статических методов с общим состоянием — это не ООП, а глобальные переменные.
- ❌ Путать `Счёт.ставка` (общее) и `self.баланс` (своё у каждого).
- ❌ «Псевдо-неизменяемость»: объект неизменяем, но отдаёт наружу изменяемый внутренний список.
