# 🚀 Проект уровня 1 · Спроектируй базу данных

> Спроектируй полную, нормализованную схему БД для реального приложения — от ER-диаграммы до
> работающих таблиц с данными. Это фундамент, на котором держится всё приложение.

> 🛠️ SQLite (для начала) или PostgreSQL. С ER-диаграммой, схемой, тестовыми данными.

---

## 🎯 Что делаем

Выбери предметную область и спроектируй её БД: ER → схема (3NF) → CREATE TABLE с ограничениями →
наполнение тестовыми данными.

```
   варианты предметной области:
   • интернет-магазин (клиенты, заказы, товары, категории).
   • система блога (пользователи, посты, комментарии, теги).
   • библиотека (книги, авторы, читатели, выдачи).
   • система задач (проекты, задачи, исполнители, метки).
```

---

## 📋 Этапы

### 1. Требования и сущности
- Опиши предметную область: что хранить, как используется.
- Выпиши сущности, их атрибуты, связи и кардинальность.

### 2. ER-диаграмма
- Нарисуй (бумага/dbdiagram.io/Mermaid). Покажи сущности, связи (1:N, N:M), кардинальность.

### 3. Схема (CREATE TABLE)
- Переведи ER в таблицы. Правильные типы (деньги → DECIMAL, даты → DATE/TIMESTAMP).
- Все ключи (PK, FK), ограничения (NOT NULL, UNIQUE, CHECK, DEFAULT).
- Связи N:M через связующие таблицы.

### 4. Нормализация
- Проверь схему по 3NF: каждый столбец описывает свою сущность? Исправь нарушения.

### 5. Наполнение данными
- Вставь реалистичные тестовые данные (несколько клиентов, заказов, товаров и т.д.).
- Убедись, что ограничения работают (попробуй вставить невалидное — отвергается).

### 6. Проверка целостности
- Попробуй нарушить FK/CHECK/UNIQUE — СУБД защищает?
- Заказ без клиента, отрицательная цена, дубликат email — отвергаются?

---

## 🌟 Бонусы

- 🥚 Документируй каждое решение (почему эта связь, этот тип, это ограничение).
- 🐣 Покажи денормализацию для одного горячего сценария (с разбором цены).
- 🦅 Добавь JSONB-поле для гибких атрибутов (где оправдано).
- 🚀 Сгенерируй много данных (тысячи строк) — пригодится для Уровня 2 (запросы) и 3 (индексы).

---

## ✅ Критерии готовности

- [ ] Есть ER-диаграмма с сущностями, связями, кардинальностью
- [ ] Схема создана (CREATE TABLE) с правильными типами и всеми ограничениями
- [ ] Реализованы все типы связей (1:N, N:M через связующие таблицы)
- [ ] Схема нормализована (3NF), нарушения исправлены
- [ ] Наполнена реалистичными данными
- [ ] Целостность работает (невалидные данные отвергаются)
