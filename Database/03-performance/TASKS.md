# ✅ Задачи уровня 3 · Производительность и надёжность

> Нужна БОЛЬШАЯ таблица (тысячи-миллионы строк), чтобы видеть эффекты. Генерируй данные. Используй
> EXPLAIN ANALYZE постоянно.

---

## 🟢 Базовые

1. **Индекс до/после.** Запрос с фильтром на большой таблице без индекса (медленно) → создай индекс →
   замерь. EXPLAIN до/после (Seq Scan → Index Scan).
2. **EXPLAIN.** Прочитай план запроса с JOIN. Используются ли индексы? Какая стратегия join?
3. **Транзакция.** Перевод между счетами в BEGIN...COMMIT; проверь ROLLBACK.

## 🟡 Средние

4. **FK-индекс.** JOIN по внешнему ключу без индекса и с ним. Сравни время и план.
5. **N+1.** Найди/воспроизведи N+1, перепиши одним JOIN. Сравни число запросов и время.
6. **Узкое место.** По EXPLAIN ANALYZE найди самый дорогой шаг медленного запроса. Почини.
7. **ACID.** Симулируй ошибку в середине транзакции — откатилось всё (атомарность)?

## 🔴 Со звёздочкой

8. ⭐ **Составной индекс.** Создай `(a,b)`, проверь ускорение для запросов по a / a,b / b. Объясни «левый префикс».
9. ⭐ **Индекс не работает.** Запрос, где индекс есть, но не используется (функция/LIKE '%x'). Исправь.
10. ⭐ **Изоляция.** Воспроизведи аномалию (lost update/non-repeatable read), реши уровнем изоляции или
    FOR UPDATE.
11. ⭐⭐ **Дедлок и MVCC.** Воспроизведи дедлок (обратный порядок); покажи, что в Postgres читатель не
    блокируется писателем (MVCC).
