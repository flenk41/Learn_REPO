# 🚀 Проект уровня 3 · Ускорь медленную базу

> Возьми базу с большими данными и медленными запросами и оптимизируй её — индексами, переписыванием
> запросов, устранением N+1 — доказывая каждый шаг через EXPLAIN и замеры. Это senior-навык работы с БД.

> 🛠️ PostgreSQL с большим объёмом данных (генерируй тысячи-миллионы строк).

---

## 🎯 Что делаем

Создай базу с большим объёмом данных, найди медленные запросы и оптимизируй их методично
(измеряй → EXPLAIN → чини → измеряй), плюс обеспечь надёжность транзакциями.

```
   большие данные → медленные запросы → EXPLAIN → индексы/переписать/убрать N+1 → быстро + надёжно
```

---

## 📋 Этапы

### 1. База с объёмом
- Схема (из Уровня 1) + сгенерируй МНОГО данных (тысячи-миллионы строк) — иначе эффекты не видны.

### 2. Найди медленное
- Напиши набор реалистичных запросов (отчёты, поиски, join'ы). Замерь время. Какие медленные?

### 3. Оптимизируй индексами
- Для медленных запросов: EXPLAIN ANALYZE → Seq Scan? → создай индексы (на фильтры, FK, сортировки).
- Замерь до/после. Покажи Seq Scan → Index Scan в плане.

### 4. Устрани N+1
- Найди (в коде/сценарии) N+1. Перепиши одним JOIN. Замерь число запросов и время.

### 5. Перепиши запросы
- Где индекс не помог — оптимизируй сам запрос (нужные столбцы, агрегация в БД, keyset-пагинация,
  убрать функции над столбцами).

### 6. Надёжность транзакциями
- Оберни связанные изменения в транзакции (ACID). Покажи атомарность (откат при сбое).
- Реши конкурентное обновление (FOR UPDATE / уровень изоляции).

### 7. Отчёт
- Для каждой оптимизации: запрос, план до, что сделал, план после, время до/после, объяснение.

---

## 🌟 Бонусы

- 🥚 График время-vs-размер-данных до/после индексов.
- 🐣 Найди оптимальный набор индексов (не лишние — замерь цену записи).
- 🦅 Денормализуй один горячий путь (с разбором цены) и сравни.
- 🚀 Настрой лог медленных запросов; смоделируй конкурентную нагрузку.

---

## ✅ Критерии готовности

- [ ] База с большим объёмом данных (эффекты видны)
- [ ] Медленные запросы найдены и оптимизированы (индексы/переписывание)
- [ ] Каждая оптимизация подтверждена EXPLAIN и замерами до/после
- [ ] Устранён N+1 (число запросов сократилось)
- [ ] Связанные изменения в транзакциях (ACID), решено конкурентное обновление
- [ ] Отчёт с планами и объяснениями
