# 02 · Инструменты: psql, клиенты, песочницы 🖼️

> 🎯 **Цель блока:** собрать инструменты для практики SQL — от мгновенных онлайн-песочниц до
> полноценной PostgreSQL.

---

## 📖 SQL учится только за клавиатурой

```
   читать про SQL бесполезно — нужно ПИСАТЬ запросы к реальным данным. инструменты:
   • ОНЛАЙН-ПЕСОЧНИЦЫ — мгновенно, ничего не ставить (для старта).
   • ЛОКАЛЬНАЯ СУБД — настоящая база (для глубины).
   • КЛИЕНТЫ — удобный интерфейс к базе.
```

---

## ⭐ Быстрый старт (ничего не ставить)

```
   • SQLite — встроенная, один файл. в браузере (sql.js) или CLI (sqlite3). мгновенно.
   • ОНЛАЙН: db-fiddle.com, sqliteonline.com, pgexercises.com (тренажёр с задачами), sql-практикумы.
     пишешь SQL → видишь результат. идеально для Уровня 2 (SQL).
   рекомендация новичку: начни с онлайн-тренажёра или sqlite3 — учи SQL без возни с установкой.
```

---

## ⭐⭐ PostgreSQL — для серьёзной практики

```
   когда дойдёшь до индексов/транзакций/масштаба — нужна настоящая СУБД. PostgreSQL:

   УСТАНОВКА:
   • локально (пакетный менеджер) или в DOCKER (проще всего):
     docker run --name pg -e POSTGRES_PASSWORD=pass -p 5432:5432 -d postgres
   • облачные бесплатные тиры (для практики без установки).

   ПОДКЛЮЧЕНИЕ:
   • psql — официальный CLI: psql -h localhost -U postgres
     команды: \l (базы), \dt (таблицы), \d table (структура), \q (выход).
   • GUI-клиенты: DBeaver (универсальный, бесплатный), pgAdmin — удобнее для визуализации схемы/данных.
```

🖼️
```
   старт:    онлайн-песочница / sqlite3   → учи SQL мгновенно
   глубина:  PostgreSQL (Docker) + psql/DBeaver → индексы, EXPLAIN, транзакции, реальная работа
```

💡 ⭐ Docker — самый простой способ поднять PostgreSQL: одна команда, изолированно, легко
удалить/пересоздать. `psql` — для команд и скриптов; DBeaver — для визуального изучения схемы,
просмотра планов, удобного редактирования. (Docker — это [DevOps-трек]; здесь — просто инструмент.)

## 📖 Учебные данные

```
   чтобы практиковаться, нужны ДАННЫЕ. варианты:
   • готовые учебные базы: pagila/dvdrental (PostgreSQL, прокат фильмов), chinook (музыка),
     northwind (классика) — реалистичные схемы с данными.
   • тренажёры с задачами: pgexercises, SQL-практикумы (запрос → проверка).
   • своя маленькая база (магазин: clients, orders, products) — для Уровня 1 (проектирование).
   совет: возьми готовую учебную базу с интересными данными — на ней приятнее учить запросы.
```

## ⚠️ Ловушки

- ❌ Только читать про SQL, не выполняя запросы (не закрепится).
- ❌ Мучиться с установкой PostgreSQL вместо Docker/онлайн на старте.
- ❌ Учить SQL на пустой базе без данных (нужны реалистичные данные).
- ❌ Игнорировать `\d`/структуру — понимать схему важно для запросов.
- ❌ Хранить пароли БД в коде (см. безопасность).

---

## ✅ Упражнения

1. **Старт.** Открой онлайн-песочницу (или sqlite3), выполни `SELECT 1 + 1;`. Работает?
2. **Учебная база.** Загрузи готовую учебную базу (dvdrental/chinook). Посмотри таблицы (`\dt`/обзор).
3. **PostgreSQL.** Подними Postgres (Docker) и подключись через `psql`. Команды `\l`, `\dt`, `\d`.
4. **Структура.** Для одной таблицы посмотри её структуру (`\d table`): столбцы, типы, ключи.
