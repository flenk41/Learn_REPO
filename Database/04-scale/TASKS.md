# ✅ Задачи · Уровень 4 — Масштаб и эксплуатация

> Закрепи масштабирование, надёжность и безопасность БД. Размышляй и пробуй на реальной СУБД.

---

## NoSQL (модуль 18)

1. **Типы.** Сопоставь 4 типа NoSQL (документная, key-value, колоночная, графовая) с задачами.
   Для каждого — пример, где он лучше реляционной.
2. **Выбор.** Для трёх систем (каталог товаров, кэш сессий, граф друзей) выбери тип БД. Обоснуй.
3. ⭐ **SQL vs NoSQL.** Когда реляционная по умолчанию? Когда оправдан NoSQL? Опиши 2 честных сценария.

---

## Репликация и шардинг (модуль 19)

4. **Лестница.** Распиши порядок масштабирования БД (оптимизация → вертикаль → кэш → реплики → шардинг).
   Почему шардинг последний?
5. ⭐ **Lag.** Опиши read-your-writes проблему с replication lag и два способа решения.
6. ⭐⭐ **Shard key.** Для соцсети выбери ключ шардирования. Какие запросы станут кросс-шардовыми (дорогими)?

---

## CAP (модуль 20)

7. **CP vs AP.** Для банка, ленты соцсети, корзины, счётчика лайков — CP или AP? Обоснуй каждый.
8. ⭐ **Eventual.** Для каких данных приложения eventual consistency приемлема, для каких нет?

---

## Кэширование (модуль 21)

9. **Cache-aside.** Реализуй: проверь кэш → miss → читай БД → положи с TTL. Замерь ускорение повтора.
10. ⭐ **Инвалидация.** При UPDATE инвалидируй кэш. Trade-off TTL против инвалидации при записи?
11. ⭐⭐ **Stampede.** Что такое cache stampede и как его избежать?

---

## Миграции (модуль 22)

12. **UP/DOWN.** Напиши миграцию (добавить столбец + индекс) с откатом.
13. ⭐ **Многошагово.** Распиши «переименовать столбец» без поломки работающего приложения.
14. ⭐⭐ **Большая таблица.** Как добавить NOT NULL-столбец в таблицу на 10млн строк без долгой блокировки?

---

## Безопасность (модуль 23)

15. **Инъекция.** Покажи уязвимый (склейка) и безопасный (параметризованный) запрос. Почему второй безопасен?
16. ⭐ **Привилегии.** Спроектируй роли БД (приложение/миграции/аналитика). Зачем разделять?
17. ⭐⭐ **Защита целиком.** Перечисли слои защиты БД (инъекции → привилегии → сеть → шифрование → бэкап → аудит).
