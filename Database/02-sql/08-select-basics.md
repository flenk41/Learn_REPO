# 08 · SELECT, WHERE, ORDER, LIMIT 🖼️⭐⭐

> 🎯 **Цель блока:** освоить основу SQL — извлечение данных запросом SELECT с фильтрацией,
> сортировкой и ограничением.

---

## 📖 SELECT — «дай мне данные»

```sql
SELECT столбцы          -- ЧТО показать
FROM таблица            -- ОТКУДА
WHERE условие           -- какие строки (фильтр)
ORDER BY столбец        -- как сортировать
LIMIT n;                -- сколько вернуть

-- пример:
SELECT name, email FROM clients WHERE age >= 18 ORDER BY name LIMIT 10;
-- «имена и email совершеннолетних клиентов, по алфавиту, первые 10»
```

```
   порядок написания: SELECT → FROM → WHERE → ORDER BY → LIMIT.
   * = все столбцы (SELECT *) — удобно для изучения, но в коде лучше явно перечислять нужные.
```

💡 ⭐⭐ SQL **декларативен**: ты описываешь, ЧТО хочешь, а СУБД решает КАК. `SELECT name FROM clients
WHERE age >= 18` — ты не пишешь циклы и поиск, СУБД сама эффективно найдёт (используя индексы).
Это основа всей работы с данными — освой её до автоматизма на практике.

## ⭐ WHERE — фильтрация строк

```sql
-- операторы сравнения и логики:
WHERE age >= 18 AND city = 'Москва'        -- И
WHERE status = 'new' OR status = 'paid'    -- ИЛИ
WHERE price BETWEEN 100 AND 500            -- диапазон
WHERE status IN ('new', 'paid', 'shipped') -- из списка
WHERE name LIKE 'А%'                       -- шаблон (% = любые символы) — имена на «А»
WHERE email IS NULL                        -- проверка NULL (не = NULL!)
WHERE NOT (status = 'cancelled')           -- отрицание
```

💡 ⭐ WHERE отбирает строки по условию. Ключевые операторы: сравнения (`=`, `<`, `>=`), логика
(`AND`/`OR`/`NOT`), `IN` (из списка), `BETWEEN` (диапазон), `LIKE` (шаблоны строк), `IS NULL`
(проверка отсутствия — помни, `= NULL` не работает!). Эти операторы покрывают большинство фильтров.

## ⭐⭐ ORDER BY и LIMIT

```sql
SELECT * FROM orders
ORDER BY created_at DESC, amount ASC    -- по дате убыванием, потом по сумме возрастанием
LIMIT 20 OFFSET 40;                      -- 20 строк, пропустив первые 40 (пагинация: страница 3)

-- ORDER BY: ASC (по возрастанию, по умолчанию) / DESC (по убыванию).
-- несколько столбцов: сначала по первому, при равенстве — по второму.
-- LIMIT n — вернуть не более n; OFFSET m — пропустить первые m (для пагинации).
```

💡 ⭐⭐ `ORDER BY` сортирует результат (важно: без него порядок НЕ гарантирован!). `LIMIT`/`OFFSET` —
основа **пагинации** (показывать данные страницами): страница 3 по 20 = `LIMIT 20 OFFSET 40`.
Частый паттерн: «последние N» = `ORDER BY created_at DESC LIMIT N`. (Большой OFFSET медленный на
больших данных — есть keyset-пагинация, но это позже.)

## 📖 Полезное: DISTINCT, вычисления, алиасы

```sql
SELECT DISTINCT city FROM clients;              -- уникальные города (без дубликатов)
SELECT name, price * qty AS total FROM items;   -- вычисление + алиас (AS — переименование столбца)
SELECT name, age FROM clients WHERE age > 18    -- комментарии: -- строка, /* блок */
   ORDER BY age DESC;
-- столбцы можно вычислять, переименовывать (AS), применять функции (UPPER, LENGTH, COALESCE для NULL...).
```

> 🧭 SQL — это [API к данным](../../Senior/03-practices/16-requirements.md): декларативно описываешь
> потребность. В приложениях запросы пишут через [параметризацию (безопасность)](../../Security/02-vulnerabilities/09-injection-sqli.md),
> не склейкой строк (защита от инъекций).

## ⚠️ Ловушки

- ❌ `= NULL` вместо `IS NULL` (не сработает — NULL «заразен»).
- ❌ Полагаться на порядок строк без ORDER BY (не гарантирован).
- ❌ `SELECT *` в production-коде (хрупко, тянет лишнее) — перечисляй нужные столбцы.
- ❌ Забыть, что LIKE чувствителен к регистру (зависит от СУБД; ILIKE в Postgres — без регистра).
- ❌ Большой OFFSET на больших данных (медленно — СУБД всё равно перебирает пропускаемые).
- ❌ Путать AND/OR без скобок (приоритет: AND выше OR).

## ✅ Задачи (на учебной базе)

1. **Базовый SELECT.** Выбери определённые столбцы из таблицы с фильтром WHERE. Несколько условий (AND/OR).
2. **Операторы.** Используй IN, BETWEEN, LIKE, IS NULL в запросах. Найди строки по шаблону.
3. **Сортировка и пагинация.** ORDER BY по нескольким столбцам; LIMIT/OFFSET для «страницы».
4. ⭐ **Вычисления.** Запрос с вычисляемым столбцом (price * qty AS total) и алиасами.
5. ⭐ **DISTINCT + функции.** Уникальные значения; примени строковую/числовую функцию (UPPER, ROUND).
