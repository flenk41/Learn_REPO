# 10 · Агрегация и GROUP BY 🖼️⭐⭐

> 🎯 **Цель блока:** освоить агрегатные функции и группировку — вычисление итогов по группам строк
> (суммы, средние, количества). Основа аналитики и отчётов.

---

## 📖 Агрегатные функции: итог по множеству строк

```sql
SELECT COUNT(*) FROM orders;              -- сколько заказов всего
SELECT SUM(amount) FROM orders;           -- общая сумма
SELECT AVG(amount) FROM orders;           -- средняя
SELECT MIN(amount), MAX(amount) FROM orders;  -- мин и макс

-- агрегатные функции «схлопывают» МНОЖЕСТВО строк в ОДНО значение:
-- COUNT (количество), SUM (сумма), AVG (среднее), MIN/MAX (мин/макс).
-- COUNT(*) — все строки; COUNT(column) — не-NULL значения; COUNT(DISTINCT column) — уникальные.
```

💡 ⭐ Агрегаты отвечают на «сколько/итого/в среднем». `COUNT(*)` считает строки, `SUM`/`AVG`/`MIN`/
`MAX` — по числовому столбцу. `COUNT(DISTINCT city)` — сколько уникальных городов. Без GROUP BY
агрегат считается по ВСЕМ строкам (одно значение на весь результат).

## ⭐⭐ GROUP BY: итоги ПО ГРУППАМ

```sql
-- «сумма заказов ПО КАЖДОМУ клиенту»:
SELECT client_id, COUNT(*) AS orders_count, SUM(amount) AS total
FROM orders
GROUP BY client_id;
-- GROUP BY делит строки на ГРУППЫ (по client_id), агрегат считается ДЛЯ КАЖДОЙ группы.
```

🖼️
```
   orders (до):              GROUP BY client_id (после):
   client_id│amount          client_id│COUNT│SUM
      1     │ 500            ───────────────────────
      1     │ 300    ───►       1     │  2  │800   ← группа клиента 1
      2     │ 200               2     │  1  │200   ← группа клиента 2
   строки группируются по значению → агрегат на каждую группу.
```

💡 ⭐⭐ `GROUP BY столбец` — делит строки на группы с одинаковым значением столбца, и агрегат
(COUNT/SUM/...) считается ОТДЕЛЬНО для каждой группы. Это основа отчётов: «продажи по месяцам»,
«заказы по клиентам», «средний чек по категориям». **Правило:** в SELECT с GROUP BY можно
только группируемые столбцы + агрегаты (не «голые» столбцы).

## ⭐ HAVING: фильтр ПО ГРУППАМ

```sql
-- «клиенты, у которых БОЛЬШЕ 5 заказов»:
SELECT client_id, COUNT(*) AS cnt
FROM orders
GROUP BY client_id
HAVING COUNT(*) > 5;          -- фильтр ПО АГРЕГАТУ (после группировки)

-- WHERE vs HAVING:
-- WHERE  — фильтрует СТРОКИ ДО группировки (нельзя использовать агрегаты).
-- HAVING — фильтрует ГРУППЫ ПОСЛЕ агрегации (можно агрегаты: HAVING SUM(...) > 1000).
```

💡 ⭐ Ключевое различие: **WHERE** отбирает строки до группировки, **HAVING** — группы после. «Сумма
заказов > 1000 у клиентов из Москвы за 2024»: `WHERE city='Москва' AND year=2024` (строки) +
`GROUP BY client_id` + `HAVING SUM(amount) > 1000` (группы). Частая ошибка — пытаться фильтровать
агрегат в WHERE (нельзя, агрегата ещё нет).

## 📖 Порядок выполнения и JOIN + GROUP BY

```
   ЛОГИЧЕСКИЙ порядок выполнения SQL (важно понимать!):
   FROM/JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
   (поэтому в WHERE нет агрегатов — группировки ещё не было; алиасы из SELECT не видны в WHERE.)

   типичный аналитический запрос — JOIN + GROUP BY:
   SELECT c.name, COUNT(o.id) AS orders, SUM(o.amount) AS total
   FROM clients c LEFT JOIN orders o ON o.client_id = c.id
   GROUP BY c.id, c.name
   ORDER BY total DESC NULLS LAST
   LIMIT 10;        -- топ-10 клиентов по сумме заказов (включая тех, у кого 0 — LEFT JOIN)
```

💡 ⭐ Понимание **порядка выполнения** объясняет многое: почему агрегаты нельзя в WHERE (FROM→WHERE
идёт ДО GROUP BY), почему алиасы SELECT не видны раньше. JOIN + GROUP BY + ORDER BY — типичный
аналитический запрос («топ N по...»).

## ⚠️ Ловушки

- ❌ «Голый» столбец в SELECT с GROUP BY (не в группировке и не агрегат) → ошибка/неоднозначность.
- ❌ Фильтровать агрегат в WHERE (нельзя — нужен HAVING).
- ❌ COUNT(column) вместо COUNT(*) когда есть NULL (COUNT(column) пропускает NULL).
- ❌ Забыть, что INNER JOIN + агрегат теряет строки без пары (нужен LEFT для «включая нулевые»).
- ❌ Не группировать по всем неагрегатным столбцам SELECT.
- ❌ Путать порядок WHERE (строки) и HAVING (группы).

## ✅ Задачи (на учебной базе)

1. **Агрегаты.** Посчитай COUNT/SUM/AVG/MIN/MAX по таблице (всего заказов, общая сумма, средний чек).
2. **GROUP BY.** Сумма и количество заказов ПО каждому клиенту (или продажи по месяцам).
3. ⭐ **HAVING.** Найди клиентов с более чем N заказами / суммой больше X (HAVING).
4. ⭐ **JOIN + GROUP BY.** Топ-10 клиентов по сумме заказов (с именами, через JOIN, включая нулевых — LEFT).
5. **WHERE vs HAVING.** Запрос с фильтром строк (WHERE) И групп (HAVING) одновременно.
