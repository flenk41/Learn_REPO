# 11 · Подзапросы и CTE 🖼️⭐

> 🎯 **Цель блока:** освоить подзапросы (запрос внутри запроса) и CTE (именованные подзапросы) —
> для сложных запросов, которые не выразить простым SELECT.

---

## 📖 Подзапрос — запрос внутри запроса

```sql
-- «клиенты, чья сумма заказов выше СРЕДНЕЙ по всем»:
SELECT name FROM clients
WHERE id IN (
    SELECT client_id FROM orders        -- ВНУТРЕННИЙ запрос (подзапрос)
    GROUP BY client_id
    HAVING SUM(amount) > (SELECT AVG(amount) FROM orders)  -- ещё подзапрос (среднее)
);
```

```
   ПОДЗАПРОС — SELECT внутри другого запроса. виды по результату:
   • СКАЛЯРНЫЙ (одно значение): WHERE amount > (SELECT AVG(amount) FROM orders).
   • СПИСОК (столбец значений): WHERE client_id IN (SELECT id FROM ...).
   • ТАБЛИЦА (в FROM): FROM (SELECT ...) AS sub — подзапрос как источник.
   используется в WHERE, FROM, SELECT.
```

💡 ⭐ Подзапросы решают «запрос, зависящий от результата другого запроса»: «выше среднего», «те,
кто есть в другом наборе», «топ по подсчёту». `IN (подзапрос)` — частый паттерн «принадлежит
множеству». Скалярный подзапрос `(SELECT AVG...)` даёт одно число для сравнения.

## ⭐ Коррелированные подзапросы и EXISTS

```sql
-- КОРРЕЛИРОВАННЫЙ — подзапрос ссылается на внешний (выполняется для каждой внешней строки):
SELECT name FROM clients c
WHERE EXISTS (                              -- есть ли хоть один заказ у этого клиента?
    SELECT 1 FROM orders o WHERE o.client_id = c.id
);                                          -- c.id — из ВНЕШНЕГО запроса (корреляция)

-- EXISTS — «существует ли хоть одна строка» (часто эффективнее IN для проверки наличия).
-- NOT EXISTS — «нет ни одной» (клиенты без заказов).
```

💡 ⭐ `EXISTS`/`NOT EXISTS` — проверка наличия связанных строк (часто читаемее и эффективнее, чем
`IN`/`NOT IN`, особенно с NULL). Коррелированный подзапрос «привязан» к внешней строке — мощно, но
может быть медленным (выполняется на каждую строку). Многое из этого выражается и через JOIN —
выбирай по читаемости/производительности.

---

## ⭐⭐ CTE (WITH) — именованные подзапросы

```sql
-- CTE (Common Table Expression) — именованный подзапрос для ЧИТАЕМОСТИ:
WITH client_totals AS (
    SELECT client_id, SUM(amount) AS total
    FROM orders GROUP BY client_id
)
SELECT c.name, ct.total
FROM clients c JOIN client_totals ct ON ct.client_id = c.id
WHERE ct.total > 1000
ORDER BY ct.total DESC;
-- client_totals определён ОДИН раз, используется как таблица. читается сверху вниз, по шагам.
```

💡 ⭐⭐ CTE (`WITH ... AS`) — это «временная именованная таблица» для запроса. Они делают сложные
запросы **читаемыми**: разбиваешь логику на именованные шаги (как переменные в коде), вместо
вложенных подзапросов-матрёшек. Можно несколько CTE, ссылающихся друг на друга. Для сложной
аналитики CTE — лучший друг читаемости (и [Senior-чистого кода](../../Senior/01-craft/03-clean-code.md)).

## 📖 Оконные функции (превью, продвинутое)

```sql
-- ОКОННЫЕ ФУНКЦИИ — агрегаты БЕЗ схлопывания строк (для рейтингов, нарастающих итогов):
SELECT name, amount,
   RANK() OVER (ORDER BY amount DESC) AS rank,           -- ранг по сумме
   SUM(amount) OVER (PARTITION BY client_id) AS client_total  -- сумма клиента рядом с каждой строкой
FROM orders;
-- OVER(...) — «окно» строк. PARTITION BY — группы, ORDER BY — порядок внутри окна.
-- даёт: ранги (ROW_NUMBER/RANK), нарастающие итоги, «N-й по...», предыдущая/следующая строка (LAG/LEAD).
```

💡 ⭐ Оконные функции — мощный продвинутый инструмент: считают агрегаты/ранги, НЕ схлопывая строки
(в отличие от GROUP BY). «Топ-3 заказа каждого клиента», «нарастающий итог», «доля от общего» —
их территория. Сложная тема; знай, что они есть, для аналитики.

---

## ⚠️ Ловушки

- ❌ Вложенные подзапросы-матрёшки вместо CTE (нечитаемо) — используй WITH.
- ❌ `NOT IN (подзапрос с NULL)` → неожиданно пустой результат (NULL ломает NOT IN; используй NOT EXISTS).
- ❌ Коррелированный подзапрос на больших данных без понимания стоимости (медленно).
- ❌ Подзапрос там, где проще/быстрее JOIN (и наоборот).
- ❌ Забыть алиас для подзапроса в FROM.

---

## ✅ Задачи (на учебной базе)

1. **Скалярный подзапрос.** Заказы с суммой выше средней (подзапрос со средним).
2. **IN.** Клиенты, сделавшие хоть один заказ (через IN с подзапросом).
3. ⭐ **EXISTS.** Клиенты с заказами (EXISTS) и без (NOT EXISTS). Сравни с LEFT JOIN-вариантом.
4. ⭐ **CTE.** Перепиши сложный вложенный запрос через WITH (несколько шагов). Сравни читаемость.
5. ⭐ **Оконная функция.** Ранг заказов по сумме / топ-N заказов каждого клиента (RANK/ROW_NUMBER OVER).
