# 🚀 Проект уровня 2 · Аналитика на SQL

> Докажи владение SQL: ответь на десятки бизнес-вопросов к реальной базе данных одними запросами —
> от простых выборок до сложной аналитики с join'ами, группировкой и подзапросами.

> 🛠️ Учебная база (dvdrental/chinook/northwind) или своя из Уровня 1. Только SQL.

---

## 🎯 Что делаем

Возьми базу с данными и составь **сборник запросов**, отвечающих на реальные вопросы бизнеса,
с нарастающей сложностью. Каждый запрос — рабочий, с комментарием «что отвечает».

```
   простые выборки → join'ы → агрегация → сложная аналитика → отчёты
```

---

## 📋 Вопросы для ответа (адаптируй под свою базу)

### Базовые (SELECT/WHERE)
- Все клиенты из города X, отсортированные по имени.
- Заказы за период с суммой больше N.
- Товары, название которых начинается на «А».

### JOIN
- Заказы с именами клиентов.
- Все клиенты и количество их заказов (включая тех, у кого 0).
- Какие товары в конкретном заказе (через N:M).

### Агрегация
- Общая выручка; средний чек; число заказов по месяцам.
- Топ-10 клиентов по сумме заказов.
- Выручка по категориям товаров.

### Сложная аналитика
- Клиенты с суммой заказов выше средней (подзапрос/CTE).
- Клиенты без единого заказа.
- Топ-3 товара в каждой категории (оконные функции).
- «Выручка по месяцам с нарастающим итогом» (оконные функции).

---

## 📋 Требования

- **10–20+ запросов** разной сложности, каждый с комментарием (какой вопрос отвечает).
- Использованы: WHERE-фильтры, INNER и LEFT JOIN, GROUP BY/HAVING, подзапросы, CTE.
- (бонус) оконные функции для рейтингов/нарастающих итогов.
- Запросы **читаемы** (CTE для сложных, алиасы, форматирование).
- Каждый запрос проверен на реальных данных (даёт осмысленный результат).

---

## 🌟 Бонусы

- 🥚 Оформи как «дашборд»: набор ключевых метрик бизнеса одними запросами.
- 🐣 Сделай представления (VIEW) для частых сложных запросов.
- 🦅 Сравни два способа одного запроса (подзапрос vs JOIN; IN vs EXISTS) — какой читаемее/быстрее?
- 🚀 Прикинь, какие из запросов будут медленными на больших данных (подготовка к Уровню 3 — индексы).

## ✅ Критерии готовности

- [ ] Использованы JOIN (INNER/LEFT), агрегация (GROUP BY/HAVING)
- [ ] Использованы подзапросы и CTE
