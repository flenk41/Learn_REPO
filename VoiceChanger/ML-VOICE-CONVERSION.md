# 🧠 Voice Changer — нейросетевая конверсия голоса (уровень B)

> Максимально «реалистичный» путь: превратить твой голос в **голос конкретной модели** (другого
> тембра/человека), а не просто сдвинуть питч/форманты. Это **voice conversion (VC)** — машинное
> обучение, не классический DSP.

> ⚠️⚠️ **Этика и закон — ЧИТАЙ ПЕРВЫМ.** Клонирование голоса РЕАЛЬНОГО человека без его согласия —
> неэтично и часто незаконно (мошенничество, выдача себя за другого, обход голосовой биометрии,
> дипфейки, диффамация). Делай VC на **свой** голос, на **синтетические/лицензированные** целевые
> модели или с **явного согласия**. Соблюдай законы о дипфейках и правила платформ. См.
> [этику влияния](../SocialEng/02-psychology/12-manipulation-vs-persuasion.md) и
> [безопасность](../Security/README.md).

## 🎯 DSP vs нейросетевая конверсия

```
   DSP (уровень A, см. DSP-DEEPDIVE):
   • меняет ХАРАКТЕРИСТИКИ твоего голоса (выше/ниже, тоньше/толще тембр, эффекты).
   • остаётся «твоим голосом, изменённым». управляемо, низкая латентность, без GPU.

   ML VOICE CONVERSION (уровень B):
   • заменяет ТЕМБР/ИДЕНТИЧНОСТЬ на ЦЕЛЕВОЙ голос (модель), сохраняя ЧТО ты сказал и интонацию.
   • «звучишь как другой человек/персонаж». максимальный реализм.
   • цена: тяжело (нужен GPU для realtime), выше латентность, надо обучать/брать модель.
```

💡 ⭐ Ключевая разница: DSP **модифицирует** твой голос, ML **подменяет идентичность** на целевую,
сохраняя содержание речи. Для «звучать как конкретный голос» нужен ML. Для «сделать голос выше/
роботизированным» хватает DSP (и это проще/быстрее).

## ⭐⭐ Как устроена конверсия (RVC / so-vits-svc)

```
   общий конвейер современных realtime-VC (RVC = Retrieval-based Voice Conversion):

   твой голос ──► [1 КОНТЕНТ-ЭНКОДЕР] ──► фонетические признаки (ЧТО сказано, БЕЗ твоего тембра)
                   (HuBERT / ContentVec)         │
   твой голос ──► [2 ДЕТЕКТ F0] ──► высота ──────┤  (+ опц. сдвиг питча под целевой диапазон)
                   (RMVPE / crepe / harvest)      │
                                                  ▼
                              [3 (опц.) RETRIEVAL] подмешать ближайшие признаки из индекса целевого
                                                  │  голоса (kNN) → вернее тембр, меньше «утечки» твоего
                                                  ▼
                              [4 ГЕНЕРАТОР/ВОКОДЕР, обучен на ЦЕЛЕВОМ голосе]
                                   (NSF-HiFiGAN: из контент+F0 синтезирует волну в ТЕМБРЕ цели)
                                                  ▼
                                         голос в ЦЕЛЕВОМ тембре
```

```
   суть: РАЗДЕЛИТЬ «что сказано» (контент, speaker-independent) и «как звучит говорящий» (тембр),
   затем СИНТЕЗИРОВАТЬ контент в тембре ЦЕЛИ. F0 переносит интонацию (можно сдвинуть под цель).
   so-vits-svc — похожая идея (SoftVC контент-энкодер + VITS-синтез).
```

💡 ⭐⭐ Главная идея VC — **разделить содержание и идентичность**: контент-энкодер (HuBERT/ContentVec)
выкидывает твой тембр, оставляя фонетику; вокодер, обученный на целевом голосе, «переозвучивает» это в
целевом тембре. F0 переносит твою интонацию. Это глубже DSP-разделения «pitch/формант» — здесь сеть
выучила, как звучит конкретный голос.

## ⏱️ Real-time: главная сложность

```
   VC тяжелее DSP → realtime требует приёмов:
   • ЧАНКИ + LOOKAHEAD: обрабатываешь короткими окнами (напр. 100-500мс) с небольшим опережением;
     КРОССФЕЙД между чанками, чтобы не было щелчков на стыках.
   • ЛАТЕНТНОСТЬ realtime-RVC ≈ 100-300мс (заметно больше DSP) — для разговора на грани, для стрима ок.
   • GPU желателен (NVIDIA) для низкой задержки; на CPU — выше задержка, меньшие модели.
   • МОДЕЛЬ в ИНФЕРЕНС-ПОТОКЕ, НЕ в аудио-callback! аудио-поток только кладёт/забирает сэмплы (ring buffer),
     инференс крутится отдельно (он может «дёргаться» — нельзя в real-time callback).
   готовые realtime-реализации (напр. w-okada Voice Changer) делают именно чанки+кроссфейд+отдельный поток.
```

💡 ⭐⭐ Архитектурное правило: **нейросеть НИКОГДА не в аудио-callback** (она не укладывается в дедлайн
и аллоцирует). Аудио-поток ↔ инференс-поток связаны lock-free ring buffer'ом
([как в C++/Rust архитектурах](ARCHITECTURE-RUST.md)). Латентность VC выше DSP — честно скажи себе:
для живого диалога DSP комфортнее, VC — когда нужен именно чужой тембр и терпима задержка.

## ⭐ Обучение целевого голоса

```
   чтобы получить «целевой» голос, модель ОБУЧАЮТ на его записях:
   • ДАННЫЕ: чистые записи целевого голоса (RVC — от ~10 мин, лучше 30-60 мин; без шума/музыки/эха).
   • препроцессинг: нарезка, ресемпл, извлечение F0 и контент-признаков.
   • ТРЕНИРОВКА генератора/вокодера на этих данных (нужен GPU, часы).
   • результат — файл модели (.pth/.onnx) + индекс для retrieval.

   ⚠️⚠️ голос человека = биометрия и личность. обучай ТОЛЬКО на:
   • своём голосе; • голосе с ЯВНЫМ согласием; • синтетических/лицензированных голосах.
   НЕ клонируй реальных людей без согласия (мошенничество/дипфейк/обход верификации — незаконно).
```

## 🔌 Интеграция в приложение (C++ / Rust)

```
   ядро DSP (уровень A) и ML (уровень B) можно совместить:
   [микрофон] → ring buffer → [ИНФЕРЕНС-ПОТОК: VC-модель (чанки+кроссфейд)] → (опц. DSP-эффекты) → выход → вирт.микрофон

   запуск модели:
   • C++:  ONNX Runtime (onnxruntime) или libtorch — загрузить .onnx/.pt, инференс на CPU/GPU.
   • Rust: ort (ONNX Runtime), candle (чистый Rust ML) или tch (libtorch). тот же принцип.
   • модель экспортируют в ONNX для портативного быстрого инференса.

   потоки: аудио-callback (real-time, только I/O буферов) ⟂ инференс-поток (модель) ⟂ UI-поток.
   связь — lock-free ring buffers; параметры (какая модель, сдвиг питча) — атомарно.
```

💡 ⭐ Переиспользуй real-time-каркас из основных архитектур: разница лишь в том, что «процессором» в
инференс-потоке стоит нейросеть (ONNX Runtime), а не DSP-функция. Эффекты уровня A можно навесить ДО/
ПОСЛЕ VC (например, реверб поверх конвертированного голоса).

## 📊 Что выбрать

```
   нужно «голос выше/ниже/толще/робот/эхо», низкая задержка, без GPU   → DSP (уровень A). проще, быстрее.
   нужно «звучать как конкретный (свой/согласованный) голос», есть GPU  → ML VC (уровень B). реалистичнее.
   гибрид: VC для тембра + DSP-эффекты (реверб/эквалайзер) поверх       → максимум характера.
   для большинства любительских задач уровня A ДОСТАТОЧНО и комфортнее по латентности.
```

## ⚠️ Ловушки

- ❌❌ Клонировать реальный голос без согласия (мошенничество/дипфейк/обход биометрии — незаконно).
- ❌ Запускать инференс В аудио-callback (не уложится, аллокации → щелчки) — только отдельный поток.
- ❌ Ждать DSP-латентности от ML (VC = 100-300мс; для живого диалога на грани).
- ❌ Обучать на грязных данных (шум/музыка/эхо) → плохая модель.
- ❌ Думать, что ML «всегда лучше» — для простых эффектов DSP проще, быстрее, без GPU.
- ❌ Игнорировать GPU-требования realtime-VC (на слабом CPU — большая задержка).
