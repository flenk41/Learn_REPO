# 🔬 Voice Changer — DSP глубоко: как реально меняют голос

> Здесь — **алгоритмы**, делающие смену голоса реалистичной (а не «бурундук»). Язык-агностично:
> математика + псевдокод. Применимо и к [C/C++](ARCHITECTURE-CPP.md), и к [Rust](ARCHITECTURE-RUST.md).

> 🧭 Предполагает базу из [обзора](README.md): голос = высота (pitch) + тембр (форманты). Цель —
> менять их **раздельно** и без артефактов.

---

## 🎯 Почему наивный питч-шифт звучит плохо

```
   НАИВНО: «ускорить/замедлить» сэмпл (resample) → меняется И высота, И форманты вместе.
   • выше → «бурундук» (форманты уехали вверх, голос «маленький»).
   • ниже → «монстр» (форманты уехали вниз).

   РЕАЛИСТИЧНО: разделить ВЫСОТУ (частота связок, F0) и ФОРМАНТЫ (резонансы тракта = «размер» головы),
   менять их НЕЗАВИСИМО. два главных семейства алгоритмов:
   • ЧАСТОТНЫЕ (phase vocoder) — работают в спектре (STFT).
   • ВРЕМЕННЫЕ (PSOLA) — работают с волной по периодам высоты. ← часто лучше для ГОЛОСА.
```

---

## ⭐⭐ Подход 1 — Phase Vocoder (частотный)

### Конвейер STFT (анализ → модификация → синтез)

```
   1. РАЗБИТЬ на перекрывающиеся кадры (окно Ханна, размер N=1024..2048, шаг hop H=N/4).
   2. FFT каждого кадра → спектр (магнитуда + фаза).
   3. МОДИФИЦИРОВАТЬ спектр (пич/форманты — ниже).
   4. IFFT → кадры → OVERLAP-ADD обратно в волну.

   псевдокод оверлап-адд:
   for each frame:
       windowed = frame * hann_window
       spectrum = FFT(windowed)
       spectrum = modify(spectrum)          # вся «магия» здесь
       out_frame = IFFT(spectrum) * hann_window
       output[pos : pos+N] += out_frame     # сложение с перекрытием
       pos += H
```

### Сдвиг времени и высоты

```
   PHASE VOCODER растягивает/сжимает ВРЕМЯ, меняя шаг СИНТЕЗА vs АНАЛИЗА (H_syn ≠ H_ana),
   при этом ПРАВИЛЬНО пересчитывая фазы (phase propagation), чтобы гармоники остались когерентны:

   true_freq[k] = (Δphase[k] / H_ana) пересчитать в мгновенную частоту бина k
   phase_syn[k] += true_freq[k] * H_syn      # накапливаем синтез-фазу

   ПИТЧ-ШИФТ = растянуть время в R раз → пересэмплировать в 1/R → длительность та же, высота × R.
   (либо чисто частотный pitch shift: сдвиг бинов с интерполяцией.)

   ⚠️ артефакты: «phasiness» (металлический/размазанный звук), смазывание транзиентов (атак).
   улучшения: PHASE LOCKING (identity locking — фазы пиков и соседей согласованы),
              детект ТРАНЗИЕНТОВ (на атаке сбрасывать фазы, не размазывать).
```

💡 ⭐⭐ Phase vocoder силён для музыки и плавных звуков, гибок. Цена — латентность (окно N большое) и
склонность к «phasiness» без phase-locking. Для голоса часто лучше PSOLA (ниже), но phase vocoder
универсальнее (работает на любом сигнале, не только на voiced-речи).

## ⭐⭐ Сохранение и сдвиг ФОРМАНТ (ключ реализма)

```
   идея: спектр = ОГИБАЮЩАЯ (spectral envelope, медленная — это ФОРМАНТЫ/тракт)
                  × ТОНКАЯ СТРУКТУРА (быстрая — гармоники, это ВЫСОТА).
   разделив их, можно двигать высоту, НЕ трогая форманты (или двигать форманты отдельно).

   как достать огибающую:
   • КЕПСТРАЛЬНЫЙ метод: log|спектр| → IFFT = кепстр → оставить НИЗКИЕ quefrency (лифтеринг) →
     обратно = гладкая огибающая (формантная).
   • LPC (линейное предсказание): подобрать фильтр порядка ~12-20, моделирующий тракт (его АЧХ = форманты).
     «отбелить» сигнал в остаток (excitation), сдвинуть высоту остатка, ВЕРНУТЬ фильтр.
   • TRUE ENVELOPE: итеративный кепстр (точнее на высоком F0).

   СДВИГ ФОРМАНТ = ВАРПИНГ огибающей по частоте на formant_scale:
       env_new[f] = env_old[f / formant_scale]     # растянуть/сжать огибающую по оси частоты
   потом «надеть» новую огибающую на (пич-сдвинутую) тонкую структуру.
```

🖼️
```
   спектр голоса:
   |огибающая (форманты F1,F2,F3) — «кто говорит / размер»
   |     ╱‾╲       ╱‾╲    ╱‾╲
   | ||||||||||||||||||||||||  ← тонкая структура (гармоники с шагом F0) — «высота»
   разделили → двигаем НЕЗАВИСИМО:
   • pitch_scale меняет шаг гармоник (высота).
   • formant_scale растягивает огибающую (тембр/«пол»).
```

💡 ⭐⭐ **Это и есть «реализм».** «Женский» голос ≈ pitch ↑ + formant ↑ (меньше тракт); «глубокий
мужской» ≈ pitch ↓ + formant ↓. Раздельный контроль формант через варпинг спектральной огибающей —
то, чего нет в наивном resample. LPC-подход дёшев и хорош для речи; кепстральный/true-envelope — точнее.

## ⭐⭐ Подход 2 — PSOLA (временной, для голоса)

```
   TD-PSOLA (Pitch-Synchronous Overlap-Add) — классика смены высоты ГОЛОСА, дёшево и качественно:

   1. ДЕТЕКТ ВЫСОТЫ (F0) → расставить «пич-метки» на каждом периоде голоса (для voiced участков).
   2. вырезать ГРЕЙНЫ (зёрна) длиной ~2 периода вокруг каждой метки (окно Ханна).
   3. СМЕНА ВЫСОТЫ: разложить грейны с НОВЫМ интервалом (ближе = выше F0, дальше = ниже),
      длительность сохраняется дублированием/пропуском грейнов.
      → форманты сохранены автоматически (форма волны грейна = отклик тракта неизменна).
   4. СМЕНА ФОРМАНТ: РЕСЭМПЛировать сами грейны (меняет форманты), сохраняя интервал (высоту).

   плюсы: низкая латентность (1-2 периода ~10-20мс), отлично для речи, дёшево по CPU.
   минусы: нужен надёжный F0; хуже на unvoiced (шипящие/шум) и полифонии → гибрид (PSOLA для voiced,
           простая обработка для unvoiced).
```

💡 ⭐ Для voice changer'а голоса **PSOLA часто предпочтительнее phase vocoder**: ниже латентность,
естественнее на речи, дешевле. Главная зависимость — **детект F0** (ниже). Многие готовые библиотеки
(WORLD-вокодер) делают высококачественный анализ-синтез голоса именно через разделение F0 / огибающей /
апериодичности.

## ⭐ Детект высоты (F0)

```
   нужен для PSOLA и для точного питч-контроля. реал-тайм методы:
   • АВТОКОРРЕЛЯЦИЯ — пик корреляции сигнала с собой со сдвигом = период.
   • YIN — разностная функция + кумулятивная нормализация + порог; точнее автокорреляции, дёшев.
   • RMVPE/CREPE (нейросетевые) — очень точны, но тяжёлые (для ML-пути, не для лёгкого DSP).

   псевдокод YIN (суть):
   d[τ] = Σ (x[i] - x[i+τ])²            # разностная функция
   d'[τ] = cumulative_mean_normalize(d) # нормализация
   F0 = sample_rate / argmin_below_threshold(d')
```

## 🎛️ Эффекты (поверх pitch/formant)

```
   РОБОТ:
   • RING MOD — out = in * sin(2π·f_c·t), f_c≈30-100Гц → металлический. простейший «робот». 1 строка.
   • ВОКОДЕР — огибающую входа по полосам наложить на КАРРИЕР (пила/шум) → классический «вокодерный» голос.

   РЕВЕРБ (пространство):
   • Schroeder/Freeverb — параллельные COMB-фильтры + последовательные ALLPASS. дёшево, классика.
   • FDN (feedback delay network) — качественнее, гибче.

   ЭКВАЛАЙЗЕР / ФОРМАНТНЫЕ ФИЛЬТРЫ:
   • BIQUAD (peaking/shelf/notch) — базовый кирпич. каскад биквадов = эквалайзер.
   • подчеркнуть/сдвинуть форманты пиковыми фильтрами на нужных частотах (доп. «характер»).

   ШУМОДАВ (gate/NR): резать тихий фон ниже порога; спектральное вычитание для шума.
   порядок цепочки: [шумодав] → [pitch/formant] → [эффекты: эквалайзер/робот/реверб] → выход.
```

💡 ⭐ Биквад-фильтр — универсальный кирпич (эквалайзер, форманты, фильтры). Робот дёшево = ring mod;
богаче = вокодер. Реверб — Freeverb для старта. Все эти эффекты — обычные DSP-блоки, одинаковые в
C++/Rust (отличается только язык).

## 🎚️ Пресеты (конкретные параметры)

```
   ПРЕСЕТ          pitch (полутоны)   formant         эффекты
   ───────         ────────────────   ─────────       ───────────────────────────
   Женский         +3 .. +5           +2 .. +4        лёгкий эквалайзер «ярче»
   Мужской глубокий −3 .. −6          −2 .. −4        лёгкий low-shelf буст
   Ребёнок         +6 .. +8           +5 .. +7        —
   Робот           0                  0               ring mod 50Гц / вокодер
   Монстр          −8 .. −12          −4 .. −6        лёгкий дисторшн + реверб
   Радио/телефон   0                  0               полосовой 300-3400Гц + лёгкий шум
   ⚠️ числа — стартовые; финал подбирается НА СЛУХ (как и игровые механики — итерациями).
```

## ⏱️ Бюджет латентности (важно для реального времени)

```
   общая задержка = захват + алгоритм (lookahead/окно) + вывод. цель < ~30-50мс.
   • размер блока B (256 @48кГц ≈ 5.3мс; 512 ≈ 10.7мс).
   • PHASE VOCODER: окно N большое (2048 ≈ 42.7мс) → существенная задержка + по hop'у. качество↑, латентность↑.
   • PSOLA: ~1-2 периода (≈10-20мс) → НИЖЕ латентность. ← плюс для разговора.

   trade-off: больше FFT → лучше частотное разрешение/качество, но больше задержка.
   для общения в Discord/игре → минимизируй (меньше блок, PSOLA/лёгкий вокодер).
   для записи/стрима (не диалог) → можно больше окно ради качества.
```

💡 ⭐⭐ Латентность — главный практический компромисс: **качество vs задержка**. Для живого разговора
(эхо мешает) держи < ~40мс → меньшие буферы, PSOLA. Для не-диалога (озвучка, запись) можно жертвовать
задержкой ради качества (большое окно, true-envelope). Это [тот же инженерный trade-off](../Senior/02-decisions/08-tradeoffs.md).

## ⚠️ Артефакты и борьба с ними

```
   • «phasiness» (металл/размазанность) phase vocoder → phase locking, меньше overlap, гибрид с PSOLA.
   • смазанные транзиенты (атаки «пш», «т») → детект транзиентов, сброс фаз на атаке.
   • «зубцы»/щелчки на стыках грейнов PSOLA → корректные пич-метки, окна Ханна, кроссфейд.
   • ДЕНОРМАЛИ в тихих хвостах реверба → flush-to-zero (CPU не тормозит).
   • искажение на unvoiced (шипящие) в PSOLA → отдельная ветка для unvoiced.
   • эхо/задержка в разговоре → уменьшай латентность (см. бюджет).
```

## 🗺️ Порядок реализации DSP-ядра

```
   1. STFT-каркас: анализ → (ничего) → синтез overlap-add. на выходе = вход без щелчков. ← фундамент.
   2. простой питч-шифт (phase vocoder или resample) — услышать эффект, измерить латентность.
   3. РАЗДЕЛЕНИЕ огибающей (LPC или кепстр) → сохранение/сдвиг формант. ← ядро реализма.
   4. (опц.) ветка PSOLA + детект F0 для голоса (ниже латентность/естественнее).
   5. эффекты: биквад-эквалайзер, ring mod (робот), Freeverb. цепочка.
   6. пресеты + подбор на слух. оптимизация (заранее аллоцировать FFT-планы, SIMD, flush-to-zero).
```

💡 Готовые библиотеки делают тяжёлую часть: **Rubber Band** (формант-сохраняющий питч), **WORLD**
(анализ-синтез голоса: F0+огибающая+апериодичность — очень качественно для голоса), **SoundTouch**.
Понимая алгоритмы выше, ты осознанно их применяешь и настраиваешь, а не «крутишь вслепую».
