# 🚀 Проект уровня 1 · «Почувствуй железо»

> Докажи себе экспериментами, что железо реально — измерь влияние кэша и предсказания ветвлений и
> объясни результаты. Это превратит теорию в интуицию, которая останется с тобой.

---

## 🎯 Что делаем

Серия маленьких бенчмарков, доказывающих ключевые эффекты железа, + отчёт с объяснениями. Язык —
C или C++ (прозрачнее всего), замеры — через таймер; на Linux — дополнительно `perf`.

```
   гипотеза → эксперимент → замер → объяснение через железо
```

---

## 📋 Эксперименты

### 1. Эффект кэша: последовательный vs случайный доступ
- Массив на ~100 МБ. Обойди (а) по порядку, (б) в случайном порядке (перемешанные индексы).
- Замерь время обоих. Ожидание: случайный в разы медленнее.
- Объясни через кэш-линии и промахи.

### 2. Эффект кэш-линий: разный «страйд»
- Проходи массив с шагом 1, 2, 4, 8, 16 элементов. Замерь.
- Найди, где скорость падает (около размера кэш-линии / 64 байта). Объясни.

### 3. Предсказание ветвлений: сортированный vs случайный
- Цикл с `if (data[i] > threshold) sum += data[i];` по большому массиву.
- Замерь на отсортированном и на случайном массиве. Объясни разницу через предсказание.

### 4. Массив vs связный список (локальность)
- Сумма N элементов: в массиве и в связном списке (узлы, разбросанные по памяти).
- Та же Big-O O(n) — замерь разницу. Объясни.

### 5. (⭐) Матрица: строки vs столбцы
- Обход/суммирование большой матрицы по строкам и по столбцам. Замерь, объясни.

---

## 📊 Отчёт

Для каждого эксперимента:
```
   • гипотеза (что ожидаешь и почему)
   • метод (как мерил, размеры, как избежал «оптимизации в ноль» компилятором)
   • результаты (числа/график)
   • объяснение через железо (кэш-линии, промахи, предсказание, локальность)
   • вывод для практики (как это применять в реальном коде)
```

## 🌟 Бонусы

- 🥚 Построй графики (время vs размер/страйд) — наглядно видно «ступеньки» уровней кэша.
- 🐣 (Linux) Подтверди `perf stat`: cache-misses и branch-misses коррелируют с замедлением?
- 🦅 Найди размеры своих L1/L2/L3 (по «ступенькам» на графике время-vs-размер массива).
- 🚀 Сравни `-O0` и `-O2` — как оптимизация меняет картину?

---

## ⚠️ Важно про замеры

- Прогревай (первый прогон «холодный»), усредняй несколько прогонов.
- Не дай компилятору выкинуть «бесполезный» цикл — используй результат (печать суммы/volatile).
- Меряй достаточно большие данные (больше кэша), чтобы эффект был виден.

## ✅ Критерии готовности

- [ ] Проведены эксперименты 1–4 (5 — бонус) с реальными замерами
- [ ] Для каждого — гипотеза, метод, результаты, объяснение через железо
- [ ] Объяснения опираются на кэш-линии/промахи/предсказание/локальность
- [ ] Есть выводы для практики (как писать быстрее)
- [ ] Замеры корректны (прогрев, усреднение, результат используется)
