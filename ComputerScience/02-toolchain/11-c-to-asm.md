# 11 · Из C в ассемблер 🖼️⭐⭐

> 🎯 **Цель блока:** научиться сопоставлять конструкции C с их ассемблером — увидеть СВОИМИ
> глазами, во что превращается код. Это закрепляет всё ядро трека.

---

## 📖 C — самый прозрачный язык для этого

```
   C близок к железу: его конструкции прямо ложатся на инструкции. поэтому C ↔ ассемблер —
   идеальная пара для понимания. (C++ добавляет абстракции, Python/JS вообще интерпретируются —
   там связь дальше.)
   рабочий инструмент: godbolt — C слева, ассемблер справа, строки связаны цветом.
```

💡 ⭐⭐ Это синтез: ты знаешь C (трек C), знаешь железо (Уровень 1), знаешь ассемблер (модуль 10).
Теперь соединяешь — видишь, как переменная становится регистром/стеком, цикл — переходом, функция —
call/ret. После этого C перестаёт быть «текстом» и становится «инструкциями, которыми ты управляешь».

---

## ⭐ Переменные → регистры и стек

```c
int f() {
    int a = 5;      // → положить 5 (в регистр или на стек [rbp-4])
    int b = a + 3;  // → взять a, прибавить 3
    return b;       // → результат в eax
}
```

```
   • простые локальные переменные → часто РЕГИСТРЫ (быстро); при нехватке → СТЕК ([rbp-N]).
   • на -O2 компилятор может вообще убрать переменные и вычислить результат сразу (модуль 21).
   • аргументы функции приходят в определённых регистрах (calling convention, модуль 18).
   • результат функции → в eax/rax.
```

---

## ⭐⭐ Управляющие конструкции

```c
// if
if (x > 0) y = 1; else y = 2;
//  → cmp x, 0 ; jle .else ; (y=1) ; jmp .end ; .else: (y=2) ; .end:

// цикл
for (int i = 0; i < n; i++) sum += a[i];
//  → инициализация i ; .loop: cmp i,n ; jge .done ; (sum += a[i]) ; inc i ; jmp .loop ; .done:

// вызов функции
int r = add(a, b);
//  → положить a,b в регистры аргументов ; call add ; результат из eax
```

💡 ⭐ Узнаёшь? Это паттерны из [модуля 10](10-assembly.md): `cmp`+`jXX` для условий, `jmp` назад для
цикла, `call`/`ret` для функций. Теперь видишь, как ИМЕННО твой C порождает их. Доступ к массиву
`a[i]` → вычисление адреса (`base + i*размер`) и загрузка из памяти.

---

## ⭐ Структуры, указатели, массивы

```
   • МАССИВ a[i]  → адрес = база + i * sizeof(элемент); инструкция загрузки по адресу (lea/mov [..]).
     видно, почему индексация O(1) и почему последовательный обход кэш-дружелюбен (модуль 08).
   • УКАЗАТЕЛЬ *p  → mov rax, [rp] (значение по адресу в rp); разыменование = обращение к памяти.
   • СТРУКТУРА s.field → база структуры + СМЕЩЕНИЕ поля (фиксированное, известно компилятору).
     отсюда padding/выравнивание (трек C) — поля по «круглым» смещениям для скорости.
```

> 🧭 Это делает наглядной [работу с памятью в C](../../C/02-memory/09-pointers.md): указатель —
> это адрес, разыменование — load из памяти, массив — арифметика адресов. В ассемблере видно буквально.

## ⭐⭐ Оптимизация удивляет (превью Уровня 4)

```c
int square(int x) { return x * x; }
```
```
   -O0:  несколько инструкций со стеком (буквальный перевод).
   -O2:  imul eax, edi, edi ; ret   — две инструкции, всё в регистрах. компилятор убрал лишнее.

   ещё ярче:
   int sum_to_n(){ int s=0; for(int i=1;i<=100;i++) s+=i; return s; }
   -O2:  mov eax, 5050 ; ret   — компилятор ВЫЧИСЛИЛ цикл на этапе компиляции! (constant folding)
```

💡 ⭐⭐ Это важнейший урок: **ассемблер на `-O2` может сильно отличаться от твоего кода** — компилятор
переписывает, убирает, вычисляет заранее. Поэтому: (1) не микрооптимизируй вслепую — компилятор
часто умнее; (2) смотри `-O0` чтобы понять логику, `-O2` чтобы понять реальную производительность.

---

## ⚠️ Ловушки

- ❌ Думать, что одна строка C = фиксированный ассемблер (зависит от оптимизации, контекста).
- ❌ Смотреть только `-O2` для понимания логики (там всё переписано) — начни с `-O0`.
- ❌ Микрооптимизировать то, что компилятор и так оптимизирует.
- ❌ Игнорировать, что `a[i]`/`*p`/`s.field` — это обращения к памяти (цена кэша).

## ✅ Упражнения (godbolt)

1. **Переменные.** Скомпилируй простую функцию на `-O0`, потом `-O2`. Как изменилось число
   инструкций? Куда делись переменные?
2. **Цикл → переходы.** Функция с циклом. Найди метку цикла и переход назад. Сопоставь со строками C.
3. **Массив.** `int g(int* a, int i){ return a[i]; }`. Найди вычисление адреса и загрузку. Как
   считается адрес?
4. **Магия -O2.** Сделай `sum_to_n` (цикл 1..100). Что выдаёт `-O2`? Почему компилятор так может?
