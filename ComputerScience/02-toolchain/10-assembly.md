# 10 · Язык ассемблера: читаем 🖼️⭐⭐

> 🎯 **Цель блока:** научиться **читать** ассемблер (не писать большие программы) — понимать, что
> делает каждая инструкция. Это «язык», на котором говорит CPU.

---

## 📖 Цель — читать, а не писать

```
   тебе НЕ нужно писать программы на ассемблере (это редко). нужно УМЕТЬ ЧИТАТЬ — чтобы:
   • понимать, во что компилируется твой код (godbolt).
   • отлаживать на низком уровне (gdb, дизассемблер).
   • видеть, что делает оптимизатор, и почему код быстрый/медленный.
   читать ассемблер проще, чем кажется: инструкции примитивны, их немного видов.
```

💡 ⭐⭐ Ассемблер выглядит страшно из-за объёма и незнакомых имён, но каждая отдельная инструкция —
**очень простая** (перемести, сложи, сравни, прыгни). Поняв десяток базовых, ты читаешь
большинство кода. Это навык «узнавания паттернов», а не зубрёжки.

---

## ⭐ Анатомия инструкции

```
   мнемоника   операнды
   mov         rax, 5        ; rax = 5  (положить 5 в регистр rax)
   add         rax, rbx      ; rax = rax + rbx
   ↑команда    ↑куда, ↑что

   ВНИМАНИЕ к синтаксису (две нотации!):
   • Intel:  mov rax, 5     (назначение СЛЕВА)  — godbolt по умолчанию, читаемее.
   • AT&T:   mov $5, %rax   (назначение СПРАВА, % у регистров, $ у констант) — gcc/objdump по умолчанию.
   смысл тот же, порядок операндов разный. выбери одну (Intel удобнее для чтения).
```

---

## ⭐⭐ Десяток инструкций, покрывающих почти всё

```
   ПЕРЕМЕЩЕНИЕ:
   mov   dst, src      — скопировать (регистр↔регистр↔память↔константа)
   lea   dst, [addr]   — вычислить адрес (часто и для арифметики)

   АРИФМЕТИКА/ЛОГИКА:
   add / sub / imul    — сложить / вычесть / умножить
   and / or / xor      — побитовые (xor rax,rax — обнулить rax, частый приём)
   inc / dec           — +1 / −1

   СРАВНЕНИЕ И ПЕРЕХОДЫ (основа if/циклов):
   cmp   a, b          — сравнить (выставить флаги)
   jmp   label         — безусловный переход
   je/jne/jg/jl/...    — условный переход (по флагам: равно/не равно/больше/меньше)

   ФУНКЦИИ (модуль 17):
   call  func          — вызвать функцию (положить адрес возврата на стек, прыгнуть)
   ret                 — вернуться (снять адрес со стека, прыгнуть туда)
   push / pop          — положить/снять со стека

   доступ к памяти: [rax], [rbp-8] — квадратные скобки = «значение по адресу».
```

💡 ⭐ Запомни **категории**, а не список: перемещение (mov/lea), арифметика (add/sub/imul/xor),
сравнение+переход (cmp/jXX — это if и циклы!), функции (call/ret/push/pop). 90% ассемблера — это они.

## ⭐ Как читать управляющие конструкции

```
   if (a > b) X(); else Y();
   →   cmp  a, b
       jle  .else        ; если a <= b → прыгнуть в .else
       <код X>
       jmp  .end
   .else:
       <код Y>
   .end:

   цикл for(i=0;i<n;i++)
   →   ... ; i=0
   .loop:
       cmp  i, n
       jge  .done        ; i>=n → выход
       <тело>
       inc  i
       jmp  .loop        ; назад
   .done:
```

💡 ⭐ Ключевой паттерн: **`cmp` + `jXX` = условие**, **`jmp` назад = цикл**, **`call`/`ret` =
функция**. Увидев их, ты «расшифровываешь» структуру высокоуровневого кода в ассемблере. Метки
(`.loop:`, `.end:`) — это адреса для переходов.

## 📖 Память и адресация

```
   [ ] = разыменование (значение по адресу):
   mov  rax, [rbp-8]     ; загрузить значение по адресу (rbp-8) в rax  (локальная переменная)
   mov  [rbx], rax       ; сохранить rax по адресу в rbx               (запись в память)
   часто увидишь [rbp-N] или [rsp+N] — это локальные переменные/аргументы на стеке (модуль 17).
   глобальные — по имени/адресу секции (модуль 16).
```

## ⚠️ Ловушки

- ❌ Пытаться «выучить весь ассемблер» — учись читать категории инструкций.
- ❌ Путать Intel и AT&T синтаксис (порядок операндов!) — выбери один.
- ❌ Игнорировать `[ ]` — это обращение к ПАМЯТИ (медленнее регистра).
- ❌ Не узнавать паттерны (cmp+jXX = if, jmp назад = цикл, call/ret = функция).
- ❌ Читать оптимизированный (`-O2`) ассемблер новичком — начни с `-O0`.

## ✅ Упражнения (godbolt, Intel-синтаксис)

1. **Базовое.** `int f(){ int a=5, b=3; return a+b; }` на `-O0`. Прочитай построчно: где 5, 3,
   сложение, возврат?
2. **if.** `int max(int a,int b){ return a>b?a:b; }`. Найди `cmp` и условный переход. Расшифруй логику.
3. **Цикл.** Функция с `for`-циклом суммирования. Найди метку цикла, `cmp`, переход назад (`jmp`).
4. **Память.** Функция с локальным массивом. Найди обращения `[rbp-…]` / `[rsp+…]`. Это стек.
