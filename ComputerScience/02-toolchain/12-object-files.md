# 12 · Объектные файлы и символы 🖼️⭐

> 🎯 **Цель блока:** понять, что внутри объектного файла (.o) и что такое символы — это ключ к
> пониманию линковки и «непонятных» ошибок сборки.

---

## 📖 Объектный файл — почти программа, но с «дырками»

```
   объектный файл (.o) = машинный код ОДНОГО исходника + метаданные для линковки. он ещё НЕ
   исполняемый, потому что:
   • ссылается на то, что определено в ДРУГИХ файлах (например, printf, твоя функция из другого .c).
   • адреса ещё не финальные (будут расставлены при линковке).

   что внутри:
   • СЕКЦИИ: .text (код), .data (инициализированные данные), .bss (нулевые/неинициализированные), .rodata (константы).
   • ТАБЛИЦА СИМВОЛОВ: имена функций/переменных — какие ОПРЕДЕЛЕНЫ здесь, какие НУЖНЫ извне.
   • РЕЛОКАЦИИ: «здесь подставь финальный адрес символа X» — инструкции для линковщика.
```

🖼️
```
   main.o:
   .text:  ...код main...  call <printf>   ← «дырка»: адрес printf пока неизвестен
   символы:  ОПРЕДЕЛЕНО: main
             НУЖНО (undefined): printf      ← разрешит линковщик (из libc)
   релокации: «адрес printf подставить вот сюда»
```

## ⭐ Символы: определённые и неопределённые

```
   СИМВОЛ — имя функции или глобальной переменной, по которому на них ссылаются.

   • ОПРЕДЕЛЁННЫЙ (defined) — здесь есть его реализация/значение (T в nm — в .text).
   • НЕОПРЕДЕЛЁННЫЙ (undefined) — используется, но определён ГДЕ-ТО ЕЩЁ (U в nm).
   • локальные vs глобальные — static-функция локальна (видна только в своём файле), обычная — глобальна.

   посмотреть: nm main.o
     T main      ← определён здесь (текст)
     U printf    ← неопределён (нужен извне)
```

💡 ⭐ Символы — это «точки соединения» между файлами. Линковщик (модуль 13) проходит по всем .o и
библиотекам и **сопоставляет**: каждый «нужен» (U) должен найти ровно один «определён» (T) где-то.
Не нашёл → `undefined reference`. Нашёл дважды → `multiple definition`.

## ⭐⭐ Объяснение «непонятных» ошибок линковки

```
   теперь типичные ошибки очевидны:

   ❌ "undefined reference to `foo`"
      = символ foo НУЖЕН (U), но НИГДЕ не определён (T).
      причины: забыл слинковать файл/библиотеку с foo; опечатка; не та сигнатура (C++ манглинг, ниже).

   ❌ "multiple definition of `bar`"
      = символ bar определён в ДВУХ .o (например, определение в заголовке, включённом дважды;
        глобальная переменная без extern в .h).
      лечение: определение — в ОДНОМ .c; в .h — только объявления (extern / прототипы).

   ❌ C++ "странные" имена в ошибках (_Z3fooi):
      = name mangling: C++ кодирует типы/перегрузки в имя символа. extern "C" отключает (для совместимости).
```

💡 ⭐⭐ Большинство страданий с линковкой сводятся к одному: **символ либо не найден, либо найден
дважды**. Понимая, что .o — это код + таблица символов, а линковщик их сопоставляет, ты
диагностируешь эти ошибки за секунды вместо «гугления наугад».

## 📖 Секции .text / .data / .bss / .rodata

```
   код и данные разложены по секциям (увидишь и в ELF, модуль 14):
   • .text   — машинный код (инструкции). только чтение + исполнение.
   • .rodata — константы, строковые литералы. только чтение.
   • .data   — глобальные/статические переменные С НАЧАЛЬНЫМ значением. чтение+запись.
   • .bss    — глобальные/статические БЕЗ значения (нулевые). не занимают места в файле (только размер).
   эта раскладка переедет в память процесса при запуске (модуль 16).
```

> 🧭 Это смыкается с [раскладкой памяти в C](../../C/02-memory/08-memory-model.md): глобальные в
> .data/.bss, код в .text, локальные на стеке (рантайм). Здесь видишь это на уровне файла.

## ⚠️ Ловушки

- ❌ Считать .o исполняемым (нужна линковка).
- ❌ Определять функции/переменные в заголовках (.h) → multiple definition.
- ❌ Забывать слинковать нужный файл/библиотеку → undefined reference.
- ❌ Не знать про C++ name mangling (странные имена в ошибках линковки).
- ❌ Путать объявление (extern/прототип) и определение (реализация).

## ✅ Упражнения

1. **nm.** Скомпилируй `main.c` (с вызовом printf) в `main.o` (`gcc -c`). Запусти `nm main.o`. Найди
   `T main` и `U printf`. Объясни.
2. **undefined reference.** Вызови функцию из другого файла, но не линкуй его. Получи ошибку.
   Слинкуй — пропала. Сопоставь с символами.
3. **multiple definition.** Определи глобальную переменную в .h, включи в два .c, слинкуй. Получи
   ошибку. Исправь через `extern`.
4. **Секции.** `objdump -h main.o` — посмотри секции (.text/.data/.bss/.rodata). Где твой код?
