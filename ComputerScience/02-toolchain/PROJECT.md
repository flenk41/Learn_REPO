# 🚀 Проект уровня 2 · «Разбери бинарник по косточкам»

> Капстоун ядра: возьми маленькую программу и проследи её путь от исходника до исполняемого файла,
> разбирая каждый артефакт. После этого сборка перестанет быть магией навсегда.

---

## 🎯 Что делаем

Берёшь крошечную программу на C (2 файла + использование стандартной библиотеки) и **документируешь
весь путь** `исходник → процесс`, разбирая каждый промежуточный артефакт инструментами.

```
   .c → препроцессор → ассемблер → объектник → линковка → ELF → (разбор каждого)
```

---

## 📋 Программа для разбора

Сделай так, чтобы было что разбирать:
```c
// utils.c  — отдельный файл (для линковки двух .o)
int add(int a, int b) { return a + b; }

// main.c
#include <stdio.h>
#define LABEL "result"
extern int add(int, int);
int main() {
    int s = add(2, 3);
    printf("%s: %d\n", LABEL, s);   // вызов библиотечной printf
    return 0;
}
```

---

## 📋 Этапы (каждый — артефакт + объяснение)

### 1. Препроцессор
- `gcc -E main.c` → найди, во что превратились `#include` и `#define LABEL`.

### 2. Компиляция → ассемблер
- `gcc -S main.c` и разбери `main.s`: найди вызов `add`, вызов `printf`, строку-литерал, возврат.
- Параллельно глянь в godbolt (`-O0` и `-O2`) — сравни.

### 3. Ассемблирование → объектные файлы
- `gcc -c main.c` и `gcc -c utils.c`. Для каждого: `nm` — какие символы определены/нужны?
- В `main.o`: `add` и `printf` — определены или нужны извне? Почему?

### 4. Линковка
- `gcc main.o utils.o -o prog`. Объясни: как линковщик связал `call add` и `call printf`?
- Эксперимент: слинкуй БЕЗ `utils.o` → получи `undefined reference to add`. Объясни.

### 5. Разбор ELF
- `file prog`, `readelf -h` (точка входа, архитектура), `readelf -S` (секции), `readelf -d` / `ldd`
  (нужные .so, динамический загрузчик).
- `objdump -d prog` — найди дизассемблированный `main` и `add`.

### 6. Карта бинарника
- Собери всё в одну «карту»: путь исходник→процесс, что на каждом шаге, ключевые символы и секции,
  какие библиотеки подгрузятся при запуске.

---

## 🌟 Бонусы

- 🥚 Сравни ассемблер `-O0` vs `-O2` для `main` — что изменилось?
- 🐣 Слинкуй статически (`-static`) и динамически — сравни размер и `ldd`. Сформулируй trade-off.
- 🦅 Найди в `readelf -d` интерпретатор (`ld-linux.so`) — это «загрузчик» (мост к Уровню 3).
- 🚀 Воспроизведи `multiple definition`, определив `add` ещё и в заголовке, включённом дважды.
