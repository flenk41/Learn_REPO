# 🏗️ Проект уровня 2 — Ядро симулятора (сбор → валюта)

> 🎯 **Цель:** оживить мир из уровня 1 — сделать главный цикл симулятора рабочим: игрок собирает
> ресурсы → копит валюту. Всё на сервере, по событиям.

---

## Что делаем

Превращаем статичную зону сбора в рабочую механику: монеты/кристаллы, которые собираются касанием и
дают валюту, плюс их периодический спавн. Это «сердце» симулятора (интерфейс и сохранение — на
уровне 3).

---

## Этапы

- [ ] **Хранилище валюты.** Серверный Script + таблица `playerCoins` (ключ — игрок). На `PlayerAdded`
      заводи запись `playerCoins[player] = 0`, на `PlayerRemoving` — убирай.
- [ ] **Заготовка ресурса.** Coin/Crystal (Model или Part) в ServerStorage/ReplicatedStorage как шаблон.
- [ ] **Спавн ресурсов.** Скрипт периодически (`task.wait`) клонирует заготовку в случайные точки зоны
      сбора (`:Clone()`, Position, Parent = workspace).
- [ ] **Сбор по касанию.** На каждой монете — `Touched` → проверка Humanoid → найти игрока
      (`Players:GetPlayerFromCharacter`) → `playerCoins[player] += 1` → `:Destroy()` монеты. С debounce!
- [ ] **Обратная связь.** Пока без GUI — печатай в Output текущую валюту игрока при сборе
      (`print(player.Name, playerCoins[player])`).
- [ ] **Чистота.** Используй ModuleScript для общих функций (например, форматирование числа). Код на
      сервере, осмысленные имена.

---

## 📦 Результат

1. Рабочий цикл: ресурсы спавнятся → игрок собирает касанием → валюта растёт (в таблице на сервере).
2. Корректная обработка событий: проверка Humanoid, debounce, привязка к нужному игроку.
3. Состояние живёт на сервере и переживает респаун игрока.
4. Хотя бы один ModuleScript с переиспользуемой функцией.

---

## 🌟 Усложнения

- Разные ресурсы с разной ценностью (монета +1, кристалл +5).
- Ограничь число монет на карте (не спавнить, если уже N штук) — для производительности.
- Эффект при сборе (смена цвета/звук — заглядывая в модуль 18).
- Случайные точки спавна в пределах зоны (через `math.random` и границы площадки).

---

## ✅ Критерии

- [ ] Сбор работает по событию (Touched + Humanoid + debounce)
- [ ] Валюта считается на сервере и привязана к нужному игроку
- [ ] Ресурсы спавнятся и удаляются корректно (без утечек)
- [ ] Код организован (сервер, ModuleScript, имена)
