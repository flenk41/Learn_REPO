# 11 · Сервисы, Script vs LocalScript 🖼️⭐⭐

> 🎯 **Цель блока:** понять сервисы (встроенные системы Roblox) и — критически важное — где код
> выполняется: на сервере или на клиенте. Путаница здесь рождает баги и дыры в безопасности.

---

## ⭐ Сервисы — встроенные системы

```
   СЕРВИС — глобальная система Roblox, доступная через game:GetService("Имя").
   частые сервисы:
   • Players          — игроки (PlayerAdded, список игроков).
   • RunService       — игровой цикл/кадры (Heartbeat, RenderStepped — модуль 12).
   • ReplicatedStorage — общее хранилище клиент↔сервер (RemoteEvents — модуль 15).
   • ServerScriptService / ServerStorage — серверная логика/хранилище.
   • DataStoreService — сохранение данных (модуль 16).
   • UserInputService — ввод с клавиатуры/мыши (на клиенте).
   • TweenService     — плавные анимации свойств (модуль 13).
   • MarketplaceService — покупки/геймпассы (модуль 21).

   правильно: local Players = game:GetService("Players")   -- надёжнее, чем game.Players
```

💡 ⭐ Бери сервисы через **`game:GetService("Имя")`** (а не `game.Players`) — это канонично и
надёжно. Сервисы — твой API к возможностям Roblox: игроки, ввод, анимации, сохранение, покупки. Каждую
новую задачу начинай с вопроса «какой сервис за это отвечает?».

---

## ⭐⭐ Script vs LocalScript — где выполняется код

```
   КЛЮЧЕВОЕ различие в Roblox: код бежит на СЕРВЕРЕ или на КЛИЕНТЕ.

   SERVER SCRIPT (Script):
   • выполняется на СЕРВЕРЕ Roblox (один на всю игру, авторитетный).
   • видит всех игроков, владеет «правдой» (очки, данные, мир).
   • место: ServerScriptService (или Workspace).
   • сюда — ВАЖНУЮ логику: начисление очков, сохранение, проверки.

   LOCAL SCRIPT (LocalScript):
   • выполняется на КЛИЕНТЕ (на устройстве КАЖДОГО игрока, своя копия).
   • видит ввод, камеру, GUI этого игрока; НЕ авторитетен.
   • место: StarterPlayerScripts, StarterGui, внутри GUI.
   • сюда — интерфейс, ввод, локальные эффекты.
```

🖼️
```
              СЕРВЕР (один, авторитетный)              КЛИЕНТЫ (у каждого свой)
   ┌────────────────────────────────┐         ┌──────────────┐  ┌──────────────┐
   │ Script в ServerScriptService   │         │ LocalScript  │  │ LocalScript  │
   │ • начисляет очки               │ ◄─────► │ • GUI, ввод  │  │ • GUI, ввод  │
   │ • хранит данные                │ Remote  │ • эффекты    │  │ • эффекты    │
   │ • проверяет (анти-чит)         │ Events  │  игрок A     │  │  игрок B     │
   └────────────────────────────────┘         └──────────────┘  └──────────────┘
                 «правда» здесь                  это можно подделать (эксплойт)
```

💡 ⭐⭐ **Сервер — источник правды; клиент — недоверенный.** Важные решения (начислить валюту, выдать
предмет, сохранить прогресс) делает только СЕРВЕР. Клиент может быть взломан игроком (модуль 19),
поэтому ему нельзя верить. Это фундамент честной игры. Связь сервер↔клиент — через RemoteEvents
(модуль 15).

## ⭐ ModuleScript — переиспользуемый код

```lua
   -- ModuleScript — не запускается сам; хранит код для других (как библиотека/модуль).
   -- ReplicatedStorage/GameUtils (ModuleScript):
   local GameUtils = {}
   function GameUtils.formatNumber(n)
       return tostring(n) .. " монет"
   end
   return GameUtils                              -- модуль ВОЗВРАЩАЕТ таблицу

   -- использование в другом скрипте:
   local GameUtils = require(game.ReplicatedStorage:WaitForChild("GameUtils"))
   print(GameUtils.formatNumber(50))            -- "50 монет"
```

💡 ⭐ ModuleScript = способ делить код на файлы и переиспользовать (как
[модули в C/Rust](../../Rust/01-basics/07b-multiple-files.md)): общая логика в ModuleScript →
`require()` из server- и local-скриптов. Не копируй код — выноси в модуль.

## ⚠️ Ловушки

- ❌❌ Делать важную логику (очки, выдача, сохранение) в LocalScript → игрок легко подделает.
- ❌ Ждать, что LocalScript увидят другие игроки (он локальный, у каждого свой).
- ❌ Класть Script не туда (LocalScript в ServerScriptService не запустится, и наоборот).
- ❌ `game.Players` вместо `game:GetService("Players")` (работает, но не канонично/надёжно).
- ❌ Копировать код между скриптами вместо ModuleScript.
