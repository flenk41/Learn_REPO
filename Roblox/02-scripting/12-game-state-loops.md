# 12 · Состояние и игровой цикл 🖼️⭐

> 🎯 **Цель блока:** хранить состояние игры (очки, фаза, таймеры) и обновлять его во времени —
> игровой цикл. Это «пульс» игры между событиями.

---

## ⭐ Состояние игры

```lua
   СОСТОЯНИЕ — данные, описывающие игру СЕЙЧАС. живёт в таблицах (на сервере!).

   -- очки игроков (по ключу-игроку):
   local playerCoins = {}                        -- { [player] = число }

   -- глобальное состояние раунда:
   local gameState = {
       phase = "waiting",                        -- waiting / playing / ended
       timeLeft = 60,
       round = 1,
   }
```

💡 ⭐ Состояние храни **на сервере** (он авторитетен, модуль 11) и в таблицах. Привязку «игрок →
данные» делают таблицей с ключом-игроком или через `leaderstats`/значения внутри Player (модуль 17).
Не теряй состояние при респауне — вешай его на Player, а не Character (модуль 06).

## ⭐⭐ Время и паузы: task.wait

```lua
   -- task.wait(сек) — пауза, НЕ блокируя весь движок (другие скрипты/события идут параллельно).
   print("старт")
   task.wait(3)                                  -- ждать 3 секунды
   print("прошло 3 секунды")

   -- таймер раунда:
   for t = 10, 0, -1 do                          -- обратный отсчёт 10..0
       gameState.timeLeft = t
       print("осталось " .. t)
       task.wait(1)
   end
   print("Раунд окончен!")
```

💡 ⭐⭐ Используй **`task.wait()`**, а не старый `wait()` (новый точнее и эффективнее). `task.wait`
приостанавливает ТОЛЬКО текущую «нить», не замораживая игру — события и другие скрипты продолжают
работать. Так делают таймеры, отсчёты, периодические действия.

## ⭐ Игровой цикл (game loop)

```lua
   -- ВАРИАНТ 1: бесконечный цикл раундов (через wait — для крупного ритма игры):
   while true do
       gameState.phase = "waiting"
       task.wait(5)                              -- ждём игроков
       gameState.phase = "playing"
       -- ...раунд: отсчёт, спавн ресурсов...
       task.wait(60)
       gameState.phase = "ended"
       -- ...подвести итоги, наградить...
       task.wait(3)
   end

   -- ВАРИАНТ 2: каждый кадр — через RunService (для плавного движения/проверок):
   local RunService = game:GetService("RunService")
   RunService.Heartbeat:Connect(function(deltaTime)
       -- deltaTime — сколько прошло с прошлого кадра (для плавности)
       -- ...двигать платформы, проверять условия...
   end)
```

🖼️
```
   два «пульса» игры:

   КРУПНЫЙ ритм (раунды/фазы):   while true do ... task.wait(N) ... end
                                  waiting → playing → ended → повтор

   КАЖДЫЙ КАДР (плавность):       RunService.Heartbeat:Connect(dt → обновить)
                                  ~60 раз/сек: движение, физика-логика, проверки
```

💡 ⭐ Два инструмента: `while ... task.wait()` — для **крупного ритма** (фазы раунда, периодический
спавн), `RunService.Heartbeat` — для **покадровых** обновлений (плавное движение, постоянные проверки).
Не делай тяжёлую работу каждый кадр — это бьёт по FPS (модуль 20).

---

## 📖 deltaTime — независимость от FPS

```
   Heartbeat даёт deltaTime — секунды с прошлого кадра. используй его для движения:
   position = position + speed * deltaTime       -- скорость в «единицах в СЕКУНДУ», не в кадр
   → на 30 и 120 FPS объект движется ОДИНАКОВО быстро. без deltaTime — на быстром ПК всё летает.
```
