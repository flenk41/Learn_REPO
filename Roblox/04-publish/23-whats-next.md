# 23 · Куда расти дальше 🖼️

> 🎯 **Цель блока:** увидеть, куда вести навыки после Roblox — вглубь Roblox-разработки и вширь, в
> «большой» геймдев и программирование. Roblox — отличная ступень, а не потолок.

---

## ⭐ Вглубь Roblox

```
   стать сильным Roblox-разработчиком:
   • ЧИСТЫЙ КОД: ModuleScript'ы, разделение клиент/сервер, паттерны (например, single-script
     architecture), типизированный Luau.
   • продвинутые системы: инвентарь, торговля, матчмейкинг, процедурная генерация, сложный AI.
   • инструменты сообщества: Rojo (разработка во внешнем редакторе + Git!), Wally (пакеты),
     тестовые фреймворки. → твой код в Git (трек 🌳) и в нормальной IDE.
   • работа в команде: дизайнер + скриптер + билдер; Roblox-студии нанимают.
```

💡 ⭐ Профессиональная Roblox-разработка — это уже «настоящая» инженерия: модульный код, Git
([трек Git](../../Git/README.md)), внешние редакторы (Rojo), команды. Многие зарабатывают на жизнь
играми Roblox. Применяй сюда чистый код и архитектуру из других треков курса.

---

## ⭐ Вширь: большой геймдев

```
   Roblox дал базу (циклы, события, клиент-сервер, физика, GUI). дальше — «настоящие» движки:
   • UNITY (C#) — самый популярный движок: 2D/3D, мобайл/ПК/консоли. ближайший шаг.
   • UNREAL (C++/Blueprints) — мощная графика, крупные проекты.
   • GODOT (GDScript/C#) — открытый, лёгкий, дружелюбный.
   • свой движок — для понимания нутра (см. капстоун: [игровой движок/ECS](../../Capstone/04-language/22-game-engine.md)).

   концепции переносятся: игровой цикл, события, состояние, сцена/объекты, клиент-сервер — везде те же.
```

💡 ⭐ То, что ты понял в Roblox (event-driven, game loop, клиент-сервер, состояние), — **универсально**.
Перейти в Unity/Godot станет легче: меняется синтаксис и инструменты, идеи остаются. Roblox — лучший
способ «попробовать геймдев и не бросить» до перехода в большие движки.

## ⭐ Вширь: программирование вообще

```
   Luau дал основы программирования. укрепи фундамент — и откроются любые направления:
   • язык глубже: [Python](../../Python/README.md) (близок по духу), потом системные C/C++/Rust.
   • как устроен компьютер: [Computer Science](../../ComputerScience/README.md).
   • структуры данных и алгоритмы: [Алгоритмы](../../Algorithms/README.md) — фундамент любой игры/программы.
   • инструменты профи: [Git и код-ревью](../../Git/README.md), [базы данных](../../Database/README.md).
   геймдев — отличная мотивация, но программирование шире игр (веб, ИИ, системы, данные).
```

> 🧭 Roblox показал, что программировать — интересно и по силам. Дальше выбирай: углубляться в геймдев
> (Unity/Unreal) или расширять программирование (языки, CS, алгоритмы) — оба пути открыты этим курсом.

## 📖 Главное, что ты унёс из трека

```
   ✅ событийная модель (реагировать, а не опрашивать).
   ✅ клиент vs сервер и «не доверяй клиенту» — фундамент сетевых приложений (не только игр!).
   ✅ состояние, игровой цикл, сохранение данных.
   ✅ цикл «сделай → опубликуй → измерь → улучши».
   ✅ ответственность: безопасность игроков, честная монетизация.
   это переносимые навыки — они работают далеко за пределами Roblox.
```
