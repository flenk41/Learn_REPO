# 21 · Монетизация (Game Passes, Robux) 🖼️⭐

> 🎯 **Цель блока:** понять, как игры Roblox зарабатывают — Game Passes и Developer Products за Robux —
> и как делать это честно.

> ⚠️ **Ответственно:** монетизация связана с **реальными деньгами**. Robux покупаются за деньги и
> могут выводиться (программа Developer Exchange) при выполнении условий Roblox. Если ты
> **несовершеннолетний** — покупки и любые выплаты обсуждай с родителями и следуй правилам и возрастным
> ограничениям Roblox. Это образовательный обзор, а не финансовый совет.

---

## 📖 Robux и экономика Roblox

```
   ROBUX — внутренняя валюта Roblox. игроки покупают её за реальные деньги и тратят в играх/на предметы.
   разработчик зарабатывает Robux, когда игроки что-то покупают в его игре.
   накопленные Robux можно (при условиях: возраст, верификация, минимум) обменять на реальные деньги
   через DevEx (Developer Exchange). Roblox берёт комиссию.
```

💡 Монетизация в Roblox реальна, но это не «лёгкие деньги»: зарабатывают единицы игр из миллионов, и
почти всё решает то, насколько игра **удерживает** игроков (модуль 22). Сначала делай хорошую игру —
монетизация прикладывается к качеству, а не наоборот.

## ⭐ Game Pass vs Developer Product

```
   GAME PASS — покупается ОДИН РАЗ, даёт постоянное преимущество/доступ:
   • примеры: «×2 валюты навсегда», VIP-зона, особый скин, больше слотов.
   • проверка владения: MarketplaceService:UserOwnsGamePassAsync(userId, gamePassId).

   DEVELOPER PRODUCT — покупается МНОГОКРАТНО (расходуемое):
   • примеры: пачка валюты, бустер на время, возрождение.
   • покупка: MarketplaceService:PromptProductPurchase(player, productId).
   • выдача: обработчик ProcessReceipt (ОБЯЗАН вернуть статус — иначе повторное списание!).

   создаются в настройках игры на сайте Roblox (Monetization), получают ID.
```

💡 ⭐ Простое правило: **постоянное преимущество → Game Pass** (купил раз — есть всегда); **расходуемое
→ Developer Product** (покупается снова и снова). Для симулятора типичны: Game Pass «×2 монет» и
Developer Product «пачка кристаллов».

## ⭐⭐ Выдача покупки — на сервере и аккуратно

```lua
   -- Game Pass: проверь владение (на сервере) и дай бонус.
   local MarketplaceService = game:GetService("MarketplaceService")
   local PASS_ID = 0000000   -- id твоего геймпасса

   local function giveBonus(player)
       local ok, owns = pcall(function()
           return MarketplaceService:UserOwnsGamePassAsync(player.UserId, PASS_ID)
       end)
       if ok and owns then
           -- например, множитель сбора ×2 (на сервере!)
       end
   end

   -- Developer Product: ProcessReceipt ДОЛЖЕН вернуть статус.
   MarketplaceService.ProcessReceipt = function(info)
       -- выдать товар игроку по info.ProductId ...
       return Enum.ProductPurchaseDecision.PurchaseGranted   -- обязательно вернуть!
   end
```

💡 ⭐⭐ Выдачу покупок делает **сервер** (как всю экономику — модули 15, 19), с `pcall` (это сетевые
вызовы). Для Developer Product критично правильно вернуть статус из `ProcessReceipt` — иначе Roblox
посчитает покупку невыполненной и **спишет/выдаст повторно**. Проверяй владение геймпассом на сервере,
не верь клиенту.

## 📖 Честная монетизация

```
   ✅ ХОРОШО: косметика, ускорители, удобство, поддержка разработчика. игра ИГРАБЕЛЬНА без покупок.
   ❌ ПЛОХО/рискованно: «pay-to-win», давление на детей, обман, скрытые списания, лутбоксы-азарт.
   Roblox имеет ПРАВИЛА (в т.ч. о монетизации и защите несовершеннолетних) — нарушение → бан/удаление.
   уважай игрока: монетизация — за ценность, а не за манипуляцию.
```

> 🧭 Тёмные паттерны и давление — это про этику (см. [соц. инженерию/манипуляции](../../SocialEng/02-psychology/12-manipulation-vs-persuasion.md)).
> Делай честно: ценность за деньги, игра полноценна и без них.
