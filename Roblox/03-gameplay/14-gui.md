# 14 · GUI: интерфейс игрока 🖼️⭐

> 🎯 **Цель блока:** делать экранный интерфейс — счётчик валюты, кнопки магазина, меню. GUI — это то,
> через что игрок видит свой прогресс и взаимодействует с игрой.

---

## ⭐ Из чего состоит GUI

```
   2D-интерфейс на экране игрока строится из объектов в StarterGui:
   • ScreenGui      — контейнер всего интерфейса (выдаётся каждому игроку).
   • TextLabel      — текст (счёт, надписи). не кликается.
   • TextButton     — кнопка с текстом (кликается).
   • ImageLabel / ImageButton — картинка / кликабельная картинка (иконки).
   • Frame          — панель/контейнер (фон меню, группировка).

   StarterGui → копируется каждому игроку в PlayerGui при спавне. правишь в StarterGui — видят все.
```

🖼️
```
   StarterGui
   └── ScreenGui
       ├── CoinLabel (TextLabel)     "💰 0"          ← счётчик валюты (верх экрана)
       └── ShopFrame (Frame)                          ← панель магазина
           ├── Title (TextLabel)     "Магазин"
           └── BuySpeed (TextButton) "Скорость +"     ← кнопка покупки апгрейда
```

💡 ⭐ GUI кладут в **StarterGui** — оттуда он копируется каждому игроку. Базовый набор симулятора:
`TextLabel` для счётчика валюты + `Frame` с `TextButton`-кнопками магазина апгрейдов. Это лицо игры.

## ⭐ Позиционирование: UDim2

```
   размер и позиция GUI задаются в UDim2 = {Scale, Offset} по X и Y:
   • Scale  — доля экрана (0..1): 0.5 = половина. адаптируется под любой экран.
   • Offset — пиксели (фикс. размер).

   UDim2.new(scaleX, offsetX, scaleY, offsetY)
   Position = UDim2.new(0.5, 0, 0, 10)     -- по центру X, 10px от верха
   Size     = UDim2.new(0, 200, 0, 50)     -- 200×50 пикселей
   AnchorPoint = Vector2.new(0.5, 0)       -- точка привязки (центр-верх)

   совет: используй Scale для адаптивности (телефоны/планшеты/ПК — разные экраны).
```

💡 ⭐ Предпочитай **Scale** (доля экрана) пикселям — тогда интерфейс хорошо смотрится и на телефоне, и
на мониторе (большинство играют с телефона!). `AnchorPoint` задаёт, какой точкой элемент «цепляется» за
свою позицию (центр/угол).

## ⭐⭐ Кнопки и обновление текста (LocalScript)

```lua
   -- GUI — это КЛИЕНТ: управляется LocalScript (внутри ScreenGui/StarterPlayerScripts).
   local button = script.Parent.ShopFrame.BuySpeed
   local coinLabel = script.Parent.CoinLabel

   -- реакция на нажатие:
   button.MouseButton1Click:Connect(function()
       print("Игрок нажал «купить скорость»")
       -- ВАЖНО: саму покупку выполняет СЕРВЕР (модуль 15)! здесь только сигнал.
   end)

   -- обновить текст счётчика:
   coinLabel.Text = "💰 " .. tostring(coins)
```

💡 ⭐⭐ GUI и нажатия живут на **клиенте** (LocalScript): кнопка реагирует через `MouseButton1Click`.
Но **сама покупка/начисление — на сервере** (модуль 15): клиент лишь шлёт сигнал «хочу купить», а
сервер проверяет валюту и выдаёт. Никогда не «выдавай апгрейд» прямо в LocalScript — это подделают.

## 📖 Удобство и адаптивность

```
   • UIListLayout / UIGridLayout — авто-расстановка кнопок (не вручную по координатам).
   • UICorner — скруглённые углы; UIPadding — отступы; UIStroke — обводка.
   • UIScale / ограничения — подгонка под размер экрана.
   • TweenService (модуль 13) — плавно показывать/прятать меню.
   красивый, адаптивный GUI сильно повышает ощущение качества игры.
```

## ⚠️ Ловушки

- ❌❌ Начислять валюту/выдавать апгрейд прямо в LocalScript по нажатию (клиент подделает — модуль 15/19).
- ❌ Только Offset (пиксели) → на телефоне всё разъезжается; используй Scale.
- ❌ GUI не в StarterGui → не выдаётся игрокам.
- ❌ Расставлять кнопки вручную вместо UIListLayout/UIGridLayout.
- ❌ Управлять GUI серверным Script (GUI — клиентский, нужен LocalScript).
