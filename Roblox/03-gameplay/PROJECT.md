# 🏗️ Проект уровня 3 — Полный цикл симулятора

> 🎯 **Цель:** довести симулятор до настоящей играбельной петли: сбор → валюта (видимая) → магазин
> апгрейдов (через сервер) → ускорение → сохранение прогресса. Это «готовая в ядре» игра.

---

## Что собираем

Полный геймплейный цикл симулятора, объединяющий всё с уровней 2-3: интерфейс, клиент-сервер,
сохранение, прогрессию. После этого уровня у тебя — играбельный симулятор, который не стыдно показать.

---

## Этапы

- [ ] **leaderstats.** На `PlayerAdded` — папка `leaderstats` с `Coins` (IntValue) и уровнями апгрейдов.
- [ ] **Сбор → валюта.** Монеты (модуль 10/12) при сборе увеличивают `Coins.Value` (видно в Tab).
- [ ] **GUI.** Счётчик валюты на экране (TextLabel, обновляется) + магазин (Frame с кнопками апгрейдов).
- [ ] **Магазин через сервер.** Кнопка → RemoteEvent `FireServer` → сервер **проверяет валюту**,
      списывает, повышает уровень апгрейда, применяет эффект (WalkSpeed/множитель). Сервер шлёт
      `FireClient` для обновления GUI.
- [ ] **Минимум 2 апгрейда.** Например, «скорость» (быстрее бегать) и «множитель» (×2 за коин).
      Цена растёт с уровнем.
- [ ] **Сохранение.** DataStore: валюта + уровни апгрейдов (таблицей, `pcall`, ключ по UserId,
      `BindToClose`). Эффекты переприменяются после респауна (`CharacterAdded`).
- [ ] **Проверка цикла.** Собрал → купил → быстрее собираешь → вышел → зашёл → всё на месте.

---

## 📦 Результат

1. Играбельный симулятор с полной петлёй (сбор → валюта → магазин → апгрейд → ускорение → сохранение).
2. Вся экономика на **сервере** с проверками (клиент только просит/отображает).
3. Прогресс **сохраняется** между сессиями и переживает респаун.
4. Понятный GUI: видно валюту и магазин.

---

## 🌟 Усложнения

- Множитель и «вместимость рюкзака» (носишь N → сдаёшь на базе).
- Вторая зона сбора, открываемая за валюту (прогресс по миру).
- Глобальный топ игроков (OrderedDataStore) на доске в мире.
- Звук/эффект при сборе и покупке (модуль 18).
- Система rebirth (перерождение ради постоянного бонуса).

---

## ✅ Критерии

- [ ] Полная петля симулятора работает от начала до конца
- [ ] Экономика на сервере, клиент не может подделать валюту/апгрейды
- [ ] Прогресс сохраняется (DataStore) и переживает респаун/сессию
- [ ] GUI отображает валюту и магазин, обновляется корректно
