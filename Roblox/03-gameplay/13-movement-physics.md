# 13 · Движение и физика 🖼️⭐

> 🎯 **Цель блока:** двигать объекты и персонажей из кода — телепорт, плавные перемещения (Tween),
> физические силы. Движение оживляет мир: двери, платформы, ускорение игрока.

---

## ⭐ CFrame и Position — куда ставить

```lua
   local part = workspace.Door

   -- мгновенно переместить (телепорт):
   part.Position = Vector3.new(0, 10, 0)               -- только позиция
   part.CFrame = CFrame.new(0, 10, 0)                  -- позиция + поворот
   part.CFrame = part.CFrame * CFrame.Angles(0, math.rad(90), 0)   -- повернуть на 90° вокруг Y

   -- ВАЖНО: для движения объект обычно Anchored=true (двигаешь сам, физика не мешает).
```

💡 ⭐ `Position` — только координаты; `CFrame` — координаты + поворот вместе (мощнее, для дверей,
поворотов, точного размещения). Телепортируя якорный (Anchored) объект через CFrame, ты полностью
контролируешь его — физика не вмешивается. `math.rad()` переводит градусы в радианы (углы в CFrame — в радианах).

## ⭐⭐ Плавное движение: TweenService

```lua
   -- TWEEN — плавная анимация свойства из A в B за время. красиво двигать двери/платформы/GUI.
   local TweenService = game:GetService("TweenService")
   local door = workspace.Door

   local info = TweenInfo.new(
       1,                                 -- длительность, сек
       Enum.EasingStyle.Quad,             -- характер (плавность)
       Enum.EasingDirection.Out
   )
   local goal = { Position = Vector3.new(0, 10, 0) }   -- куда (целевые свойства)

   local tween = TweenService:Create(door, info, goal)
   tween:Play()                            -- запустить плавное движение
```

💡 ⭐⭐ **TweenService** — главный инструмент красивого движения: вместо рывка объект плавно «едет» в
целевое значение за заданное время. Им двигают двери, платформы-лифты, всплывающий GUI, плавную смену
цвета/прозрачности. Гораздо чище, чем двигать вручную по кадрам.

## ⭐ Управление персонажем

```lua
   -- влияем на игрока через Humanoid (модуль 06):
   local character = player.Character
   local humanoid = character:WaitForChild("Humanoid")

   humanoid.WalkSpeed = 32                 -- ускорить (апгрейд скорости в симуляторе!)
   humanoid.JumpPower = 75                 -- выше прыжок
   humanoid.Health = humanoid.Health - 10  -- урон

   -- телепорт персонажа (через опорную часть):
   character:MoveTo(Vector3.new(0, 5, 0))               -- безопасный телепорт
   -- или: character.HumanoidRootPart.CFrame = CFrame.new(...)
```

💡 ⭐ Скорость/прыжок/здоровье игрока меняются через **Humanoid** — это типичные апгрейды симулятора
(быстрее бегать = быстрее собирать). Телепорт персонажа — через `:MoveTo()` или `HumanoidRootPart.CFrame`
(не за случайную часть тела).

## 📖 Физические движения (по желанию)

```
   для «физичного» поведения (не Anchored объекты):
   • применять силы/скорости — современные Constraints: LinearVelocity, AngularVelocity,
     AlignPosition, VectorForce (вместо устаревших BodyMover'ов).
   • для большинства игр (двери, платформы, лифты) хватает Anchored + TweenService/CFrame — проще и надёжнее.
   физику с силами бери, когда нужен реальный «вес/импульс» (мячи, транспорт).
```
