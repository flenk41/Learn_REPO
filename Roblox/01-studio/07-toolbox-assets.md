# 07 · Toolbox, ассеты, импорт 🖼️

> 🎯 **Цель блока:** использовать готовые ассеты (модели, звуки, картинки) с умом — ускорять работу,
> но не тащить в игру мусор и небезопасный код.

---

## ⭐ Toolbox — библиотека ассетов

```
   TOOLBOX (View → Toolbox) — каталог моделей, изображений, звуков, плагинов от Roblox и сообщества.
   вкладки: Marketplace (всё подряд), Inventory (твои), Creations (твои опубликованные), Recent.

   как использовать: найди модель → перетащи/кликни → она вставится в Workspace.
   так быстро наполняешь мир: деревья, мебель, декор, эффекты — не лепить руками.
```

💡 ⭐ Toolbox экономит часы: не моделируй дерево с нуля — возьми готовое. Но это палка о двух концах:
качество разное, а в моделях бывают **скрытые скрипты** (см. ниже). Бери у проверенных авторов (Roblox,
с высоким рейтингом), осматривай содержимое.

---

## ⭐⭐ Безопасность ассетов

```
   ⚠️⚠️ модели из Toolbox могут содержать ВРЕДОНОСНЫЕ СКРИПТЫ:
   • вставил красивую модель → внутри Script, который ломает игру, спамит, крадёт доступ, ставит бэкдор.
   • особенно опасны бесплатные модели «всё в одном» от ноунейм-авторов.

   ✅ правила безопасности:
   • после вставки РАСКРОЙ модель в Explorer → проверь, нет ли неожиданных Script/LocalScript.
   • удали любые скрипты, которых ты не ждёшь.
   • используй плагин/поиск по дереву на Script'ы перед публикацией.
   • бери ассеты у официальных/проверенных авторов; для своего важного кода — пиши сам.
```

💡 ⭐⭐ Главная опасность Toolbox — **чужой код внутри моделей**. Это та же мысль, что в треке
безопасности про [supply chain / чужие зависимости](../../Security/03-defensive-code/19-secrets-dependencies.md):
не доверяй слепо чужому коду в своём проекте. Всегда осматривай вставленное на скрытые скрипты.

## ⭐ Свои ассеты: импорт

```
   • ИЗОБРАЖЕНИЯ (иконки, текстуры) — Asset Manager → Import (загружаются на Roblox, получают ID).
   • ЗВУКИ — аналогично; используешь по AudioId/SoundId (модуль 18 — эффекты).
   • МЕШИ (3D-модели из Blender и т.п.) — MeshPart / import (.fbx/.obj).
   ⚠️ загружаемый контент проходит МОДЕРАЦИЮ Roblox; должен быть твоим/разрешённым (авторские права).

   Asset Manager (View) — управляет твоими загруженными ассетами и их ID.
```

💡 ⭐ Свои картинки/иконки для GUI (модуль 14) загружай через Asset Manager — получишь `rbxassetid://...`,
который вставляешь в свойства (Image, SoundId). Уважай авторские права: не загружай чужое.

---

## 📖 Когда брать, а когда делать самому

```
   ✅ брать из Toolbox: декор, природа, мебель, готовые звуки/эффекты (не критично, экономит время).
   ✅ делать самому: игровую ЛОГИКУ (скрипты!), ключевые механики, всё, что влияет на честность игры.
   причина: чужой скрипт = чужой контроль и риск. декоративный меш — безопасно; чужой «движок игры» — нет.
```

## ⚠️ Ловушки

- ❌❌ Вставить модель и не проверить на скрытые скрипты → заражённая/сломанная игра.
- ❌ Строить игру целиком на чужих «бесплатных моделях с логикой» (теряешь контроль и безопасность).
- ❌ Загружать чужой контент (нарушение авторских прав, бан/модерация).
- ❌ Тащить тяжёлые/детализированные ассеты пачками → просадка производительности (модуль 20).
