# 03 · Parts: свойства, материалы, якорь 🖼️⭐⭐

> 🎯 **Цель блока:** глубже освоить Parts — главный строительный материал Roblox — и их свойства,
> включая физику (Anchored, CanCollide) и внешний вид.

---

## ⭐⭐ Part и его ключевые свойства

```
   PART — деталь, кирпичик мира. почти всё (платформы, стены, монеты, кнопки) — это Parts.
   формы: Block, Sphere, Cylinder, Wedge (свойство Shape у Part, или отдельные типы).

   ВНЕШНИЙ ВИД:
   • Size (Vector3 X,Y,Z) — размеры.   • Color / BrickColor — цвет.
   • Material — Plastic, Wood, Metal, Neon (светится), Glass, Grass...
   • Transparency (0..1) — прозрачность.   • Reflectance — отражение.

   ФИЗИКА:
   • Anchored (true/false) — закреплён ли. true = висит на месте, физика не двигает.
   • CanCollide (true/false) — сталкивается ли с другими (false = сквозь него проходят).
   • Massless, CustomPhysicalProperties — тонкая настройка (позже).
```

🖼️
```
   Anchored × CanCollide — четыре поведения детали:

   Anchored=true,  CanCollide=true   → стена/пол: стоит, не пройти. (платформы)
   Anchored=true,  CanCollide=false  → декор/триггер: стоит, проходим насквозь. (зоны)
   Anchored=false, CanCollide=true   → физический предмет: падает, сталкивается. (ящик)
   Anchored=false, CanCollide=false  → падает и проходим (редко).
```

💡 ⭐⭐ Запомни два главных физических свойства: **Anchored** (двигает ли физика) и **CanCollide**
(можно ли пройти насквозь). 90% багов новичка — забытый Anchored (пол улетел) или лишний CanCollide
(застрял в декоре). Сознательно выбирай оба для каждой детали.

---

## ⭐ Позиция: Position, CFrame, Pivot

```
   • Position (Vector3) — где центр детали в мире.
   • Orientation — поворот по осям (в градусах).
   • CFrame — позиция + поворот вместе (мощнее; из скриптов чаще работают с CFrame, модуль 13).
   • двигать в Studio: инструмент Move (стрелки), Rotate (кольца), Scale (квадратики).

   привязка к сетке: Model → Snap to Grid (шаг перемещения/поворота) — для аккуратной сборки.
```

💡 ⭐ В Studio двигай мышью (Move/Rotate/Scale), а точные значения подправляй в Properties. Snap to
Grid помогает строить ровно (платформы стык в стык). Из кода позицию задаёшь через `Position` или
`CFrame` (модуль 13).

## ⭐ Группировка: Model и сборка

```
   связанные детали объединяй в МОДЕЛЬ (Model):
   • выдели несколько Parts → ПКМ → Group (Ctrl+G) → станут одной Model в Explorer.
   • Model двигается/копируется целиком; у неё может быть PrimaryPart (опорная деталь).

   копирование: Ctrl+C / Ctrl+V (или Alt+перетаскивание — дублировать).
   так строят: одна платформа → копируешь → собираешь уровень.
```

💡 ⭐ Группируй логически связанное (дом = стены+крыша+дверь → одна Model). Это как функция/модуль в
коде: единица, которую удобно двигать, копировать и находить из скриптов целиком. Связь с
[организацией кода](../../C/01-basics/07b-multiple-files.md) — тот же принцип группировки.

## ⚠️ Ловушки

- ❌ Забыть Anchored на статичной детали → при Play пол/платформа падает.
- ❌ Лишний/недостающий CanCollide → игрок проваливается или застревает.
- ❌ Neon/Glass материалы без понимания (Neon светится сам, не освещает сцену).
- ❌ Не группировать связанное → потом не подвинуть/не скопировать целиком.
- ❌ Гигантский Transparency=1 + CanCollide=true → невидимая стена (иногда нужно, но путает).
