# 🚀 Мини-проект · Обобщённый контейнер на C

> Примени продвинутый C: построй типобезопасный обобщённый контейнер (как `std::vector`/`map`, но
> на C), используя макро-генерацию, intrusive-приёмы или void* — с тестами и чистотой под санитайзерами.

---

## 🎯 Что делаем

Выбери контейнер и сделай его обобщённым (работающим с разными типами) одним из изученных способов,
качественно, как библиотека.

```
   варианты:
   • generic Vec через макро-генерацию DEFINE_VEC(T, name) — типобезопасно.
   • generic hash map через макросы или void*.
   • intrusive linked list (как в ядре Linux) с container_of.
   • tagged-union «значение» (int/float/string/list) — основа для интерпретатора.
```

---

## 📋 Требования (на примере generic Vec через макросы)

- **Обобщённость:** `DEFINE_VEC(T, name)` генерирует типизированную структуру и функции (push/pop/get/grow).
- **Корректность:** авторост ×2, освобождение, нет утечек.
- **Типобезопасность:** компилятор проверяет тип элементов (преимущество макро-подхода над void*).
- **Безопасные макросы:** скобки, `do-while(0)`, без побочных эффектов в аргументах.
- **Качество:** тесты на нескольких типах (int, struct), чистый прогон под **ASan/UBSan/Valgrind**,
  README (как использовать, какой подход и почему).
