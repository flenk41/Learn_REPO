# 11 · Динамическая память: malloc / free 🖼️⭐

> 🎯 **Цель блока:** научиться выделять память **вручную** в куче (heap) — когда размер
> заранее неизвестен. Это вершина работы с памятью в C.

---

## 📖 Зачем нужна динамическая память

Обычный массив требует знать размер **на этапе компиляции**:

```c
int arr[100];          // а если нужно 5? а если 1 000 000? а если "сколько введёт юзер"?
```

Проблемы статических массивов:
- размер фиксирован заранее;
- живут на стеке (мало места);
- умирают в конце функции.

🎯 **Динамическая память** решает всё это: ты просишь у системы **столько байт, сколько
надо, во время работы программы**, и память живёт, пока ты её не освободишь.

---

## ⭐ `malloc` — выделить память

```c
#include <stdlib.h>

int *p = malloc(sizeof(int));     // выделить 4 байта в куче
```

🖼️ Что произошло:

```
   STACK              HEAP (куча)
  ┌──────┐          ┌────────────┐
  │  p ●─┼─────────►│  ???? (4б) │  ← malloc вернул адрес этого блока
  └──────┘          └────────────┘
  p хранит адрес    память выделена, но значение пока МУСОР
```

- `malloc(n)` выделяет `n` **байт** и возвращает **адрес** начала блока.
- `sizeof(int)` = 4, поэтому `malloc(sizeof(int))` = выделить под один `int`.
- Возвращённую память надо использовать через указатель.

```c
*p = 42;              // записать значение в выделенную память
printf("%d\n", *p);   // 42
```

---

## ⭐ `free` — освободить память

Куча **не чистится сама**! Закончил работу с памятью — верни её системе:

```c
free(p);          // вернуть память системе
p = NULL;         // ✅ хорошая привычка: обнулить указатель после free
```

🖼️

```
ДО free:   p ●──► [42] (занято)
free(p):   p ●──► [свободно]   ← память возвращена системе
p = NULL:  p → NULL            ← p больше никуда не указывает
```

> 💡 **Золотое правило:** на каждый `malloc` должен быть ровно один `free`.
> Это самое важное правило управления памятью в C.

---

## ⭐ Выделение массива динамически

Вот где сила — массив нужного размера во время работы:

```c
int n;
printf("Сколько чисел? ");
scanf("%d", &n);

int *arr = malloc(n * sizeof(int));   // массив на n элементов
if (arr == NULL) {                    // ⚠️ ВСЕГДА проверяй!
    printf("Не хватило памяти!\n");
    return 1;
}

for (int i = 0; i < n; i++)
    arr[i] = i * i;                   // работаем как с обычным массивом!

for (int i = 0; i < n; i++)
    printf("%d ", arr[i]);

free(arr);                            // освобождаем весь массив одним free
arr = NULL;
```

🖼️ malloc выделил `n` ячеек подряд в куче, `arr` указывает на начало:

```
arr ──► [0][1][4][9][16]...   (n элементов подряд в куче)
```

💡 С динамическим указателем работаешь как с обычным массивом: `arr[i]` работает!

---

## ⭐ Проверка результата — обязательна!

`malloc` может вернуть `NULL`, если памяти не хватило. Разыменование `NULL` = краш.

```c
int *p = malloc(1000000000L * sizeof(int));   // вдруг не хватит
if (p == NULL) {
    fprintf(stderr, "Ошибка выделения памяти\n");
    return 1;
}
```

> ⚠️ Профи **всегда** проверяют результат `malloc`. Это отличает аккуратный код от «упало
> непонятно почему».

---

## 📖 Семейство функций выделения

| Функция | Что делает |
|---------|-----------|
| `malloc(size)` | выделить `size` байт (содержимое — **мусор**) |
| `calloc(n, size)` | выделить `n*size` байт и **обнулить** их |
| `realloc(ptr, newsize)` | **изменить размер** ранее выделенного блока |
| `free(ptr)` | освободить блок |

### `calloc` — выделить и обнулить

```c
int *arr = calloc(n, sizeof(int));   // все элементы = 0 (в отличие от malloc)
```

### `realloc` — изменить размер (например, растущий массив)

```c
int *arr = malloc(2 * sizeof(int));
// ...нужно больше места...
int *tmp = realloc(arr, 5 * sizeof(int));   // расширить до 5
if (tmp == NULL) { /* обработать ошибку, старый arr ещё валиден */ }
arr = tmp;                                  // ✅ присваивай через временную!
```

🖼️ `realloc` может **перенести** данные в новое место, если рядом нет места:

```
было:  arr ──► [a][b] (рядом занято)
realloc: система находит место побольше, КОПИРУЕТ данные:
стало: arr ──► [a][b][_][_][_]  (возможно, по новому адресу!)
```

> ⚠️ Никогда не пиши `arr = realloc(arr, ...)` напрямую: если `realloc` вернёт `NULL`,
> ты потеряешь старый указатель → утечка. Используй временную переменную.

## ⚠️ ЧЕТЫРЕ смертных греха динамической памяти

### 1. 💧 Утечка памяти (memory leak) — забыл `free`

```c
void leak(void) {
    int *p = malloc(100);
    // ... забыли free(p)
}                              // p удалён со стека, но память в куче ОСТАЛАСЬ занятой
                               // → её адрес потерян, освободить уже нельзя!
```
Программа постепенно съедает всю память. В долгоживущих программах (серверы) — смертельно.

### 2. 💀 Висячий указатель (use after free) — использование после `free`

```c
int *p = malloc(sizeof(int));
free(p);
*p = 5;          // ⚠️ память уже освобождена! Запись в чужое → краш или порча
```
Лечение: `p = NULL;` после `free`.

### 3. ☠️ Двойное освобождение (double free)

```c
free(p);
free(p);         // ⚠️ освобождаем уже освобождённое → краш / порча кучи
```
Лечение: `p = NULL;` после первого `free` (free(NULL) безопасен).

### 4. 🚫 Использование без инициализации

```c
int *p;
*p = 5;          // ⚠️ p указывает в случайное место!
```

🖼️ Жизненный цикл правильной работы с памятью:

```
malloc ──► проверить != NULL ──► использовать ──► free ──► p = NULL
   │                                                          ▲
   └──────────────── ровно один на один ──────────────────────┘
```

## 📖 Динамическая память живёт дольше функции

В отличие от стека, память из кучи **переживает** функцию, которая её создала:

```c
int* make_array(int n) {
    int *arr = malloc(n * sizeof(int));   // в куче
    for (int i = 0; i < n; i++) arr[i] = i;
    return arr;          // ✅ можно вернуть! Память в куче не умирает
}                        // (сравни: возвращать &local нельзя — стек умирает)

int main(void) {
    int *a = make_array(5);
    // ... используем ...
    free(a);             // освобождаем там, где закончили
    return 0;
}
```

💡 Тот, кто выделил, не обязан освобождать — но **кто-то** обязан. Договорённость
«кто отвечает за free» — важная часть дизайна программ на C.

## ✅ Задачи

1. **Один int.** Выдели через `malloc` один `int`, запиши, выведи, освободи.
2. **Динамический массив.** Спроси у пользователя `n`, создай массив, заполни числами,
   выведи сумму, освободи.
3. **calloc.** Сделай то же через `calloc`, убедись, что элементы изначально нули.
4. **Копия массива.** Функция, которая принимает массив и его размер, возвращает
   **новый** динамический массив-копию.
5. **Конкатенация строк.** Функция `char* my_strcat(const char *a, const char *b)`,
   которая выделяет память под результат и возвращает новую строку. Не забудь `+1` под `'\0'`!
6. **realloc — растущий массив.** Считывай числа, пока не введут 0, храня их в массиве,
   который увеличивается через `realloc` по мере необходимости.
7. **Поиск утечки.** Намеренно напиши код с утечкой, потом найди и исправь.
8. ⭐ **Матрица.** Выдели динамически двумерную матрицу `n×m` (массив указателей на
   массивы), заполни, выведи, корректно освободи (free каждой строки + free массива строк).
