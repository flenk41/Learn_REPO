## 🟢 Блок A — структура проекта

1. ⭐ **Раздели монолит** на `main.c` + модуль логики (`.h`/`.c`).
2. ⭐⭐ **Раскладка по папкам** `include/ src/ build/` + Makefile с `-Iinclude`.
3. ⭐⭐ **static-приватность** — спрячь внутренние функции, проверь недоступность извне.
4. ⭐⭐⭐ **Три модуля** с правильным направлением зависимостей (без циклов) + один Makefile.

---

## 🟢 Блок B — проектирование API

5. ⭐⭐ **Opaque-тип** — перепиши Stack/Vector со скрытыми полями.
6. ⭐⭐ **Парный жизненный цикл** create/destroy, проверка ASan на утечки.
7. ⭐⭐ **Единый стиль ошибок** + документация в `.h`.
8. ⭐⭐ **const-аудит** всего API.
9. ⭐⭐⭐ **API-ревью** — дай заголовок другому, собери обратную связь, улучши.

## 🟢 Блок C — внешние API

10. ⭐⭐ **GET-запрос** через libcurl к публичному API, вывод статуса и тела.
11. ⭐⭐ **Обработка кодов** 200/404, ошибок сети.
12. ⭐⭐⭐ **Парсинг JSON** через cJSON — достать конкретные поля.
13. ⭐⭐⭐ **Мини-клиент** с чистым API-обёрткой над запросом.
