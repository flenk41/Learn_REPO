## 🎯 Что такое API — одним предложением

> **API (Application Programming Interface) — это «контракт»: набор функций и правил,
> через которые с твоим кодом (или чужим сервисом) общаются, не зная, что у него внутри.**

Есть две стороны медали, и мы разберём обе:

| Сторона | Пример | Где |
|---------|--------|-----|
| **Ты проектируешь API** | другие зовут твои `vec_push()`, не зная про `realloc` внутри | модули 1–2 |
| **Ты используешь чужой API** | зовёшь сервис по сети и получаешь JSON | модуль 3 |

💡 В обоих случаях идея одна: **граница между «что снаружи» и «как сделано внутри»**.
Хороший разработчик мыслит этими границами.
