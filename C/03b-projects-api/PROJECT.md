## 📋 Что делаем

Библиотеку `strbuf` — «растущую строку» (string builder), к которой удобно дописывать
текст без ручного управления буферами. Внутри — `realloc` (Уровень 2), снаружи — чистый
API (модуль 2).

```c
StrBuf *sb = strbuf_create();
strbuf_append(sb, "Привет, ");
strbuf_append(sb, "мир!");
strbuf_append_int(sb, 42);
printf("%s\n", strbuf_cstr(sb));     // "Привет, мир!42"
strbuf_destroy(sb);
```

## ✅ Требования (минимум)

- [ ] `StrBuf` — **opaque-тип** (поля скрыты в `.c`, в `.h` только `typedef struct StrBuf StrBuf;`).
- [ ] API: `create`, `append`, `append_int`, `cstr` (получить C-строку), `len`, `clear`,
      `destroy`.
- [ ] Парный жизненный цикл `create`/`destroy`, рост через `realloc`.
- [ ] Раскладка по папкам + Makefile с `-Iinclude`, цели `make`, `make test`, `make clean`.
- [ ] Документация каждой функции в `strbuf.h`.
- [ ] **Ноль утечек** под `-fsanitize=address` (и demo, и тесты).
- [ ] `README.md` с описанием API и примером.

## 🪜 Пошаговый план

1. **API на бумаге.** Сначала напиши `strbuf.h` — спроектируй интерфейс, **не думая о
   реализации**. Какие функции нужны пользователю? Это и есть «API-first» подход.
2. **Opaque-тип.** `typedef struct StrBuf StrBuf;` в `.h`; `struct StrBuf { char *data;
   size_t len, cap; };` — в `.c`.
3. **Реализация.** `strbuf.c`: create (malloc), append (realloc при нехватке — вспомни
   Vector!), cstr, destroy.
4. **Demo.** `main.c` использует библиотеку **только через `strbuf.h`** — ни разу не
   залезая в поля.
5. **Тесты.** `test_strbuf.c`: проверь append, рост, clear, len. Простые `assert`.
6. **Makefile.** Папки, `-Iinclude`, цели сборки/тестов/очистки.
7. **ASan + README.** Чистый прогон, описание API.

## 🧱 Скелет `include/strbuf.h`

```c
#ifndef STRBUF_H
#define STRBUF_H
#include <stddef.h>

/* Растущая строка. Детали скрыты — работай только через эти функции. */
typedef struct StrBuf StrBuf;

/* Создать пустой буфер. Вернёт NULL при нехватке памяти.
   Освобождать через strbuf_destroy. */
StrBuf *strbuf_create(void);

/* Дописать C-строку в конец. Вернёт 0 при успехе, !=0 при ошибке. */
int strbuf_append(StrBuf *sb, const char *text);

/* Дописать целое число. */
int strbuf_append_int(StrBuf *sb, int value);

/* Текущее содержимое как C-строка (валидна до следующего изменения). */
const char *strbuf_cstr(const StrBuf *sb);

/* Длина строки (без '\0'). */
size_t strbuf_len(const StrBuf *sb);

/* Очистить (длина = 0), буфер не освобождается. */
void strbuf_clear(StrBuf *sb);

/* Освободить всю память. */
void strbuf_destroy(StrBuf *sb);

#endif
```

Дальше реализуй `strbuf.c` сам — это и есть проект.

## 💪 Усложнения (для «отлито»)

- ➕ `strbuf_appendf(sb, "%d-%s", n, s)` — форматированная дозапись (через `vsnprintf`).
- ➕ `strbuf_insert` / `strbuf_remove`.
- ➕ Юнит-тесты с подсчётом пройденных/упавших.
- ➕ Версионирование: `STRBUF_VERSION` в заголовке, раздел в README.
- ➕ Превратить в статическую библиотеку (`libstrbuf.a` через `ar`) и подключить к demo.
- ➕ Бонус-связь с веб-API: используй `strbuf` для накопления ответа libcurl (модуль 3).
