# 03 · Переменные и типы данных 🖼️

> 🎯 **Цель блока:** понять, что переменная — это **имя для ячеек памяти**, и сколько
> байт занимает каждый тип. Здесь начинается связь с памятью.

---

## 📖 Что такое переменная

Переменная — это **именованный кусочек памяти**, где хранится значение.

```c
int age = 25;
```

Что произошло в памяти:

```
       имя: age
        ▼
     ┌────────────────────┐
RAM: │  25                │   ← 4 байта подряд (тип int)
     └────────────────────┘
      адрес, например 0x1000
```

- `int` — **тип** (целое число). Он определяет, **сколько байт** выделить.
- `age` — **имя**, по которому ты обращаешься к этим байтам.
- `25` — **значение**, которое там лежит.

💡 Ты пишешь `age`, а компьютер работает с **адресом** этих байтов. Имя — для тебя,
адрес — для машины.

---

## 📐 Основные типы и их размер

Размер типа = сколько байт он занимает в памяти. Это критично для понимания памяти!

| Тип | Что хранит | Обычный размер | Диапазон |
|-----|-----------|----------------|----------|
| `char` | один символ / маленькое число | **1 байт** | −128…127 |
| `int` | целое число | **4 байта** | ≈ ±2 млрд |
| `short` | короткое целое | 2 байта | ≈ ±32 тыс |
| `long long` | большое целое | 8 байт | очень большое |
| `float` | дробное число | 4 байта | ~7 знаков точности |
| `double` | дробное (точное) | 8 байт | ~15 знаков точности |
| `unsigned int` | целое **без знака** (только ≥0) | 4 байта | 0 … ≈4 млрд |

> ⚠️ Размеры могут отличаться на разных системах! Поэтому есть оператор `sizeof`.

### Узнать размер типа — `sizeof`

```c
#include <stdio.h>

int main(void)
{
    printf("char       = %zu байт\n", sizeof(char));
    printf("int        = %zu байт\n", sizeof(int));
    printf("double     = %zu байт\n", sizeof(double));
    printf("long long  = %zu байт\n", sizeof(long long));
    return 0;
}
```

> `%zu` — формат для вывода `sizeof` (тип `size_t`). Запусти и посмотри числа на своей машине.

## 🖼️ Как разные типы лежат в памяти

```
char c = 'A';        ┌────┐
                     │ 65 │                1 байт ('A' = код 65)
                     └────┘

int n = 1000;        ┌────┬────┬────┬────┐
                     │    4 байта        │   4 байта
                     └────┴────┴────┴────┘

double d = 3.14;     ┌────┬────┬────┬────┬────┬────┬────┬────┐
                     │            8 байт                     │
                     └────┴────┴────┴────┴────┴────┴────┴────┘
```

💡 Чем больше тип — тем больше памяти. Если хранишь миллион чисел `int` — это 4 МБ.
Тех же чисел в `char` (если влезают) — всего 1 МБ. **Выбор типа = выбор расхода памяти.**

## ✍️ Объявление и инициализация

```c
int x;          // объявили — память выделена, но значение МУСОР (что было в ячейках)
x = 10;         // присвоили значение
int y = 20;     // объявили И сразу задали значение (инициализация) — так лучше

int a = 1, b = 2, c = 3;   // несколько сразу
```

> ⚠️ **Частая ошибка новичка:** использовать переменную **до** присваивания.
> В C она содержит «мусор» — случайные байты, которые были в памяти. Всегда инициализируй!

```c
int sum;                    // мусор!
printf("%d\n", sum + 5);    // ⚠️ непредсказуемый результат (undefined behavior)
```

---

## 🏷️ Константы

```c
const double PI = 3.14159;   // значение нельзя изменить
PI = 3;                      // ⚠️ ошибка компиляции — и это хорошо!
```

💡 `const` защищает от случайного изменения и говорит компилятору «это не поменяется» —
он может оптимизировать код.

## 🔤 Спецификаторы формата для `printf` / `scanf`

| Тип | Формат | Пример |
|-----|--------|--------|
| `int` | `%d` | `printf("%d", 42);` |
| `float` / `double` | `%f` | `printf("%f", 3.14);` |
| `char` | `%c` | `printf("%c", 'A');` |
| строка | `%s` | `printf("%s", "hi");` |
| `unsigned` | `%u` | |
| `sizeof` (size_t) | `%zu` | |
| шестнадцатеричный | `%x` | `printf("%x", 255); // ff` |
| указатель/адрес | `%p` | (понадобится в блоке про указатели) |

## 🧪 Эксперимент: переполнение

Положи в `char` число больше 127:

```c
char x = 127;
x = x + 1;          // что будет?
printf("%d\n", x);  // напечатает -128 !!!
```

🖼️ Почему? `char` — это всего 1 байт (8 бит). Когда переполняешь — число
«заворачивается» по кругу:

```
... 125, 126, 127, -128, -127 ...
                 └─ переполнение, прыжок в минимум
```

💡 Это первый намёк, что **типы ограничены размером памяти**. Запомни — переполнение
встретится ещё не раз.
