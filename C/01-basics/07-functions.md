# 07 · Функции 🖼️

> 🎯 **Цель блока:** научиться разбивать программу на функции. И впервые заглянуть, как
> функции живут в памяти (стек вызовов) — это мостик к блоку про память.

---

## 📖 Что такое функция

Функция — это **именованный блок кода**, который можно вызывать много раз.
Ты уже знаешь две: `main` и `printf`.

```c
int square(int x)      // тип результата | имя | (параметры)
{
    return x * x;      // return — вернуть результат
}

int main(void)
{
    int result = square(5);   // вызов функции
    printf("%d\n", result);   // 25
    return 0;
}
```

🖼️ Анатомия функции:

```
   int   square ( int x )
    │      │       └──┬──┘
    │      │      параметр (вход)
    │   имя функции
 тип возвращаемого значения (что отдаёт наружу)
```

---

## 📖 Зачем функции

| Без функций | С функциями |
|-------------|-------------|
| код повторяется | пишешь один раз, вызываешь много |
| сложно читать | каждая функция = одна задача |
| трудно искать ошибки | тестируешь по частям |

💡 Правило: **одна функция — одна задача**. Имя функции должно говорить, что она делает:
`calculate_average`, `is_prime`, `print_menu`.

---

## 📖 Параметры и возвращаемое значение

```c
// несколько параметров
int sum(int a, int b) {
    return a + b;
}

// ничего не возвращает — тип void
void greet(const char *name) {
    printf("Привет, %s!\n", name);
    // нет return со значением
}

// ничего не принимает
int get_lucky_number(void) {
    return 7;
}
```

## ⭐ Важно: аргументы передаются ПО ЗНАЧЕНИЮ (копией)

```c
void try_change(int x) {
    x = 100;          // меняем КОПИЮ
}

int main(void) {
    int a = 5;
    try_change(a);
    printf("%d\n", a);   // ВСЁ ЕЩЁ 5 ! Оригинал не изменился
    return 0;
}
```

🖼️ Почему? При вызове создаётся **копия** значения в новой ячейке памяти:

```
main:           try_change:
 a               x
┌───┐  копия    ┌───┐
│ 5 │ ────────► │ 5 │ → стало 100, но это другая ячейка!
└───┘           └───┘
оригинал не     эта копия
тронут          умрёт после функции
```

💡 Чтобы функция **изменила** оригинал — надо передать **адрес** переменной (указатель).
Это мы подробно разберём в [блоке про указатели](../02-memory/09-pointers.md). Запомни этот
момент — он ключевой!

## 🖼️ Стек вызовов — где живут функции в памяти

Когда вызывается функция, под её локальные переменные выделяется кусок памяти —
**кадр стека** (stack frame). Когда функция заканчивается — кадр удаляется.

```c
int twice(int n) { return n * 2; }
int quad(int n)  { return twice(twice(n)); }
int main(void)   { return quad(3); }
```

🖼️ Стек растёт вниз при вызовах и сворачивается при возвратах:

```
Вызвали main → quad → twice          Стек памяти:
                                     ┌──────────────┐
                                     │ twice (n=...)│ ← добавляется сверху
                                     ├──────────────┤
                                     │ quad  (n=3)  │
                                     ├──────────────┤
                                     │ main         │
                                     └──────────────┘
   Каждая закончившаяся функция убирается со стека.
```

💡 Поэтому он и называется **стек** (стопка): кто вызван последним — уходит первым.
Полную картину памяти (стек + куча) ты увидишь в
[модуле 08 · Модель памяти](../02-memory/08-memory-model.md).

> ⚠️ Если функция вызывает сама себя бесконечно (плохая рекурсия) — стек переполняется:
> **stack overflow**, программа падает.

---

## 📖 Рекурсия — функция вызывает саму себя

```c
int factorial(int n) {
    if (n <= 1) return 1;          // базовый случай (выход!)
    return n * factorial(n - 1);   // вызов самой себя
}
```

🖼️ Как считается `factorial(3)`:

```
factorial(3) = 3 * factorial(2)
                    factorial(2) = 2 * factorial(1)
                                        factorial(1) = 1   ← база
             = 3 * (2 * (1)) = 6
```

> ⚠️ У рекурсии **обязан** быть базовый случай (условие выхода), иначе — stack overflow.

## 📖 Прототипы функций

Если функцию определяешь **после** `main`, компилятору нужно знать о ней заранее —
объяви **прототип** сверху:

```c
#include <stdio.h>

int sum(int a, int b);     // прототип (объявление) — обещание «функция будет ниже»

int main(void) {
    printf("%d\n", sum(2, 3));
    return 0;
}

int sum(int a, int b) {    // определение (реализация)
    return a + b;
}
```

💡 В больших проектах прототипы выносят в **заголовочные файлы** (`.h`) — об этом в
Middle-блоке про модульность.
