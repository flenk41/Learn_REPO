# 🚀 Пет-проект уровня 3 — Менеджер задач (Todo) с сохранением

> 🎯 **Цель:** собрать ВСЁ из Middle в один проект: структуры, динамическую память,
> связный список (или динамический массив), файлы и разбиение на модули. Это уже
> «портфолио-проект».

---

## 📋 Что должна уметь программа

Консольный менеджер задач, который **запоминает задачи между запусками** (сохраняет в файл):

```
==== МОИ ЗАДАЧИ ====
1. [ ] Купить хлеб           (приоритет: 2)
2. [x] Сделать домашку       (приоритет: 1)
3. [ ] Позвонить маме        (приоритет: 3)

Команды: add / done N / delete N / list / save / quit
> add Помыть посуду 2
Задача добавлена (id=4)
> done 1
Задача 1 отмечена выполненной
> quit
Задачи сохранены в tasks.txt. Пока!
```

---

## ✅ Требования (минимум)

- [ ] Структура `Task { int id; char title[100]; int done; int priority; }`.
- [ ] Хранение задач в **динамической** структуре (динамический массив **или** связный
      список — на твой выбор).
- [ ] Команды: `add`, `done N`, `delete N`, `list`, `save`, `quit`.
- [ ] **Сохранение в файл** при выходе и **загрузка** при старте.
- [ ] Проект разбит минимум на 2 модуля (`task.h`/`task.c` + `main.c`) + **Makefile**.
- [ ] Никаких утечек памяти (проверь ASan).

---

## 🪜 Пошаговый план

1. **Структура и хранилище.** Опиши `Task`. Выбери: динамический массив (Vector из
   уровня 2!) или связный список (из модуля 14). Реализуй добавление в память.
2. **Вывод списка.** Команда `list` — печать всех задач с галочками `[x]/[ ]`.
3. **Цикл команд.** Считывай команды в цикле (`fgets` + разбор), пока не `quit`.
4. **add.** Парси `add <текст> <приоритет>`, создавай задачу с новым id.
5. **done / delete.** Найди задачу по id, измени/удали.
6. **Сохранение.** При `save`/`quit` запиши все задачи в `tasks.txt`
   (текстом — по строке на задачу, или бинарно через `fwrite`).
7. **Загрузка.** При старте, если файл есть — прочитай задачи в память.
8. **Модули + Makefile.** Раздели на файлы, напиши Makefile с ASan-флагами.
9. **ASan.** Добейся чистого вывода.

## 🧱 Скелет

**task.h:**
```c
#ifndef TASK_H
#define TASK_H
#include <stddef.h>

typedef struct {
    int  id;
    char title[100];
    int  done;
    int  priority;
} Task;

typedef struct {
    Task  *items;
    size_t count;
    size_t capacity;
} TaskList;

void tasklist_init(TaskList *list);
void tasklist_add(TaskList *list, const char *title, int priority);
int  tasklist_done(TaskList *list, int id);
int  tasklist_delete(TaskList *list, int id);
void tasklist_print(const TaskList *list);
int  tasklist_save(const TaskList *list, const char *filename);
int  tasklist_load(TaskList *list, const char *filename);
void tasklist_free(TaskList *list);

#endif
```

**main.c** — цикл чтения команд, разбор и вызов функций из `task.c`.

**Makefile:**
```makefile
CC = gcc
CFLAGS = -Wall -Wextra -g -fsanitize=address

todo: main.o task.o
	$(CC) $(CFLAGS) main.o task.o -o todo

main.o: main.c task.h
	$(CC) $(CFLAGS) -c main.c
task.o: task.c task.h
	$(CC) $(CFLAGS) -c task.c

clean:
	rm -f *.o todo
```

## 💪 Усложнения (для «отлично»)

- ➕ Сортировка задач по приоритету (через `qsort` + указатель на функцию).
- ➕ Фильтры: показать только невыполненные / только с приоритетом N.
- ➕ Срок выполнения (дата) у задачи + предупреждение о просроченных.
- ➕ Категории/теги задач.
- ➕ Подтверждение перед удалением, защита от кривого ввода.
- ➕ Цветной вывод в терминал (ANSI-коды).
- ➕ Юнит-тесты для функций `task.c`.
