## 📖 Препроцессор — первый шаг компиляции

Вспомни 4 стадии из модуля 02. Первая — **препроцессор**. Он обрабатывает строки,
начинающиеся с `#`, **до** компиляции: подставляет, вырезает, заменяет текст.

### `#define` — макрос-константа

```c
#define PI 3.14159
#define MAX_USERS 100

double area = PI * r * r;        // препроцессор заменит PI на 3.14159
```

🖼️ Что делает препроцессор — **текстовая замена** перед компиляцией:

```
ты написал:   area = PI * r * r;
препроцессор: area = 3.14159 * r * r;   ← подставил, потом уже компилирует
```

### `#define` — макрос-функция

```c
#define SQUARE(x) ((x) * (x))
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int y = SQUARE(5);      // → ((5) * (5)) = 25
```

> ⚠️ **Всегда оборачивай аргументы в скобки!** Иначе:
> ```c
> #define SQUARE(x) x * x
> SQUARE(2 + 3)   // → 2 + 3 * 2 + 3 = 11, а не 25 !!!
> ```
> Со скобками `((2+3)*(2+3))` = 25. Это классическая ловушка макросов.

💡 В современном C для констант лучше `const`/`enum`, для функций — обычные `static
inline` функции. Макросы оставляй для условной компиляции и особых случаев.

## 📖 Условная компиляция

```c
#define DEBUG 1

#if DEBUG
    printf("Отладка: x = %d\n", x);
#endif

#ifdef WINDOWS
    // код только для Windows
#else
    // код для остальных систем
#endif
```

Полезно для отладочных вставок и кода под разные платформы.

## ⭐ Разбиваем программу на файлы

Когда программа растёт, держать всё в одном `.c` неудобно. Делим на модули:

- **`.h` (заголовок)** — «что умеет модуль»: объявления функций, типы, константы (интерфейс).
- **`.c` (реализация)** — «как это сделано»: тела функций.

### Пример: модуль `mymath`

**mymath.h** — интерфейс:
```c
#ifndef MYMATH_H        // include guard — защита от двойного включения
#define MYMATH_H

int add(int a, int b);
int multiply(int a, int b);

#endif
```

**mymath.c** — реализация:
```c
#include "mymath.h"

int add(int a, int b)      { return a + b; }
int multiply(int a, int b) { return a * b; }
```

**main.c** — использование:
```c
#include <stdio.h>
#include "mymath.h"     // "..." для своих файлов, <...> для системных

int main(void) {
    printf("%d\n", add(2, 3));
    return 0;
}
```

### ⭐ Include guard — зачем `#ifndef`

🖼️ Если один заголовок включить дважды (напрямую и через другой файл) — будут
повторные объявления → ошибка. Include guard это предотвращает:

```
первое  #include "mymath.h":  MYMATH_H не определён → включаем, определяем MYMATH_H
второе  #include "mymath.h":  MYMATH_H уже определён → пропускаем содержимое
```

> 💡 Альтернатива: одна строка `#pragma once` в начале файла (поддерживается всеми
> современными компиляторами).

## ⭐ Компиляция нескольких файлов

🖼️ Каждый `.c` компилируется в `.o` (объектный файл), потом линковщик их связывает:

```
 main.c ──компилятор──► main.o   ┐
                                 ├──линковщик──► program.exe
mymath.c ──компилятор──► mymath.o ┘
```

Вручную:
```bash
gcc -c main.c -o main.o          # компилируем каждый файл отдельно
gcc -c mymath.c -o mymath.o
gcc main.o mymath.o -o program   # линкуем в одну программу
```

Или одной командой:
```bash
gcc main.c mymath.c -o program
```

## 🛠️ Makefile — автоматизация сборки

Печатать команды вручную каждый раз утомительно. **Make** делает это за тебя, причём
**пересобирает только изменённые файлы**.

Создай файл `Makefile`:

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -g -fsanitize=address

program: main.o mymath.o
	$(CC) $(CFLAGS) main.o mymath.o -o program

main.o: main.c mymath.h
	$(CC) $(CFLAGS) -c main.c

mymath.o: mymath.c mymath.h
	$(CC) $(CFLAGS) -c mymath.c

clean:
	rm -f *.o program
```

> ⚠️ В Makefile отступы — это **табы**, а не пробелы! Иначе ошибка.

Использование:
```bash
make           # собрать (только изменённое)
make clean     # удалить артефакты
```

🖼️ Make строит граф зависимостей и пересобирает по цепочке:

```
изменил mymath.c → пересобрать mymath.o → пересобрать program
(main.o трогать не надо — он не зависел от изменения)
```

> На Windows `make` ставится через MSYS2: `pacman -S make`. Или используй `mingw32-make`.

## ✅ Задачи

1. **Свой модуль.** Вынеси математические функции из своих прошлых задач в `mymath.h` +
   `mymath.c`. Подключи из `main.c`. Собери вручную тремя командами.
2. **Makefile.** Напиши Makefile для проекта из задачи 1. Проверь `make` и `make clean`.
3. **Безопасные макросы.** Напиши макросы `MIN`, `MAX`, `ABS`, `SWAP`. Проверь их на
   «коварных» аргументах вроде `MAX(a++, b)` — пойми, почему макросы тут опасны.
4. **DEBUG-логирование.** Сделай макрос `LOG(msg)`, который печатает сообщение только
   если определён `DEBUG`. Управляй через `gcc -DDEBUG`.
5. ⭐ **Многофайловый проект.** Возьми свой Vector (пет-проект уровня 2) и раздели на
   `vector.h` + `vector.c` + `main.c` с Makefile.
