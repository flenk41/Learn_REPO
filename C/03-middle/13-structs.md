# 13 · Структуры, union, enum 🖼️

> 🎯 **Цель блока:** научиться создавать **свои типы данных**, объединяя поля. И увидеть,
> как структура лежит в памяти (подряд, но с «дырками»-выравниванием).

---

## 📖 Зачем нужны структуры

До сих пор у тебя были отдельные переменные. Но что, если нужно описать «точку»
(x, y) или «человека» (имя, возраст, рост)? Держать кучу отдельных переменных неудобно.

**Структура** (`struct`) — это новый тип, который объединяет несколько полей в одно целое.

```c
struct Point {
    int x;
    int y;
};

struct Point p;     // создали переменную типа "struct Point"
p.x = 10;           // доступ к полям через точку .
p.y = 20;
printf("(%d, %d)\n", p.x, p.y);
```

---

## 🖼️ Как структура лежит в памяти

Поля структуры располагаются в памяти **подряд**, одно за другим:

```c
struct Point { int x; int y; };   // два int по 4 байта
```

```
        ┌──────────┬──────────┐
struct: │    x     │    y     │     8 байт подряд
        │ (4 байта)│ (4 байта)│
        └──────────┴──────────┘
        0x1000     0x1004
```

💡 Поэтому `sizeof(struct Point)` = 8. Структура — это просто «склейка» полей в памяти.

> ⚠️ Иногда `sizeof` структуры **больше** суммы полей — из-за **выравнивания (padding)**.
> Например `struct { char c; int n; }` занимает не 5, а 8 байт: компилятор добавляет
> 3 «пустых» байта после `char`, чтобы `int` начинался с адреса, кратного 4. Подробно —
> в Senior-модуле [20 · Выравнивание](../04-senior/20-alignment-ub.md).

## 📖 typedef — короткое имя для типа

Писать `struct Point` каждый раз утомительно. `typedef` создаёт псевдоним:

```c
typedef struct {
    int x;
    int y;
} Point;            // теперь тип называется просто "Point"

Point p = {10, 20};         // инициализация по порядку полей
Point q = {.x = 1, .y = 2}; // именованная инициализация (нагляднее)
```

---

## ⭐ Структуры и указатели: оператор `->`

Когда у тебя **указатель** на структуру, для доступа к полям используют `->` вместо `.`:

```c
Point p = {10, 20};
Point *ptr = &p;

printf("%d\n", (*ptr).x);   // так можно, но громоздко
printf("%d\n", ptr->x);     // ✅ так принято: ptr->x означает (*ptr).x
```

🖼️

```
ptr ●──► Point { x=10, y=20 }
          ptr->x читает поле x по адресу
```

💡 Правило простое: есть **сама структура** → пиши `.`; есть **указатель** на неё → пиши `->`.

## ⭐ Передача структур в функции

Структуры тоже передаются **по значению** (копируется вся структура!). Для больших
структур это дорого — передавай **указатель**:

```c
// ❌ копирует всю структуру (медленно для больших)
double distance_bad(Point a, Point b) { ... }

// ✅ передаём адреса — копируется только указатель (8 байт)
double distance(const Point *a, const Point *b) {
    int dx = a->x - b->x;
    int dy = a->y - b->y;
    return sqrt(dx*dx + dy*dy);
}

distance(&p1, &p2);
```

💡 `const Point *a` означает «не буду менять структуру, только читаю» — хороший тон.

## 📖 Вложенные структуры

```c
typedef struct {
    char  name[50];
    int   age;
    Point position;     // структура внутри структуры
} Person;

Person hero = { "Гена", 30, {5, 10} };
printf("%s стоит в (%d,%d)\n", hero.name, hero.position.x, hero.position.y);
```

## 📖 enum — именованные константы

`enum` создаёт набор именованных целочисленных констант. Делает код читаемым:

```c
typedef enum {
    MON,    // 0
    TUE,    // 1
    WED,    // 2
    THU,    // 3
    FRI,    // 4
    SAT,    // 5
    SUN     // 6
} Weekday;

Weekday today = WED;
if (today == SAT || today == SUN)
    printf("Выходной!\n");
```

💡 Вместо «магических чисел» (`if (state == 2)`) пиши `if (state == PAUSED)` — понятнее.
По умолчанию нумерация с 0, но можно задать свои значения: `enum { LOW=1, MID=5, HIGH=10 }`.

---

## 📖 union — несколько полей в ОДНОЙ памяти

`union` похож на `struct`, но все поля **делят одну и ту же память** (занимает размер
самого большого поля). В каждый момент «активно» только одно поле.

```c
union Value {
    int    i;
    float  f;
    char   bytes[4];
};

union Value v;
v.i = 65;
printf("%d\n", v.i);        // 65
printf("%c\n", v.bytes[0]); // 'A' — те же байты, прочитанные как char!
```

🖼️ В отличие от struct, union кладёт поля «друг на друга»:

```
struct: [ i ][ f ][ bytes ]   ← рядом (12 байт)
union:  [ i / f / bytes ]     ← в одном месте (4 байта, наибольшее поле)
```

💡 union используют для экономии памяти и для «тегированных объединений» (значение,
которое может быть разных типов). Часто комбинируют с `enum`-тегом.

## ✅ Задачи

1. **Точка и расстояние.** Структура `Point`, функция расстояния между двумя точками
   (через указатели). Считай две точки, выведи расстояние.
2. **Прямоугольник.** Структура `Rect { Point top_left; int w, h; }`. Функции площади,
   периметра, проверки попадания точки внутрь.
3. **Каталог книг.** Структура `Book { char title[100]; char author[100]; int year; }`.
   Массив из нескольких книг, функции печати, поиска по году, сортировки по названию.
4. **enum-состояния.** Опиши `enum TrafficLight { RED, YELLOW, GREEN }`. Функция, которая
   по текущему сигналу возвращает следующий.
5. **Дробь.** Структура `Fraction { int num, den; }`. Функции сложения, умножения,
   сокращения дробей (НОД).
6. ⭐ **Тегированное объединение.** Структура `Variant { enum Type tag; union { int i;
   double d; char *s; } data; }`. Функция печати в зависимости от `tag`.
