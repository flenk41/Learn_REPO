# 17 · Работа с файлами 🖼️

> 🎯 **Цель блока:** научиться сохранять данные на диск и читать их обратно. Теперь твои
> программы смогут **запоминать** данные между запусками.

---

## 📖 Файл vs память

Вспомни модуль 00: память (RAM) стирается при выключении, диск — сохраняет навсегда.
Чтобы данные пережили закрытие программы, их надо **записать в файл**.

🖼️

```
   Программа (RAM)              Диск (навсегда)
  ┌──────────────┐    fwrite   ┌──────────────┐
  │  данные      │ ──────────► │  файл.txt    │
  │  в памяти    │ ◄────────── │              │
  └──────────────┘    fread    └──────────────┘
```

## ⭐ Три шага работы с файлом: открыть → работать → закрыть

```c
FILE *f = fopen("data.txt", "w");   // 1. ОТКРЫТЬ
if (f == NULL) {                    //    всегда проверяй!
    perror("Не удалось открыть файл");
    return 1;
}

fprintf(f, "Привет, файл!\n");      // 2. РАБОТАТЬ (писать/читать)

fclose(f);                          // 3. ЗАКРЫТЬ (обязательно!)
```

> ⚠️ Забыл `fclose`? Данные могут не записаться (они в буфере) + утечка ресурса.
> `FILE*` — это как malloc: открыл → обязан закрыть.

---

## 📖 Режимы открытия `fopen`

| Режим | Что делает |
|-------|-----------|
| `"r"` | чтение (файл должен существовать) |
| `"w"` | запись (создаёт новый / **стирает** существующий!) |
| `"a"` | дозапись в конец (append) |
| `"r+"` | чтение и запись |
| `"rb"`, `"wb"` | то же, но **бинарный** режим (добавь `b`) |

> ⚠️ Режим `"w"` **удаляет** содержимое файла! Если надо дописать — используй `"a"`.

## 📖 Запись и чтение текста

### Запись

```c
FILE *f = fopen("scores.txt", "w");
fprintf(f, "%s %d\n", "Гена", 100);   // как printf, но в файл
fprintf(f, "%s %d\n", "Чебур", 95);
fclose(f);
```

### Чтение

```c
FILE *f = fopen("scores.txt", "r");
char name[50];
int score;
while (fscanf(f, "%49s %d", name, &score) == 2) {   // пока читаются 2 значения
    printf("%s набрал %d\n", name, score);
}
fclose(f);
```

💡 `fscanf` возвращает количество прочитанных значений — по нему ловим конец файла.

### Построчное чтение через `fgets`

```c
char line[256];
FILE *f = fopen("text.txt", "r");
while (fgets(line, sizeof(line), f) != NULL) {   // читаем по строке
    printf("Строка: %s", line);
}
fclose(f);
```

> 💡 `fgets` безопаснее `fscanf("%s")`: ты задаёшь максимальный размер буфера и не
> переполнишь его. Профи предпочитают `fgets`.

## ⭐ Бинарные файлы: fread / fwrite

Текст удобен для людей, но громоздок. Для структур эффективнее писать **сырые байты**:

```c
typedef struct { char name[50]; int age; } Person;

Person people[3] = { {"A", 20}, {"B", 30}, {"C", 40} };

// записать массив структур одним вызовом
FILE *f = fopen("people.bin", "wb");
fwrite(people, sizeof(Person), 3, f);   // (откуда, размер элемента, кол-во, файл)
fclose(f);

// прочитать обратно
Person loaded[3];
FILE *g = fopen("people.bin", "rb");
fread(loaded, sizeof(Person), 3, g);
fclose(g);
```

🖼️ `fwrite` копирует байты структуры **как есть** из памяти в файл:

```
память: [Person][Person][Person]  ──fwrite──►  файл (сырые байты)
                                   ◄──fread───
```

💡 Быстро и компактно, но файл не читается глазами и зависит от платформы (выравнивание,
порядок байт). Для обмена между системами используют текст/JSON или продуманный формат.

## 📖 Позиционирование в файле

```c
fseek(f, 0, SEEK_END);     // перейти в конец файла
long size = ftell(f);      // узнать размер (текущую позицию)
rewind(f);                 // вернуться в начало
```

Полезно, чтобы узнать размер файла или прыгать к нужной записи.

### Пример: прочитать весь файл в память

```c
FILE *f = fopen("data.txt", "rb");
fseek(f, 0, SEEK_END);
long size = ftell(f);
rewind(f);

char *buffer = malloc(size + 1);
fread(buffer, 1, size, f);
buffer[size] = '\0';        // завершаем строку
fclose(f);
// ... работаем с buffer ...
free(buffer);
```
