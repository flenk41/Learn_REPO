## 🛠️ GDB — продвинутая отладка

Базу ты видел в модуле 12. Теперь — приёмы профи.

### Watchpoint — поймать, КТО меняет переменную

Самый мощный приём для отлова порчи памяти:

```gdb
watch counter        # остановиться, как только counter изменится
run
# GDB покажет ТОЧНОЕ место, где переменная поменялась — старое и новое значение
```

🖼️ Бесценно, когда переменная «непонятно почему» испортилась — watchpoint ловит виновника.

### Условная точка останова

```gdb
break process.c:42 if id == 1000     # стоп только когда id == 1000
```
Не нужно жать `continue` тысячу раз — остановишься точно на нужной итерации.

### Анализ падения через core dump

Когда программа упала, можно вскрыть «труп»:

```bash
ulimit -c unlimited           # разрешить core-файлы
./program                     # упала → создался core
gdb ./program core            # вскрываем
(gdb) bt                      # backtrace: точная цепочка вызовов до краша
(gdb) frame 2                 # перейти к кадру
(gdb) print variable          # посмотреть значения в момент краха
```

### Шпаргалка команд GDB

| Команда | Действие |
|---------|----------|
| `watch x` | остановка при изменении `x` |
| `rwatch x` | остановка при **чтении** `x` |
| `break f if cond` | условная точка |
| `info locals` | все локальные переменные |
| `bt` / `bt full` | стек вызовов (с переменными) |
| `frame N` | перейти к кадру стека N |
| `x/16xb ptr` | дамп 16 байт памяти по адресу |
| `finish` | доработать текущую функцию и выйти |

## 🛠️ Санитайзеры — твой ежедневный набор

Собери в одну таблицу всё, что ты уже встречал:

| Санитайзер | Флаг | Что ловит |
|-----------|------|-----------|
| **ASan** | `-fsanitize=address` | утечки, выход за границы, use-after-free, double-free |
| **UBSan** | `-fsanitize=undefined` | переполнение, кривые сдвиги, деление на 0, невыровненный доступ |
| **TSan** | `-fsanitize=thread` | гонки данных |
| **MSan** | `-fsanitize=memory` | чтение неинициализированной памяти (только Clang) |

> 💡 **Дев-сборка профи:**
> ```bash
> gcc -Wall -Wextra -g -fsanitize=address,undefined program.c -o program
> ```
> (ASan и TSan несовместимы вместе — TSan гоняй отдельным прогоном.)

## 🛠️ Профилирование — где программа тратит время

### gprof — классический профилировщик

```bash
gcc -pg program.c -o program       # -pg включает профилирование
./program                          # создаётся gmon.out
gprof program gmon.out             # отчёт: какие функции «съели» время
```

🖼️ Отчёт покажет таблицу:
```
  %time   функция        вызовов
  68.2    compute_heavy   1000000     ← вот где узкое место!
  15.1    parse_line      50000
   ...
```

### perf — мощный профилировщик Linux

```bash
perf stat ./program           # общая статистика: циклы, кэш-промахи, ветвления
perf record ./program         # записать профиль
perf report                   # интерактивный отчёт по «горячим» местам
```

`perf` показывает реальные счётчики CPU: cache-misses, branch-misses — то, что ты учил в
модуле 22.

### Valgrind callgrind + KCachegrind

```bash
valgrind --tool=callgrind ./program
# открой callgrind.out.* в KCachegrind — визуальный граф вызовов
```

## 🛠️ Статический анализ — найти баги БЕЗ запуска

Анализаторы читают код и находят проблемы, не выполняя его:

```bash
# Встроенный в GCC статический анализатор
gcc -fanalyzer program.c

# clang-tidy — проверки качества и багов
clang-tidy program.c --

# cppcheck — отдельный анализатор
cppcheck --enable=all program.c
```

Они находят: утечки по путям выполнения, разыменование NULL, неинициализированные
переменные, мёртвый код — ещё до запуска.

## 📋 Рабочий процесс Senior-разработчика

🖼️ Как профи доводит код до качества:

```
1. Пишу код с -Wall -Wextra              (компилятор — первый рубеж)
2. Прогоняю под ASan + UBSan              (ошибки памяти и UB)
3. Многопоточное — под TSan               (гонки)
4. Статический анализ -fanalyzer          (баги без запуска)
5. Профилирую (perf/gprof) ЕСЛИ медленно  (оптимизирую измеримо)
6. Отладка GDB при конкретном баге         (watchpoint, core dump)
```

> 💡 Заметь: оптимизация — **предпоследний** шаг, и только при доказанной необходимости.
> Сначала корректность (санитайзеры), потом скорость (профайлер).
