# 🚀 Финальные пет-проекты — стань Senior

> 🎯 **Цель:** выбрать **один (а лучше два)** серьёзный проект, который объединяет всё:
> память, структуры данных, аллокаторы, файлы, возможно потоки и сеть. Это проект для
> портфолио и собеседований на Senior.

> 💡 Любой из них требует 1–4 недели работы. Это нормально. Реальные системы так и
> пишутся. Веди код в git, разбей на модули, покрой тестами.

---

## 🏆 Проект 1 — Key-Value база данных (рекомендую первой)

Мини-аналог Redis: хранилище пар «ключ → значение» с сохранением на диск.

**Что внутри:**
- 🗃️ **Хеш-таблица** для быстрого поиска (модуль 14).
- 💾 **Персистентность**: сохранение/загрузка в файл (модуль 17).
- 🧠 **Свой аллокатор** для записей (модуль 19).
- ⌨️ Командный интерфейс: `SET key value`, `GET key`, `DEL key`, `KEYS`, `SAVE`.

**Усложнения:** TTL (время жизни ключа), бинарный формат файла, сетевой доступ (модуль
23), потокобезопасность (модуль 21).

**Прокачивает:** хеш-таблицы, управление памятью, файлы, архитектуру.

---

## 🏆 Проект 2 — Интерпретатор/калькулятор выражений

Программа, которая разбирает и вычисляет выражения вроде `(3 + 4) * 2 - 7`.

**Что внутри:**
- 🔤 **Лексер** (строка → токены).
- 🌳 **Парсер** → абстрактное синтаксическое дерево (AST) на указателях (модуль 14).
- 🧮 **Вычислитель** дерева (рекурсия).
- 🧠 **Arena-аллокатор** под узлы дерева (модуль 19) — выделил дерево, посчитал, освободил всё разом.

**Усложнения:** переменные (`x = 5`), функции (`sin`, `sqrt`), стековая виртуальная машина.

**Прокачивает:** деревья, рекурсию, аллокаторы, парсинг — фундамент компиляторов.

---

## 🏆 Проект 3 — Свой аллокатор памяти (для любителей низкого уровня)

Полноценная замена `malloc`/`free`.

**Что внутри:**
- 🧱 Получение памяти от ОС через `mmap` (модуль 23).
- 📋 Управление свободными блоками (free-list), заголовки, выравнивание (модули 19–20).
- 🔗 Слияние соседних свободных блоков (борьба с фрагментацией).
- 🏎️ Бенчмарки против системного malloc.

**Усложнения:** разные стратегии (best-fit / first-fit), потокобезопасность, отладочный
режим с детекцией утечек.

**Прокачивает:** глубочайшее понимание памяти. После этого malloc для тебя — открытая книга.

---

## 🏆 Проект 4 — Многопоточный HTTP-сервер

Сервер, отдающий файлы по HTTP, обрабатывающий несколько клиентов разом.

**Что внутри:**
- 🌐 **Сокеты** TCP (модуль 23).
- 🧵 **Пул потоков** для параллельных клиентов (модуль 21).
- 📥 Разбор HTTP-запросов, работа с буферами.
- 📤 Отдача файлов (модуль 17), возможно через `mmap`.

**Прокачивает:** сеть, многопоточность, буферы, реальную архитектуру серверов.

---

## 🏆 Проект 5 — Мини-shell (командная оболочка)

Свой `bash`: читает команды, запускает программы.

**Что внутри:**
- 🔄 Цикл чтения команд, разбор аргументов.
- 🚀 **fork + exec** для запуска программ (модуль 23).
- 🔗 Пайпы (`ls | grep x`), перенаправление (`> file`).
- 📜 Встроенные команды (`cd`, `exit`, история).

**Прокачивает:** процессы, системные вызовы, парсинг.

## 📋 Требования к ЛЮБОМУ финальному проекту

- [ ] Разбит на модули (`.h`/`.c`), собирается через Makefile или CMake.
- [ ] Ведётся в **git** с осмысленными коммитами.
- [ ] **Ноль** утечек и ошибок под ASan/UBSan.
- [ ] Есть набор тестов (хотя бы базовых).
- [ ] README с описанием, сборкой и примерами.
- [ ] Обработка ошибок (не падает на кривом вводе/нехватке памяти).

## 🎓 Что дальше после C

Ты понимаешь память глубже большинства разработчиков. Следующие шаги курса:

- **C++** — RAII, умные указатели, контейнеры поверх той же ручной памяти.
- **Rust** — безопасность памяти без сборщика мусора (borrow checker). После C ты
  оценишь, зачем он нужен.
- **Go / Python** — управляемая память; теперь ты понимаешь, что GC делает «под капотом».
