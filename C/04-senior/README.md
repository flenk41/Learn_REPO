## 📂 Подробнее о модулях

### 19 · Свои аллокаторы памяти
- зачем писать свой аллокатор (скорость, контроль)
- **arena / bump allocator** 🖼️ — выделение «вперёд», освобождение всё сразу
- **pool allocator** 🖼️ — пул блоков одного размера
- free-list, фрагментация
- 🚀 проект: написать свой `malloc`/`free` поверх большого буфера

### 20 · Выравнивание, padding и UB
- 🖼️ выравнивание (alignment) и padding в структурах — почему `sizeof` больше суммы полей
- `alignof`, `alignas`, упаковка структур
- что такое **undefined behavior** и почему он опасен
- `const`, `volatile`, `restrict`
- строгий алиасинг (strict aliasing)

### 21 · Многопоточность и атомарность
- потоки (`pthreads` / C11 `threads.h`)
- 🖼️ гонки данных (data races) — почему два потока ломают память
- мьютексы, блокировки
- атомарные операции, `_Atomic`
- memory ordering (модель памяти C11) — продвинутая тема
- 🚀 проект: многопоточный счётчик / пул потоков

### 22 · Кэш и оптимизация
- 🖼️ иерархия памяти: регистры → L1/L2/L3 → RAM
- cache lines, локальность данных
- почему обход массива по строкам быстрее, чем по столбцам
- структура «массив структур» vs «структура массивов» (SoA/AoS)
- профилирование, измерение, `perf`
- 🚀 проект: оптимизировать обработку большого массива, замерить ускорение

### 23 · Системное программирование
- системные вызовы, `mmap`
- работа с процессами, `fork`/`exec` (Linux)
- сокеты, основы сети
- 🚀 проект: мини-shell / простой HTTP-сервер

### 24 · Отладка и профилирование (продвинуто)
- GDB глубоко: watchpoints, условные точки, core dump
- профилировщики: `gprof`, `perf`, Valgrind `callgrind`
- санитайзеры: ASan, UBSan, TSan, MSan
- статический анализ: `clang-tidy`, `cppcheck`
