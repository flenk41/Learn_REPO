## 📖 Системный вызов — просьба к ядру ОС

Программа не может напрямую трогать железо, диск, сеть. Она **просит** ядро ОС через
**системный вызов** (syscall). `printf`, `malloc`, `fopen` — все они в итоге вызывают
syscalls.

🖼️

```
  Твоя программа (user space)
        │  системный вызов (write, read, mmap...)
        ▼
  ┌───────────────────┐
  │   Ядро ОС (kernel) │  ← управляет памятью, диском, сетью, процессами
  └───────────────────┘
        │
        ▼
     Железо
```

Низкоуровневый ввод-вывод (без буферизации stdio):
```c
#include <unistd.h>
write(1, "Hello\n", 6);    // 1 = stdout, напрямую в ядро (так работает printf внутри)
```

## ⭐ mmap — память напрямую от ядра

Помнишь карту памяти (модуль 08)? `malloc` для больших блоков сам зовёт **`mmap`** —
системный вызов, который просит у ядра целые страницы памяти (обычно по 4 КБ).

```c
#include <sys/mman.h>

size_t size = 1024 * 1024;     // 1 МБ
void *mem = mmap(NULL, size,
                 PROT_READ | PROT_WRITE,        // права: чтение+запись
                 MAP_PRIVATE | MAP_ANONYMOUS,   // приватная, не из файла
                 -1, 0);
if (mem == MAP_FAILED) { /* ошибка */ }

// ... используем mem как обычную память ...

munmap(mem, size);             // вернуть ядру
```

💡 `mmap` — это «сырая» память от ОС, на которой можно построить **свой аллокатор**
(модуль 19). Так и работают настоящие malloc-и внутри.

### mmap файла — файл как массив в памяти

```c
int fd = open("data.bin", O_RDONLY);
struct stat st; fstat(fd, &st);
char *data = mmap(NULL, st.st_size, PROT_READ, MAP_PRIVATE, fd, 0);
// теперь к файлу можно обращаться как к массиву: data[i] !
```

🖼️ ОС «проецирует» файл в адресное пространство — читаешь память, а ОС подгружает с диска:

```
  файл на диске  ──mmap──►  data[0], data[1], ... (как обычный массив)
                            ОС сама подгружает нужные куски
```

## 📖 Процессы: fork / exec

`fork()` создаёт **копию** текущего процесса. `exec()` заменяет процесс другой программой.
Так работают шеллы и запуск программ.

```c
#include <unistd.h>
#include <sys/wait.h>

pid_t pid = fork();            // создать дочерний процесс
if (pid == 0) {
    // код ДОЧЕРНЕГО процесса
    execlp("ls", "ls", "-l", NULL);   // заменить себя программой ls
} else {
    // код РОДИТЕЛЬСКОГО процесса
    wait(NULL);                // дождаться завершения ребёнка
}
```

🖼️

```
       родитель
          │ fork()
     ┌────┴────┐
  родитель   ребёнок (копия)
     │          │ exec("ls")
   wait      становится программой ls
```

## 📖 Сеть: сокеты (основы)

Сокет — это «конец трубы» для обмена данными по сети. Базовый TCP-сервер:

```
socket()  → создать сокет
bind()    → привязать к порту (например 8080)
listen()  → слушать входящие
accept()  → принять подключение клиента
read()/write() → обмен данными
close()   → закрыть
```

🖼️

```
  СЕРВЕР                          КЛИЕНТ
  socket→bind→listen→accept ◄──── socket→connect
         │                              │
         └────── read / write ──────────┘  (обмен байтами)
```

💡 Поверх этого строятся HTTP-серверы, чаты, игры. Данные ходят как **байты** — снова
работа с буферами памяти.

## 📖 Сборка больших проектов: CMake

`make` хорош для маленьких проектов. Для больших и кроссплатформенных используют **CMake**:

```cmake
cmake_minimum_required(VERSION 3.10)
project(MyApp C)
set(CMAKE_C_STANDARD 11)
add_executable(myapp main.c utils.c)
```

```bash
cmake -B build
cmake --build build
```

CMake генерирует сборку под любую систему (Make, Ninja, Visual Studio).

## ✅ Задачи

> Делай в Linux или WSL.

1. **write напрямую.** Выведи текст через `write(1, ...)` без `printf`. Сравни.
2. **mmap-память.** Выдели 1 МБ через `mmap`, заполни, освободи через `munmap`.
3. **cat через mmap.** Реализуй упрощённый `cat`: спроецируй файл через `mmap` и выведи.
4. **fork.** Создай дочерний процесс, пусть родитель и ребёнок печатают разные сообщения.
   Используй `wait`.
5. **Запуск программы.** Через `fork` + `exec` запусти системную команду (`ls`, `date`).
6. ⭐ **Эхо-сервер.** TCP-сервер, который принимает подключение и возвращает обратно всё,
   что прислал клиент. Проверь через `telnet`/`nc`.
7. ⭐⭐ **CMake.** Переведи любой свой многофайловый проект с Makefile на CMake.
