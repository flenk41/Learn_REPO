# 00 · Как устроен компьютер и память 🖼️

> 🎯 **Цель блока:** понять, что такое память, байт и адрес — без этого C не имеет смысла.
> Это самый важный фундамент всего курса. Читай не спеша.

---

## 📖 Компьютер — это очень быстрый, но очень тупой исполнитель

Компьютер умеет всего три вещи:
1. **Хранить** числа (в памяти).
2. **Считать** (складывать, вычитать — в процессоре).
3. **Перекладывать** числа из одного места в другое.

Всё остальное — игры, браузер, музыка — это комбинации этих трёх действий,
выполненные миллиарды раз в секунду.

---

## 🖼️ Что такое память (RAM)

Представь **очень длинную улицу из одинаковых ячеек-домиков**. Каждый домик:
- хранит **ровно один байт** (число от 0 до 255);
- имеет уникальный **адрес** (номер домика): 0, 1, 2, 3, …

```
Адрес:   0      1      2      3      4      5      6      7
       +------+------+------+------+------+------+------+------+
RAM:   |  72  |  101 |  108 |  108 |  111 |   0  |  255 |  42  |
       +------+------+------+------+------+------+------+------+
          ^                                              ^
          |                                              |
     первый байт                                   восьмой байт
```

💡 **Главная идея:** память — это просто **пронумерованный ряд байтов**.
Когда ты создаёшь переменную, ОС выделяет тебе несколько таких ячеек.
Когда работаешь с указателями (дальше в курсе) — ты работаешь именно с **адресами** этих ячеек.

---

## 🔢 Бит, байт и почему 0–255

- **Бит** — самая маленькая единица. Это 0 или 1 (выключено/включено).
- **Байт** — это 8 бит подряд.

Сколько разных значений можно записать в 8 бит? Каждый бит — 2 варианта, битов 8:

```
2 × 2 × 2 × 2 × 2 × 2 × 2 × 2 = 2⁸ = 256 вариантов
```

Значит байт хранит число от **0 до 255**. Пример байта:

```
бит:    1   0   0   1   0   0   0   1
вес:  128  64  32  16   8   4   2   1
        ^                           ^
      128 + 16 + 1 = 145
```

📐 **Единицы измерения памяти:**

| Единица | Сколько |
|---------|---------|
| 1 байт (byte) | 8 бит |
| 1 КБ (KB) | 1024 байта |
| 1 МБ (MB) | 1024 КБ |
| 1 ГБ (GB) | 1024 МБ |

Когда говорят «у меня 16 ГБ оперативки» — это значит ≈ 17 миллиардов домиков-байтов.

---

## 🧠 RAM против диска — важное различие

| | **Оперативная память (RAM)** | **Диск (SSD/HDD)** |
|--|------------------------------|--------------------|
| Скорость | 🚀 очень быстрая | 🐢 в сотни раз медленнее |
| Объём | меньше (8–32 ГБ) | больше (256 ГБ – терабайты) |
| Что хранит | данные **работающих** программ | файлы, которые лежат «навсегда» |
| При выключении | ❌ всё стирается | ✅ сохраняется |

💡 Когда программа **запущена**, её код и данные живут в **RAM**. Именно с RAM ты будешь
работать в C через переменные и указатели. Когда программа закрывается — её память
освобождается.

## ⚙️ Процессор (CPU) и регистры

Процессор — это «калькулятор», который берёт числа из памяти, что-то с ними делает
и кладёт обратно. У него есть свои сверхбыстрые ячейки — **регистры** (их очень мало,
но они мгновенные).

```
        ┌─────────────┐         читает/пишет        ┌──────────────┐
        │     CPU     │ <─────────────────────────> │   Память     │
        │  регистры   │        по адресам           │   (RAM)      │
        │  [AX][BX]…  │                             │  [0][1][2]…  │
        └─────────────┘                             └──────────────┘
```

Иерархия скорости (от самого быстрого к медленному):

```
Регистры CPU  >  Кэш L1/L2/L3  >  RAM  >  SSD  >  HDD
  ⚡ 1 такт      ⚡ ~3-40 тактов   ~100      медленно   очень медленно
```

📌 Эту иерархию мы подробно разберём в Senior-блоке про **кэш и оптимизацию** —
именно понимание памяти отличает быстрый код от медленного.

## 🤔 Зачем всё это знать для C

В языках вроде Python память спрятана — ты не видишь адреса и байты.
В **C ты работаешь с памятью напрямую**:

- объявил переменную → выделились байты в RAM;
- взял `&переменная` → получил её **адрес** (номер домика);
- использовал указатель → читаешь/пишешь по адресу;
- забыл освободить память → **утечка**.

Поэтому без модели «память = ряд пронумерованных байтов» дальше никак.
Запомни эту картинку — мы будем возвращаться к ней постоянно. 🧠
