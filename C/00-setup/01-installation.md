# 01 · Установка компилятора и редактора 🛠️

> 🎯 **Цель блока:** установить всё необходимое и убедиться, что компилятор работает.
> В конце ты напечатаешь команду и увидишь версию компилятора — это значит «всё готово».

---

## 📖 Что нам нужно

Чтобы писать на C, нужны **две вещи**:

| Что | Зачем | Аналогия |
|-----|-------|----------|
| **Компилятор** | превращает твой текст-код в программу, которую понимает процессор | «переводчик» с C на язык машины |
| **Редактор кода** | удобно писать и подсвечивать код | «умный блокнот» |

Мы возьмём:
- Компилятор **GCC** (через пакет MinGW-w64) — самый популярный для C.
- Редактор **Visual Studio Code** — бесплатный, удобный, под все системы.

## 🪟 Установка на Windows (твоя система)

### Шаг 1. Установить компилятор через MSYS2

Самый надёжный способ получить свежий GCC на Windows — **MSYS2**.

1. Зайди на сайт **https://www.msys2.org**
2. Скачай установщик (кнопка вида `msys2-x86_64-XXXXXXXX.exe`).
3. Запусти, оставь путь по умолчанию `C:\msys64`, жми **Next** до конца.
4. После установки откроется чёрное окно терминала **MSYS2**.

> 🖼️ Терминал MSYS2 выглядит как обычная командная строка с приглашением вида:
> ```
> user@DESKTOP MSYS ~
> $
> ```

### Шаг 2. Установить пакет с GCC

В открытом окне MSYS2 введи команду и нажми Enter:

```bash
pacman -S mingw-w64-ucrt-x86_64-gcc
```

Когда спросит `Proceed with installation? [Y/n]` — нажми `Y` и Enter.
Дождись окончания загрузки.

### Шаг 3. Добавить компилятор в PATH (чтобы вызывать из любого места)

«PATH» — это список папок, где Windows ищет программы. Добавим туда папку с GCC.

1. Нажми **Win** → набери `переменные среды` → открой
   **«Изменение системных переменных среды»**.
2. Кнопка **«Переменные среды…»**.
3. В разделе **«Системные переменные»** найди переменную `Path` → **Изменить**.
4. **Создать** → вставь путь:
   ```
   C:\msys64\ucrt64\bin
   ```
5. **ОК** во всех окнах.

> 🖼️ Схема того, что произошло:
> ```
> Ты в терминале пишешь:  gcc
>          │
>          ▼
> Windows ищет "gcc.exe" по папкам из PATH ──► C:\msys64\ucrt64\bin\gcc.exe ✅ нашёл!
> ```

### Шаг 4. Проверить установку ✅

Открой **новую** командную строку (PowerShell): нажми **Win**, набери `powershell`, Enter.
Введи:

```powershell
gcc --version
```

Если видишь что-то вроде:

```
gcc.exe (Rev3, Built by MSYS2 project) 14.2.0
Copyright (C) 2024 Free Software Foundation, Inc.
```

🎉 **Поздравляю, компилятор установлен!**

> ⚠️ Если пишет `gcc не является внутренней или внешней командой` — значит PATH
> прописан неверно или ты не открыл **новое** окно терминала. Закрой все окна
> PowerShell, открой заново и повтори. Проверь, что путь `C:\msys64\ucrt64\bin`
> существует и в нём есть файл `gcc.exe`.

## 🍎 Установка на macOS (если вдруг)

```bash
# Установить инструменты разработчика (включают clang — совместим с gcc)
xcode-select --install

# Проверить
clang --version
```

В курсе можешь везде вместо `gcc` писать `clang` — команды одинаковые.

---

## 🐧 Установка на Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install build-essential gdb
gcc --version
```

## 💻 Установка редактора VS Code

1. Скачай с **https://code.visualstudio.com**
2. Установи (галочку *«Добавить в PATH»* и *«Открывать через контекстное меню»* — оставь включёнными).
3. Открой VS Code → слева иконка **Extensions** (квадратики) → найди и установи:
   - **C/C++** (от Microsoft) — подсветка, автодополнение, отладка.
   - *(по желанию)* **Code Runner** — запуск кода одной кнопкой.

> 🖼️ После установки расширения C/C++ редактор будет подсвечивать ошибки прямо во время
> написания кода — красные подчёркивания под проблемными местами.

---

## 🗂️ Создай рабочую папку

Договоримся хранить весь код курса в одном месте. Создай папку, например:

```
D:\LearnProject\my-code\C\
```

Внутри будем создавать по папке на каждый блок. Открывай эту папку в VS Code через
**Файл → Открыть папку**.
