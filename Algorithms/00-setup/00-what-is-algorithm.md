# 00 · Что такое алгоритм и структура данных 🖼️

> 🎯 **Цель блока:** понять два фундаментальных понятия — алгоритм и структура данных — и
> почему они неразделимы.

---

## 📖 Алгоритм — рецепт решения задачи

**Алгоритм** — чёткая последовательность шагов, которая из входных данных получает результат.
Как рецепт: конкретные действия в конкретном порядке, ведущие к цели.

```
   задача: найти максимум в списке чисел
   алгоритм: возьми первое как «текущий максимум»;
             пройди по остальным; если больше — обнови максимум;
             верни максимум
```

💡 Один и ту же задачу можно решить **разными** алгоритмами — и они будут отличаться по
**скорости** и **памяти**. Искусство — найти не просто работающий, а **эффективный** алгоритм
(ядро трека, уровень 2).

---

## 📖 Структура данных — как хранятся данные

**Структура данных** — способ организации данных в памяти, чтобы с ними было удобно и быстро
работать.

```
   массив      — данные подряд, доступ по индексу     [a][b][c][d]
   связный список — узлы, ссылающиеся друг на друга    [a]→[b]→[c]
   хеш-таблица  — ключ → значение, мгновенный доступ   {ключ: значение}
   дерево       — иерархия (родитель → дети)
   граф         — сеть связей
```

💡 ⭐ Структура определяет, какие операции **быстрые**, а какие **медленные**. В массиве доступ
по индексу мгновенный, но вставка в середину медленная. В связном списке — наоборот. Выбор
структуры под задачу — половина успеха.

---

## ⭐ Алгоритмы и структуры неразделимы

```
   «Алгоритмы + структуры данных = программы» (Никлаус Вирт)
```

🖼️
```
   ЗАДАЧА
     ├─ какие ДАННЫЕ? → выбираем СТРУКТУРУ (как хранить)
     └─ какие ОПЕРАЦИИ? → выбираем АЛГОРИТМ (как обрабатывать)
   правильная структура часто ДЕЛАЕТ алгоритм простым и быстрым
```

💡 ⭐ Они работают в паре: правильная структура данных может превратить сложную задачу в
тривиальную. Например, проверка «есть ли элемент» — медленная в списке (перебор), мгновенная в
хеш-таблице. Часто «оптимизация алгоритма» — это на самом деле **смена структуры данных**.

---

## 📖 Зачем это разработчику

```
   💼 технические собеседования — алгоритмы спрашивают почти везде
   ⚡ эффективность — код, работающий на 100 элементах, может «лечь» на миллионе
   🧠 мышление — учит раскладывать задачи и оценивать решения
   🏗️ фундамент — БД, поисковики, ИИ, игры — всё построено на алгоритмах и структурах
```

💡 Это не «академическая теория ради теории». Понимание алгоритмов напрямую влияет на то,
сможешь ли ты написать код, который работает **на реальном масштабе** данных.

---

## ⚠️ Заблуждения

- ❌ «Алгоритмы не нужны, есть готовые библиотеки». Библиотеки **состоят** из алгоритмов; чтобы
  выбрать правильную и не злоупотребить — надо понимать, что внутри.
- ❌ «Главное — чтобы работало». На масштабе «работает медленно» = «не работает».
- ❌ «Структура данных — это просто хранилище». Она определяет скорость всех операций.

## 🛠️ Практика

1. Запиши алгоритм поиска максимума своими словами по шагам.
2. Для задачи «проверить, есть ли число в коллекции» подумай: как это в списке и как в
   хеш-таблице (множестве)? Где быстрее?
3. Приведи 3 примера структур данных из жизни (очередь в магазине, дерево папок, …).
