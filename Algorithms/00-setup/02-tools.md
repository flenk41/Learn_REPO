# 02 · Где практиковать алгоритмы 🛠️

> 🎯 **Цель блока:** выбрать язык и площадки для практики, и понять, как **правильно** решать
> задачи, чтобы реально учиться, а не просто «прорешать».

---

## 📖 Алгоритмы — на любом языке

Алгоритмы и структуры **не зависят от языка** — это идеи. Но для практики удобнее язык с
читаемым синтаксисом и готовыми структурами «из коробки».

```
   🐍 Python  — минимум синтаксиса, фокус на идеях (примеры трека — в этом стиле); list, dict, set
   ☕ Java / C# — строгая типизация, популярны на собеседованиях
   ➕ C++     — если нужна скорость и контроль (есть STL: vector, map, set)
   🦀 Rust    — безопасность + скорость (если уже знаешь)
```

💡 Для **изучения** алгоритмов Python — отличный выбор: меньше отвлекаешься на синтаксис, больше
думаешь об идее. Для собеседований в конкретную компанию узнай, какие языки они принимают.

⚠️ Полезно понимать, что готовые `list`/`dict` внутри — это структуры данных со своей сложностью
(уровень 2). Использовать `dict` для «быстрого поиска» — значит понимать, что это хеш-таблица.

## 🛠️ Площадки для практики

```
   📗 LeetCode      — главная площадка; задачи по темам и сложности (Easy/Medium/Hard)
   🥋 Codewars      — задачи-«ката», уровни kyu, можно по языку
   🎓 HackerRank    — структуры данных и алгоритмы отдельными треками
   🇷🇺 Stepik / e-olymp — курсы и задачи на русском
   🏆 Codeforces    — спортивное программирование (для продвинутых)
```

💡 Для старта: **LeetCode** (раздел Easy, потом Medium), фильтруя по теме (Array, Hash Table,
Two Pointers…). Решай **по темам**, а не случайно — так закрепляешь паттерны (уровень 4).

---

## ⭐ Как решать задачи ПРАВИЛЬНО

Просто «прорешать сотню задач» — не значит научиться. Метод важнее количества:

```
   1. ПОЙМИ задачу — что на входе, что на выходе, крайние случаи (пустой ввод, дубли)
   2. ПРИДУМАЙ решение «в лоб» (brute force) — даже медленное, главное правильное
   3. ОЦЕНИ его сложность (Big-O) — а можно ли быстрее? (уровень 2)
   4. УЛУЧШИ — какая структура/приём ускорит? (хеш? два указателя? сортировка?)
   5. НАПИШИ и проверь на крайних случаях
   6. РАЗБЕРИ чужие решения после своего — учись новым приёмам
```

💡 ⭐ Ключевое: **сначала своё решение, потом подсмотреть**. Если сразу читать ответ — учишься
узнавать, а не решать. И всегда **оценивай сложность** своего решения — это главный навык трека.
Застрял на 30–40 минут — подсмотри **идею** (не весь код), потом допиши сам.

---

## 📖 Не гонись за количеством

```
   ❌ «прорешал 300 задач, ничего не понял» — поверхностно, забывается
   ✅ «решил 50 задач глубоко, понял паттерны» — переносится на новые задачи
```

💡 Лучше глубоко разобрать **меньше** задач (понять, почему решение работает, какие паттерны,
какая сложность), чем поверхностно прорешать много. Цель — научиться **думать**, а не запомнить
ответы.

---

## ⚠️ Ловушки

- ❌ Читать решение, не попробовав сам — учишься узнавать, а не решать.
- ❌ Решать случайные задачи вместо тем — паттерны не закрепляются.
- ❌ Гнаться за количеством в ущерб пониманию.
- ❌ Не оценивать сложность своего решения (главный пропускаемый шаг).

## 🛠️ Практика

1. Заведи аккаунт на LeetCode (или аналоге). Реши 2–3 задачи раздела Easy на массивы/строки.
2. Для каждой: сначала своё решение, оцени его Big-O, потом посмотри чужие — нашёл лучше?
3. Выбери язык для трека (рекомендуется Python) и убедись, что знаешь его `list`, `dict`, `set`.
