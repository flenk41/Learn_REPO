# 🚀 Пет-проект уровня 3 — «Библиотека алгоритмов»

> 🎯 **Цель:** реализовать классические алгоритмы своими руками — лучший способ понять их
> устройство, сложность и применимость.

---

## 📋 Что делаем

Собираем собственную «библиотеку» классических алгоритмов с тестами, анализом сложности и
демонстрацией применения.

### Алгоритмы для реализации

1. **Поиск:** линейный (O(n)) и бинарный (O(log n)). Покажи на замерах разницу шагов.
2. **Сортировки:** одна простая (вставками, O(n²)) и одна эффективная (слиянием, O(n log n)).
   Сравни время на растущих данных; сравни с встроенной.
3. **Рекурсия + ДП-намёк:** fib наивный (O(2ⁿ)) и с мемоизацией (O(n)). Покажи разницу.
4. **Дерево (BST):** вставка, поиск, удаление, три обхода. Покажи, что in-order = сортировка.
5. **Граф:** список смежности + BFS и DFS. Примени BFS для кратчайшего пути, DFS для связности.

### Требования

- Для **каждого** алгоритма укажи время и память (Big-O).
- Напиши тесты (проверка на разных входах, включая крайние случаи).
- Для каждого — **где применяется** (1-2 примера реальных задач).

### Демонстрационная задача

Выбери одну задачу, которую решаешь **двумя** алгоритмами из библиотеки, и сравни (например,
«найти, есть ли путь» через BFS и DFS; или «отсортировать и найти» бинарным поиском).

### Отчёт

`README.md`: код алгоритмов, таблица «алгоритм → сложность → применение», результаты замеров,
демонстрация выбора алгоритма под задачу.

- [ ] Есть тесты и замеры, подтверждающие сложность
