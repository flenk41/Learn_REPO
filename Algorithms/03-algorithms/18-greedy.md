# 18 · Жадные алгоритмы и приёмы 🖼️⭐

> 🎯 **Цель блока:** познакомиться с жадными алгоритмами — когда «брать лучшее сейчас» работает,
> а когда нет — и закрепить хеширование как универсальный ускоритель.

---

## ⭐ Жадный алгоритм — «бери лучшее сейчас»

**Жадный алгоритм** на каждом шаге делает выбор, который кажется **лучшим в данный момент**,
надеясь, что это приведёт к лучшему общему решению.

```
   пример: размен монетами 50/10/5/1 — бери самую крупную, что влезает
   нужно 67: 50 + 10 + 5 + 1 + 1 = 5 монет → жадный даёт ОПТИМУМ (для таких монет)
```

💡 ⭐ Жадный подход **прост и быстр** (обычно O(n) или O(n log n) с сортировкой). Но ⚠️ он работает
**не всегда**! Локально лучший выбор не гарантирует глобально лучший. Жадный даёт оптимум только
для задач с особым свойством (когда локальный оптимум ведёт к глобальному).

---

## ⚠️ Когда жадность подводит

```
   ❌ размен «странными» монетами 1/3/4, нужно 6:
      жадный: 4 + 1 + 1 = 3 монеты
      оптимум: 3 + 3 = 2 монеты!  → жадный ОШИБСЯ

   для таких задач нужно динамическое программирование (уровень 4)
```

💡 ⚠️ Главный навык — понять, **можно ли** применить жадность к конкретной задаче. Иногда да
(минимальное остовное дерево, расписание, обычные монеты), иногда нет (нужен полный перебор или
ДП). Проверяй на контрпримерах: всегда ли локально лучшее даёт глобально лучшее? Если нашёл
контрпример — жадность не подходит.

---

## ⭐ Когда жадность работает

```
   ✅ выбор активностей (максимум непересекающихся встреч) — бери с ранним концом
   ✅ минимальное остовное дерево (Краскал, Прим)
   ✅ кратчайшие пути Дейкстры (уровень 4) — жадный по расстоянию
   ✅ когда «локально лучшее = шаг к глобально лучшему» (доказуемо)
```

💡 ⭐ Жадные алгоритмы — мощны там, где применимы: просты и эффективны. Классические жадные задачи
(расписание, остовное дерево) стоит знать. Но всегда **обоснуй**, почему жадность даёт оптимум для
твоей задачи (или проверь на примерах).

---

## ⭐⭐ Хеширование — универсальный ускоритель (повторение)

Из модуля 06: хеш-таблица превращает многие O(n²) в O(n). Это **самый частый** приём решения
задач — закрепим как метод:

```
   паттерн: «для каждого элемента быстро проверить/найти что-то о других элементах»
   → используй хеш-таблицу/множество для O(1) проверки

   примеры:
   - «есть ли дубликаты» → set, O(n)
   - «два числа с суммой target» → хеш, O(n)
   - «частоты», «группировки», «видел ли уже» → хеш-таблица
```

💡 ⭐⭐ При виде задачи спрашивай: **«поможет ли здесь хеш-таблица?»**. Очень часто ответ — да, и
она даёт лучшее решение. Это первый инструмент в арсенале. Память O(n) обычно того стоит
(компромисс время/память, модуль 12).

---

## 📖 Обзор: как выбрать подход к задаче

```
   нужен быстрый поиск/проверка наличия → ХЕШ-ТАБЛИЦА
   данные отсортированы / можно отсортировать → БИНАРНЫЙ ПОИСК / два указателя
   связи между объектами → ГРАФ (BFS/DFS)
   иерархия / упорядоченные данные с вставками → ДЕРЕВО (BST)
   «лучшее на каждом шаге» (и это работает) → ЖАДНЫЙ
   перебор с пересчётом подзадач → ДИНАМИЧЕСКОЕ ПРОГРАММИРОВАНИЕ (уровень 4)
```

💡 Это карта инструментов. Распознать **тип** задачи → выбрать подход. Этот навык — суть
алгоритмического мышления, и он оттачивается практикой (уровень 4: паттерны).

---

## ⚠️ Ловушки

- ❌ Применять жадность, не проверив, даёт ли она оптимум (контрпримеры!).
- ❌ Не замечать, что задача решается хеш-таблицей (главный пропускаемый приём).
- ❌ Считать жадность «всегда оптимальной» — она часто ошибается.
- ❌ Брать сложный подход (ДП), когда хватает простого (жадный/хеш).

---

## 🛠️ Практика

1. Реши «выбор активностей» (максимум непересекающихся встреч) жадно — бери с ранним концом.
2. Найди контрпример, где жадный размен монетами даёт не оптимум (монеты 1/3/4, сумма 6).
3. Реши 3 задачи через хеш-таблицу, превратив наивные O(n²) в O(n).
