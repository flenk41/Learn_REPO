# 05 · Стек и очередь 🖼️⭐

> 🎯 **Цель блока:** освоить стек (LIFO) и очередь (FIFO) — простые, но фундаментальные
> структуры, лежащие в основе множества алгоритмов.

---

## ⭐ Стек — «последним пришёл, первым вышел» (LIFO)

**Стек** (stack) — структура, где добавляешь и забираешь элементы **только с одного конца**
(вершины). Последний добавленный — первый извлекаемый.

🖼️
```
   push(1) push(2) push(3):        pop():
        ┌───┐                       вернёт 3 (последний вошёл)
        │ 3 │ ← вершина (top)            ┌───┐
        ├───┤                            │ 2 │ ← теперь вершина
        │ 2 │                            ├───┤
        ├───┤                            │ 1 │
        │ 1 │                            └───┘
        └───┘
```

```
   push(x) — положить на вершину      O(1)
   pop()   — снять с вершины          O(1)
   peek()  — посмотреть вершину       O(1)
```

💡 ⭐ Аналогия — стопка тарелок: кладёшь и берёшь сверху. Все операции **O(1)**. Знакомо? Это
**стек вызовов** из языков курса — при вызове функции её кадр кладётся на стек, при возврате —
снимается ([модель памяти C](../../C/02-memory/08-memory-model.md), [процессы ОС](../../OS/01-processes/03-process.md)).

```
   где применяют стек:
   📞 стек вызовов функций (рекурсия)
   ↩️ отмена действий (Ctrl+Z) — история как стек
   () проверка скобок, разбор выражений
   🔍 обход графа в глубину (DFS, модуль 17)
```

---

## ⭐ Очередь — «первым пришёл, первым вышел» (FIFO)

**Очередь** (queue) — добавляешь с одного конца (хвост), забираешь с другого (голова). Первый
вошёл — первый вышел.

🖼️
```
   enqueue: добавляем в ХВОСТ →  [1][2][3]  ← dequeue: забираем из ГОЛОВЫ (вернёт 1)
            (последний)            ↑голова
```

```
   enqueue(x) — добавить в хвост     O(1)
   dequeue()  — забрать из головы    O(1)
```

💡 ⭐ Аналогия — очередь в магазине: кто встал первым, того и обслужат первым. Применения:

```
   🖨️ очередь задач (печать, обработка)
   🌐 обработка запросов на сервере (трек Сети/ОС)
   🔍 обход графа в ширину (BFS, модуль 17)
   ⚖️ планировщик процессов (трек ОС)
```

---

## 📖 Как реализуют

```
   СТЕК   — легко на динамическом массиве (push/pop в конец — O(1)) или на связном списке
   ОЧЕРЕДЬ — на связном списке (добавление в хвост, удаление из головы — O(1))
            или на массиве-кольце (circular buffer)
   ⚠️ очередь на обычном массиве с удалением из начала = O(n) (сдвиг) — так НЕ делают
```

💡 Стек и очередь — это **абстракции** (как они себя ведут), а внутри — массив или список. В
Python: стек — обычный `list` (append/pop), очередь — `collections.deque` (а не list — иначе
dequeue из начала будет O(n)).

---

## ⭐ Дек (deque) — двусторонняя очередь

```
   DEQUE — можно добавлять/забирать с ОБОИХ концов, всё O(1)
   универсальнее: может работать и как стек, и как очередь
```

💡 `deque` (double-ended queue) — практичный инструмент: используется как быстрая очередь и стек
одновременно. В Python — `collections.deque`.

---

## ⚠️ Ловушки

- ❌ Реализовать очередь на массиве с `pop(0)` / удалением из начала — это O(n), убивает
  производительность.
- ❌ Путать LIFO (стек) и FIFO (очередь).
- ❌ Забыть проверить пустоту перед pop/dequeue (ошибка на пустой структуре).
- ❌ Использовать стек там, где нужен порядок очереди (и наоборот).
