# 04 · Связные списки 🖼️⭐

> 🎯 **Цель блока:** понять связный список — структуру, где элементы **разбросаны** в памяти и
> связаны ссылками, — и чем он отличается от массива.

---

## ⭐ Связный список — узлы со ссылками

**Связный список** — последовательность **узлов**, где каждый узел хранит значение и **ссылку**
(указатель) на следующий узел. В отличие от массива, узлы **не лежат подряд** — они разбросаны по
памяти.

🖼️
```
   МАССИВ:   [10][20][30][40]   ← подряд в памяти, доступ по индексу

   СВЯЗНЫЙ СПИСОК:
   [10|•]──►[20|•]──►[30|•]──►[40|null]
    значение↑  ↑ссылка на следующий
   узлы где угодно в памяти, связаны ссылками
```

```
   узел = { значение, ссылка_на_следующий }
   список знает только ссылку на ПЕРВЫЙ узел (голову)
```

💡 ⭐ Ключевое отличие: нет индексов и непрерывности. Чтобы дойти до 3-го элемента — надо пройти
по ссылкам от головы. Зато вставка/удаление — это просто **переставить ссылки**, не сдвигая
ничего. Связь с указателями из [C](../../C/02-memory/09-pointers.md) и `unique_ptr`-списком из
[C++](../../Cpp/02-memory/11-smart-pointers.md) — это они.

---

## ⭐ Сложность: зеркало массива

```
   доступ к i-му элементу     → O(n)   надо пройти по ссылкам от головы 😟
   поиск элемента             → O(n)   перебор 😐
   вставка/удаление в НАЧАЛЕ  → O(1)   переставил пару ссылок ✅✅
   вставка/удаление в середине (если уже стоишь там) → O(1) ✅
```

💡 ⭐ Профиль списка — **зеркало** массива: медленный доступ по индексу, но **быстрая вставка/
удаление** (если знаешь, куда). Где массив силён (индекс), список слаб; где массив слаб (вставка в
середину), список силён. Выбор — от того, что ты делаешь чаще.

---

## 📖 Виды списков

```
   ОДНОСВЯЗНЫЙ   — узел ссылается только на СЛЕДУЮЩИЙ        [a]→[b]→[c]
   ДВУСВЯЗНЫЙ    — узел ссылается на следующий И предыдущий  [a]⇄[b]⇄[c]  (можно идти назад)
   КОЛЬЦЕВОЙ     — последний ссылается на первый (цикл)
```

💡 Двусвязный список позволяет ходить в обе стороны и удалять узел, зная только его (есть ссылка
назад) — ценой лишней ссылки в каждом узле. Большинство языковых «list» внутри — динамические
массивы, а связный список используют точечно, когда нужен его профиль.

---

## ⭐ Массив vs связный список — когда что

```
   МАССИВ                          СВЯЗНЫЙ СПИСОК
   ─────────────────────────────────────────────────────────
   доступ по индексу O(1)          доступ O(n)
   вставка в середину O(n)         вставка O(1) (если стоишь там)
   компактно, дружит с кэшем       разбросан, кэш-недружелюбен
   фиксированный/растёт копией      легко растёт (новый узел)

   нужен быстрый доступ → МАССИВ
   много вставок/удалений в начале/середине → СПИСОК
```

💡 ⭐ На практике **массив чаще побеждает**: O(1) доступ + кэш-дружелюбность часто перевешивают
теоретическое преимущество списка во вставке (потому что обход списка по разбросанной памяти
медленный из-за промахов кэша — привет, [иерархия памяти ОС](../../OS/02-memory/12-tlb-hierarchy.md)).
Связный список — фундамент для других структур (стеки, очереди, хеш-таблицы с цепочками).

---

## ⚠️ Ловушки

- ❌ Использовать список ради «быстрой вставки», когда чаще нужен доступ по индексу.
- ❌ Забыть переставить ссылки при удалении (потеря части списка или утечка).
- ❌ Зациклить ссылки случайно (бесконечный обход).
- ❌ Недооценивать кэш: обход списка медленнее обхода массива той же длины.

## 🛠️ Практика

1. Реализуй односвязный список: добавление в начало (O(1)), в конец, печать, удаление по значению.
2. Сравни: доступ к середине массива (O(1)) и к середине списка (O(n) обходом).
3. Реализуй разворот связного списка (классическая задача на переставление ссылок).
