# 🚀 Пет-проект уровня 1 — «Свои структуры данных»

> 🎯 **Цель:** реализовать базовые структуры данных самому — это лучший способ понять их
> устройство и сложность изнутри.

---

## 📋 Что делаем

Реализуем набор базовых структур с нуля (не используя готовые там, где учимся), с тестами и
анализом сложности каждой операции.

### Структуры для реализации

1. **Динамический массив** — `push` (с ростом ёмкости ×2), `get(i)`, `set(i)`, `pop`, `size`.
   Объясни амортизированную O(1) для push.
2. **Связный список** (односвязный) — `push_front`, `push_back`, `find`, `remove`, `reverse`,
   `print`. Объясни O(1) вставки в начало и O(n) доступа.
3. **Стек** (на массиве) — `push`, `pop`, `peek`, `is_empty`. Примени для проверки скобок.
4. **Очередь** (на деке/списке) — `enqueue`, `dequeue`. Примени для модели обслуживания.
5. **Хеш-множество** (упрощённое) — `add`, `contains`, `remove` через хеш-функцию + цепочки
   (связные списки в ячейках). Объясни O(1) в среднем и роль коллизий.

### Требования

- Для **каждой** операции каждой структуры укажи её **Big-O** (готовим уровень 2).
- Напиши простые тесты (добавил → проверил → удалил → проверил).
- Где уместно — сравни свою реализацию с готовой языковой по скорости.

### Применения (свяжи структуры с задачами)

- стеком — проверка скобок;
- очередью — обслуживание клиентов;
- хеш-множеством — убрать дубли / проверить наличие за O(1).

### Отчёт

`README.md`: код структур, таблица «операция → сложность» для каждой, тесты, и вывод — какая
структура для каких задач подходит.
