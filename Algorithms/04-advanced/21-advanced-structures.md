# 21 · Продвинутые структуры данных 🖼️⭐

> 🎯 **Цель блока:** познакомиться со структурами, решающими специальные задачи: куча (priority
> queue), префиксное дерево (trie), система непересекающихся множеств (union-find).

---

## ⭐ Куча / очередь с приоритетом (heap)

**Куча** — структура, дающая быстрый доступ к **минимуму** (или максимуму) набора. Реализует
**очередь с приоритетом**: элементы выходят не по порядку прихода, а по приоритету.

🖼️
```
   куча (мин-куча) — бинарное дерево, где родитель ≤ детей:
            [1]              минимум всегда наверху
           /   \
        [3]    [2]
        /  \
     [5]  [4]
```

```
   вставка        → O(log n)
   взять минимум  → O(log n)
   посмотреть мин → O(1)
```

💡 ⭐ Куча — когда нужно **многократно брать минимум/максимум** из меняющегося набора: Дейкстра
(модуль 20, берём ближайшую вершину), планировщик задач по приоритету, «топ-K элементов», слияние
отсортированных потоков. В Python — `heapq`, в C++ — `priority_queue`. Не путай: куча даёт быстрый
минимум, но **не** быстрый поиск произвольного элемента.

---

## ⭐ Префиксное дерево (trie)

**Trie** — дерево для хранения строк, где общие префиксы хранятся **один раз**. Каждый путь от
корня — это префикс.

🖼️
```
   слова: "cat", "car", "card"
          (root)
            │ c
            │ a
           ╱ ╲
          t   r ──► card
        (cat) (car)
   общий префикс "ca" хранится ОДИН раз
```

```
   поиск слова / префикса → O(длина слова), не зависит от числа слов!
```

💡 ⭐ Trie блестит для задач с **префиксами**: автодополнение (поиск всех слов на «ca»), словари,
проверка орфографии, поиск по началу. Поиск зависит от длины слова, а не от размера словаря.
Ценой памяти (хранит дерево символов) даёт быстрый префиксный поиск.

## ⭐ Система непересекающихся множеств (union-find)

**Union-Find** (DSU) отвечает на вопрос «**в одной ли группе** два элемента?» и умеет
**объединять** группы — очень быстро.

```
   операции:
   find(x)      — в какой группе элемент x
   union(x, y)  — объединить группы x и y
   обе ~O(1) (почти константа с оптимизациями)
```

💡 ⭐ Union-Find — для задач **группировки/связности**: «связаны ли два узла в сети», компоненты
связности графа, минимальное остовное дерево (Краскал), «друзья друзей в одной компании». Когда
задача про «объединять группы и проверять принадлежность» — это union-find.

---

## 📖 Когда нужны продвинутые структуры

```
   многократно брать минимум/максимум      → КУЧА (priority queue)
   префиксы, автодополнение, словари        → TRIE
   группировка, связность, объединение      → UNION-FIND
   диапазонные запросы (сумма на отрезке)   → дерево отрезков (segment tree, для интереса)
```

💡 ⭐ Эти структуры — специализированные инструменты. Не нужны для большинства задач, но для
**своих** задач они оптимальны, и базовыми структурами их не заменить эффективно. Знать об их
существовании и назначении — признак зрелости: при виде подходящей задачи вспомнишь нужный
инструмент.

## ⚠️ Ловушки

- ❌ Использовать кучу для поиска произвольного элемента (она про минимум/максимум).
- ❌ Решать задачи на префиксы перебором, когда trie даёт O(длины слова).
- ❌ Проверять связность перебором, когда union-find даёт ~O(1).
- ❌ Тащить сложную структуру, когда хватает базовой (бритва Оккама).

---

## 🛠️ Практика

1. Используй кучу (`heapq`) чтобы найти K наименьших/наибольших элементов в большом списке.
2. Построй простой trie для автодополнения: добавь слова, найди все на заданный префикс.
3. Используй union-find, чтобы посчитать число компонент связности в графе.
