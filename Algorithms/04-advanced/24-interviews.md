## 📖 Зачем алгоритмы на собеседованиях

Технические собеседования (особенно в крупные компании) почти всегда включают алгоритмические
задачи. Это проверка не «знаешь ли ты ответ», а **как ты думаешь**: разбираешь задачу, выбираешь
структуру/алгоритм, оцениваешь сложность, пишешь и тестируешь код.

💡 ⭐ Хорошая новость: собеседовательные задачи — это в основном **паттерны** (модуль 22) на
**базовых структурах** (уровень 1) с **анализом сложности** (уровень 2). Весь этот трек — и есть
подготовка. Дальше — практика по методу.

---

## ⭐ Метод решения задачи на собеседовании

```
   1. УТОЧНИ — переспроси условие, крайние случаи (пустой ввод? дубли? размер?)
   2. ПРИМЕРЫ — проговори на примере, что вход → что выход
   3. ПОДХОД «в лоб» — озвучь простое решение и его сложность (даже если медленное)
   4. УЛУЧШИ — «можно ли быстрее?» → паттерн/структура (хеш? два указателя? ДП?)
   5. ОЦЕНИ сложность улучшенного (время И память)
   6. ПИШИ код — аккуратно, проговаривая
   7. ТЕСТИРУЙ — прогони на примерах и крайних случаях
```

💡 ⭐⭐ Главное на собеседовании — **думать вслух**. Интервьюер оценивает процесс мышления, а не
только финальный код. Даже если застрял — проговаривай идеи. «Начну с brute force O(n²), потом
подумаю, как ускорить хешем» — это уже хороший сигнал.

## ⭐ План подготовки

```
   1. ОСНОВЫ — этот трек: структуры (L1), сложность (L2), алгоритмы (L3-4) ✅
   2. ПАТТЕРНЫ — реши по 5-10 задач каждого паттерна (модуль 22)
   3. ПО ТЕМАМ — LeetCode по тегам: Array, Hash, Two Pointers, Tree, Graph, DP...
   4. РАЗБОР — после своего решения смотри чужие, учись приёмам
   5. ИМИТАЦИЯ — решай на время, проговаривая вслух (как на собесе)
   6. РЕГУЛЯРНОСТЬ — лучше 1-2 задачи в день стабильно, чем рывками
```

💡 ⭐ Не «прорешать 500 задач», а **понять паттерны** на ~100-150 хорошо разобранных. Глубина
важнее количества (модуль 02). Решай по темам, разбирай, повторяй забытое.

## 📖 Итог трека: что ты теперь умеешь

```
   ✅ знаешь структуры данных и их сложность (массив, список, стек/очередь, хеш, дерево, граф)
   ✅ владеешь Big-O — анализируешь время И память любого кода (ЯДРО)
   ✅ знаешь классические алгоритмы (поиск, сортировки, рекурсия, обходы)
   ✅ умеешь ДП, продвинутые графы/структуры, паттерны решения
   ✅ понимаешь реальную производительность (кэш, профилирование)

   главный навык: распознать ТИП задачи → выбрать структуру/алгоритм → оценить сложность
```

💡 ⭐ Это и есть алгоритмическое мышление: не зубрить решения, а **видеть тип задачи** и применять
подходящий инструмент. Big-O — язык, на котором ты теперь обоснуешь любой выбор.

## ⭐ Куда расти дальше

```
   📗 продолжай LeetCode по темам (Medium → Hard постепенно)
   📚 книги: «Грокаем алгоритмы» (для начала), «Cracking the Coding Interview», CLRS (глубоко)
   🏆 соревновательное программирование (Codeforces) — для скорости и сложных тем
   🔬 специализация: алгоритмы для БД (индексы — следующий трек!), ИИ, графика, сети
   🛠️ применяй в работе: выбирай структуры осознанно, оценивай сложность кода
```

💡 Алгоритмы — фундамент, на котором строится всё остальное (БД, системы, ИИ). Связь с курсом:
индексы БД — это деревья; маршрутизация — графы; кэш — компромисс память/скорость. Понимание
алгоритмов делает тебя сильнее во всех областях.
