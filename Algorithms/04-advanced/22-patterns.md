# 22 · Паттерны решения задач 🖼️⭐⭐

> 🎯 **Цель блока:** освоить универсальные паттерны — два указателя, скользящее окно и др. —
> которые закрывают огромную долю алгоритмических задач (и собеседований).

---

## ⭐⭐ Паттерны — ключи к большинству задач

Большинство алгоритмических задач решаются **типовыми приёмами**. Узнал паттерн → знаешь
структуру решения. Это превращает «не знаю, как подступиться» в «это задача на скользящее окно».

```
   главные паттерны:
   👉👈 два указателя (two pointers)
   🪟 скользящее окно (sliding window)
   🐢🐰 быстрый и медленный указатель
   🔢 хеш-таблица (модули 06, 18)
   🔀 бинарный поиск / по ответу (модуль 13)
   🔙 backtracking (перебор с возвратом)
```

💡 ⭐⭐ Изучение паттернов — самый эффективный способ готовиться к собеседованиям и решать задачи
быстро. Разберём главные.

---

## ⭐ Два указателя (two pointers)

Два индекса двигаются по массиву/строке — навстречу или в одну сторону.

```
   ПАЛИНДРОМ: указатели с краёв к центру, сравниваем
   [a][b][c][b][a]
    ↑           ↑   → сравнили, сдвинули внутрь
   ОТСОРТИРОВАННАЯ ПАРА с суммой target: один в начале, один в конце
   мала сумма → двигай левый вправо; велика → правый влево → O(n) вместо O(n²)
```

💡 ⭐ Два указателя превращают многие O(n²) (двойной цикл) в **O(n)**. Работает на отсортированных
данных и симметричных задачах (палиндром, пары, слияние). Один проход, два бегунка.

---

## ⭐⭐ Скользящее окно (sliding window)

Окно (подотрезок) «скользит» по массиву/строке, расширяясь и сжимаясь — для задач о
**подотрезках/подстроках**.

```
   «максимальная сумма подмассива длины k» / «самая длинная подстрока без повторов»
   [a b c | d e f | g h]
          └ окно ┘ → двигаем правую границу (добавляем), левую (убираем) → O(n)
```

🖼️
```
   вместо пересчёта каждого подотрезка (O(n²)):
   двигаем окно, обновляя результат инкрементально → O(n)
```

💡 ⭐⭐ Скользящее окно — для задач «найти подотрезок/подстроку с условием». Вместо перебора всех
подотрезков (O(n²)) — двигаем границы окна, поддерживая текущее состояние → **O(n)**. Частый
паттерн на собеседованиях (подстроки, суммы, частоты в окне).

---

## ⭐ Быстрый и медленный указатель

```
   два указателя двигаются с РАЗНОЙ скоростью (один на 1, другой на 2 шага)
   ПРИМЕНЕНИЕ: поиск цикла в связном списке (если быстрый догонит медленного — есть цикл)
              найти середину списка (быстрый дошёл до конца — медленный в середине)
```

💡 Элегантный приём для связных списков и циклов (алгоритм Флойда «черепаха и заяц»). Решает за
O(n) время и O(1) память задачи, которые иначе требовали бы хеш-таблицы.

---

## 📖 Как распознать паттерн

```
   «пара/тройка с суммой», палиндром, отсортированный массив → ДВА УКАЗАТЕЛЯ
   «подстрока/подмассив с условием», «окно размера k»         → СКОЛЬЗЯЩЕЕ ОКНО
   цикл в списке, середина списка                             → БЫСТРЫЙ/МЕДЛЕННЫЙ
   «есть ли», «частота», «видел ли»                           → ХЕШ-ТАБЛИЦА
   «отсортировано» / «найти границу»                          → БИНАРНЫЙ ПОИСК
   «все варианты/комбинации/перестановки»                     → BACKTRACKING
   «оптимум через подзадачи с перекрытием»                    → ДП (модуль 19)
```

💡 ⭐ Это шпаргалка распознавания. Прочитал задачу → определил **сигналы** → выбрал паттерн.
Практика на LeetCode по темам закрепляет это: решаешь 5-10 задач одного паттерна — и начинаешь
узнавать его мгновенно.

---

## ⚠️ Ловушки

- ❌ Решать каждую задачу «с нуля», не узнавая паттерн.
- ❌ Перебор O(n²) там, где два указателя/окно дают O(n).
- ❌ Заучивать решения конкретных задач вместо паттернов (не переносится).
- ❌ Применять паттерн не к той задаче (сначала проверь сигналы).

---

## 🛠️ Практика

1. Реши «пара с суммой» в отсортированном массиве двумя указателями (O(n)).
2. Реши «самая длинная подстрока без повторяющихся символов» скользящим окном.
3. Найди цикл в связном списке быстрым/медленным указателем.
