# 20 · Продвинутые алгоритмы на графах 🖼️⭐

> 🎯 **Цель блока:** познакомиться с алгоритмами на графах сверх BFS/DFS — кратчайшие пути
> (Дейкстра), топологическая сортировка — решающими реальные задачи.

---

## 📖 За пределами простого обхода

BFS/DFS (модуль 17) обходят граф, но реальные задачи требуют большего: «кратчайший путь с
**учётом расстояний**», «в каком порядке выполнять задачи с зависимостями». Для этого —
специальные алгоритмы.

💡 Эти алгоритмы — надстройки над обходами. Понимать их **идею** важнее, чем зубрить код: на
собеседованиях и в работе ты узнаёшь тип задачи и применяешь подходящий.

---

## ⭐ Кратчайший путь во взвешенном графе: Дейкстра

```
   BFS находит кратчайший путь по ЧИСЛУ рёбер (невзвешенный граф).
   а если у рёбер РАЗНЫЙ вес (расстояния, время, стоимость)?
   → алгоритм ДЕЙКСТРЫ
```

🖼️
```
   [A]--4--[B]
    │       │
    1       2
    │       │
   [C]--3--[D]   кратчайший A→D? A-C-D = 1+3 = 4 (а не A-B-D = 4+2 = 6)
```

```
   идея Дейкстры (жадный + очередь с приоритетом):
   - храни «лучшее известное расстояние» до каждой вершины
   - всегда обрабатывай ближайшую необработанную (priority queue, модуль 21)
   - обновляй расстояния до соседей («релаксация»)
```

💡 ⭐ Дейкстра — это «BFS с учётом весов»: вместо обычной очереди — **очередь с приоритетом** (куча,
модуль 21), всегда берём ближайшую вершину. Применения: навигаторы (кратчайший маршрут), сети
(маршрутизация — привет [трек Сети](../../Network/01-basics/05-routing.md)!). ⚠️ Не работает с
отрицательными весами (для них — другие алгоритмы).

## ⭐ Топологическая сортировка

```
   задача: есть зависимости «A нужно сделать ДО B». В каком ПОРЯДКЕ выполнять?
   (сборка проекта, установка пакетов, учебный план с пререквизитами)
```

🖼️
```
   надеть носки → надеть ботинки
   надеть рубашку → надеть пиджак
   → топосортировка даёт допустимый порядок: носки, рубашка, ботинки, пиджак
```

💡 ⭐ **Топологическая сортировка** упорядочивает вершины ориентированного графа так, что все
зависимости идут раньше. Делается через DFS или через «убирание вершин без входящих рёбер».
⚠️ Работает только на графах **без циклов** (DAG): если A→B→A (циклическая зависимость) —
порядок невозможен (это, кстати, способ **обнаружить циклы** в зависимостях). Применения:
системы сборки, планировщики задач, разрешение зависимостей.

---

## 📖 Обзор графовых задач

```
   обход / связность / есть путь        → BFS / DFS (модуль 17)
   кратчайший путь, рёбра равны          → BFS
   кратчайший путь, веса рёбер           → ДЕЙКСТРА
   порядок с зависимостями (DAG)         → ТОПОСОРТИРОВКА
   минимальное остовное дерево           → Краскал / Прим (жадные)
   поиск циклов                          → DFS / топосортировка
```

💡 ⭐ Это карта графовых алгоритмов. Большинство «сложных» задач — это распознать, что это граф, и
выбрать алгоритм. Глубоко знать реализацию каждого не обязательно для старта; важно понимать,
**какой инструмент под какую задачу**.

---

## ⚠️ Ловушки

- ❌ Применять Дейкстру к графу с отрицательными весами (не работает).
- ❌ Использовать BFS для кратчайшего пути во **взвешенном** графе (нужен Дейкстра).
- ❌ Делать топосортировку на графе с циклами (невозможно — но так находят циклы).
- ❌ Реализовывать сложный алгоритм вместо распознавания типа задачи.

## 🛠️ Практика

1. Нарисуй взвешенный граф и найди кратчайший путь вручную (логика Дейкстры).
2. Возьми набор задач с зависимостями (сборка/учёба) и построй топологический порядок.
3. Для 5 задач из жизни определи: какой графовый алгоритм подходит?
