# 07 · Slash-команды 🖼️

> 🎯 **Цель блока:** освоить slash-команды — короткие команды на `/`, которые управляют
> самим Claude Code, и узнать, что можно делать **свои** команды.

---

## 📖 Что это и зачем

**Slash-команда** — это команда Claude Code, а не задача агенту. Начинается с `/`. Она
управляет инструментом: сменить модель, очистить контекст, посмотреть статус и т.д.

```
   обычный текст  →  задача для агента («добавь тест»)
   /команда       →  управление Claude Code («/clear», «/model»)
```

---

## 📖 Команды, которые нужны сразу

| Команда | Что делает |
|---|---|
| `/help` | Список команд и подсказки |
| `/clear` | **Очистить контекст** — начать «с чистого листа» (память сессии обнуляется) |
| `/compact` | **Сжать контекст** — сохранить суть, выгрузить лишнее (ядро уровня 2) |
| `/model` | Посмотреть/сменить модель |
| `/status` | Аккаунт, состояние, объём контекста |
| `/init` | Создать `CLAUDE.md` — файл памяти проекта |
| `/login` `/logout` | Вход/выход аккаунта |
| `/mcp` | Управление MCP-серверами (уровень 3) |
| `/agents` | Управление саб-агентами (уровень 3) |

🖼️
```
   контекст забит мусором?
        ├─ /clear    → всё стереть, начать заново
        └─ /compact  → сжать, оставить суть (умнее, чем /clear)
```

💡 `/clear` и `/compact` — твои инструменты гигиены памяти. Подробно — в уровне 2, но
запомни их уже сейчас: длинная сессия = замусоренный контекст = хуже и дороже.

---

## ⭐ Свои slash-команды

Повторяющиеся промпты можно превратить в **свою команду**. Обычно это markdown-файл в
папке проекта:

```
   мой-проект/
   └── .claude/
       └── commands/
           └── review.md      → команда /review
```

Внутри `review.md` — текст промпта-шаблона, например:

```markdown
Сделай ревью изменений в этом PR: найди баги, проблемы безопасности и стиль.
Для каждого замечания укажи файл:строку и severity.
```

Теперь вместо того чтобы каждый раз печатать этот промпт, ты пишешь `/review`.

💡 Это первый шаг к автоматизации: часто повторяемое — выноси в команду. Точный формат
(папка, аргументы в команде, namespace) сверяй с документацией — он развивается.

---

## ⚠️ Ловушки

- ❌ Путать slash-команду с задачей: `/добавь тест` не сработает — это просто текст.
- ❌ Никогда не чистить контекст в длинной сессии → агент «тупеет» от мусора.
- ❌ Каждый раз печатать один и тот же длинный промпт вместо своей команды.

---

## 🛠️ Практика

1. Выполни `/help` и пролистай доступные команды.
2. Поработай, потом сделай `/compact` и заметь, как «уплотнился» контекст; затем попробуй
   `/clear` и почувствуй разницу (после clear агент не помнит прошлое).
3. Создай свою команду `/review` (файл в `.claude/commands/`) и вызови её.
