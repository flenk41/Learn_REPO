# 🚀 Пет-проект уровня 1 — «Маленькая фича от плана до коммита»

> 🎯 **Цель:** пройти полный цикл работы с агентом на реальной маленькой фиче: промпт →
> Plan Mode → итерации → проверка → коммит → своя команда.

---

## 📋 Что делаем

Берём небольшой проект (можно расширить песочницу из уровня 0) и **аккуратно**, по всем
правилам уровня 1, добавляем одну осмысленную фичу.

Пример: к простому скрипту/приложению добавить функцию «поиск по списку» с тестами. Или к
заметкам — команду «удалить по id». Главное — фича маленькая и **проверяемая**.

### Шаги

1. **ТЗ (контекст вперёд).** Напиши одно полное ТЗ: цель, где менять, критерий готовности
   («тест X зелёный, поведение такое-то»).
2. **Plan Mode.** Включи режим планирования, получи план, **разбери и поправь** хотя бы один
   пункт, утверди.
3. **Итерации.** Дай агенту делать по шагам. После каждого — **проверяй** (запусти тест/сценарий).
4. **Коррекция.** Хотя бы раз поправь агента контрастом «не X, а Y».
5. **Коммит.** Зафиксируй работающую фичу.
6. **Своя команда.** Заметил повторяющийся промпт (например, «прогони тесты и покажи, что
   красное»)? Вынеси в `.claude/commands/`.

---

## ✅ Критерии готовности

- [ ] Написал полное ТЗ (цель + контекст + критерий)
- [ ] Провёл задачу через Plan Mode с правкой плана
- [ ] Сделал фичу итерациями с проверкой на каждом шаге
- [ ] Применил коррекцию «не X, а Y»
- [ ] Зафиксировал результат коммитом
- [ ] Создал хотя бы одну свою slash-команду

---

## 🌟 Усложнение (по желанию)

- Проведи **ту же фичу** дважды: один раз «как попало», другой — по правилам. Сравни
  количество переделок и итоговое качество. Запиши вывод — это твой личный аргумент за
  дисциплину.
- Перед коммитом попроси агента **сам** объяснить, что и зачем он изменил (саммари diff).
