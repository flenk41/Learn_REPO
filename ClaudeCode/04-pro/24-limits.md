# 24 · Границы, лимиты и дисциплина 🖼️⭐

> 🎯 **Цель блока:** трезво понять, чего Claude Code **не** делает хорошо, какие у него
> лимиты, и какая инженерная дисциплина делает работу с ним надёжной.

---

## 📖 Где агент силён, где слаб

```
   👍 силён:                          👎 слабее / рискованно:
   - рутина, бойлерплейт              - задачи без проверяемого критерия
   - поиск/навигация по коду          - очень специфический домен без контекста
   - рефактор по ясному ТЗ            - «придумать» факты, которых нет в контексте
   - тесты, ревью, объяснения         - молчаливые предположения вместо вопросов
   - многошаговая работа с целью      - ответственность за итог (она на тебе)
```

💡 Главное правило трезвости: **агент усиливает тебя, но не снимает с тебя ответственность**.
Финальное «работает и безопасно» — твоё решение, подтверждённое проверкой.

---

## ⭐ Лимиты, о которых нужно помнить

```
   🧠 окно контекста — конечно (модуль 09): не влезает всё, нужна гигиена
   💸 стоимость/квоты — токены платные; лимиты по тарифу/ключу
   ⏱️ скорость — больше контекста и effort = дольше и дороже
   🌍 доступность — зависит от региона и тарифа (модуль 02)
   📚 знания модели — есть граница обучения; свежее/специфичное даёшь ты
                      (через контекст, поиск, MCP, заметки)
```

💡 Управление лимитами — это ровно навыки трека: чистый контекст, верный выбор модели/effort,
делегирование саб-агентам, вынос знаний в `CLAUDE.md`/Skills/заметки.

---

## ⭐ Инженерная дисциплина вокруг ИИ

Сведём весь трек в принципы надёжной работы:

```
   1. Чёткая цель + проверяемый критерий «готово».
   2. Контекст вперёд: полное ТЗ сразу, а не по кусочкам.
   3. Дроби на проверяемые шаги, проверяй каждый («готово» ≠ работает).
   4. Гигиена памяти: /clear, /compact, важное — в CLAUDE.md/заметки.
   5. Ограничители: разрешения (allow/ask/deny), hooks, git-чек-поинты, ветки.
   6. Минимум прав: токены, MCP, коннекторы — только нужное.
   7. Безопасность: секреты вне кода/памяти; помни про prompt injection.
   8. Ты — последний рубеж: проверяешь и отвечаешь за результат.
```

🖼️
```
   плохо:  «сделай всё» → принял вслепую → удивился, что сломалось
   хорошо: ясная цель → план → шаги с проверкой → ограничители → проверил → ок
```
