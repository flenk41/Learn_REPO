# 22 · Connectors и внешние сервисы 🖼️

> 🎯 **Цель блока:** понять Connectors — подключение Claude к внешним сервисам (на базе MCP) —
> и как это расширяет работу за пределы кода.

---

## 📖 Что такое Connectors

**Connectors (коннекторы)** — это готовые подключения Claude к внешним сервисам (например,
хранилища файлов, таск-менеджеры, базы знаний, инструменты команды). Технически под капотом
часто тот же **MCP** (модуль 13), но поданный как «подключения» в продуктах Claude (desktop,
web), которые можно включить парой кликов.

🖼️
```
   MCP (протокол)  ──основа──►  Connectors (готовые подключения к сервисам)
        │                            │
   «розетка»                    «уже подобранные вилки» к популярным сервисам
```

💡 Грубо: MCP — это **как** подключать инструменты, Connectors — **готовые** подключения,
которые тебе не нужно собирать руками. Идея та же: дать ассистенту доступ к твоим реальным
системам.

---

## 📖 Зачем это вне кодинга

Claude Code и Claude в целом полезны **не только для кода**. С коннекторами ассистент может:

```
   📄 работать с твоими документами и базами знаний
   ✅ смотреть/обновлять задачи в таск-менеджере
   📊 тянуть данные для анализа и отчётов
   🔎 искать по внутренним источникам команды
```

💡 «Хватит делать руками то, что можно делегировать» — коннекторы превращают ассистента в
помощника по **рабочим процессам**, а не только по коду: подготовка отчётов, разбор данных,
рутина с документами.

---

## ⭐ Безопасность и доступ — те же правила

Коннектор = доступ к реальному сервису под твоей учёткой/токеном. Значит, всё из модулей
14/21 в силе:

```
   ✅ подключай доверенные сервисы и только нужные
   ✅ минимальные права (только чтение, только нужный проект)
   ✅ помни про prompt injection: данные из сервиса могут содержать «инструкции»
   ✅ не разрешай деструктивные действия не глядя
```

⚠️ Конкретный список доступных коннекторов и способ подключения зависит от продукта
(desktop/web) и быстро меняется — смотри актуальное в интерфейсе и документации.

---

## 📖 Connectors vs свой MCP

```
   Connectors → готовое подключение к популярному сервису (быстро, кликом)
   свой MCP   → твой уникальный/внутренний инструмент (пишешь сам, модуль 15)
```

💡 Нужен популярный сервис — ищи готовый коннектор. Нужен твой внутренний — пиши свой
MCP-сервер. Это две стороны одной экосистемы.

---

## ⚠️ Ловушки

- ❌ Подключать всё подряд → лишний доступ и шум.
- ❌ Давать токены с избыточными правами.
- ❌ Забывать про injection в данных из сервисов.
- ❌ Ждать готовый коннектор под уникальный внутренний сервис — для него нужен свой MCP.

---

## 🛠️ Практика

1. Посмотри в интерфейсе Claude (desktop/web), какие коннекторы доступны.
2. Подключи один безопасный (например, к хранилищу заметок/документов) с минимальными правами.
3. Дай задачу, использующую сервис («собери выжимку из этих документов»).
4. Сформулируй, что бы ты подключил, а что — нет, и почему.
