# 14 · Подключение MCP-серверов 🛠️⭐

> 🎯 **Цель блока:** научиться подключать готовые MCP-серверы к Claude Code и понимать
> разные транспорты и вопросы доступа/секретов.

---

## 📖 Три способа «общения» с MCP-сервером (транспорты)

MCP-сервер может работать по-разному — это **транспорт**:

```
   📦 stdio  — сервер запускается как локальный процесс на твоей машине
              (Claude Code сам его поднимает и общается через ввод/вывод)
   🌐 HTTP / SSE — сервер где-то по сети (удалённый), общение по HTTP
```

💡 Локальные инструменты (доступ к локальной БД, файлам, утилитам) — обычно **stdio**.
Облачные сервисы (хостящийся MCP какого-то продукта) — обычно **HTTP/SSE**, часто с авторизацией.

---

## 🛠️ Как подключить

Claude Code управляет MCP через команду и/или конфиг:

```bash
# добавить локальный (stdio) сервер
claude mcp add my-tools -- npx -y some-mcp-server

# добавить удалённый (HTTP/SSE) сервер
claude mcp add --transport http docs https://example.com/mcp

# посмотреть список / статус
claude mcp list
```

Внутри сессии — команда `/mcp` показывает подключённые серверы, их инструменты и статус
авторизации.

💡 Точный синтаксис (`claude mcp add`, флаги, файл `.mcp.json` для общих серверов проекта)
сверяй с документацией — он развивается. Идея устойчива: **указываешь, как запустить/найти
сервер, и он появляется в наборе инструментов агента**.

🖼️
```
   .mcp.json (в репо)  → общие серверы проекта (вся команда получает их)
   личная настройка    → твои персональные серверы
   claude mcp add ...  → добавить на лету
        ▼
   /mcp → видишь список и статус (connected / нужна авторизация)
```

---

## ⭐ Доступ и секреты — самое важное

Многим серверам нужен доступ к внешнему сервису → нужна **авторизация** (токен, OAuth).

```
   ❌ НЕ вписывай токены прямо в конфиг, который попадёт в git.
   ✅ Используй переменные окружения / защищённое хранилище / OAuth-вход.
```

💡 Подключая удалённый MCP, ты даёшь агенту реальный доступ к сервису под этим токеном.
Думай о **правах**: лучше токен с минимально нужными правами (только чтение, конкретный
проект), чем «всё могу».

⚠️ MCP-сервер — это сторонний код. Подключай серверы, которым доверяешь. Сервер видит то,
что ты ему дал, и может делать то, что умеют его инструменты.

---

## 📖 Что подключают чаще всего

Типичные категории готовых MCP-серверов:

```
   🐙 системы контроля версий / трекеры (GitHub, GitLab, issue-трекеры)
   🗄️ базы данных и хранилища
   📚 документация / базы знаний / поиск
   💬 мессенджеры и таск-менеджеры
   🌐 браузер / веб-автоматизация
```

💡 Начни с одного полезного сервера, освойся, потом добавляй. «Подключи нужное», а не «всё».

---

## ⚠️ Ловушки

- ❌ Токены в конфиге под git.
- ❌ Подключить десять серверов сразу — агент тонет в инструментах, контекст распухает.
- ❌ Дать токен с правами «администратор всего» там, где хватило бы чтения.
- ❌ Доверять незнакомому MCP-серверу без понимания, что он делает.

---

## 🛠️ Практика

1. Подключи один простой/локальный MCP-сервер, выполни `/mcp`, посмотри его инструменты.
2. Дай агенту задачу, где он должен использовать инструмент сервера; проследи вызов.
3. Проверь, что секрет (если нужен) хранится в переменной окружения, а не в репо.
4. Отключи сервер и убедись, что инструменты исчезли из набора.
