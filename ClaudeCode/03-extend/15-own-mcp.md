# 15 · Свой MCP-сервер на Python 🛠️⭐

> 🎯 **Цель блока:** написать **свой** MCP-сервер на Python и подключить его к Claude Code.
> Это единственный «программистский» блок трека — но он несложный.

---

## 📖 Когда нужен свой MCP-сервер

Готовых серверов много, но не на всё. Свой сервер нужен, когда хочешь дать агенту доступ к
**твоему** внутреннему: специфическому API, базе, бизнес-логике, скриптам компании.

```
   готовый MCP   → есть под популярный сервис? подключай его
   свой MCP      → твой уникальный инструмент/система? пиши сам (это просто)
```

💡 Свой MCP-сервер — это маленькая программа, которая объявляет несколько функций-инструментов
и реализует их. Агент видит эти функции как инструменты и вызывает по необходимости.

---

## 🛠️ Минимальный сервер (Python, FastMCP)

Самый простой путь — официальный Python-SDK `mcp` (FastMCP). Инструмент = обычная функция с
декоратором.

```python
# server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("my-tools")

@mcp.tool()
def add(a: int, b: int) -> int:
    """Сложить два числа. Вызывай, когда нужно посчитать сумму."""
    return a + b

@mcp.tool()
def get_user_status(user_id: str) -> str:
    """Вернуть статус пользователя по id из нашей системы.
    Вызывай, когда спрашивают про конкретного пользователя."""
    # тут — реальный запрос к твоему API/БД
    return f"Пользователь {user_id}: активен"

if __name__ == "__main__":
    mcp.run()  # по умолчанию транспорт stdio
```

```bash
pip install mcp        # официальный Python SDK (точный пакет — см. документацию MCP)
```

💡 Описание (docstring) — это то, что прочитает агент. Пиши **когда вызывать**, не только
что делает. Имена функций и типы аргументов превращаются в схему инструмента автоматически.

🖼️
```
   server.py  ──объявляет──►  add(a,b), get_user_status(id)
        │                          ▲
        │ запускается как stdio     │ агент вызывает по описанию
        ▼                          │
   Claude Code ◄── MCP-протокол ──► инструменты в наборе агента
```

---

## 🛠️ Подключение к Claude Code

```bash
claude mcp add my-tools -- python /path/to/server.py
# затем внутри сессии: /mcp  → должен появиться "my-tools" с инструментами
```

Дай задачу: «посчитай 17 + 25 моим инструментом» / «какой статус у пользователя u-42» —
агент вызовет твою функцию.

💡 Часто свои MCP-серверы упаковывают в **Docker** (изоляция, воспроизводимость окружения) —
это удобно для командного и серверного использования. Для учёбы достаточно локального запуска.

---

## ⭐ Хорошие инструменты — узкие и понятные

```
   ✅ один инструмент = одно понятное действие (get_user_status)
   ✅ ясное описание: что делает + когда вызывать
   ✅ типизированные аргументы (id: str, limit: int)
   ✅ предсказуемый результат, понятные ошибки
   ❌ один «мега-инструмент do_everything(action, payload)» — агент запутается
```

💡 Думай об инструменте как об API для агента: чем чётче контракт, тем лучше им пользуются.
Для опасных действий (запись, удаление) полагайся на систему разрешений Claude Code.

---

## ⚠️ Ловушки

- ❌ Расплывчатые описания → агент вызывает не вовремя или не тот инструмент.
- ❌ «Мега-инструмент» на все случаи → хуже, чем несколько узких.
- ❌ Зашитые секреты прямо в `server.py` → выноси в переменные окружения.
- ❌ Инструмент с побочными эффектами без осторожности (полагайся на разрешения).

## 🛠️ Практика

1. Установи SDK, напиши `server.py` с 2 инструментами (один — чистая функция, второй —
   обращение к чему-то твоему: файл/локальные данные).
2. Подключи через `claude mcp add`, проверь `/mcp`.
3. Дай агенту задачу, требующую обоих инструментов; проследи вызовы.
4. Ухудши описание одного инструмента и заметь, как агент начинает ошибаться с выбором.
