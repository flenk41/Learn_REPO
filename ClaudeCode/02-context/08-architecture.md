# 08 · Как устроен Claude Code 🖼️⭐

> 🎯 **Цель блока:** заглянуть «под капот» агента: из чего складывается то, что видит модель
> на каждом шаге, и почему понимание устройства = контроль над результатом.

---

## 📖 Агент = модель + цикл + инструменты + контекст

Claude Code — это не модель сама по себе. Это **цикл вокруг модели**:

🖼️
![Архитектура Claude Code](../assets/claude-code-arch.svg)

```
   ┌─────────────────────────────────────────────────────┐
   │ КОНТЕКСТ (то, что модель «видит» на этом шаге):       │
   │   • системный промпт (правила, кто она, инструменты) │
   │   • CLAUDE.md проекта (память про проект)            │
   │   • история сессии (что говорили и делали)          │
   │   • результаты инструментов (что прочитал/запустил)  │
   └───────────────────────┬─────────────────────────────┘
                           ▼
                    МОДЕЛЬ думает
                           ▼
                  вызывает ИНСТРУМЕНТ (read/edit/bash/...)
                           ▼
              результат добавляется в КОНТЕКСТ → новый шаг
```

💡 На каждом шаге модель получает **весь контекст заново** — у неё нет «памяти» вне того,
что лежит в окне контекста прямо сейчас. Поэтому **управление контекстом = управление
агентом**. Это и есть ядро уровня.

---

## ⭐ Что модель не знает по умолчанию

- Она **не помнит** прошлые сессии (если это не записано в `CLAUDE.md`/память).
- Она **не видит** файлы, которые не прочитала в этой сессии.
- Она **не знает** твоих негласных договорённостей и стиля — если ты их не сказал.

💡 Всё, что важно, должно **попасть в контекст**: либо ты это скажешь, либо агент прочитает
файл, либо это лежит в `CLAUDE.md`. Нет в контексте — для модели не существует.

---

## 📖 Системный промпт и инструменты

Claude Code сам формирует **системный промпт** — невидимую инструкцию, где описано: кто
агент, как себя вести, какие у него инструменты (чтение/правка файлов, bash, поиск и т.д.),
правила безопасности. Ты этим не управляешь напрямую, но влияешь через:

- `CLAUDE.md` — твои постоянные указания проекту (модуль 11);
- Skills, команды, агентов — добавляют поведение и инструменты (уровень 3);
- сам диалог — твои сообщения тоже часть контекста.

🖼️
```
   системный промпт (от Claude Code)  ← задаёт рамки
        +
   CLAUDE.md (от тебя)                ← правила проекта
        +
   история сессии + результаты tools  ← «оперативка»
        =
   КОНТЕКСТ, который модель видит на шаге
```

---

## ⭐ Почему это важно понимать

Когда агент «тупит», «забыл» что-то или «выдумал» — почти всегда дело в контексте:

```
   «забыл» → этого нет в текущем окне контекста (или вытеснилось)
   «выдумал» → не было фактов в контексте, модель достроила
   «не учёл файл» → он не был прочитан в этой сессии
   «противоречит правилу» → правило не попало в контекст (нет в CLAUDE.md)
```

💡 Диагностика «что не так с агентом» = вопрос «что сейчас в его контексте и чего там не
хватает». Это ключевая инженерная привычка трека.

## ⚠️ Ловушки

- ❌ Ожидать, что агент «помнит» сказанное в прошлой сессии. Не помнит — запиши в CLAUDE.md.
- ❌ Думать, что модель «видит весь проект». Она видит только прочитанное в этой сессии.
- ❌ Считать «выдумывание» багом инструмента — чаще это нехватка фактов в контексте.

---

## 🛠️ Практика

1. Спроси агента про файл, который он ещё не читал, — увидишь, что он не знает деталей.
   Потом попроси прочитать и спроси снова. Почувствуй роль контекста.
2. В новой сессии спроси про что-то из прошлой — убедись, что «памяти» нет.
3. Запиши на бумаге: «всё важное должно попасть в контекст».
