# 11 · .claude и CLAUDE.md — постоянная память проекта 🖼️⭐

> 🎯 **Цель блока:** освоить долговременную память агента — файл `CLAUDE.md` и папку
> `.claude/` — то, что переживает сессии и компакцию.

---

## 📖 Сессия забывает — файл помнит

Из модулей 09–10: память сессии конечна и стирается. Чтобы важное **не терялось** между
сессиями и при `/clear`/`/compact`, его кладут на диск. Главное место — файл **`CLAUDE.md`**.

🖼️
```
   КОНТЕКСТ СЕССИИ (RAM)              ДИСК (постоянное)
   ┌────────────────────┐            ┌────────────────────────┐
   │ история, файлы,     │  /clear →  │  CLAUDE.md             │
   │ вывод команд        │  ✗ стёрто   │  (правила, факты,      │
   │ (временно)          │            │   команды проекта)     │
   └────────────────────┘            └──────────┬─────────────┘
            ▲ при старте сессии Claude Code       │
            └──────── читает CLAUDE.md ←───────────┘
```

💡 `CLAUDE.md` автоматически попадает в контекст в начале сессии. Это «памятка проекту»,
которую агент видит **всегда**. Прямая параллель: контекст — RAM, `CLAUDE.md` — диск.

---

## ⭐ Что писать в CLAUDE.md

Короткие, полезные факты и правила про **этот** проект:

```markdown
# Проект: магазин на FastAPI

## Команды
- Запуск: `uvicorn app.main:app --reload`
- Тесты: `pytest -q`
- Линтер: `ruff check .`

## Правила
- Код на Python 3.12, типизация обязательна.
- Бизнес-логика — в `app/services/`, не в роутерах.
- Не добавлять зависимости без необходимости.

## Контекст
- БД — Postgres, миграции через Alembic.
```

Создать заготовку можно командой `/init` — она просканирует проект и предложит черновик.

💡 Пиши **факты, а не команды-приказы**. «Тесты запускаются так-то», «код в таком стиле» —
это контекст. А не «ТЫ ОБЯЗАН…»: на новых моделях агрессивные приказы вредят (модуль 04).

## 📖 Папка `.claude/` — всё про этот проект

Рядом живёт папка `.claude/`, где хранятся настройки и расширения проекта:

```
   мой-проект/
   ├── CLAUDE.md                 ← память проекта (правила, факты)
   └── .claude/
       ├── settings.json         ← настройки (разрешения, хуки) — уровень 4
       ├── commands/             ← свои slash-команды (модуль 07)
       ├── agents/               ← свои саб-агенты (уровень 3)
       └── skills/               ← свои навыки (уровень 3)
```

💡 Всё это **версионируется в git вместе с проектом** — то есть команда делит общие правила,
команды, агентов. Новый разработчик клонирует репо — и у его Claude Code уже есть «память»
и инструменты проекта.

---

## ⭐ Уровни памяти: проект и личное

Память бывает разной «области видимости»:

```
   📁 проектная   — CLAUDE.md в репозитории: правила для всех в команде
   🏠 личная       — твои глобальные предпочтения (вне репо проекта)
```

💡 Командные договорённости → в проектный `CLAUDE.md` (в git). Личные привычки («всегда
объясняй кратко») → в личную память, чтобы не навязывать команде.

---

## ⚠️ Ловушки

- ❌ Раздувать `CLAUDE.md` в полотно. Он **всегда** в контексте — большой файл ест окно
  каждую сессию. Держи кратким и по делу.
- ❌ Писать туда секреты (ключи, пароли) — файл в git, секреты утекут.
- ❌ Превращать в свалку устаревших заметок — обновляй, удаляй неактуальное.
- ❌ Использовать приказной тон вместо фактов.

---

## 🛠️ Практика

1. Выполни `/init` в проекте, отредактируй черновик `CLAUDE.md`: добавь команды и 3 правила.
2. Сделай `/clear` и спроси агента про команду запуска — убедись, что он знает её из `CLAUDE.md`.
3. Загляни в `.claude/` — посмотри структуру (команды/агенты/настройки).
4. Договорённость, которая раньше «терялась», запиши в `CLAUDE.md` и проверь, что она держится.
