# 12 · Режимы и разрешения 🖼️

> 🎯 **Цель блока:** разобраться в режимах работы Claude Code и в системе разрешений —
> твоём «руле безопасности» при работе с агентом.

---

## 📖 Режимы: насколько самостоятелен агент

Claude Code умеет работать с разной степенью автономии. Грубо:

```
   📝 обычный        — агент спрашивает разрешение на правки/команды
   ⚡ авто-принятие  — правки применяются без отдельного «да» на каждую
   🧭 Plan Mode      — только планирует и читает, файлы не трогает (модуль 05)
```

🖼️
```
   осторожно ◄──────────────────────────────► автономно
     Plan Mode      обычный режим      авто-принятие правок
   (думает, не      (спрашивает        (делает сам, ты
    меняет)          по ходу)           проверяешь итог)
```

💡 Режимы обычно переключаются **Shift+Tab** (циклически). Выбирай по риску задачи:
незнакомый/опасный код → Plan Mode; рутина в знакомом коде → можно авто-принятие.

---

## ⭐ Система разрешений

На потенциально опасные действия (запуск команд, правка файлов, сетевые вызовы) агент
**спрашивает разрешение**. Варианты:

```
   [ Разрешить ]                       — один раз
   [ Разрешить и не спрашивать ]       — для этого типа действий в сессии
   [ Отклонить ]                       — нет, и скажи почему/как иначе
```

💡 «Не спрашивать» удобно для безобидного (чтение файлов, `git status`). Но осторожно с
тем, что **меняет систему, удаляет данные, ходит в сеть или тратит деньги**.

## ⭐ Настройка разрешений в settings.json

Чтобы не отвечать вручную каждый раз, в `.claude/settings.json` можно заранее задать
политику: что разрешать автоматически, что — запрещать, что — спрашивать.

```jsonc
// .claude/settings.json (упрощённо; точный формат — в документации)
{
  "permissions": {
    "allow": ["Read", "Grep"],          // безобидное — без вопросов
    "ask":   ["Bash"],                  // команды — всегда спрашивать
    "deny":  ["Bash(rm -rf:*)"]         // опасное — запретить совсем
  }
}
```

💡 Это «правила дорожного движения» для агента, общие для проекта (файл в git). Точные
ключи/синтаксис уточняй в доках — они эволюционируют. Идея неизменна: **заранее очерти, что
можно автоматически, а что только с твоего ведома**.

- ❌ Разрешать сетевые/деструктивные действия не глядя.
- ❌ Считать, что разрешения = недоверие. Это нормальная инженерная страховка.

## 🛠️ Практика

1. Переключи режимы (Shift+Tab), заметь, как меняется поведение (особенно Plan Mode).
2. На запрос разрешения попробуй все три ответа и посмотри последствия.
3. Загляни в `.claude/settings.json` (создай при необходимости) и добавь простое правило
   `ask` для команд.
4. Сформулируй для себя: какие действия ты **никогда** не разрешаешь автоматически.
