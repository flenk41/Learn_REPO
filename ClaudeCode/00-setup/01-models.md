# 01 · Anthropic и семейство моделей Claude 🖼️

> 🎯 **Цель блока:** понять, какая модель «думает» внутри Claude Code, чем модели
> отличаются и как осознанно выбирать модель и «усилие» (effort).

---

## 📖 Кто такая Anthropic и что такое Claude

**Anthropic** — компания, создающая семейство моделей **Claude**. Claude Code — это
инструмент-агент, внутри которого работает одна из моделей Claude. Грубо: Claude Code —
«тело» (инструменты, цикл, интерфейс), а модель Claude — «мозг».

---

## 📖 Семейство моделей (актуально на момент написания)

| Модель | ID | Окно контекста | Кому/зачем |
|---|---|---|---|
| **Claude Opus 4.8** | `claude-opus-4-8` | 1M | Самая мощная Opus: сложные многошаговые задачи, рефакторинг, агентная работа. **По умолчанию в Claude Code.** |
| **Claude Sonnet 4.6** | `claude-sonnet-4-6` | 1M | Лучший баланс скорость/интеллект для большого объёма работы. |
| **Claude Haiku 4.5** | `claude-haiku-4-5` | 200K | Самая быстрая и дешёвая: простые/скоростные задачи, под-агенты. |
| **Claude Fable 5** | `claude-fable-5` | 1M | Топовый тир выше Opus для самых тяжёлых задач. |

🖼️
```
   быстро/дёшево  ←─────────────────────────────→  умнее/дороже
       Haiku 4.5      Sonnet 4.6      Opus 4.8      Fable 5
       (под-агенты,   (объём,         (сложное,     (самое
        классиф.)      продакшн)       агентное)     тяжёлое)
```

💡 «Окно контекста» (1M / 200K токенов) — это сколько информации модель удерживает
одновременно. Это и есть её **рабочая память** — ядро уровня 2. Чем больше окно, тем
больше кода и истории она «видит» сразу.

⚠️ Модели и версии **быстро меняются**. Точный список и ID всегда смотри в документации
или через `/model` внутри Claude Code — не полагайся на память (свою или модели).

---

## ⭐ Параметр «усилие» (effort)

У современных моделей есть рычаг **effort** — сколько «думать» и сколько токенов тратить:

```
   low      — мелкие/скоростные задачи, под-агенты
   medium   — баланс
   high     — большинство задач, требующих интеллекта
   xhigh    — лучший выбор для кода и агентной работы (по умолчанию в Claude Code)
   max      — когда правильность важнее стоимости
```

💡 В Claude Code по умолчанию используется высокий уровень усилия (`xhigh`) — именно
поэтому он хорош в коде. Больше усилие → больше размышлений и аккуратных шагов, но дороже
и медленнее.

---

## 📖 «Adaptive thinking» — модель сама решает, когда думать

У моделей 4.6+ есть **адаптивное мышление**: модель сама определяет, когда и насколько
глубоко размышлять перед ответом, а когда отвечать сразу. Тебе не нужно вручную выставлять
«бюджет размышлений» — это устаревший подход. В Claude Code это уже настроено разумно.

---

## ⚠️ Ловушки

- ❌ Брать самую дешёвую модель «ради экономии» на сложной задаче → плохой результат
  обойдётся дороже (потерянное время, переделки). Выбор модели — твоё решение по задаче.
- ❌ Считать, что «больше окно = всегда лучше». Большое окно полезно, но **забивать его
  мусором вредно** (об этом — уровень 2).
- ❌ Запоминать ID моделей наизусть и хардкодить с датами-суффиксами. ID берут из доков.
