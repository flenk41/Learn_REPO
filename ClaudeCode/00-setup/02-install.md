# 02 · Установка, подписка и доступ 🛠️

> 🎯 **Цель блока:** установить Claude Code, разобраться со способами оплаты/доступа и
> запустить первую сессию в своём проекте.

---

## 🛠️ Установка (CLI)

Claude Code ставится как обычный CLI-инструмент. Самый частый путь:

```bash
# через npm (нужен Node.js 18+)
npm install -g @anthropic-ai/claude-code

# затем — из папки проекта
cd мой-проект
claude
```

💡 Есть и другие способы установки (нативный установщик, скрипт) — актуальные команды
смотри в документации. После установки команда `claude` запускает агента в **текущей папке**
(он видит файлы этого проекта).

⚠️ На Windows бывает удобнее запускать через WSL/терминал; в IDE — поставить расширение
(VS Code/JetBrains) и пользоваться встроенной панелью. Суть одна — это тот же агент.

---

## 📖 Способы доступа и оплаты

Есть два основных способа «питать» Claude Code:

```
   1) Подписка Claude (Pro / Max)  — вход через аккаунт claude.ai (/login).
      Удобно для личной работы; лимиты по тарифу.

   2) API-ключ Anthropic           — оплата по токенам (платформа разработчика).
      Гибко для команд/автоматизации; платишь за использование.
```

💡 Для старта проще всего **подписка** и вход командой `/login`. Если ты строишь
автоматизацию (CI, скрипты, серверы) — обычно используют **API-ключ**.

⚠️ Никогда не коммить API-ключ в репозиторий и не вставляй его в код. Ключ — в переменную
окружения (`ANTHROPIC_API_KEY`) или в защищённое хранилище.

## 🛠️ Первый запуск — что увидишь

```
   $ claude
   ╭──────────────────────────────────────────╮
   │  Claude Code                              │
   │  cwd: /home/me/мой-проект                 │
   ╰──────────────────────────────────────────╯
   >                ← сюда пишешь задачу обычным текстом
```

Полезные команды первым делом:

- `/help` — список команд и подсказки.
- `/login` — войти под аккаунтом Claude (если используешь подписку).
- `/model` — посмотреть/сменить модель.
- `/status` — состояние, аккаунт, контекст.
- `/init` — создать файл памяти проекта `CLAUDE.md` (об этом — уровень 2).

---

## 📖 Доступность по регионам

Доступность сервисов Anthropic **зависит от региона**. Если в твоей стране доступ
ограничен, используются стандартные легальные способы стабильного подключения к интернету
(надёжный провайдер/сеть). Конкретику и список поддерживаемых стран всегда уточняй в
официальной документации Anthropic — она меняется.

💡 Принцип трека: мы учим **инструменту**, а вопросы оплаты/доступа решаются твоими
обычными средствами доступа в интернет и актуальными правилами Anthropic.

---

## 🛠️ Практика

1. Установи Claude Code любым доступным способом, проверь `claude --version`.
2. Зайди в **тестовую** папку (не рабочий продакшн!) и запусти `claude`.
3. Выполни `/help`, затем `/model` и `/status` — осмотрись.
4. Если используешь подписку — `/login`. Если API-ключ — проверь, что переменная окружения
   задана и **не** лежит в git.

---

## ⚠️ Ловушки

- ❌ Запускать `claude` сразу в большом продакшн-репозитории «вживую». Сначала — песочница.
- ❌ Хранить ключ в коде/`.env`, который попадает в git. Используй переменные окружения.
- ❌ Думать, что нужно «и подписка, и ключ». Достаточно одного способа доступа.
