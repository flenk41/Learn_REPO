# 04 · MAC-адреса и Ethernet 🖼️

> 🎯 **Цель блока:** разобраться с канальным уровнем — MAC-адресами, кадрами Ethernet, ролью
> коммутатора — и понять, чем MAC отличается от IP.

---

## 📖 Два адреса: зачем и MAC, и IP

У устройства есть **два** адреса, и это не дублирование:

```
   MAC-адрес  — «вшит» в сетевую карту, уникален, не меняется.
                Работает ВНУТРИ одного сегмента сети (канальный, 2-й слой).
                Пример: a4:5e:60:c1:2f:8b

   IP-адрес   — логический, назначается, может меняться.
                Работает МЕЖДУ сетями по всему миру (сетевой, 3-й слой).
                Пример: 192.168.1.10
```

🖼️
```
   аналогия:
   IP   = почтовый адрес дома (логический, по нему ищут по миру)
   MAC  = серийный номер конкретного почтового ящика (физический, локальный)
```

💡 IP отвечает на «в какой сети искать и каким путём», MAC — «какому именно устройству в
**этом** сегменте отдать кадр прямо сейчас». На каждом «прыжке» MAC меняется, а IP назначения
остаётся прежним. Это фундаментальная связка.

---

## 📖 Ethernet-кадр

На канальном уровне данные едут в **кадрах** (frame):

```
   [MAC получателя | MAC отправителя | тип | ДАННЫЕ (IP-пакет внутри) | контрольная сумма]
```

💡 Внутри Ethernet-кадра лежит IP-пакет, внутри него — TCP-сегмент, внутри — данные (та самая
матрёшка, модуль 01). Контрольная сумма (FCS) позволяет отбросить повреждённый кадр.

## ⭐ Коммутатор (switch) и ARP

**Коммутатор** соединяет устройства в одной локальной сети и пересылает кадры **по MAC-адресам**:
он запоминает, какое устройство на каком порту, и шлёт кадр только нужному.

А как узнать MAC по известному IP соседа? Протоколом **ARP**:

```
   ПК хочет послать на 192.168.1.20, но знает только IP.
   ARP-запрос (всем): «у кого 192.168.1.20? сообщите свой MAC»
   ответ: «это я, мой MAC = a4:5e:...»
   теперь ПК шлёт кадр на этот MAC.
```

💡 ARP связывает мир IP (3-й слой) и мир MAC (2-й слой). Результаты кэшируются (`arp -a`).

---

## 📖 Коммутатор vs роутер

```
   КОММУТАТОР (switch)  — внутри ОДНОЙ сети, работает по MAC (2-й слой)
   РОУТЕР (router)      — соединяет РАЗНЫЕ сети, работает по IP (3-й слой)
```

💡 Дома роутер обычно совмещает обе роли (+ Wi-Fi + DHCP + NAT). Но концептуально это разные
функции: коммутация — внутри сегмента, маршрутизация — между сегментами (модуль 05).

## ⚠️ Ловушки

- ❌ Думать, что MAC и IP — одно и то же или что один лишний. Они работают на **разных слоях**.
- ❌ Считать, что MAC «едет» через весь интернет. Он меняется на **каждом** хопе; сквозной — IP.
- ❌ Путать коммутатор (по MAC, внутри сети) и роутер (по IP, между сетями).

---

## 🛠️ Практика

1. Узнай MAC своей сетевой карты (`ipconfig /all` / `ip link`).
2. Посмотри ARP-кэш: `arp -a` — увидишь соответствия IP↔MAC соседей.
3. Объясни на своей домашней сети: где коммутация (между твоими устройствами), а где
   маршрутизация (выход в интернет).
