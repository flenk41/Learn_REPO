# 05 · Маршрутизация: как пакет находит путь 🖼️⭐

> 🎯 **Цель блока:** понять, как пакет добирается из одной сети в другую через цепочку
> роутеров — что такое шлюз по умолчанию, таблица маршрутизации и хопы.

---

## 📖 Проблема: сети не соединены «все со всеми»

В мире миллионы сетей. Никто не тянет провод от каждой к каждой. Вместо этого пакет идёт
**по цепочке роутеров**, каждый передаёт его «ближе к цели». Это и есть **маршрутизация**.

🖼️
```
   твой ПК ──► домашний роутер ──► роутер провайдера ──► ... ──► роутер сети сервера ──► сервер
        (приватная LAN)    (шлюз)        магистрали интернета            (чужая LAN)
```

💡 Каждый роутер смотрит на **IP получателя** и решает: «куда переслать дальше?». Он не знает
весь путь — знает только **следующий шаг**. Как навигатор, который на каждом перекрёстке
говорит лишь «следующий поворот».

---

## ⭐ Шлюз по умолчанию (default gateway)

Когда твой ПК хочет послать пакет **за пределы своей подсети**, он отправляет его на
**шлюз по умолчанию** — обычно это твой домашний роутер.

```
   получатель в МОЕЙ подсети?  → шлю напрямую (через коммутатор, по MAC)
   получатель в ДРУГОЙ сети?   → шлю на шлюз (роутер), пусть он разбирается
```

💡 Поэтому в настройках сети всегда есть «шлюз» (`192.168.1.1`). Без него ты общаешься только
внутри своей LAN, но не выходишь в интернет.

---

## ⭐ Таблица маршрутизации

У каждого роутера (и у твоего ПК) есть **таблица маршрутов**: «для такой-то сети — следующий
прыжок туда-то».

```
   назначение            следующий хоп
   192.168.1.0/24    →   напрямую (моя сеть)
   0.0.0.0/0         →   192.168.1.1   (всё остальное — на шлюз)
```

💡 `0.0.0.0/0` — «маршрут по умолчанию»: всё, для чего нет конкретного правила, идёт на шлюз.
Роутеры провайдеров обмениваются маршрутами друг с другом (протоколы вроде BGP) — так
строится «карта интернета».

---

## 📖 Хопы и TTL

Каждый проход через роутер — **хоп** (hop). У пакета есть поле **TTL** (Time To Live):
каждый роутер уменьшает его на 1; если TTL дошёл до 0 — пакет отбрасывают.

```
   TTL=64 → 63 → 62 → ... → 0 (отброшен)
```

💡 TTL защищает от «вечно кружащих» пакетов при ошибках маршрутизации. Именно на TTL основан
`traceroute`: он шлёт пакеты с растущим TTL и собирает адреса роутеров, которые «жалуются» на
истёкший TTL. Так ты видишь путь.

---

## ⚠️ Ловушки

- ❌ Думать, что роутер знает весь путь до цели. Он знает только **следующий хоп**.
- ❌ Забывать про шлюз: без него нет выхода за пределы своей подсети.
- ❌ Путать «маршрутизацию» (между сетями, по IP) с «коммутацией» (внутри сети, по MAC).

---

## 🛠️ Практика

1. Посмотри свою таблицу маршрутов: `ip route` / `route print` (Windows) — найди default gateway.
2. Сделай `traceroute`/`tracert` к зарубежному сайту — посчитай хопы, увидь рост задержки.
3. Объясни на своём выводе: первый хоп — твой роутер, дальше — провайдер, дальше — магистрали.
