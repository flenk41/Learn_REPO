# 🚀 Пет-проект уровня 1 — «Анатомия одного запроса»

> 🎯 **Цель:** проследить ровно один веб-запрос на всех уровнях — от имени до пакетов — и
> собрать целостную картину «как данные доходят».

---

## 📋 Что делаем

Берём один сайт и **разбираем по слоям** весь путь запроса, объединяя всё из уровня 1
(IP, MAC, маршрутизация, DNS, пакеты) в одном исследовании с отчётом.

### Шаги

1. **Имя → адрес (DNS).** `dig site.com` → запиши IP и тип записи. Объясни, по какой иерархии
   это разрешилось.
2. **Своя сеть (IP/MAC).** Узнай свой IP, маску, шлюз, MAC. Определи подсеть. Посмотри `arp -a` —
   найди MAC шлюза.
3. **Путь (маршрутизация).** `traceroute site.com` → выпиши хопы, отметь свой роутер, провайдера,
   рост задержки. Объясни роль TTL.
4. **Пакеты (Wireshark).** Запусти захват, открой сайт. Поймай:
   - DNS-запрос и ответ;
   - TCP-пакеты к IP сайта;
   - разверни один пакет по слоям (Eth → IP → TCP → данные), найди MAC, IP, порты.
5. **Сборка картины.** Нарисуй схему: `имя → DNS → IP → твой ПК (MAC/IP) → шлюз → провайдер →
   … → сервер`, подписав, какой слой за что отвечает.

### Отчёт

`report.md` с разделами по шагам, ключевыми находками и итоговой схемой. Отдельно ответь:
«где здесь канальный, сетевой, транспортный, прикладной слои?».

- [ ] Определил свою подсеть, шлюз и его MAC (ARP)
- [ ] Проследил путь хопами и объяснил TTL
- [ ] Поймал и разобрал пакет по слоям в Wireshark
- [ ] Собрал единую схему запроса по слоям
