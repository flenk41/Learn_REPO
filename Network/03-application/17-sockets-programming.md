## 📖 Сокет в коде — как файл

Из модуля 08: **сокет** — конечная точка соединения (IP+порт), для программы это «труба» для
данных. Работа с ним похожа на файл: создать → подключить/слушать → читать/писать → закрыть.

🖼️
```
   СЕРВЕР                              КЛИЕНТ
   socket()                           socket()
   bind(IP:порт)   ← занять адрес
   listen()        ← слушать
   accept()  ──────── connect() ────► (3-way handshake под капотом!)
   recv()/send()  ◄──── send()/recv() ──►  обмен данными
   close()                            close()
```

💡 Под `connect()`/`accept()` скрывается то самое TCP-рукопожатие (модуль 09). Сетевая теория
живёт прямо в этих вызовах.

---

## 🛠️ Простой TCP-сервер (Python)

```python
# server.py — эхо-сервер: что прислали, то и вернёт
import socket

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # TCP-сокет
s.bind(("0.0.0.0", 5000))     # слушать порт 5000 на всех интерфейсах
s.listen()                    # режим ожидания подключений
print("Жду подключения на :5000")

conn, addr = s.accept()       # блокируется, пока кто-то не подключится
print("Подключился:", addr)
data = conn.recv(1024)        # прочитать до 1024 байт
print("Получено:", data.decode())
conn.sendall(data)            # отправить обратно (эхо)
conn.close()
```

## 🛠️ Простой TCP-клиент (Python)

```python
# client.py
import socket

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.connect(("127.0.0.1", 5000))   # подключиться к серверу (handshake)
s.sendall(b"Привет, сокет!")     # отправить байты
print("Ответ:", s.recv(1024).decode())
s.close()
```

💡 Запусти `python server.py`, в другом терминале `python client.py` — увидишь обмен. Это
полноценный TCP-обмен «вживую», который ты можешь поймать в Wireshark (увидишь SYN/ACK и данные).

⭐ Заметь: ты пишешь **байты** (`b"..."`). Данные из памяти программы уходят в сокет → по сети →
в память другой программы. Сокет соединил два буфера памяти — связь с темой курса.

## 📖 UDP в коде — ещё проще

```python
# UDP: без listen/accept/connect — просто шлёшь датаграммы
import socket
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)   # SOCK_DGRAM = UDP
s.sendto(b"ping", ("127.0.0.1", 5001))                 # отправил и забыл
```

💡 У UDP нет соединения (модуль 10) — поэтому нет `connect/accept`, есть `sendto`/`recvfrom`.
Код короче, но и гарантий нет: датаграмма может не дойти.

---

## ⭐ Что важно понять (а не зазубрить API)

```
   bind     — «занять» адрес+порт (сервер)
   listen   — готовность принимать (сервер)
   accept   — принять входящее (под капотом — handshake)
   connect  — подключиться (клиент, под капотом — handshake)
   send/recv — обмен байтами (TCP — поток!)
   close    — закрыть (под капотом — FIN/ACK)
```

⚠️ Так как TCP — **поток байт** (модуль 09), один `send` может прийти несколькими `recv` и
наоборот. Поэтому в реальных протоколах добавляют **границы сообщений** (длину впереди, или
разделитель). Это типичная ошибка новичков.

---

## ⚠️ Ловушки

- ❌ Ждать, что один `send` = один `recv` в TCP. Это поток; нужны свои границы сообщений.
- ❌ Забыть `close()` — утечка сокетов/портов (аналог утечки ресурсов в памяти).
- ❌ Слать строки вместо байт — сокеты работают с байтами (кодируй/декодируй явно).
- ❌ Захардкодить адрес `127.0.0.1` и удивляться, что снаружи не подключиться (нужен `0.0.0.0`).
