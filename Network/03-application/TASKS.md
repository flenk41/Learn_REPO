# ✅ Задачи уровня 3 — Прикладной уровень

> Закрепляем: HTTP/HTTPS, TLS, прикладные протоколы, REST API/JSON, сокет-программирование,
> HTTP/2 и HTTP/3.

---

## 🟢 Базовые

1. **HTTP.** Разбери `curl -v https://example.com`: метод, заголовки, код состояния.
2. **Коды.** Объясни классы 2xx/3xx/4xx/5xx и приведи по 2 примера каждого.
3. **TLS.** Назови три гарантии TLS и роль сертификата/CA.
4. **Протоколы.** Составь таблицу протокол→порт→задача (мин. 6 строк).

---

## 🟡 Средние

5. **API.** Дёрни публичный REST API (`curl https://api.github.com/users/...`), разбери JSON и код.
6. **CRUD.** Сопоставь GET/POST/PUT/PATCH/DELETE с действиями и примерами URL.
7. **Сертификат.** Открой «замочек» в браузере, опиши, кем выдан сертификат, кому и до какого срока.
8. **Версии HTTP.** В DevTools определи, по какой версии (h2/h3/http1.1) грузятся 3 разных сайта.

---

## 🔴 Со звёздочкой

9. ⭐ **Свой сервер.** Напиши TCP эхо-сервер и клиент (Python). Добейся обмена и **поймай его в
   Wireshark** (handshake + данные + закрытие).
10. ⭐ **Границы сообщений.** Доработай сервер из задачи 9 так, чтобы он корректно обрабатывал
    несколько сообщений (придумай разделитель или префикс длины). Объясни, почему это нужно.
11. ⭐ **QUIC.** Найди сайт на HTTP/3, поймай в Wireshark UDP-трафик QUIC. Объясни, зачем веб
    ушёл на UDP.

---

## 🎯 Критерий готовности

- [ ] Понимаю HTTP (методы, коды, заголовки) и HTTPS=HTTP+TLS
- [ ] Понимаю TLS, сертификаты, CA
- [ ] Понимаю REST API и JSON, прикладные протоколы
- [ ] Написал сокет сервер+клиент, понимаю HTTP/2 и HTTP/3
