# 02 · Инструменты сетевика 🛠️

> 🎯 **Цель блока:** познакомиться с инструментами, которыми мы будем щупать сеть весь трек:
> `ping`, `traceroute`, `curl`, `dig/nslookup`, `netcat`, `ss/netstat`, Wireshark.

---

## 🛠️ Базовый набор (есть почти везде)

```
   ping        — проверить «жив ли хост» и измерить задержку
   traceroute  — показать путь пакета через роутеры (tracert на Windows)
   curl        — сделать HTTP(S)-запрос из терминала
   dig / nslookup — спросить DNS: какой IP у имени
   nc (netcat) — «швейцарский нож»: открыть TCP/UDP-соединение руками
   ss / netstat — какие соединения и порты открыты на твоей машине
```

💡 На Windows часть команд называется иначе: `tracert`, `nslookup`, `Test-NetConnection`
(PowerShell). На Linux/macOS — `traceroute`, `dig`, `ss`. Идея одна.

---

## 🛠️ Что показывает каждая (коротко)

```bash
ping google.com
#  → отвечает за ~20 мс  → хост доступен, задержка ~20 мс

traceroute google.com        # tracert google.com на Windows
#  → список роутеров (хопов) на пути

curl -v https://example.com
#  → весь HTTP-обмен: DNS, TCP, TLS, заголовки, тело

dig example.com              # nslookup example.com — попроще
#  → IP-адрес(а) имени и какой DNS-сервер ответил

ss -tunap                    # netstat -tunap (Linux) / netstat -ano (Windows)
#  → открытые TCP/UDP-соединения и порты
```

🖼️
```
   вопрос                          инструмент
   «хост жив? как далеко?»     →   ping
   «каким путём идут пакеты?»  →   traceroute / tracert
   «что отвечает сайт?»        →   curl -v
   «какой IP у имени?»         →   dig / nslookup
   «кто слушает порты у меня?» →   ss / netstat
   «покажи реальные пакеты»    →   Wireshark / tcpdump
```

---

## ⭐ Wireshark — рентген сети

**Wireshark** (GUI) и **tcpdump** (CLI) перехватывают и показывают **реальные пакеты**. Это
самый мощный учебный инструмент: ты видишь TCP-рукопожатие, заголовки, содержимое — всё, о
чём мы говорим в треке, «вживую».

💡 Установи Wireshark уже сейчас. В уровне 2 (TCP/UDP) и далее мы будем смотреть в нём
реальные handshake и пакеты — это превращает теорию в наглядность.

⚠️ Перехват чужого трафика без разрешения — незаконно и неэтично. Смотри **свой** трафик в
учебных целях (например, открой Wireshark и пингани сайт — увидишь свои же пакеты).

---

## 🛠️ Установка

```
   Wireshark   — wireshark.org (Win/macOS/Linux), включает tcpdump-движок
   curl        — обычно уже есть; иначе из пакетного менеджера
   dig         — пакет dnsutils (Linux) / bind (macOS brew); на Windows — nslookup есть из коробки
   netcat (nc) — пакет netcat/ncat; на Windows — ncat из проекта Nmap
```

💡 Минимум для старта: рабочие `ping`, `curl`, `nslookup`/`dig` (есть почти везде) + установленный
Wireshark. Остальное доставишь по ходу.

---

## ⚠️ Ловушки

- ❌ Имена команд на Windows и Linux отличаются (`tracert` vs `traceroute`) — не пугайся.
- ❌ `ping` может «не отвечать», хотя хост жив: некоторые серверы блокируют ICMP. Это не всегда
  значит «недоступен».
- ❌ Перехватывать чужой трафик. Только свой, в учебных целях.

---

## 🛠️ Практика

1. Выполни `ping`, `traceroute`, `curl -v`, `dig`/`nslookup` к любимому сайту — посмотри вывод.
2. Запусти `ss`/`netstat` и найди соединения своего браузера.
3. Установи Wireshark, запусти захват, открой сайт — увидь свои пакеты (мы вернёмся к ним).
