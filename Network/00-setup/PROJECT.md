# 🚀 Пет-проект уровня 0 — «Карта пути до сайта»

> 🎯 **Цель:** собственными руками проследить и описать, как твои данные доходят до сайта —
> от DNS до ответа сервера, используя базовые инструменты.

---

## 📋 Что делаем

Берём один сайт и **исследуем весь путь** до него инструментами из модуля 02, оформляя
наблюдения в небольшой отчёт (markdown-файл).

### Шаги

1. **Выбери сайт** (например, `example.com` или любимый ресурс).
2. **DNS.** `dig site.com` / `nslookup site.com` → запиши IP-адрес(а) и какой DNS ответил.
3. **Доступность и задержка.** `ping site.com` → средняя задержка, потери пакетов.
4. **Путь.** `traceroute site.com` / `tracert site.com` → сколько хопов (роутеров), где
   задержка растёт (часто на выходе из страны).
5. **HTTP-обмен.** `curl -v https://site.com` → найди в выводе: разрешение DNS, установку
   TCP-соединения, TLS-рукопожатие, HTTP-заголовки ответа.
6. **Свои пакеты.** В Wireshark запусти захват и повтори `ping` — найди ICMP-пакеты, посмотри
   их адреса и время.

### Отчёт

Оформи `report.md`: для каждого шага — команда, ключевой результат, твой вывод. В конце —
схема пути: `твой ПК → роутер → провайдер → … → сервер` с подписанными задержками, где смог
их различить.

  вывод, почему «далеко» = «медленнее».
- Сопоставь каждый шаг проекта со **слоем** TCP/IP (DNS — прикладной, ping/ICMP — сетевой,
  TCP в curl — транспортный).
