# ✅ Задачи · Уровень 3 — Совместная работа и ревью

> Нужен аккаунт на GitHub/GitLab. Многое удобно проверять «на двух машинах» = две локальные копии
> одного удалённого репозитория.

---

## Remotes (модуль 13)

1. Создай репо на GitHub, склонируй. Проверь `git remote -v` и `git branch -a`.
2. Настрой SSH-ключ, переключи origin на SSH, сделай тестовый push.
3. ⭐ Объясни разницу локального `main` и `origin/main`.

## Fetch/pull/push (модуль 14)

4. Две копии репо: в одной коммит+push, в другой fetch+смотри `origin/main`, потом слей.
5. Спровоцируй отклонённый push (сервер ушёл вперёд). Реши через pull → push.
6. ⭐ Сравни `git pull` и `git pull --rebase` при расхождении.
7. ⭐⭐ Объясни, почему `push --force` в общую ветку опасен, и что делать вместо.

## Pull requests (модуль 15)

8. Открой PR с понятным описанием (что/зачем/как проверить).
9. Слей учебные PR тремя способами (merge/squash/rebase). Сравни историю main.
10. ⭐ Настрой защиту ветки main (require PR + review) в учебном репо.

## Код-ревью (модуль 16)

11. Самостоятельно отревьюй свой PR построчно — найди, что улучшить.
12. Напиши 3 конструктивных комментария к чужому/учебному PR (баг / ясность / nit).
13. ⭐ Перепиши 3 грубых/расплывчатых комментария в конструктивные.
14. ⭐⭐ Составь чек-лист ревьюера для своей команды.

## Workflow (модуль 17)

15. Опиши GitHub Flow по шагам и проживи его в учебном репо (ветка → PR → merge → тег).
16. ⭐ Сравни GitHub Flow vs Git Flow: для какого продукта что? Обоснуй.

---

## 🎯 Итоговое

17. Пройди ПОЛНЫЙ командный цикл в одиночку: ветка → коммиты → push → PR с описанием → «ревью»
    (комментарии + правки) → merge → удалить ветку. Это твой ежедневный рабочий процесс.
