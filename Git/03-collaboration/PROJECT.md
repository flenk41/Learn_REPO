# 🏗️ Проект уровня 3 — Внеси вклад через полный PR-цикл

> 🎯 **Цель:** пройти реальную командную разработку через GitHub: от issue до влитого, отревьюенного
> pull request'а. Это то, как работают в индустрии каждый день.

---

## Вариант A — свой проект с «командным» процессом

- [ ] Запушь один из своих проектов курса на GitHub.
- [ ] **Защити main**: настрой required pull request + required review (Settings → Branches).
- [ ] Заведи 3 issue (задачи: фича, баг, улучшение).
- [ ] На каждую — ветка `feature/...` или `fix/...` → коммиты → push → **PR** с описанием (что/зачем/
      как проверить, `Fixes #N`).
- [ ] **Отревьюй сам себя** в каждом PR: пройди по диффу, оставь и реши комментарии, внеси правки.
- [ ] Слей через подходящую стратегию (squash для мелочи, merge для крупного). Issue закрылись автоматически?
- [ ] В конце — чистый main со всей работой, история читается, ветки удалены.

## Вариант B — вклад в open source (сильнее)

- [ ] Найди дружелюбный к новичкам проект (метка `good first issue`).
- [ ] **Форкни** репозиторий, склонируй форк, добавь `upstream` remote.
- [ ] Заведи ветку, внеси небольшое улучшение (документация/тест/мелкий фикс).
- [ ] Синхронизируйся с upstream (`git fetch upstream` + rebase), запушь в свой форк.
- [ ] Открой **PR в оригинальный проект** по их CONTRIBUTING-гайдлайнам.
- [ ] Пройди реальное ревью мейнтейнеров, отреагируй на замечания.

---

## 📦 Результат

1. Ссылка на репозиторий/PR с **образцовым описанием** и историей обсуждения.
2. Минимум 3 PR (вариант A) или 1 принятый/отревьюенный PR в чужой проект (вариант B).
3. Защищённый main, осмысленные ветки, чистая история.
4. Краткая рефлексия: что было сложно в ревью/синхронизации, что вынес.

---

## 🌟 Усложнения

- Добавь шаблон PR (`.github/pull_request_template.md`) и `CODEOWNERS`.
- Настрой простой CI (GitHub Actions: запуск тестов на PR) — заглядывая в DevOps-трек.
- Поучаствуй ревьюером в чужом учебном PR — дай конструктивную обратную связь.

---

## ✅ Критерии

- [ ] Прошёл полный цикл issue → ветка → PR → ревью → merge
- [ ] PR хорошо описаны, маленькие, атомарные
- [ ] Применил конструктивное ревью (как автор и/или ревьюер)
- [ ] main защищён, история и ветки в порядке
