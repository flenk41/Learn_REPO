# 13 · Удалённые репозитории 🖼️⭐

> 🎯 **Цель блока:** понять, как локальный репозиторий связывается с сервером (GitHub/GitLab) —
> remotes, clone, origin. Это вход в командную работу.

---

## 📖 Что такое remote

```
   REMOTE — ссылка на копию репозитория в другом месте (на сервере). Git распределённый: у тебя
   ПОЛНАЯ история локально, а remote — общая «точка встречи», куда вы все пушите и откуда тянете.

   origin — имя remote по умолчанию (главный сервер репозитория). можно несколько remotes
   (origin, upstream — для форков).

   git remote -v                 # показать настроенные remotes (адреса)
   git remote add origin <url>   # привязать remote по имени origin
   git remote remove origin      # отвязать
```

🖼️
```
   твой компьютер                       сервер (GitHub)
   ┌──────────────┐   push (отдать)    ┌──────────────┐
   │  локальный   │ ─────────────────► │   origin     │
   │  репозиторий │ ◄───────────────── │  (общий)     │
   └──────────────┘   fetch (забрать)  └──────────────┘
        ▲                                     ▲
        │ у тебя полная история               │ у коллег тоже свои копии
```

💡 ⭐ Remote — это просто «адрес другой копии». `origin` — соглашение об имени для главного сервера.
Поскольку Git распределён, твой локальный репо самодостаточен; remote нужен, чтобы **обмениваться**
работой с другими (и иметь резервную копию в облаке).

## ⭐ Clone и связь

```
   git clone <url>               # скачать репозиторий + историю + настроить origin автоматически
   git clone <url> my-folder     # в конкретную папку

   после clone:
   • есть полная история, рабочая копия ветки по умолчанию,
   • origin уже привязан → можно сразу fetch/pull/push (модуль 14).

   создать новый на сервере:
   • на GitHub: New repository → следуй инструкции (git remote add origin ... + git push -u origin main).
```

## ⭐ Доступ: SSH и токены

```
   приватные операции (push) требуют аутентификации. два способа:

   SSH-ключ (удобно для постоянной работы):
   ssh-keygen -t ed25519 -C "you@example.com"     # сгенерировать пару ключей

   git remote set-url origin git@github.com:user/repo.git    # SSH-адрес

   # вместо пароля при push вводишь токен (Settings → Developer settings → tokens)
```

💡 ⭐ GitHub давно не принимает пароль для git-операций. Настрой SSH-ключ (один раз) — и push/pull
работают без ввода пароля. Никогда не вкладывай токен/ключ в код или коммиты (модуль 06).

## 📖 remote-tracking ветки

```
   origin/main — «remote-tracking» ветка: твоё локальное ЗНАНИЕ о том, где main на сервере
   (на момент последнего fetch). Это не то же, что твой локальный main.

   git branch -a                 # все ветки, включая remotes/origin/*
   git fetch                     # обновить знание о сервере (origin/*) — модуль 14
```

💡 `origin/main` обновляется только при `fetch`/`pull` — это «снимок сервера у тебя». Разница между
`main` и `origin/main` показывает, насколько ты впереди/позади сервера.
