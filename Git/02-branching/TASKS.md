# ✅ Задачи · Уровень 2 — Ветки и слияние (ядро)

> Это ядро трека. Делай всё руками и СМОТРИ `git log --oneline --graph --all` после каждого шага —
> визуализация графа закрепляет модель.

---

## Ветки (модуль 08)

1. Создай `feature/a`, сделай 2 коммита. Вернись в main — изменения исчезли из файлов? Почему?
2. Заведи две ветки от main, в каждой свой коммит. Смотри обе в `--graph --all`.
3. ⭐ Войди в detached HEAD, сделай коммит, выйди — как сохранить его веткой?

## Merge (модуль 09)

4. Слей ветку в main fast-forward (main не трогал). Merge-коммита нет — почему?
5. Создай расхождение (коммиты и в main, и в ветке) → слей. Теперь merge-коммит есть.
6. ⭐ Слей с `--no-ff`. Сравни граф с обычным слиянием.

## Конфликты (модуль 10)

7. Спровоцируй конфликт (две ветки правят одну строку). Реши руками, заверши `add`+`commit`.
8. Спровоцируй конфликт и отмени через `git merge --abort`.
9. ⭐ Реши конфликт через UI своей IDE. Сравни с ручным способом.

## Rebase (модуль 11)

10. Создай расхождение, сделай `git rebase main` на feature. История линейна? Хеши изменились?
11. ⭐ Вызови конфликт в rebase, реши, `git rebase --continue`.
12. ⭐ Попробуй `git pull --rebase` — чем история отличается от обычного pull?

## Merge vs rebase (модуль 12)

13. Один сценарий разреши и через merge, и через rebase. Сравни `--graph`.
14. ⭐⭐ Для 3 ситуаций выбери merge/rebase с обоснованием (влить фичу / актуализировать ветку /
    причесать черновики).
