# 02 · Установка и настройка 🖼️

> 🎯 **Цель блока:** установить Git, настроить имя/почту, познакомиться с интерфейсами (CLI, GUI,
> IDE) и завести аккаунт на хостинге.

---

## 🛠️ Установка

```
   Windows:  скачай с git-scm.com (включает Git Bash — удобный терминал). Или: winget install Git.Git
   macOS:    brew install git   (или установится с Xcode Command Line Tools)
   Linux:    sudo apt install git   /   sudo dnf install git

   проверка:
   git --version       → должно вывести версию (например, git version 2.43.0)
```

---

## ⭐ Первая настройка (обязательно)

```
   Git подписывает каждый коммит твоим именем и почтой. Настрой ОДИН раз (глобально):

   git config --global user.name  "Имя Фамилия"
   git config --global user.email "you@example.com"

   полезное:
   git config --global init.defaultBranch main      # ветка по умолчанию — main
   git config --global core.editor "code --wait"    # редактор для сообщений (VSCode)
   git config --global pull.rebase false            # поведение pull (модуль 14)

   посмотреть всё:
   git config --list
```

💡 ⭐ Имя и почта — не для входа, а для **авторства коммитов** в истории. Используй ту же почту, что
на GitHub/GitLab, чтобы коммиты связывались с твоим профилем. Это первое, что делают на новой машине.

---

## 📖 Как работать с Git: три интерфейса

```
   1. КОМАНДНАЯ СТРОКА (CLI) — git add, git commit, git push... ⭐ учись на ней:
      она одинакова везде, мощнее всего, и за GUI всё равно стоят эти команды.
   2. GUI-клиенты — GitHub Desktop, Sourcetree, GitKraken: визуальные, удобны для истории/диффов.
   3. ВСТРОЕНО В IDE — VSCode, JetBrains: коммиты, ветки, диффы прямо в редакторе (удобно ежедневно).

   рекомендация: понимай CLI (этот трек — на ней), а в быту используй что удобно. GUI скрывает
   суть, пока не освоишь модель — учись на командах.
```

---

## ⭐ Аккаунт на хостинге

```
   заведи аккаунт на GitHub (github.com) — стандарт де-факто (или GitLab/Bitbucket).
   зачем: хранить репозитории в облаке, сотрудничать, делать pull request'ы, портфолио.

   доступ к приватным репо — по SSH-ключу или токену (не паролю):
   • SSH: ssh-keygen → добавь публичный ключ в настройки GitHub (модуль 13 подробнее).
   • HTTPS + Personal Access Token.
```

> 🧭 GitHub — твоё портфолио разработчика. Публичные репозитории с проектами курса
> ([капстоун](../../Capstone/README.md)) — то, что смотрят работодатели.

---

## ⚠️ Ловушки

- ❌ Не настроить user.name/email → коммиты с непонятным/чужим авторством.
- ❌ Учиться только в GUI → не понимаешь, что происходит «под капотом».
- ❌ Хранить пароль в открытую вместо SSH-ключа/токена.

---

## ✅ Задачи

1. Установи Git, проверь `git --version`.
2. Настрой `user.name` и `user.email` глобально. Проверь через `git config --list`.
3. Заведи аккаунт на GitHub (если ещё нет).
4. Открой Git-панель в своей IDE — посмотри, какие операции там есть.
