# 01 · Что такое Git: снимки, а не дельты 🖼️⭐

> 🎯 **Цель блока:** понять ментальную модель Git — коммит как снимок проекта, история как цепочка
> снимков. Это фундамент: кто понимает модель, тот не «заучивает команды», а думает.

---

## 📖 Коммит — это снимок (snapshot)

```
   многие VCS хранят ИЗМЕНЕНИЯ (дельты): «в файле X добавили строку 5, удалили строку 8».
   Git хранит СНИМКИ: каждый коммит — это фотография ВСЕГО проекта в этот момент.
   (технически неизменные файлы не дублируются — ссылка на прежнюю версию; но модель — «снимок».)

   коммит = { снимок файлов + автор + дата + сообщение + ссылка на родителя(ей) }
```

🖼️
```
   история = цепочка снимков, каждый ссылается на предыдущий (родителя):

   [C1] ◄── [C2] ◄── [C3] ◄── [C4]   ← каждый коммит знает своего родителя
   снимок   снимок   снимок   снимок
                                 ▲
                                HEAD (где ты сейчас)
```

💡 ⭐ Думай о коммите как о **сохранении в игре**: точка, к которой можно вернуться. История — это
последовательность сохранений. Эта модель «снимков + родитель» объясняет почти всё в Git: ветка —
это просто указатель на коммит, merge — коммит с двумя родителями, откат — переход к другому снимку.

---

## ⭐ Три ключевых объекта

```
   • КОММИТ (commit) — снимок + метаданные. Идентифицируется ХЕШЕМ (SHA-1, напр. a1b2c3d...).
   • ВЕТКА (branch) — лёгкий ПОДВИЖНЫЙ указатель на коммит (по умолчанию `main`). Создать — мгновенно.
   • HEAD — указатель на «где ты сейчас» (обычно на текущую ветку → на её последний коммит).

   хеш коммита — отпечаток содержимого: если что-то изменить, хеш другой. Поэтому история
   неподделываема: каждый коммит «запечатывает» всё предыдущее.
```

💡 ⭐ Ветка в Git — это **не копия файлов**, а просто ярлык (41 байт) на коммит. Поэтому ветки
дёшевы и быстры — в этом сила Git (модуль 08). Когда делаешь коммит, ярлык текущей ветки сдвигается
на новый коммит.

---

## 📖 Три состояния файла

```
   файл в Git проходит через три «зоны»:

   РАБОЧАЯ ПАПКА  ──git add──►  STAGING (индекс)  ──git commit──►  РЕПОЗИТОРИЙ (история)
   (твои правки)               (что войдёт в                     (сохранённый снимок)
                                следующий коммит)

   • modified — изменил файл, но ещё не отметил.
   • staged — отметил (add): попадёт в следующий коммит.
   • committed — сохранено в историю.
```

💡 Staging (промежуточная зона) — особенность Git: ты сам решаешь, ЧТО именно войдёт в коммит (не
обязательно все изменения сразу). Это позволяет делать аккуратные, осмысленные коммиты (модуль 03, 07).
