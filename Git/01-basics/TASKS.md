# ✅ Задачи · Уровень 1 — Основы Git

> Всё руками в учебном репозитории. Запускай `git status` после каждого шага — это рефлекс.

---

## Базовый цикл (модуль 03)

1. Создай репозиторий `git init`, добавь 3 файла, сделай первый коммит.
2. Измени два файла под РАЗНЫЕ задачи. Закоммить их двумя отдельными коммитами.
3. ⭐ Используй `git add -p`, чтобы закоммитить только ЧАСТЬ изменений одного файла.

## История (модуль 04)

4. Сделай 6+ коммитов. Посмотри `git log --oneline --graph`.
5. Открой один коммит через `git show`. Что за метаданные?
6. ⭐ Сделай `git blame` на файл. Найди, в каком коммите появилась конкретная строка, и открой его.

## Отмена (модуль 05)

7. Выкинь правку в рабочей папке (`git restore`).
8. Убери файл из staging (`git restore --staged`).
9. ⭐ Сделай коммит, отмени через `git reset --soft HEAD~1`. Куда делись изменения?
10. ⭐⭐ Сделай `git reset --hard`, потом восстанови «потерянный» коммит через `git reflog`.

## .gitignore (модуль 06)

11. Добавь `.gitignore` с `*.log` и `build/`. Проверь, что Git игнорирует такие файлы.
12. ⭐ Случайно закоммить «секрет» (`secret.txt`), затем убери его из отслеживания (`git rm --cached`).
    Почему в реальности этого мало?

## Хорошие коммиты (модуль 07)

13. Перепиши 5 плохих сообщений в хорошие (императив + «почему»).
14. ⭐⭐ Возьми крупное «грязное» изменение и разбей на серию атомарных коммитов.

    изменения между ними в обе стороны.
