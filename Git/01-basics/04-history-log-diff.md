# 04 · История: log и diff 🖼️⭐

> 🎯 **Цель блока:** научиться читать историю проекта — смотреть коммиты, сравнивать версии,
> выяснять, кто и зачем менял строку.

---

## ⭐ Просмотр истории: git log

```
   git log                       # полная история: хеш, автор, дата, сообщение
   git log --oneline             # компактно: один коммит = одна строка (хеш + сообщение)
   git log --oneline --graph     # + визуальный граф веток и слияний
   git log --oneline -10         # последние 10 коммитов
   git log --author="Имя"        # коммиты конкретного автора
   git log --since="2 weeks ago" # за период
   git log -- path/to/file       # история конкретного файла
   git log -p                    # с диффами (что именно менялось)
```

🖼️
```
   git log --oneline --graph:

   * a1b2c3d (HEAD -> main) Добавить валидацию email
   * d4e5f6a Исправить расчёт скидки
   * 9z8y7x6 Добавить страницу профиля
   * 1a2b3c4 Первый коммит
   ↑ хеш      ↑ сообщение (зачем сделано изменение)
```

💡 ⭐ `git log --oneline --graph --all` — твоя «карта» проекта: видно все ветки, слияния, куда
указывает HEAD. Привыкай читать историю — это половина работы с Git (понять, что происходило).

---

## ⭐ Сравнение: git diff

```
   git diff                  # незастейдженные изменения (рабочая папка ↔ staging)
   git diff --staged         # застейдженные (staging ↔ последний коммит)
   git diff HEAD             # все изменения относительно последнего коммита
   git diff main feature     # разница между двумя ветками
   git diff a1b2c3d 9z8y7x6  # между двумя коммитами
   git diff -- file.txt      # только по одному файлу

   чтение диффа:
   - удалённая строка (была)        ← красным
   + добавленная строка (стала)     ← зелёным
   @@ -10,7 +10,8 @@                 ← с какой строки, сколько строк
```

💡 ⭐ Diff — язык code review: PR показывает именно дифф. Научись бегло читать `-`/`+` блоки —
это то, что обсуждают на ревью (модуль 16).

---

## 📖 Кто и зачем: blame и show

```
   git show <хеш>            # показать конкретный коммит целиком (метаданные + дифф)
   git show HEAD             # последний коммит
   git blame file.txt        # для КАЖДОЙ строки: кто, когда, в каком коммите её написал
   git log -S "функция"      # найти коммиты, где добавили/убрали этот текст («поиск по истории»)
```

💡 `git blame` + `git show` — связка для расследования: «почему здесь так написано?» → blame даёт
коммит → show показывает контекст и сообщение (зачем). Поэтому важны осмысленные сообщения (модуль 07).

## 📖 Ссылки на коммиты

```
   способы указать коммит:
   • полный хеш  a1b2c3d4e5f6...  или короткий a1b2c3d (хватает 7 символов)
   • HEAD        — текущий коммит
   • HEAD~1      — на 1 назад (родитель), HEAD~3 — на 3 назад
   • main        — последний коммит ветки main
   • тег/ветка   — по имени
```

---

## ⚠️ Ловушки

- ❌ Не уметь читать `git log` → не понимаешь, что происходило в проекте.
- ❌ Путать `git diff` (рабочая) и `git diff --staged` (staged) — показывают разное.
- ❌ Игнорировать `git blame` при вопросе «почему так» (он отвечает за минуты).

## ✅ Задачи

1. Сделай 5 коммитов. Посмотри историю через `git log --oneline --graph`.
2. Измени файл, не добавляя в staging. Сравни `git diff` и `git diff --staged`.
3. Через `git show` открой один из своих коммитов. Что в нём за метаданные?
4. Сделай `git blame` на файл — кто автор каждой строки?
5. Сравни две точки истории: `git diff HEAD~3 HEAD`.
