# 🏗️ Проект уровня 4 — Профессиональный репозиторий «под ключ»

> 🎯 **Цель:** собрать всё в один образцовый репозиторий с автоматизацией, релизами и
> восстановлением — такой, какой не стыдно показать на собеседовании.

---

## Что делаем

Возьми реальный проект (например, [капстоун](../../Capstone/README.md) или библиотеку) и доведи его
Git-инфраструктуру до профессионального уровня.

---

## Этапы

### Качество и автоматизация
- [ ] **pre-commit framework**: форматтер + линтер + проверка секретов (gitleaks/detect-secrets) +
      trailing-whitespace. Закоммить конфиг в репозиторий.
- [ ] **CI на GitHub Actions**: сборка + тесты + линт на каждый PR. Бейдж статуса в README.
- [ ] **Защита main**: required PR, required review, required green CI.

### Чистая история
- [ ] Веди работу через фиче-ветки и PR (никаких прямых пушей в main).
- [ ] Перед каждым PR причёсывай ветку интерактивным rebase (черновики → атомарные коммиты).
- [ ] Conventional Commits на всех коммитах.

### Релизы
- [ ] Выпусти минимум 2 версии аннотированными тегами по **semver** (например, v0.1.0 → v0.2.0).
- [ ] Создай GitHub **Releases** с заметками. Веди `CHANGELOG.md`.
- [ ] Покажи на примере: какое изменение → какой рост версии (PATCH/MINOR/MAJOR).

### Мастерство восстановления
- [ ] Намеренно устрой 3 «аварии» и восстановись, задокументировав команды:
      1) `reset --hard` → восстановление через `reflog`;
      2) удаление ветки → восстановление;
      3) баг в истории → поиск через `git bisect`.
- [ ] Приложи `RECOVERY.md` — твою личную аптечку Git.

---

## 📦 Результат

1. Публичный репозиторий с зелёным CI, защищённым main, pre-commit и бейджем в README.
2. Чистая история (атомарные коммиты, PR, причёсанные ветки).
3. Минимум 2 релиза (теги + Releases + CHANGELOG) по semver.
4. `RECOVERY.md` с разобранными авариями и командами восстановления.

---

## 🌟 Усложнения

- Шаблоны issue/PR (`.github/`), `CODEOWNERS`, авто-changelog из Conventional Commits.
- Workflow, собирающий артефакт и публикующий его в Release по пушу тега `v*`.
- `git bisect run` с автотестом — автоматический поиск регрессии в CI.

---

## ✅ Критерии

- [ ] Репозиторий выглядит профессионально (CI, защита main, pre-commit, README с бейджем)
- [ ] История чистая и атомарная, всё через PR с ревью
- [ ] Версионирование по semver с тегами и Releases
- [ ] Уверенно восстанавливаюсь после аварий (документировано)
