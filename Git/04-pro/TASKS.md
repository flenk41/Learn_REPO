# ✅ Задачи · Уровень 4 — Профи и восстановление

> Продвинутые инструменты. Делай в учебном репо, не бойся ломать — ты уже умеешь восстанавливать.

---

## Интерактивный rebase (модуль 18)

1. 5 черновых коммитов (с «fix typo», «debug») → собери в 2 осмысленных через `rebase -i`.
2. Переименуй плохое сообщение (`reword`), выкинь отладочный коммит (`drop`).
3. ⭐ `git commit --fixup` + `rebase -i --autosquash`.

## stash / cherry-pick / reflog (модуль 19)

4. `stash` → переключись на хотфикс → вернись `stash pop`.
5. Перенеси один коммит между ветками через `cherry-pick`.
6. ⭐⭐ `reset --hard`, затем восстанови через `reflog`.

## Теги и semver (модуль 20)

7. Аннотированный тег `v1.0.0` + push на сервер.
8. Для багфикса/фичи/ломающего изменения определи рост semver (PATCH/MINOR/MAJOR).
9. ⭐ Создай GitHub Release из тега с заметками.

## Хуки и CI (модуль 21)

10. Напиши `pre-commit` хук, блокирующий коммит при `TODO!` в диффе.
11. ⭐ Настрой pre-commit framework (формат + проверка секретов).
12. ⭐⭐ GitHub Actions: прогон тестов на PR; сделай PR с падающим тестом → CI красный.

## Bisect и масштаб (модуль 22)

13. Внеси баг среди 15+ коммитов, найди через `git bisect`. Сколько шагов?
14. ⭐ `git bisect run ./test.sh` — автоматический поиск.
15. ⭐ `git clone --depth 1` крупного репо — чем отличается история?

## Восстановление (модуль 23)

16. Собери личную шпаргалку восстановления (ситуация → команды).
17. Удали ветку с работой и восстанови из `reflog`.
18. ⭐ Перед `git clean -fd` сделай `clean -n` — оцени опасность.
19. ⭐⭐ Распиши план «секрет попал в историю и запушен» (ротация → чистка → команда).
