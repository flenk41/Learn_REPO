## 📖 Золотое правило: сначала измерь

> «Преждевременная оптимизация — корень всех зол» (Кнут).

Сначала **профилируй**, найди узкое место, потом оптимизируй именно его. Узкое место почти
всегда не там, где кажется.

```cpp
#include <chrono>
auto start = std::chrono::high_resolution_clock::now();
// ... код ...
auto end = std::chrono::high_resolution_clock::now();
auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();
std::cout << ms << " мс\n";
```

Профайлеры: **perf** (Linux), **Valgrind callgrind**, **VTune**, встроенный профайлер в
Visual Studio.

## ⭐ Флаги оптимизации компилятора

```bash
g++ -std=c++17 -O2 program.cpp -o program    # -O2 — разумный дефолт
g++ -std=c++17 -O3 -march=native program.cpp # агрессивнее + под твой CPU
```

| Флаг | Что делает |
|------|-----------|
| `-O0` | без оптимизации (для отладки) |
| `-O2` | хорошая оптимизация (релиз) |
| `-O3` | агрессивная |
| `-march=native` | использовать инструкции твоего CPU |

💡 Часто `-O2` даёт **больше**, чем ручные трюки. Всегда собирай релиз с оптимизацией.

## ⭐ Кэш-дружелюбный код

Как и в C (см. [C-курс модуль 22](../../C/04-senior/22-cache-optimization.md)), главный
фактор скорости — **расположение данных в памяти**. Последовательный доступ = кэш-хиты.

```cpp
// 🚀 vector — данные подряд, кэш-дружелюбно
std::vector<int> v(1'000'000);
for (int x : v) sum += x;        // последовательно → быстро

// 🐢 list — узлы разбросаны по куче → промахи кэша
std::list<int> lst;             // каждый узел отдельно → медленный обход
```

💡 **`vector` почти всегда быстрее `list`** на современных процессорах — даже для вставок,
потому что кэш важнее алгоритмической сложности на практике. Выбирай `vector` по умолчанию.

## ⭐ Избегай лишних копий

```cpp
// 🐢 копирование больших объектов
void process(std::vector<int> data);     // копирует весь вектор!
std::string get() { return huge_string; }

// 🚀 ссылки и перемещение
void process(const std::vector<int>& data);   // без копии
v.push_back(std::move(temp));                  // переместить, не копировать

// range-for без копий
for (const auto& item : container) { ... }     // const& вместо копии
```

💡 Самые частые потери производительности у новичков — **лишние копирования**. Передавай
по `const&`, перемещай где можно, бери `const auto&` в циклах.

## 📖 Другие приёмы (после профилирования!)

```cpp
// reserve — избежать перевыделений вектора
std::vector<int> v;
v.reserve(1000);                 // выделить заранее

// emplace_back вместо push_back — строит на месте, без временного объекта
v.emplace_back(args);

// inline для крошечных функций (компилятор обычно сам решает)
inline int square(int x) { return x * x; }
```

| Приём | Эффект |
|-------|--------|
| `-O2`/`-O3` | главный рычаг |
| последовательные данные (vector) | кэш-хиты |
| `const&` и move | нет лишних копий |
| `reserve` | нет перевыделений |
| `emplace_back` | нет временных объектов |
| `constexpr` | вычислено при компиляции |

## 🛠️ Инструменты Senior

```bash
# Санитайзеры (на этапе разработки)
g++ -fsanitize=address,undefined -g program.cpp    # память + UB
g++ -fsanitize=thread -g program.cpp               # гонки

# Статический анализ
clang-tidy program.cpp --
cppcheck --enable=all program.cpp

# Профилирование
perf stat ./program                                # счётчики CPU, кэш-промахи
valgrind --tool=callgrind ./program
```

## ✅ Задачи

1. **vector vs list.** Замерь время обхода и суммирования 1М элементов в `vector` и
   `list`. Объясни разницу (кэш).
2. **-O0 vs -O2.** Замерь тяжёлый цикл с разными уровнями оптимизации.
3. **Копия vs ссылка.** Замерь функцию, принимающую большой вектор по значению и по
   `const&`.
4. **reserve.** Замерь заполнение вектора на 1М элементов с `reserve` и без.
5. **emplace vs push.** Сравни `emplace_back` и `push_back` для объектов с конструктором.
6. ⭐ **perf.** Прогони cache-friendly и cache-unfriendly код под `perf stat`, сравни
   `cache-misses`.
