# 22 · Шаблоны и метапрограммирование

> 🎯 **Цель блока:** освоить продвинутые шаблоны — вариативные шаблоны, `constexpr`,
> концепты (C++20). Это вычисления **на этапе компиляции** — уникальная сила C++.

---

## ⭐ constexpr — вычисления при компиляции

`constexpr` означает «может быть вычислено компилятором, ещё до запуска программы»:

```cpp
constexpr int square(int x) { return x * x; }

constexpr int val = square(5);     // вычислено при компиляции → 25
int arr[square(4)];                // можно как размер массива (16)
```

🖼️
```
   обычная функция:   square(5) считается в РАНТАЙМЕ
   constexpr:         square(5) считается КОМПИЛЯТОРОМ → в программе уже готовое 25
```

💡 `constexpr` переносит вычисления из времени выполнения во время компиляции — программа
работает быстрее (результат уже посчитан). В C++20 есть `consteval` (обязательно при
компиляции) и `constexpr`-контейнеры.

---

## ⭐ Вариативные шаблоны (variadic)

Шаблон, принимающий **любое число** аргументов любых типов:

```cpp
template<typename... Args>              // пакет параметров
void print(Args... args) {
    ((std::cout << args << " "), ...);  // fold expression (C++17)
    std::cout << "\n";
}

print(1, "два", 3.0, 'x');             // работает с любым числом и типами
```

💡 Так устроены `printf`-подобные функции, `make_unique`, `emplace_back`. Пакет `Args...`
разворачивается в конкретные аргументы при компиляции.

---

## ⭐ Концепты (C++20) — ограничения на типы

Раньше ошибки шаблонов были нечитаемыми «портянками». **Концепты** задают требования к
типам понятно:

```cpp
#include <concepts>

template<typename T>
concept Numeric = std::integral<T> || std::floating_point<T>;

template<Numeric T>                     // T обязан быть числом
T sum(T a, T b) { return a + b; }

sum(2, 3);        // ✅
sum(2.5, 1.5);    // ✅
// sum("a", "b"); // ❌ понятная ошибка: "a" не Numeric
```

💡 Концепты делают шаблоны самодокументируемыми и дают человекочитаемые ошибки. Это одно
из главных улучшений современного C++.

---

## 📖 Type traits — информация о типах

```cpp
#include <type_traits>

template<typename T>
void process(T value) {
    if constexpr (std::is_integral_v<T>) {       // ветвление ПРИ КОМПИЛЯЦИИ
        std::cout << "целое\n";
    } else {
        std::cout << "не целое\n";
    }
}
```

💡 `if constexpr` выбирает ветку **на этапе компиляции** в зависимости от типа — ненужная
ветка вообще не компилируется. Мощный инструмент обобщённого кода.

---

## 📖 Зачем это всё

- **Производительность:** вычисления при компиляции = ноль затрат в рантайме.
- **Обобщённость:** один код для многих типов (вся STL так устроена).
- **Безопасность:** концепты ловят ошибки типов рано и понятно.

> ⚠️ Метапрограммирование — мощно, но легко переусложнить. Используй, когда это реально
> упрощает код или нужна производительность. Не превращай простую задачу в шаблонный ребус.

## ✅ Задачи

1. **constexpr.** Напиши `constexpr` факториал, используй его как размер массива.
2. **Вариативный print.** Реализуй `print(...)` для любого числа аргументов через fold.
3. **Вариативная сумма.** `sum(1, 2, 3, 4)` для любого числа чисел.
4. **Концепт.** Напиши концепт `Printable` (есть оператор `<<`) и функцию, работающую
   только с такими типами (`-std=c++20`).
5. **if constexpr.** Функция, по-разному обрабатывающая целые и дробные типы.
6. ⭐ **constexpr-вычисления.** Посчитай таблицу простых чисел при компиляции.
