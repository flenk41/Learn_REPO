## 📖 Rvalue-ссылки `&&`

Из Уровня 2 ты знаешь move. Под капотом он работает через **rvalue-ссылки** (`&&`) —
ссылки на временные объекты, из которых можно «забрать кишки».

```cpp
void take(std::string&& s) {        // && — принимает только rvalue (временные)
    std::string local = std::move(s);  // забрать данные
}

take(std::string{"временный"});     // ✅ rvalue
std::string named = "именованный";
// take(named);                     // ❌ named — lvalue
take(std::move(named));             // ✅ std::move превращает в rvalue
```

🖼️
```
   T&   — ссылка на lvalue (именованный объект)
   T&&  — ссылка на rvalue (временный) → можно забрать данные
   std::move(x) — «приведи x к rvalue, разрешаю забрать»
```

## ⭐ Перемещающий конструктор и присваивание

Класс, владеющий ресурсом, должен уметь его **перемещать** (а не только копировать):

```cpp
class Buffer {
    int* data; size_t size;
public:
    Buffer(size_t n) : data(new int[n]), size(n) {}
    ~Buffer() { delete[] data; }

    // перемещающий конструктор — «крадёт» указатель
    Buffer(Buffer&& o) noexcept
        : data(o.data), size(o.size) {
        o.data = nullptr;           // источник обнуляем (чтобы не было double delete)
        o.size = 0;
    }
    // перемещающее присваивание
    Buffer& operator=(Buffer&& o) noexcept {
        if (this != &o) {
            delete[] data;          // освободить своё
            data = o.data; size = o.size;
            o.data = nullptr; o.size = 0;
        }
        return *this;
    }
};
```

🖼️
```
   копирование:  выделить новую память + переписать всё  🐢
   перемещение:  data = o.data; o.data = nullptr         ⚡ (перецепили указатель)
```

> 💡 `noexcept` на move-операциях важен: контейнеры (vector) используют перемещение при
> росте, только если оно `noexcept` — иначе откатываются на копирование ради безопасности.

## ⭐⭐ Правило нуля — лучшее правило

Правило пяти (Уровень 3) требует писать 5 функций для класса с ресурсом. Но это легко
сделать неправильно. **Правило нуля:** не управляй ресурсами вручную — используй
RAII-обёртки (`vector`, `unique_ptr`), и тогда **не пиши ни одной из пяти**.

```cpp
// ❌ правило пяти — много кода, легко ошибиться
class Buffer {
    int* data;
    // нужно писать деструктор, копирование, перемещение... 5 функций
};

// ✅ правило нуля — ноль кода управления ресурсами
class Buffer {
    std::vector<int> data;          // vector сам всё умеет
public:
    Buffer(size_t n) : data(n) {}
    // компилятор сам сгенерирует ПРАВИЛЬНЫЕ копирование/перемещение/деструктор!
};
```

💡 **Правило нуля — главный совет по управлению памятью в C++:** оборачивай ресурсы в
готовые RAII-типы, и компилятор сделает копирование/перемещение правильно за тебя. Пиши
свои 5 функций, только когда оборачиваешь «сырой» ресурс (это редко).

## ⭐ Copy elision и RVO — копий нет вообще

Компилятор часто **полностью убирает** копирование/перемещение при возврате из функции —
это **RVO** (Return Value Optimization):

```cpp
std::vector<int> create() {
    std::vector<int> result(1000);
    return result;          // НЕ копируется и даже НЕ перемещается —
}                           // объект строится сразу в месте вызова (RVO)

std::vector<int> v = create();   // ноль копий, ноль перемещений
```

🖼️
```
   Наивно:  построить result → переместить в v → уничтожить result
   RVO:     построить сразу в v                  (промежуток исключён)
```

> ⚠️ Поэтому **не пиши `return std::move(x)`** — это мешает RVO и делает код медленнее.
> Просто `return x;` — компилятор выберет оптимальный вариант (часто вообще без копий).

## 📖 Perfect forwarding (идеальная передача)

Шаблонная функция может «прозрачно» передать аргументы дальше, сохранив их категорию
(lvalue/rvalue) — через `std::forward`:

```cpp
template<typename T>
void wrapper(T&& arg) {              // && здесь — "универсальная ссылка"
    process(std::forward<T>(arg));  // передать как есть: lvalue как lvalue, rvalue как rvalue
}
```

💡 Используется в реализации `make_unique`, `emplace_back` и др. — чтобы создавать объекты
без лишних копий. Тонкая тема; на старте достаточно знать, что она существует и зачем.
