## 📖 Эволюция C++

C++ кардинально изменился. «Современный C++» (с C++11) — это почти другой язык: безопаснее,
выразительнее, без ручного управления памятью.

| Стандарт | Ключевое |
|----------|----------|
| **C++11** | auto, умные указатели, move, лямбды, range-for, nullptr |
| **C++14** | make_unique, обобщённые лямбды |
| **C++17** | structured bindings, `if constexpr`, `optional`, `std::filesystem` |
| **C++20** | концепты, ranges, корутины, модули, `<=>` |
| **C++23** | `std::expected`, улучшения ranges |

## ⭐ Полезные типы современного C++

### std::optional — «значение или ничего»
```cpp
#include <optional>
std::optional<int> findUser(const std::string& name) {
    if (/* нашли */) return 42;
    return std::nullopt;            // «ничего» (вместо -1 или nullptr)
}

if (auto user = findUser("Гена")) {
    std::cout << *user << "\n";
}
```
💡 `optional` честно выражает «может не быть значения» — безопаснее, чем магические числа.

### std::variant — «один из нескольких типов»
```cpp
#include <variant>
std::variant<int, std::string> value = 42;
value = "теперь строка";           // type-safe объединение
```

### structured bindings (C++17)
```cpp
std::map<std::string, int> m = {{"a", 1}};
for (const auto& [key, val] : m)   // распаковка пары
    std::cout << key << "=" << val << "\n";
```

### std::string_view — строка без копирования
```cpp
void print(std::string_view sv) { std::cout << sv; }   // не копирует строку
```

## ⭐ Ranges (C++20) — конвейеры обработки

```cpp
#include <ranges>
#include <vector>

std::vector<int> v = {1, 2, 3, 4, 5, 6};

auto result = v
    | std::views::filter([](int x){ return x % 2 == 0; })   // только чётные
    | std::views::transform([](int x){ return x * x; });    // квадраты
// result: 4, 16, 36 — ленивый конвейер (как генераторы Python!)
```

💡 Ranges — это питоничные ленивые конвейеры в C++. Читается сверху вниз, без временных
векторов.

## 📋 Гайдлайны современного C++

```
   ✅ Умные указатели вместо new/delete
   ✅ RAII для всех ресурсов
   ✅ const везде, где возможно (const correctness)
   ✅ auto для очевидных типов
   ✅ Правило нуля (vector/string/умные указатели вместо ручной памяти)
   ✅ Передача больших объектов по const&
   ✅ Алгоритмы STL вместо ручных циклов
   ✅ {} -инициализация
   ✅ enum class вместо обычного enum
   ❌ Голые new/delete, сырые владеющие указатели, C-массивы, макросы
```

## 🛠️ Инструменты профессионала

```bash
# Сборка
cmake -B build && cmake --build build      # стандарт для проектов

# Качество кода
clang-format -i *.cpp                       # автоформатирование
clang-tidy *.cpp --                         # линтер, находит проблемы
cppcheck --enable=all *.cpp                 # статический анализ

# Проверки во время разработки
g++ -std=c++20 -Wall -Wextra -fsanitize=address,undefined -g
```

| Инструмент | Зачем |
|-----------|-------|
| **CMake** | сборка проектов (кроссплатформенно) |
| **clang-format** | единый стиль кода |
| **clang-tidy** | линтер и модернизация кода |
| **санитайзеры** | память, UB, гонки |
| **Catch2 / GoogleTest** | юнит-тесты |
