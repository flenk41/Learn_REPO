# 21 · Многопоточность 🖼️⭐

> 🎯 **Цель блока:** запускать потоки в C++ и безопасно работать с общей памятью. В
> отличие от Python (GIL), потоки C++ — **настоящий параллелизм** на разных ядрах.

---

## ⭐ std::thread — настоящий параллелизм

```cpp
#include <thread>
#include <iostream>

void work(int id) {
    std::cout << "Поток " << id << " работает\n";
}

int main() {
    std::thread t1(work, 1);     // запустить поток
    std::thread t2(work, 2);
    t1.join();                   // дождаться завершения
    t2.join();
    return 0;
}
```

🆚 В Python из-за GIL потоки **не** считают параллельно (см.
[Python-курс модуль 23](../../Python/04-senior/23-concurrency.md)). В C++ **нет GIL** —
потоки реально работают на разных ядрах одновременно. Это мощно, но опаснее: ты сам
отвечаешь за синхронизацию.

> ⚠️ Каждый `std::thread` нужно `join()` (дождаться) или `detach()` (отпустить) до
> разрушения, иначе программа упадёт. Современный `std::jthread` (C++20) делает `join` сам.

## ⚠️ Гонка данных (та же проблема, что в C)

```cpp
int counter = 0;
void increment() {
    for (int i = 0; i < 1000000; ++i)
        counter++;               // ⚠️ ГОНКА! несколько потоков портят counter
}
// два потока → результат НЕ 2000000, а случайный
```

🖼️ `counter++` — это «прочитать → +1 → записать», три операции. Два потока вклиниваются
друг в друга и теряют изменения (та же картина, что в [C-курсе](../../C/04-senior/21-concurrency.md)).

---

## ⭐ Решение 1: mutex + RAII (lock_guard)

```cpp
#include <mutex>

std::mutex mtx;
int counter = 0;

void increment() {
    for (int i = 0; i < 1000000; ++i) {
        std::lock_guard<std::mutex> lock(mtx);   // RAII: захват замка
        counter++;                               // критическая секция
    }                                            // ← lock снимается автоматически
}
```

💡 `std::lock_guard` — это **RAII для блокировки**: захватывает мьютекс в конструкторе,
освобождает в деструкторе. Даже при исключении замок снимется. Это та же идея RAII, что
и для памяти — никогда не «забытый» unlock.

> 💡 `std::unique_lock` — более гибкая версия (можно блокировать/разблокировать вручную,
> использовать с условными переменными).

## ⭐ Решение 2: атомарные операции

Для простых счётчиков мьютекс — перебор. `std::atomic` делает операции неделимыми:

```cpp
#include <atomic>

std::atomic<int> counter{0};

void increment() {
    for (int i = 0; i < 1000000; ++i)
        counter++;               // атомарно — без замка, без гонки
}
```

✅ Быстрее мьютекса, нет deadlock. Подходит для счётчиков, флагов.

## ⭐ std::async и future — результат из потока

Удобный способ запустить задачу и получить результат:

```cpp
#include <future>

int compute() {
    return 42;
}

int main() {
    std::future<int> result = std::async(std::launch::async, compute);
    // ... делаем что-то параллельно ...
    std::cout << result.get() << "\n";   // дождаться и забрать результат
    return 0;
}
```

🖼️ `future` — «обещание будущего результата». `get()` ждёт, пока задача закончится, и
возвращает значение. Удобнее ручного `thread` + общая переменная.

## 📖 Параллельные алгоритмы (C++17)

STL-алгоритмы умеют распараллеливаться одним флагом:

```cpp
#include <execution>
std::sort(std::execution::par, v.begin(), v.end());   // параллельная сортировка!
```

## ⚠️ Deadlock и советы

> ⚠️ **Deadlock** — два потока ждут замки друг друга вечно. Чтобы избежать: захватывай
> мьютексы всегда в одном порядке, держи блокировки минимально, используй `std::lock`
> для нескольких сразу.

Правила безопасной многопоточности:
- защищай **любые** общие изменяемые данные (mutex/atomic);
- держи критические секции короткими;
- предпочитай `atomic` для простых случаев, `lock_guard` для сложных;
- лови гонки инструментом **ThreadSanitizer**:
  ```bash
  g++ -std=c++17 -fsanitize=thread -g program.cpp -o program
  ```

## ✅ Задачи

1. **Два потока** печатают свои id; дождись обоих через `join`.
2. **Гонка.** Воспроизведи неверный счётчик на 2–4 потоках, подтверди через TSan.
3. **Почини mutex.** Добавь `lock_guard`, получи правильный результат.
4. **Почини atomic.** Перепиши через `std::atomic`, сравни скорость с мьютексом.
5. **async.** Посчитай сумму большого массива в нескольких `std::async`-задачах, сложи
   результаты через `future.get()`.
6. ⭐ **Параллельная сортировка** через `std::execution::par`, сравни со скоростью обычной.
7. ⭐ **Пул потоков** (producer-consumer) с очередью задач, мьютексом и
   `condition_variable`.
