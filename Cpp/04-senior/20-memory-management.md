# 20 · Управление памятью и аллокаторы 🖼️

> 🎯 **Цель блока:** заглянуть глубже — как C++ выделяет память на низком уровне,
> выравнивание, placement new и кастомные аллокаторы. Это уровень системного программиста.

---

## 📖 Что происходит при new

`new` делает **два** шага:
1. выделяет сырую память (вызывает `operator new`, внутри — `malloc`);
2. вызывает конструктор объекта в этой памяти.

`delete` — наоборот: вызывает деструктор, потом освобождает память.

```cpp
MyClass* p = new MyClass(args);
// = operator new(sizeof(MyClass)) + конструктор MyClass(args)

delete p;
// = деструктор ~MyClass() + operator delete(p)
```

💡 Понимание этого разделения (память отдельно, конструирование отдельно) — ключ к
кастомным аллокаторам и контейнерам.

---

## ⭐ Выравнивание (alignment)

Как и в C (см. [C-курс, модуль 20](../../C/04-senior/20-alignment-ub.md)), объекты
выравниваются по адресам, кратным их размеру. В структурах появляется padding.

```cpp
struct Bad  { char a; int b; char c; };   // sizeof = 12 (с padding)
struct Good { int b; char a; char c; };   // sizeof = 8  (поля от больших к меньшим)

std::cout << alignof(int) << "\n";         // 4
std::cout << sizeof(Bad) << " " << sizeof(Good) << "\n";  // 12 8
```

```cpp
alignas(64) int cacheLine[16];             // выровнять по 64 байта (cache line)
```

💡 `alignas` управляет выравниванием — полезно для кэш-оптимизаций (модуль 23) и SIMD.

---

## ⭐ Placement new — конструирование в готовой памяти

`placement new` строит объект в **уже выделенной** памяти (не выделяет новую). Это основа
контейнеров и пулов:

```cpp
#include <new>

char buffer[sizeof(MyClass)];                  // сырая память (на стеке)
MyClass* p = new (buffer) MyClass(args);       // построить ЗДЕСЬ, без выделения

p->~MyClass();                                 // деструктор вызываем ВРУЧНУЮ
// освобождать buffer не нужно — это стек
```

🖼️
```
   обычный new:    выделить память + построить объект
   placement new:  построить объект в ГОТОВОЙ памяти (выделение отдельно)
```

💡 Так `std::vector` отделяет выделение (capacity) от конструирования (size): резервирует
память заранее, а объекты строит по мере `push_back` через placement new.

## ⭐ Кастомный аллокатор (идея)

Как и в C (arena/pool, [C-курс модуль 19](../../C/04-senior/19-allocators.md)), в C++
можно подменить стратегию выделения. Контейнеры STL принимают **аллокатор** параметром:

```cpp
std::vector<int, MyAllocator<int>> v;   // вектор с твоим аллокатором
```

Простой pool-аллокатор раздаёт куски из заранее выделённого блока — выделение O(1), ноль
фрагментации. Это ускоряет программы, создающие миллионы мелких объектов (игры, парсеры).

```cpp
// Можно переопределить глобальные operator new/delete для всей программы:
void* operator new(size_t size) {
    std::cout << "Выделено " << size << " байт\n";
    return std::malloc(size);
}
void operator delete(void* p) noexcept { std::free(p); }
```

💡 Кастомные аллокаторы — продвинутая тема. Важно понимать: **C++ даёт полный контроль над
тем, как выделяется память**, в отличие от Python. Это его суперсила для
высокопроизводительных систем.

## 📖 Умные указатели и аллокация

```cpp
auto p = std::make_unique<int>(42);     // одно выделение
auto sp = std::make_shared<int>(42);    // make_shared делает ОДНО выделение
                                        // под объект И счётчик вместе (эффективнее)
```

💡 `make_shared` объединяет память объекта и управляющего блока (счётчика) в одно
выделение — быстрее и кэш-дружелюбнее, чем `shared_ptr<int>(new int)`.

## ✅ Задачи

1. **Padding.** Создай структуры с разным порядком полей, сравни `sizeof`. Оптимизируй.
2. **alignof/alignas.** Выведи выравнивание типов; выровняй массив по 64 байта.
3. **Свой operator new.** Переопредели глобальные `operator new`/`delete` с логом —
   посмотри, сколько и когда выделяется памяти в программе.
4. **Placement new.** Построй объект в буфере на стеке, вызови деструктор вручную.
5. ⭐ **Pool-аллокатор.** Реализуй простой пул фиксированных блоков, замерь скорость против
   обычного `new` на миллионе аллокаций.
6. ⭐ **make_shared vs shared_ptr(new).** Сравни число выделений (через свой operator new).
