## ⭐ Передача аргументов: три способа

Это **важнейшая** тема. От способа передачи зависит, копируется ли объект и можно ли его
изменить.

### 1. По значению (копия)
```cpp
void byValue(int x) {
    x = 100;             // меняем КОПИЮ
}
int a = 5;
byValue(a);              // a остаётся 5
```
🖼️ Создаётся копия — оригинал не меняется. Для больших объектов — **дорого** (копируется всё).

### 2. По ссылке (`&`) — можно менять оригинал
```cpp
void byRef(int& x) {     // & = ссылка на оригинал
    x = 100;             // меняем ОРИГИНАЛ
}
int a = 5;
byRef(a);                // a стало 100!
```
🖼️
```
   a ──┐
       ├── x — это ДРУГОЕ ИМЯ для той же переменной (псевдоним)
   x ──┘
   меняем x → меняется a
```

🆚 В C для этого нужен был указатель: `void byPtr(int* x) { *x = 100; }` и вызов
`byPtr(&a)`. В C++ **ссылка** делает то же самое, но чище — без `*` и `&` при вызове.

### 3. По const-ссылке — читаем без копирования
```cpp
void print(const std::string& s) {   // не копируем И не меняем
    std::cout << s << "\n";
}
```
💡 **Золотое правило C++:** большие объекты передавай по `const&` — нет копирования
(быстро) и нет случайных изменений (безопасно). Это самый частый способ передачи.

### Шпаргалка
| Способ | Копирует? | Меняет оригинал? | Когда |
|--------|-----------|------------------|-------|
| `int x` | да | нет | мелкие типы, нужна копия |
| `int& x` | нет | **да** | надо изменить оригинал |
| `const int& x` | нет | нет | **читать большой объект** (по умолчанию) |

## ⭐ Ссылка vs указатель (коротко)

```cpp
int a = 5;
int& ref = a;        // ссылка: ДРУГОЕ ИМЯ для a, всегда указывает на a
int* ptr = &a;       // указатель: хранит АДРЕС a, можно перенаправить

ref = 10;            // меняет a (просто как a = 10)
*ptr = 20;           // меняет a через разыменование
```

| Ссылка `&` | Указатель `*` |
|------------|---------------|
| всегда ссылается на объект | может быть `nullptr` |
| нельзя перенаправить | можно менять, куда смотрит |
| синтаксис как у обычной переменной | нужны `*` и `&` |

💡 В C++ предпочитают **ссылки**, где можно — они безопаснее (не бывают «никуда»). Полно
разберём в [модуле 08](../02-memory/08-memory-references.md).

## ⭐ Перегрузка функций

В C++ можно иметь **несколько функций с одним именем** — компилятор выберет по типам
аргументов:

```cpp
int add(int a, int b) { return a + b; }
double add(double a, double b) { return a + b; }
std::string add(const std::string& a, const std::string& b) { return a + b; }

add(2, 3);          // вызовет int-версию → 5
add(2.5, 1.5);      // вызовет double-версию → 4.0
add("Привет, ", "мир");  // вызовет string-версию
```

🆚 В C так нельзя — там имена функций уникальны. Перегрузка делает API удобнее.

## 📖 Аргументы по умолчанию

```cpp
void greet(const std::string& name, const std::string& greeting = "Привет") {
    std::cout << greeting << ", " << name << "!\n";
}

greet("Гена");                // Привет, Гена!
greet("Гена", "Здравствуй");  // Здравствуй, Гена!
```

## 📖 Возврат нескольких значений

```cpp
#include <tuple>
std::tuple<int, int> minmax(int a, int b) {
    if (a < b) return {a, b};
    return {b, a};
}

auto [mn, mx] = minmax(7, 3);   // структурное связывание (C++17)
std::cout << mn << " " << mx;   // 3 7
```
💡 `auto [a, b] = ...` — современный способ распаковать несколько значений (как в Python).
