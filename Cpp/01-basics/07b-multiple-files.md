# 07b · Разбиваем проект на файлы 🖼️⭐⭐

> 🎯 **Цель блока:** выносить код в отдельные файлы и вызывать функции/классы между ними. В C++ это
> связка **заголовок (.hpp) + реализация (.cpp)** — как в C, но с нюансами C++.

> 🧭 Базовый навык; организацию проекта с **CMake** и дизайн API углубляет
> [раздел «Проекты и API»](../03b-projects-api/01-project-structure.md).

---

## 📖 Тот же принцип, что в C

```
   C++ наследует модель C: чтобы вызвать функцию из другого файла, нужно её ОБЪЯВЛЕНИЕ (заголовок) +
   ОПРЕДЕЛЕНИЕ (скомпилированный .cpp). линковщик соединит.

   модуль = пара:
   • заголовок .hpp (или .h) — ЧТО предлагается: объявления функций, классов, шаблонов.
   • реализация .cpp        — КАК это сделано: тела функций/методов.
```

💡 ⭐ Расширения по соглашению: заголовки — `.hpp`/`.h`, реализация — `.cpp`/`.cc`. Главное —
последовательность в проекте. (До C++20 модулей это стандартный способ; `import` из C++20 пока не
везде удобен, поэтому учим классику `.hpp`/`.cpp`.)

## ⭐⭐ Связка .hpp + .cpp (на примере)

🖼️
```
   math_utils.hpp — ЗАГОЛОВОК
   ─────────────────────────────────────────────
   #pragma once                 // защита от двойного включения (короче, чем #ifndef-гард)

   namespace mu {               // namespace — чтобы имена не сталкивались (особенность C++)
       int add(int a, int b);   // объявления, БЕЗ тел
       int mul(int a, int b);
   }

   math_utils.cpp — РЕАЛИЗАЦИЯ
   ─────────────────────────────────────────────
   #include "math_utils.hpp"

   namespace mu {
       int add(int a, int b) { return a + b; }
       int mul(int a, int b) { return a * b; }
   }

   main.cpp — ИСПОЛЬЗОВАНИЕ
   ─────────────────────────────────────────────
   #include <iostream>
   #include "math_utils.hpp"     // подключаем объявления → можно вызывать

   int main() {
       std::cout << mu::add(2, 3) << "\n";   // 5
   }
```

💡 ⭐⭐ Схема: **`.hpp` — объявления (часто в `namespace`), `.cpp` — определения, другие файлы
`#include "модуль.hpp"`**. `#pragma once` в начале каждого заголовка — защита от двойного включения
(аналог include guard из C). `namespace` группирует имена модуля и вызывается через `mu::add`.

## ⭐⭐ Классы по файлам

```
   класс обычно: ОБЪЯВЛЕНИЕ (интерфейс) в .hpp, РЕАЛИЗАЦИЯ методов в .cpp.

   counter.hpp
   ─────────────────────────────
   #pragma once
   class Counter {
   public:
       void inc();          // объявления методов (без тел)
       int value() const;
   private:
       int n_ = 0;
   };

   counter.cpp
   ─────────────────────────────
   #include "counter.hpp"
   void Counter::inc()        { ++n_; }        // Counter:: — метод из класса Counter
   int  Counter::value() const { return n_; }
```

💡 ⭐⭐ Тело метода вне класса пишут с префиксом `Counter::` (область видимости). Объявление класса —
в заголовке (его видят пользователи), реализация методов — в `.cpp`. Короткие методы можно определять
прямо в классе (inline), но крупные выноси в `.cpp` — быстрее компиляция, чище интерфейс.

## ⭐ Компиляция и линковка

```
   как в C: каждый .cpp → объектный .o, потом линковка. заголовки только #include-ятся.

   g++ -std=c++17 main.cpp math_utils.cpp counter.cpp -o app    # все .cpp разом

   по шагам:
   g++ -std=c++17 -c math_utils.cpp     # → math_utils.o
   g++ -std=c++17 -c main.cpp           # → main.o
   g++ main.o math_utils.o -o app       # линковка

   на практике сборкой управляет CMake (раздел 03b) — руками перечислять .cpp неудобно в больших проектах.
```

💡 ⭐ Перечисляй ВСЕ свои `.cpp` при сборке — иначе «undefined reference to `mu::add`» (линковщик не
нашёл определение). `.hpp` в команду сборки НЕ передают — они вставляются через `#include`.

## ⚠️ Ловушки

- ❌ «undefined reference» — забыл `math_utils.cpp` в команде компиляции.
- ❌ Определение обычной (не-inline, не-шаблонной) функции в `.hpp` → «multiple definition» при включении в несколько `.cpp`.
- ❌ Шаблоны (`template`) — наоборот, их определения ОБЫЧНО держат в заголовке (компилятору нужно тело в точке использования).
- ❌ Забыть `#pragma once`/include guard → переопределения.
- ❌ `#include "file.cpp"` — не включай реализацию; включай только заголовки.
- ❌ Несоответствие сигнатуры в `.hpp` и `.cpp` (или забыл `namespace`/`Counter::`) → ошибки линковки.
