# 03 · Переменные, типы, auto, const

> 🎯 **Цель блока:** освоить переменные и типы C++, познакомиться с `auto` и современной
> инициализацией. Если знаешь C — здесь много знакомого плюс новинки.

---

## 📖 Объявление переменных

```cpp
int age = 25;
double price = 19.99;
char grade = 'A';
bool is_ready = true;        // bool — настоящий логический тип (в C его не было!)
std::string name = "Гена";   // строка как объект (нужен #include <string>)
```

🆚 В отличие от C, в C++ есть полноценный тип `bool` (`true`/`false`) и удобный
`std::string` вместо массивов `char`.

---

## 📐 Основные типы и размеры

| Тип | Что хранит | Размер |
|-----|-----------|--------|
| `int` | целое | 4 байта |
| `double` | дробное (точное) | 8 байт |
| `float` | дробное | 4 байта |
| `char` | символ | 1 байт |
| `bool` | true/false | 1 байт |
| `long long` | большое целое | 8 байт |
| `std::string` | строка (объект) | переменный |

```cpp
#include <iostream>
int main() {
    std::cout << sizeof(int) << " " << sizeof(double) << std::endl;  // 4 8
    return 0;
}
```

💡 Размеры — как в C (см. [C-курс, модуль 03](../../C/01-basics/03-variables-types.md)),
ведь это тот же машинный уровень.

## ⭐ Современная инициализация `{}`

C++ ввёл **единый синтаксис инициализации** через фигурные скобки:

```cpp
int a = 5;        // классически (из C)
int b(5);         // через скобки
int c{5};         // ✅ современно (uniform initialization)
double d{3.14};
std::string s{"привет"};
```

💡 Форма `{}` лучше: она **запрещает опасные сужающие преобразования**:
```cpp
int x = 3.9;      // молча обрежет до 3 (опасно)
int y{3.9};       // ❌ ошибка компиляции! — и это хорошо, ловит баг
```

> 💡 Привыкай к `{}` — это современный безопасный стиль C++.

## ⭐ `auto` — вывод типа

Ключевое слово `auto` просит компилятор **сам определить тип** по значению:

```cpp
auto age = 25;            // int
auto price = 19.99;       // double
auto name = std::string{"Гена"};  // std::string
auto x = 5 + 3.0;         // double (компилятор знает)
```

🖼️
```
   auto age = 25;
        │      └─ значение целое → компилятор делает age типом int
   "сам определи тип"
```

💡 `auto` особенно полезен с длинными типами (итераторы STL — Уровень 3), где писать тип
вручную утомительно. Но для простых случаев иногда яснее указать тип явно.

> ⚠️ `auto` определяет тип **в момент объявления**, переменная не «динамическая» как в
> Python. Это всё ещё статическая типизация — просто компилятор пишет тип за тебя.

## ⭐ `const` — неизменяемость

```cpp
const double PI = 3.14159;
PI = 3;                    // ❌ ошибка компиляции — и это правильно

const int MAX_USERS = 100;
```

💡 `const` в C++ используется **повсюду** — это важная часть культуры языка («const
correctness»). Помечай `const` всё, что не должно меняться: это документация + защита +
помощь компилятору в оптимизации. Мы вернёмся к этому много раз.

### `constexpr` — вычисляется на этапе компиляции
```cpp
constexpr int SIZE = 10 * 10;   // вычислено компилятором, ещё до запуска
int arr[SIZE];                  // можно использовать как размер массива
```

## 📖 Преобразование типов

```cpp
int n = static_cast<int>(3.9);     // 3 — современное приведение типа
double d = static_cast<double>(7) / 2;   // 3.5

// Старый C-стиль (избегай в C++):
int m = (int)3.9;                  // работает, но менее безопасно
```

💡 В C++ предпочитают `static_cast<>` вместо C-приведения `(int)` — оно явнее и безопаснее.
