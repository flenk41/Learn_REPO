# 🚀 Мини-проект · Своя type-safe библиотека

> Примени всю мощь шаблонов: построй небольшую обобщённую, типобезопасную, zero-cost библиотеку.
> Это доказывает, что ты не «пользуешься» абстракциями C++, а **создаёшь** их.

---

## 🎯 Что делаем

Выбери один компонент и сделай его обобщённым, ограниченным концептами, с использованием
вариативных шаблонов и метапрограммирования (как настоящие библиотеки).

```
   варианты (выбери один, доведи качественно):
   • свой type-safe контейнер (Vector/StaticVector) с концептами и итераторами.
   • свой tuple (вариативные шаблоны + метапрограммирование).
   • свой variant / optional (type-safe union).
   • свой format-подобный typed printf (вариативные шаблоны + fold).
   • свой обобщённый result/expected с понятными ошибками.
```

---

## 📋 Требования (на примере своего tuple)

- **Вариативные шаблоны:** хранит любое число элементов любых типов (`my_tuple<int, string, double>`).
- **Метапрограммирование:** `get<N>()` (доступ по индексу на этапе компиляции), `size()`.
- **Концепты:** ограничения на типы там, где осмысленно (например, требуем копируемость/печатаемость).
- **Perfect forwarding:** конструирование из любых аргументов без лишних копий.
- **Качество:** тесты (включая разные типы/число элементов), godbolt-проверка zero-cost (нет
  рантайм-оверхеда сверх ручного кода), README с примерами.

---

## 🌟 Бонусы

- 🥚 `if constexpr` для разной обработки типов внутри.
- 🐣 fold-выражения (например, печать всех элементов, сравнение).
- 🦅 ranges-совместимость / итераторы (для контейнера).
- 🚀 сравни ассемблер своей абстракции и ручного кода (godbolt) — докажи zero-cost.
