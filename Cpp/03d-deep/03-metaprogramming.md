# 3 · Метапрограммирование, SFINAE, type traits 🖼️⭐

> 🎯 **Цель блока:** понять шаблонное метапрограммирование — вычисления над типами на этапе
> компиляции: type traits, SFINAE, `if constexpr`. То, на чём построена STL.

---

## 📖 Вычисления на этапе компиляции

```
   ШАБЛОННОЕ МЕТАПРОГРАММИРОВАНИЕ — программирование, где «данные» это ТИПЫ и КОНСТАНТЫ, а
   «вычисления» происходят при КОМПИЛЯЦИИ. результат — код без рантайм-оверхеда (zero-cost).
   примеры: «является ли T указателем?», «убрать const с типа», «выбрать алгоритм по свойству типа».
```

💡 ⭐ Это «программа, генерирующая программу». Компилятор выполняет метакод, порождая
специализированный обычный код. Так STL подбирает оптимальные реализации под тип (например, быстрое
копирование для POD через memcpy). Современный C++ сделал это проще (`constexpr`, концепты), но
понимать основы важно.

## ⭐ Type traits: интроспекция типов

```cpp
#include <type_traits>

std::is_integral_v<int>           // true  — целое?
std::is_pointer_v<int*>           // true  — указатель?
std::is_same_v<int, long>         // false — одинаковые типы?
std::remove_const_t<const int>    // int   — преобразование типа (убрать const)
std::decay_t<int[5]>              // int*  — «распад» как при передаче в функцию
std::is_base_of_v<Base, Derived>  // отношения наследования
```

```
   TYPE TRAITS — шаблоны, отвечающие на вопросы о типах (..._v → значение bool) или ПРЕОБРАЗУЮЩИЕ
   типы (..._t → новый тип). это «стандартная библиотека для метапрограммирования».
```

💡 ⭐ Свой простой trait:

```cpp
template <typename T> struct is_pointer_simple        { static constexpr bool value = false; };
template <typename T> struct is_pointer_simple<T*>    { static constexpr bool value = true;  };
// частичная специализация для T* → срабатывает для указателей. так устроены реальные traits.
```

## ⭐⭐ SFINAE → if constexpr (эволюция выбора по типу)

```
   ЗАДАЧА: разный код в зависимости от свойства типа.

   СТАРЫЙ способ — SFINAE (Substitution Failure Is Not An Error):
   «если подстановка типа в шаблон не удалась — это не ошибка, а отбрасывание перегрузки».
   через std::enable_if включают/выключают перегрузки. МОЩНО, но многословно и нечитаемо.
```

```cpp
// SFINAE (старо, многословно):
template <typename T, std::enable_if_t<std::is_integral_v<T>, int> = 0>
void f(T x) { /* для целых */ }

// if constexpr (C++17 — НАМНОГО яснее):
template <typename T>
void f(T x) {
    if constexpr (std::is_integral_v<T>) { /* для целых */ }
    else if constexpr (std::is_floating_point_v<T>) { /* для дробных */ }
    else { /* остальное */ }
}
```

💡 ⭐⭐ `if constexpr` — это «ветвление на этапе компиляции»: компилятор выбирает ОДНУ ветку по
условию-константе, остальные **отбрасывает** (даже не компилирует). Это заменило большую часть
SFINAE-трюков читаемым кодом. А C++20 концепты (модуль 2) заменили остаток. Эволюция: SFINAE →
`if constexpr` → концепты. Знай все три, пиши на новейшем.

## 📖 constexpr: вычисления при компиляции

```cpp
constexpr int factorial(int n) {              // может выполниться при КОМПИЛЯЦИИ
    return n <= 1 ? 1 : n * factorial(n - 1);
}
constexpr int x = factorial(5);               // 120 вычислено компилятором, в бинарнике уже константа

// consteval (C++20) — ОБЯЗАТЕЛЬНО при компиляции; constinit — инициализация при компиляции.
```

💡 `constexpr` функции/переменные вычисляются при компиляции, если возможно — современная замена
старому «вычислениям через шаблоны» (template metaprogramming в стиле рекурсивных шаблонов). Гораздо
читаемее: пишешь обычную функцию с `constexpr`.

## ⚠️ Ловушки

- ❌ Использовать SFINAE/`enable_if`, когда есть `if constexpr` или концепты (многословно).
- ❌ Путать `if` и `if constexpr` (обычный if не отбрасывает ветки — обе компилируются).
- ❌ Писать свои traits, когда есть стандартные.
- ❌ Глубокая рекурсия в шаблонах → медленная компиляция, лимиты глубины.
- ❌ Ожидать рантайм-поведения от метакода (всё на компиляции).

---

## ✅ Задачи

1. **Traits.** Используй `std::is_integral_v`, `std::is_same_v`, `std::remove_const_t` в маленьких примерах.
2. **Свой trait.** Напиши `is_pointer` через частичную специализацию.
3. ⭐ **if constexpr.** Функция `to_string`, которая по типу (число/строка/контейнер) выбирает реализацию.
4. ⭐ **constexpr.** Напиши `constexpr` функцию (факториал/проверка простого) и убедись (static_assert),
   что вычисляется при компиляции.
5. **Эволюция.** Возьми SFINAE-пример и перепиши через `if constexpr`, потом через концепт.
