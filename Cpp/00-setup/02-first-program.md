# 02 · Первая программа и компиляция 🛠️

> 🎯 **Цель блока:** написать, скомпилировать и запустить первую программу на C++ и
> понять, чем её устройство отличается от C.

---

## 📖 «Hello, world!» на C++

Создай файл `hello.cpp`:

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, world!" << std::endl;
    return 0;
}
```

### Разбираем построчно

| Строка | Что значит |
|--------|-----------|
| `#include <iostream>` | подключаем библиотеку ввода-вывода (потоки cin/cout) |
| `int main()` | главная функция — с неё начинается программа |
| `std::cout` | «стандартный вывод» (поток в консоль) |
| `<<` | оператор вывода: «отправить в поток» |
| `std::endl` | перевод строки (как `\n`) + сброс буфера |
| `return 0;` | успешное завершение |

🆚 **Сравни с C:**
```c
// C:    printf("Hello, world!\n");
// C++:  std::cout << "Hello, world!" << std::endl;
```
В C++ для вывода используют **потоки** (`cout`), а не `printf`. Это безопаснее (нет
проблем с `%d`/`%s`) и работает с любыми типами.

---

## 📖 Что такое `std::`

`std` — это **пространство имён** (namespace) стандартной библиотеки. `std::cout`
означает «`cout` из пространства `std`». Это защищает от конфликтов имён.

Можно избавиться от приставки `std::` (но в учебных целях лучше писать явно):
```cpp
#include <iostream>
using namespace std;        // теперь можно писать cout без std::

int main() {
    cout << "Hello!" << endl;
    return 0;
}
```

> ⚠️ `using namespace std;` удобно для учёбы, но в больших проектах его избегают (может
> вызвать конфликты имён). Привыкай писать `std::` — это хороший тон.

## 🛠️ Компиляция и запуск

Открой терминал в папке с файлом (VS Code: **Terminal → New Terminal**).

### Скомпилировать
```bash
g++ -std=c++17 hello.cpp -o hello
```
```
g++   -std=c++17   hello.cpp    -o    hello
 │        │            │         │       │
компилятор стандарт  исходник  "вывод" имя программы
```

### Запустить
**Windows:** `.\hello.exe`
**macOS/Linux:** `./hello`

Вывод:
```
Hello, world!
```
🎉 Твоя первая программа на C++!

## ⚠️ Флаги, которые стоит включать всегда

```bash
g++ -std=c++17 -Wall -Wextra -g hello.cpp -o hello
```

| Флаг | Что делает |
|------|-----------|
| `-std=c++17` | современный стандарт языка |
| `-Wall -Wextra` | включить предупреждения (твой друг!) |
| `-g` | отладочная информация (для отладчика) |

💡 На этапе изучения памяти (Уровень 2) добавим ещё `-fsanitize=address` — как в C.

## 📖 Чем компиляция C++ похожа на C

Те же 4 стадии: препроцессор → компилятор → ассемблер → линковщик (см.
[C-курс, модуль 02](../../C/00-setup/02-first-program.md)). Результат — машинный код.

> ⚠️ Компиляция C++ **медленнее**, чем C: язык сложнее, шаблоны разворачиваются, много
> заголовков. В больших проектах это заметно — поэтому используют системы сборки (CMake)
> и инкрементальную сборку.

## 🧪 Эксперимент: вывод разных типов

Потоки C++ автоматически понимают тип — не нужно `%d`/`%f`:

```cpp
#include <iostream>
int main() {
    int age = 25;
    double pi = 3.14;
    std::string name = "Гена";

    std::cout << "Имя: " << name << ", возраст: " << age
              << ", pi: " << pi << std::endl;
    return 0;
}
```

В C тебе пришлось бы помнить `%d`, `%f`, `%s`. В C++ поток сам разбирается — это и
безопаснее, и проще.

## ✅ Задачи

1. **Своя визитка.** Выведи три строки: имя, город, любимый язык.
2. **Ёлочка.** Нарисуй пирамиду из звёздочек через несколько `cout`.
3. **Типы.** Выведи в одну строку число, дробь, символ и строку через `<<`.
4. **Сравни с C.** Напиши «Hello» и через `printf` (подключив `<cstdio>`), и через
   `cout`. Оба работают в C++ — но почувствуй разницу.
5. **Предупреждения.** Скомпилируй с `-Wall -Wextra`, попробуй намеренно сделать ошибку
   (убери `;`), прочитай сообщение компилятора.
