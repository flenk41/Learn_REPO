# 00 · Что такое C++ и его модель памяти 🖼️

> 🎯 **Цель блока:** понять, что такое C++, чем он отличается от C и Python, и какую
> проблему памяти он решает. Это задаёт смысл всему курсу.

---

## 📖 Что такое C++

C++ создал Бьёрн Страуструп в 1980-х как **«C с классами»**. Сегодня это один из самых
мощных языков: на нём пишут игры (Unreal Engine), браузеры, базы данных, операционные
системы, торговые системы, где важна **скорость и контроль**.

C++ почти полностью включает C, но добавляет:
- **классы и объекты** (ООП);
- **RAII** и **умные указатели** — безопасную работу с памятью;
- **шаблоны** (обобщённое программирование);
- **STL** — огромную библиотеку готовых контейнеров и алгоритмов;
- **move-семантику** — эффективную передачу владения.

---

## ⭐ Главная проблема, которую решает C++

Вспомни (или представь) C. Там работа с памятью — ручная и **опасная**:

```c
char* buffer = malloc(100);   // выделили
// ... много кода ...
// ❌ забыли free(buffer)  → утечка памяти
// ❌ или free дважды       → краш
// ❌ или использовали после free → краш
```

В C **легко** забыть освободить память или ошибиться. В больших программах это —
источник 70% серьёзных багов и уязвимостей.

🎯 **C++ решает это деструкторами и RAII:** объект сам освобождает свои ресурсы, когда
выходит из области видимости. Тебе **не нужно** помнить про `free` — это происходит
автоматически и предсказуемо.

```cpp
{
    std::string buffer(100, ' ');   // выделили память
    // ... много кода ...
}   // ← здесь buffer автоматически освобождает память. Гарантированно.
```

## 🖼️ Три подхода к памяти — где C++

```
   C: ручная память              Python: автоматическая
   ┌──────────────┐              ┌──────────────┐
   │ malloc       │              │ создал объект│
   │ ...          │              │ ...          │
   │ free ← сам!  │              │ (GC уберёт   │
   │ (забыл? утечка)│            │  сам когда-то)│
   └──────────────┘              └──────────────┘
   мощно, но опасно              удобно, но без контроля

              C++: лучшее из обоих миров
              ┌────────────────────────────┐
              │ создал объект               │
              │ ...                         │
              │ } ← деструктор освобождает  │
              │     САМ, в точный момент    │
              └────────────────────────────┘
              контроль C + безопасность Python
```

💡 Ключевое слово — **детерминированно**. В Python сборщик мусора уберёт объект
«когда-нибудь». В C++ деструктор сработает **точно** в момент выхода из блока `{}`. Ты
знаешь, **когда** и освобождается память, и закрываются файлы, и снимаются блокировки.

## 📖 Модель памяти: что нужно помнить

C++ использует ту же модель памяти, что и C (мы разберём её подробно в Уровне 2):

```
   STACK (стек)  — локальные переменные, чистятся автоматически
   HEAP  (куча)  — динамическая память (new/delete или умные указатели)
   DATA / TEXT   — глобальные данные и код
```

🎯 Главная мысль курса: **в C++ ты предпочитаешь стек и RAII куче с ручным управлением.**
Вместо «выделил в куче — не забудь освободить» ты пишешь объекты, которые сами за собой
убирают. Это называется **«не управляй памятью вручную — пусть объекты делают это за
тебя»**.

## 📖 C++ — компилируемый и быстрый

Как и C, C++ **компилируется** в машинный код (не интерпретируется, как Python). Поэтому:
- 🚀 очень быстрый (близко к C);
- 🔧 работает без виртуальной машины;
- 📦 даёт готовый исполняемый файл.

```
   твой .cpp ──компилятор(g++)──► машинный код ──► программа.exe
```

Цена — компиляция занимает время, а ошибки строже. Но взамен — скорость и контроль,
которых нет в Python.

## 🤔 Когда выбирать C++

| Бери C++ | Бери Python |
|----------|-------------|
| игры, движки | скрипты, автоматизация |
| высокая производительность | быстрая разработка |
| системное/встраиваемое ПО | анализ данных, ML-прототипы |
| контроль над памятью | веб-бэкенды, боты |

💡 Понимая C, C++ и Python, ты выбираешь правильный инструмент под задачу — и понимаешь
память на всех уровнях.
