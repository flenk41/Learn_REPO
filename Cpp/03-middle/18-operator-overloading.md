# 18 · Перегрузка операторов

> 🎯 **Цель блока:** научить свои типы работать с операторами `+`, `==`, `<<` и др. —
> чтобы объекты вели себя естественно, как встроенные типы.

---

## 📖 Зачем это нужно

Хочется, чтобы свой тип `Vector2D` складывался через `+`, сравнивался через `==`,
печатался через `<<`:

```cpp
Vector2D a{1, 2}, b{3, 4};
Vector2D c = a + b;          // хотим, чтобы это работало
std::cout << c;              // и это
```

Для этого операторы **перегружают** — задают, что они делают для твоего типа.

---

## ⭐ Арифметические операторы

```cpp
class Vector2D {
public:
    double x, y;
    Vector2D(double x_, double y_) : x(x_), y(y_) {}

    // оператор + как метод
    Vector2D operator+(const Vector2D& other) const {
        return Vector2D(x + other.x, y + other.y);
    }
    Vector2D operator*(double scalar) const {
        return Vector2D(x * scalar, y * scalar);
    }
};

Vector2D a{1, 2}, b{3, 4};
Vector2D c = a + b;          // (4, 6) — вызвался operator+
Vector2D d = a * 2.0;        // (2, 4)
```

🖼️
```
   a + b   →   a.operator+(b)   →   возвращает новый Vector2D
```

## ⭐ Операторы сравнения

```cpp
class Vector2D {
public:
    double x, y;
    bool operator==(const Vector2D& o) const {
        return x == o.x && y == o.y;
    }
    bool operator!=(const Vector2D& o) const {
        return !(*this == o);
    }
};

// C++20: можно одним оператором <=> (spaceship) получить все сравнения
// auto operator<=>(const Vector2D&) const = default;
```

> 💡 В C++20 оператор `<=>` (`operator<=>() = default;`) автоматически даёт все сравнения
> (`< > <= >= == !=`) сразу — большое удобство.

---

## ⭐ Оператор вывода `<<`

Чтобы `std::cout << v` работал, перегружают `<<`. Это **свободная функция** (не метод),
часто `friend`:

```cpp
class Vector2D {
public:
    double x, y;
    friend std::ostream& operator<<(std::ostream& os, const Vector2D& v) {
        os << "(" << v.x << ", " << v.y << ")";
        return os;           // вернуть поток для цепочки <<
    }
};

Vector2D v{3, 4};
std::cout << v << "\n";      // (3, 4)
```

💡 `friend` даёт функции доступ к приватным полям. Возврат `os&` позволяет цепочки:
`cout << a << b << c`.

## ⭐ Оператор индексирования `[]`

```cpp
class IntArray {
    std::vector<int> data;
public:
    IntArray(size_t n) : data(n) {}
    int& operator[](size_t i) { return data[i]; }              // для записи
    int  operator[](size_t i) const { return data[i]; }        // для чтения
};

IntArray arr(5);
arr[0] = 42;                 // вызвался operator[]
std::cout << arr[0];
```

💡 Так твой класс ведёт себя как массив. Ты уже использовал это в RAII-обёртках Уровня 2.

## 📖 Какие операторы можно перегружать

```
   Арифметика:  + - * / %      Сравнение: == != < > <= >=
   Присваивание: = += -= ...    Доступ:    [] () -> *
   Поток:       << >>           Инкремент: ++ --
```

> ⚠️ **Не злоупотребляй!** Перегружай операторы только когда смысл **очевиден** (`+` для
> векторов — да; `+` для «добавить пользователя в базу» — нет, лучше метод `addUser`).
> Операторы должны делать то, что от них интуитивно ожидают.
