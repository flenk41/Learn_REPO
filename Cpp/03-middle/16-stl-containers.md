# 16 · STL-контейнеры 🖼️

> 🎯 **Цель блока:** освоить готовые контейнеры STL, которые **сами управляют памятью**
> (RAII!). Это то, что заменяет ручные массивы и структуры данных из C.

---

## 📖 STL — стандартная библиотека шаблонов

STL даёт готовые, эффективные, **безопасные по памяти** контейнеры. Тебе почти никогда не
нужно писать свой список или хеш-таблицу — они уже есть и сами освобождают память.

| Контейнер | Что это | Аналог |
|-----------|---------|--------|
| `std::vector` | динамический массив | список Python, ArrayList |
| `std::string` | строка | str |
| `std::array` | массив фикс. размера | C-массив, но безопасный |
| `std::map` | отсортированные пары ключ→значение | dict (на дереве) |
| `std::unordered_map` | хеш-таблица ключ→значение | dict Python |
| `std::set` / `unordered_set` | уникальные элементы | set Python |
| `std::list` | двусвязный список | |
| `std::deque` | очередь с двух концов | |

## ⭐ std::vector — рабочая лошадка

```cpp
#include <vector>

std::vector<int> v = {10, 20, 30};
v.push_back(40);              // добавить в конец
v[0] = 100;                  // доступ по индексу
std::cout << v.size() << "\n";  // 4
v.pop_back();                // удалить последний

for (int x : v) std::cout << x << " ";   // перебор

// безопасный доступ с проверкой границ
v.at(1);                     // бросит исключение при выходе за границы
```

🖼️ `vector` — это динамический массив, который сам растёт (как твой Vector из C-курса, но
готовый и безопасный):
```
   vector ──► [10][20][30][40]  в куче, лежат подряд
   при заполнении сам делает realloc с запасом
   при разрушении сам освобождает память (RAII) — delete не нужен!
```

💡 `vector` управляет памятью сам: выделяет, растит, освобождает в деструкторе. Никаких
утечек. Это «правило нуля» в действии.

> 💡 `reserve(n)` заранее выделяет место под `n` элементов — избегает лишних перевыделений
> при росте. Полезно, когда знаешь примерный размер.

## ⭐ std::map и unordered_map

```cpp
#include <map>
#include <unordered_map>

std::unordered_map<std::string, int> ages;   // хеш-таблица (быстрее)
ages["Гена"] = 30;
ages["Чебур"] = 5;

if (ages.count("Гена"))                       // есть ключ?
    std::cout << ages["Гена"] << "\n";        // 30

for (const auto& [name, age] : ages)          // структурное связывание (C++17)
    std::cout << name << ": " << age << "\n";
```

| | `std::map` | `std::unordered_map` |
|--|-----------|----------------------|
| Внутри | красно-чёрное дерево | хеш-таблица |
| Порядок | отсортирован по ключу | без порядка |
| Поиск | O(log n) | O(1) в среднем |
| Когда | нужен порядок | нужна скорость |

💡 `unordered_map` — это `dict` из Python (хеш-таблица, см.
[Python-курс](../../Python/03-middle/14-dict-set-hashing.md)). `map` — отсортированная
версия на дереве.

## ⭐ std::set — уникальные элементы

```cpp
#include <set>
#include <unordered_set>

std::set<int> s = {3, 1, 2, 1, 3};   // дубликаты убраны → {1, 2, 3} (отсортировано)
s.insert(5);
if (s.count(2)) std::cout << "есть 2\n";

// убрать дубликаты из вектора
std::vector<int> v = {1, 1, 2, 3, 3};
std::set<int> unique(v.begin(), v.end());   // {1, 2, 3}
```

## 📖 std::array и std::string

```cpp
#include <array>
std::array<int, 5> arr = {1, 2, 3, 4, 5};   // фикс. размер, но безопаснее C-массива
arr.size();                                  // знает свой размер!

std::string s = "привет";
s += " мир";                                 // конкатенация
s.length();
s.substr(0, 6);                              // подстрока
s.find("мир");                               // поиск
```

## 📋 Какой контейнер выбрать

```
   Нужен список значений с индексами   → vector (по умолчанию!)
   Ключ → значение, нужна скорость     → unordered_map
   Ключ → значение, нужен порядок      → map
   Уникальные значения                 → set / unordered_set
   Фиксированный размер на стеке        → array
   Частые вставки в середину/начало     → list / deque
```

> 💡 **По умолчанию бери `vector`** — он быстрый, кэш-дружелюбный, подходит в 90% случаев.
> Остальные — когда есть конкретная причина.

## ✅ Задачи

1. **vector-базовое.** Заполни вектор, выведи, посчитай сумму/min/max, удвой каждый элемент.
2. **Частоты слов.** Через `unordered_map<string,int>` посчитай, сколько раз каждое слово
   встречается в тексте.
3. **Уникальные.** Убери дубликаты из вектора через `set`.
4. **Телефонная книга** на `map<string,string>`: добавить, найти, удалить, вывести всех
   (по алфавиту — это даст `map`).
5. **Пересечение множеств** — найди общие элементы двух `set`.
6. **reserve.** Замерь время заполнения вектора на миллион элементов с `reserve` и без.
7. ⭐ **Гистограмма оценок** — посчитай распределение оценок класса через map.
