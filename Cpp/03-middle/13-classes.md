## 📖 Класс — чертёж объекта

```cpp
class Dog {
public:                          // публичное — доступно снаружи
    std::string name;
    int age;

    void bark() {                // метод
        std::cout << name << ": Гав!\n";
    }
};

int main() {
    Dog rex;
    rex.name = "Рекс";
    rex.age = 3;
    rex.bark();                  // Рекс: Гав!
}
```

## ⭐ Инкапсуляция: public / private

```cpp
class BankAccount {
private:                         // приватное — только внутри класса
    double balance = 0;

public:                          // публичное — интерфейс (API)
    void deposit(double amount) { balance += amount; }
    bool withdraw(double amount) {
        if (amount > balance) return false;
        balance -= amount;
        return true;
    }
    double getBalance() const { return balance; }  // const-метод: не меняет объект
};
```

🖼️
```
   public:  ← дверь наружу (методы, которыми пользуются)
   ┌─────────────────────────┐
   │ private: balance        │ ← спрятано, снаружи не достать
   └─────────────────────────┘
```

💡 Прячь данные в `private`, открывай только методы — это инкапсуляция (как opaque-тип в
C и `_private` в Python). `const` после метода = «этот метод не меняет объект».

## ⭐ Конструктор и деструктор

```cpp
class Dog {
    std::string name;
public:
    Dog(const std::string& n) : name(n) {     // конструктор (список инициализации)
        std::cout << name << " родился\n";
    }
    ~Dog() {                                   // деструктор (RAII!)
        std::cout << name << " ушёл\n";
    }
};

Dog rex("Рекс");     // вызовется конструктор
// в конце области видимости — деструктор
```

💡 `: name(n)` — **список инициализации**: инициализирует поля до тела конструктора. Это
эффективнее и иногда обязательно (для `const`-полей и ссылок). Привыкай к нему.

## 📖 Несколько конструкторов (перегрузка)

```cpp
class Point {
    int x, y;
public:
    Point() : x(0), y(0) {}                    // по умолчанию
    Point(int x_, int y_) : x(x_), y(y_) {}    // с параметрами

    void print() const { std::cout << "(" << x << ", " << y << ")\n"; }
};

Point a;          // (0, 0)
Point b(3, 4);    // (3, 4)
```

## ⭐⭐ Правило трёх/пяти — связь с памятью

Если твой класс **управляет ресурсом** (например, владеет `new`-памятью), то компилятор
по умолчанию копирует его **неправильно** (поверхностно — копирует указатель, а не данные).

```cpp
class Buffer {
    int* data;
    size_t size;
public:
    Buffer(size_t n) : data(new int[n]), size(n) {}
    ~Buffer() { delete[] data; }
    // ⚠️ если скопировать Buffer по умолчанию — два объекта будут владеть ОДНИМ
    //    указателем → двойной delete → краш!
};
```

🖼️
```
   Buffer b1(10);  →  b1.data ──► [память]
   Buffer b2 = b1; →  b2.data ──► [та же память!]   (поверхностная копия)
   при разрушении: b1 удаляет память, b2 удаляет ЕЁ ЖЕ → ☠️ double free
```

**Правило пяти:** если определяешь **один** из этих пяти — определи (или удали) **все**:
1. деструктор,
2. копирующий конструктор,
3. копирующее присваивание,
4. перемещающий конструктор,
5. перемещающее присваивание.

```cpp
class Buffer {
    int* data; size_t size;
public:
    Buffer(size_t n) : data(new int[n]), size(n) {}
    ~Buffer() { delete[] data; }

    // глубокое копирование
    Buffer(const Buffer& o) : data(new int[o.size]), size(o.size) {
        std::copy(o.data, o.data + size, data);
    }
    // ... и остальные (или = delete / = default)
};
```

> 💡 **Правило нуля (лучшее!):** не управляй ресурсами вручную вообще — используй
> `std::vector`, `std::string`, умные указатели. Тогда **ни одно** из пяти писать не
> нужно — компилятор сгенерирует правильно. Подробно — в [Senior, модуль 19](../04-senior/19-move-deep.md).

```cpp
class Buffer {
    std::vector<int> data;       // ✅ vector сам управляет памятью
public:
    Buffer(size_t n) : data(n) {}
    // никаких деструкторов/копирований писать НЕ нужно — всё работает правильно!
};
```

## 📖 this и статические члены

```cpp
class Counter {
    static int count;            // общий для всех объектов
    int id;
public:
    Counter() { id = ++count; }
    int getId() const { return this->id; }   // this — указатель на текущий объект
};
int Counter::count = 0;          // определение статического члена
```
