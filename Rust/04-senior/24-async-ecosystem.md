## ⭐ async / await — асинхронность

Для тысяч одновременных операций ввода-вывода (сеть, диск) Rust использует `async`/`await` —
как в Python. Один поток обслуживает множество задач, переключаясь в точках ожидания.

```rust
async fn fetch_data(id: u32) -> String {
    // ... асинхронная операция (запрос, чтение) ...
    format!("данные {}", id)
}

async fn run() {
    let result = fetch_data(1).await;       // .await — дождаться, не блокируя поток
    println!("{}", result);
}
```

🖼️
```
   async fn        — функция возвращает Future (отложенное вычисление)
   .await          — «дождись результата, пока другие задачи работают»
   один поток обслуживает тысячи async-задач (как asyncio в Python)
```

> 💡 Для запуска async-кода нужна **среда выполнения** (runtime) — обычно `tokio` или
> `async-std` (внешние крейты). `async`/`await` идеальны для веб-серверов, сетевых
> сервисов, где много ожидания I/O. Подробное освоение — отдельная большая тема.

```toml
# Cargo.toml
[dependencies]
tokio = { version = "1", features = ["full"] }
```
```rust
#[tokio::main]
async fn main() {
    run().await;
}
```

## ⭐ Тестирование — встроено в язык

Тесты в Rust — часть языка, не нужна внешняя библиотека:

```rust
fn add(a: i32, b: i32) -> i32 { a + b }

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
        assert_eq!(add(-1, 1), 0);
    }

    #[test]
    fn test_panic() {
        assert!(add(2, 2) == 4);
    }

    #[test]
    #[should_panic]
    fn test_should_fail() {
        panic!("ожидаемая паника");
    }
}
```
```bash
cargo test                   # запустить все тесты
```

| Макрос | Что делает |
|--------|-----------|
| `#[test]` | помечает тест-функцию |
| `assert!` | проверить условие |
| `assert_eq!` | проверить равенство |
| `#[should_panic]` | тест ожидает панику |

💡 Тесты живут рядом с кодом (`#[cfg(test)]` исключает их из релиза). Документационные тесты
(в комментариях `///`) тоже выполняются — Rust проверяет даже примеры в документации!

## ⭐ Cargo-экосистема и инструменты

```bash
cargo new / cargo build / cargo run / cargo test   # основа
cargo add serde                  # добавить зависимость
cargo clippy                     # линтер — находит проблемы и неидиоматичный код
cargo fmt                        # форматтер (единый стиль)
cargo doc --open                 # сгенерировать и открыть документацию
cargo bench                      # бенчмарки
cargo install имя                # установить инструмент глобально
```

| Инструмент | Зачем |
|-----------|-------|
| **cargo** | сборка, зависимости, всё |
| **clippy** | линтер (must-have!) |
| **rustfmt** | форматирование |
| **crates.io** | репозиторий библиотек (как PyPI/npm) |
| **docs.rs** | автодокументация всех крейтов |

💡 Экосистема Rust — одна из лучших. `cargo` объединяет всё, что в C++ разбросано по
разным инструментам (CMake, conan, clang-format...). Это огромное удобство.

## 📋 Гайдлайны идиоматичного Rust

```
   ✅ Владение и заимствование вместо clone везде
   ✅ Option/Result вместо null/исключений
   ✅ match для разбора вариантов
   ✅ Итераторы вместо ручных циклов
   ✅ &str в параметрах, String в возврате
   ✅ #[derive(Debug)] на структурах
   ✅ ? для передачи ошибок
   ✅ cargo clippy + cargo fmt
   ❌ unwrap() в продакшене, лишние clone, unsafe без нужды
```

> 💡 Слушай **clippy** — он учит идиоматичному Rust, предлагая улучшения. И читай ошибки
> компилятора: в Rust они исключительно подробные и часто подсказывают точное решение.
