# 20 · Бесстрашная многопоточность 🖼️⭐⭐

> 🎯 **Цель блока:** запускать потоки в Rust и понять, почему гонки данных в нём
> **невозможны на этапе компиляции**. Это «бесстрашная многопоточность» — киллер-фича Rust.

---

## ⭐ Запуск потоков

```rust
use std::thread;

fn main() {
    let handle = thread::spawn(|| {       // запустить поток (замыкание)
        for i in 1..5 {
            println!("поток: {}", i);
        }
    });

    for i in 1..3 {
        println!("main: {}", i);
    }

    handle.join().unwrap();               // дождаться завершения потока
}
```

🆚 Как `std::thread` в C++ — настоящий параллелизм на разных ядрах (нет GIL, в отличие от
[Python](../../Python/04-senior/23-concurrency.md)).

---

## ⭐⭐ Главное: borrow checker предотвращает гонки

Вспомни правило из [модуля 10](../02-memory/10-borrow-checker.md): «много `&` ИЛИ один
`&mut`, не вместе». В многопоточности оно становится сверхспособностью.

```rust
let mut counter = 0;
let handle = thread::spawn(|| {
    counter += 1;            // ❌ ОШИБКА КОМПИЛЯЦИИ! поток не может одолжить counter
});
```

🖼️
```
   C/C++:  два потока пишут в counter → ГОНКА → краш/мусор (ловится с трудом, в рантайме)
   Rust:   компилятор НЕ ДАЁТ расшарить изменяемые данные между потоками без защиты
           → гонка данных НЕВОЗМОЖНА на этапе компиляции
```

💡 Это и есть «бесстрашная многопоточность»: если многопоточный код **скомпилировался**, в
нём гарантированно нет гонок данных. Та же система владения, что защищает память, защищает и
от гонок.

---

## ⭐ move-замыкания и передача владения в поток

Чтобы поток использовал данные, нужно **передать ему владение** через `move`:

```rust
let data = vec![1, 2, 3];
let handle = thread::spawn(move || {      // move — владение data уходит в поток
    println!("{:?}", data);
});
handle.join().unwrap();
// data здесь больше недоступна — владение ушло в поток (безопасно!)
```

---

## ⭐ Arc — разделяемое владение между потоками

`Rc` (модуль 19) не потокобезопасен. Его потокобезопасная версия — **Arc** (Atomic Rc):

```rust
use std::sync::Arc;
use std::thread;

let data = Arc::new(vec![1, 2, 3]);
let mut handles = vec![];

for i in 0..3 {
    let data = Arc::clone(&data);         // клон для каждого потока (счётчик +1)
    let handle = thread::spawn(move || {
        println!("поток {}: {:?}", i, data);
    });
    handles.push(handle);
}
for h in handles { h.join().unwrap(); }
```

💡 `Arc` позволяет нескольким потокам **читать** общие данные. Для **изменения** нужен ещё
мьютекс.

---

## ⭐ Mutex — безопасное изменение из потоков

`Mutex` (взаимная блокировка) даёт изменять данные из потоков — только один поток за раз:

```rust
use std::sync::{Arc, Mutex};
use std::thread;

let counter = Arc::new(Mutex::new(0));    // Arc для разделения + Mutex для изменения
let mut handles = vec![];

for _ in 0..10 {
    let counter = Arc::clone(&counter);
    let handle = thread::spawn(move || {
        let mut num = counter.lock().unwrap();   // захватить блокировку
        *num += 1;                               // безопасно изменить
    });                                          // блокировка снимается здесь (RAII!)
    handles.push(handle);
}
for h in handles { h.join().unwrap(); }
println!("{}", *counter.lock().unwrap());        // 10 — всегда правильно!
```

🖼️
```
   Arc<Mutex<T>>
    │     └─ Mutex: только один поток меняет за раз (без гонки)
    └─ Arc: несколько потоков владеют

   counter.lock() → захват → изменение → авто-разблокировка в конце области (RAII)
```

💡 `Arc<Mutex<T>>` — стандартный паттерн «разделяемое изменяемое состояние между потоками».
Mutex в Rust возвращает данные **через** блокировку — невозможно забыть залочить (в отличие
от C++, где mutex и данные раздельны).

## ⭐ Send и Sync — трейты безопасности потоков

Компилятор знает, какие типы безопасно передавать между потоками, через трейты:
- **`Send`** — тип можно **переместить** в другой поток (большинство типов).
- **`Sync`** — на тип можно **ссылаться** из нескольких потоков.

```rust
// Rc НЕ Send → компилятор не даст использовать его в потоках
// Arc — Send + Sync → можно
```

💡 Эти трейты — основа гарантий. Тебе обычно не нужно их писать: компилятор сам проверяет и
не даст передать в поток то, что небезопасно. Например, `Rc` не `Send`, поэтому код с `Rc` в
потоке **не скомпилируется** — компилятор направит к `Arc`.

## 📖 Каналы — передача сообщений

Альтернатива общему состоянию — передавать данные через каналы (как в Go):

```rust
use std::sync::mpsc;
use std::thread;

let (tx, rx) = mpsc::channel();           // transmitter, receiver
thread::spawn(move || {
    tx.send("привет из потока").unwrap();  // отправить
});
println!("{}", rx.recv().unwrap());        // получить
```

💡 «Не общайтесь через разделяемую память — делите память через общение». Каналы часто
безопаснее и проще мьютексов.
