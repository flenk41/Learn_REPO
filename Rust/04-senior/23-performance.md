# 23 · Производительность 🖼️

> 🎯 **Цель блока:** научиться писать быстрый Rust и понять «zero-cost абстракции» —
> почему высокоуровневый код Rust не медленнее ручного низкоуровневого.

---

## ⭐ Zero-cost абстракции — главная идея

Девиз Rust: **«вы не платите за то, что не используете, а за то, что используете — нельзя
написать лучше вручную»**.

```rust
// высокоуровневый итератор
let sum: i32 = (1..=1000).filter(|x| x % 2 == 0).map(|x| x * x).sum();

// компилируется в такой же машинный код, как ручной цикл на C!
```

🖼️
```
   Python: абстракции дорогие (объекты, интерпретатор) → медленно
   C++:    абстракции в основном zero-cost
   Rust:   абстракции zero-cost ПО ДИЗАЙНУ → высокоуровневый код = скорость C
```

💡 Итераторы, замыкания, обобщения, `Option` — всё это компилируется в оптимальный код. Ты
пишешь выразительно, а получаешь скорость C. Это уникальное сочетание.

## ⭐ Релизная сборка обязательна для замеров

```bash
cargo build --release        # с оптимизацией (в десятки раз быстрее debug!)
cargo run --release
```

> ⚠️ **Никогда не меряй производительность в debug-сборке!** Debug-версия не оптимизирована
> и медленнее в 10–50 раз. Все замеры — только `--release`.

```toml
# Cargo.toml — настройка оптимизации релиза
[profile.release]
opt-level = 3            # максимальная оптимизация (по умолчанию)
lto = true              # link-time optimization (ещё быстрее)
```

## ⭐ Кэш-дружелюбный код

Как и в C/C++ (см. [C-курс](../../C/04-senior/22-cache-optimization.md)), главный фактор
скорости — расположение данных. `Vec` хранит данные подряд → кэш-хиты.

```rust
// 🚀 Vec — данные подряд, кэш-дружелюбно
let v: Vec<i32> = (0..1_000_000).collect();
let sum: i64 = v.iter().map(|&x| x as i64).sum();   // быстро

// 🐢 связный список (узлы разбросаны) — медленнее из-за промахов кэша
```

💡 По умолчанию используй `Vec` — он почти всегда быстрее структур с разбросанными узлами,
даже когда алгоритмическая сложность хуже. Кэш важнее.

## 📖 Избегай лишних аллокаций и клонов

```rust
// 🐢 лишний clone
fn process(s: String) { }          // забирает владение → нужен clone у вызова
process(data.clone());

// 🚀 заимствование
fn process(s: &str) { }            // без копий
process(&data);

// 🐢 создание промежуточных Vec
let result: Vec<i32> = v.iter().map(|x| x * 2).collect();
let sum: i32 = result.iter().sum();

// 🚀 ленивый конвейер без промежуточного Vec
let sum: i32 = v.iter().map(|x| x * 2).sum();
```

💡 Частые потери — лишние `.clone()` и промежуточные коллекции. Заимствуй (`&`) и используй
ленивые итераторы (без `collect`, пока не нужно).

## 📖 Полезные приёмы

```rust
// reserve — заранее выделить ёмкость Vec
let mut v = Vec::with_capacity(1000);

// &str вместо String где можно
// итераторы вместо ручных циклов (zero-cost + читаемо)

// inline (компилятор обычно сам, но можно подсказать)
#[inline]
fn small(x: i32) -> i32 { x * 2 }
```

| Приём | Эффект |
|-------|--------|
| `--release` | главный рычаг |
| `Vec` (последовательность) | кэш-хиты |
| заимствование вместо clone | нет копий |
| ленивые итераторы | нет промежуточных коллекций |
| `with_capacity`/`reserve` | нет перевыделений |
