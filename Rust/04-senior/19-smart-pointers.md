# 19 · Умные указатели (Box, Rc, RefCell) 🖼️⭐

> 🎯 **Цель блока:** освоить умные указатели Rust — они расширяют систему владения для
> случаев, которые не покрываются обычными правилами (рекурсия, разделяемое владение).

---

## 📖 Зачем нужны умные указатели

Обычное владение покрывает 90% случаев. Но иногда нужно больше:
- хранить данные в куче явно (рекурсивные структуры) → **Box**;
- разделить владение между несколькими (граф) → **Rc**;
- менять данные через неизменяемую ссылку (контролируемо) → **RefCell**.

---

## ⭐ Box<T> — данные в куче

`Box` помещает значение в кучу и владеет им. Простейший умный указатель.

```rust
let b = Box::new(5);             // 5 живёт в куче, b владеет
println!("{}", *b);              // 5 — разыменование
// при выходе из области b освобождает память автоматически (RAII)
```

🖼️
```
   STACK        HEAP
   b ──────►    [5]
   Box владеет данными в куче; освободит при выходе из области
```

### Главное применение — рекурсивные типы
Структура не может содержать саму себя напрямую (бесконечный размер). `Box` решает это:

```rust
enum List {
    Cons(i32, Box<List>),        // Box даёт фиксированный размер (указатель)
    Nil,
}
use List::*;
let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
```

💡 Без `Box` компилятор не знал бы размер `List` (он бесконечно вложенный). `Box` — это
указатель фиксированного размера на данные в куче. Аналог `unique_ptr` из C++.

---

## ⭐ Rc<T> — разделяемое владение (счётчик ссылок)

Иногда у данных должно быть **несколько владельцев** (например, узел графа, на который
ссылаются многие). `Rc` (Reference Counted) считает владельцев:

```rust
use std::rc::Rc;

let a = Rc::new(String::from("общие данные"));
let b = Rc::clone(&a);           // НЕ копия данных — ещё один владелец (счётчик +1)
let c = Rc::clone(&a);           // счётчик = 3

println!("{}", Rc::strong_count(&a));   // 3
// данные удалятся, когда счётчик дойдёт до 0
```

🖼️
```
   a ──┐
   b ──┼──► [данные | счётчик: 3]    несколько владельцев одних данных
   c ──┘
   каждый уходит → счётчик -1 → 0 → данные освобождены
```

💡 `Rc` — прямой аналог `shared_ptr` из [C++](../../Cpp/02-memory/11-smart-pointers.md) и
подсчёта ссылок из [Python](../../Python/02-memory/11-refcount-gc.md). `Rc::clone` дёшев —
копирует не данные, а лишь увеличивает счётчик.

> ⚠️ `Rc` — **только для однопоточного** кода. Для многопоточности есть `Arc` (atomic Rc) —
> Уровень 20.

---

## ⭐ RefCell<T> — внутренняя изменяемость

Borrow checker проверяет заимствования при **компиляции**. Иногда нужно менять данные через
неизменяемую ссылку, а правила проверять в **рантайме**. Это `RefCell`:

```rust
use std::cell::RefCell;

let data = RefCell::new(5);

*data.borrow_mut() += 10;         // изменить, хотя data не mut!
println!("{}", data.borrow());    // 15

// правила всё ещё действуют, но проверяются в рантайме:
let b1 = data.borrow();
// let b2 = data.borrow_mut();     // ⚠️ ПАНИКА в рантайме: уже заимствовано
```

💡 `RefCell` переносит проверку «один &mut ИЛИ много &» из компиляции в **рантайм**. Если
нарушишь — паника (а не порча памяти). Это «внутренняя изменяемость» — лазейка для случаев,
где статической проверки недостаточно.

---

## ⭐ Rc<RefCell<T>> — разделяемые изменяемые данные

Частая комбинация: несколько владельцев **и** возможность менять:

```rust
use std::rc::Rc;
use std::cell::RefCell;

let shared = Rc::new(RefCell::new(vec![1, 2, 3]));
let clone1 = Rc::clone(&shared);

clone1.borrow_mut().push(4);      // изменить через один владелец
println!("{:?}", shared.borrow()); // [1, 2, 3, 4] — видно через другой
```

🖼️ `Rc` даёт несколько владельцев, `RefCell` даёт изменяемость. Вместе — разделяемое
изменяемое состояние (например, узлы дерева/графа со ссылками на родителя).

> ⚠️ `Rc<RefCell<T>>` мощно, но может создать **циклы ссылок** (как `shared_ptr` в C++ →
> утечка). Для «обратных» ссылок используют `Weak<T>` (слабая ссылка, не считается) —
> аналог `weak_ptr`/`weakref` из C++/Python.

---

## 📋 Какой указатель выбрать

```
   Данные в куче, один владелец         → Box<T>
   Несколько владельцев (1 поток)        → Rc<T>
   Несколько владельцев (много потоков)  → Arc<T>  (модуль 20)
   Изменяемость через неизменяемую ссылку → RefCell<T> (1 поток) / Mutex (потоки)
   Разделяемое изменяемое (1 поток)      → Rc<RefCell<T>>
   Разрыв циклов                          → Weak<T>
```

> 💡 По умолчанию — обычное владение и ссылки (Уровень 2). Умные указатели — когда правил
> владения не хватает. Не усложняй без необходимости.
