## ⭐ Трейт-объекты (dyn) — динамический полиморфизм

Обобщения (`<T: Trait>`) выбирают тип при компиляции. Но иногда нужна **коллекция разных
типов** с общим трейтом — для этого трейт-объекты (`dyn`):

```rust
trait Draw {
    fn draw(&self) -> String;
}

struct Circle;
struct Square;
impl Draw for Circle { fn draw(&self) -> String { "○".to_string() } }
impl Draw for Square { fn draw(&self) -> String { "□".to_string() } }

// вектор РАЗНЫХ типов, объединённых трейтом
let shapes: Vec<Box<dyn Draw>> = vec![
    Box::new(Circle),
    Box::new(Square),
    Box::new(Circle),
];

for shape in &shapes {
    println!("{}", shape.draw());        // вызовется нужная реализация
}
```

🖼️
```
   Vec<Box<dyn Draw>>
        └─ dyn Draw: «любой тип, реализующий Draw»
   вызов draw() идёт через таблицу методов в рантайме (как vtable в C++)
```

💡 `dyn Trait` — это динамическая диспетчеризация (как виртуальные функции/vtable в
[C++](../../Cpp/03-middle/14-inheritance-polymorphism.md)). Чуть медленнее обобщений
(косвенный вызов), но позволяет хранить разные типы вместе.

| | Обобщения `<T>` | Трейт-объекты `dyn` |
|--|-----------------|---------------------|
| Когда выбор | при компиляции | в рантайме |
| Скорость | быстрее (zero-cost) | косвенный вызов |
| Разные типы вместе | нет | **да** |

## ⭐ Ассоциированные типы

Трейт может объявлять «тип-заполнитель», который конкретизирует реализация. Главный пример —
`Iterator`:

```rust
trait Iterator {
    type Item;                            // ассоциированный тип
    fn next(&mut self) -> Option<Self::Item>;
}

struct Counter { count: u32 }
impl Iterator for Counter {
    type Item = u32;                      // конкретизируем
    fn next(&mut self) -> Option<u32> {
        if self.count < 5 {
            self.count += 1;
            Some(self.count)
        } else {
            None
        }
    }
}
```

💡 Реализовав `Iterator` для своего типа, ты получаешь **все** методы итераторов
(`map`/`filter`/`collect` из [модуля 18](../03-middle/18-closures-iterators.md)) бесплатно!

## ⭐ Перегрузка операторов через трейты

Операторы в Rust — это трейты. Хочешь `+` для своего типа — реализуй `Add`:

```rust
use std::ops::Add;

#[derive(Debug, Clone, Copy)]
struct Vector2D { x: f64, y: f64 }

impl Add for Vector2D {
    type Output = Vector2D;
    fn add(self, other: Vector2D) -> Vector2D {
        Vector2D { x: self.x + other.x, y: self.y + other.y }
    }
}

let a = Vector2D { x: 1.0, y: 2.0 };
let b = Vector2D { x: 3.0, y: 4.0 };
let c = a + b;                            // вызовет Add::add → (4.0, 6.0)
```

💡 Аналог перегрузки операторов из C++, но через трейты (`Add`, `Sub`, `Mul`, `PartialEq` и
др.). Единый, явный механизм.

## 📖 Трейты с параметрами и супертрейты

```rust
// супертрейт: Summary требует, чтобы тип реализовал Display
trait Summary: std::fmt::Display {
    fn summarize(&self) -> String {
        format!("Краткое: {}", self)      // можно использовать методы Display
    }
}
```

## 📖 Когда что использовать

```
   Один тип, известен при компиляции, нужна скорость → обобщения <T: Trait>
   Разные типы в одной коллекции / выбор в рантайме   → Box<dyn Trait>
   Свой итератор                                       → реализуй Iterator (ассоц. тип)
   Свои операторы                                      → трейты Add/Sub/...
```
