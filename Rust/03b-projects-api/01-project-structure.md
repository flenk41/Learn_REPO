# 1 · Структура проекта: модули и крейты 🖼️

> 🎯 **Цель блока:** научиться раскладывать Rust-проект по модулям, файлам и крейтам.
> Cargo делает структуру простой и единообразной — это сильная сторона Rust.

---

## 📖 Крейт (crate) — единица компиляции

В Rust **крейт** — это компилируемая единица. Бывает двух видов:

| Крейт | Файл | Что это |
|-------|------|---------|
| **бинарный** | `src/main.rs` | программа (есть `fn main`) |
| **библиотечный** | `src/lib.rs` | библиотека (переиспользуемый код) |

```
myproject/
├── Cargo.toml
└── src/
    ├── main.rs        ← бинарный крейт (программа)
    └── lib.rs         ← библиотечный крейт (опционально)
```

💡 Проект может быть и тем, и другим: `lib.rs` содержит логику (библиотека), `main.rs`
использует её. Это позволяет переиспользовать код и тестировать его отдельно.

## ⭐ Модули (mod) — организация кода

**Модуль** группирует код. В отличие от C++ (заголовки) или Python (файлы), в Rust модули —
часть языка через `mod`.

### Модули в одном файле
```rust
mod math {                       // модуль math
    pub fn add(a: i32, b: i32) -> i32 {   // pub — видно снаружи модуля
        a + b
    }
    fn helper() {}               // приватная (по умолчанию)
}

fn main() {
    println!("{}", math::add(2, 3));   // обращение через ::
}
```

### Модули в отдельных файлах
```
src/
├── main.rs
├── math.rs          ← модуль math
└── utils.rs         ← модуль utils
```
```rust
// main.rs
mod math;            // подключить файл math.rs как модуль
mod utils;

fn main() {
    math::add(2, 3);
}
```

### Модули в папках
```
src/
├── main.rs
└── shapes/
    ├── mod.rs        ← или shapes.rs рядом
    ├── circle.rs
    └── square.rs
```
```rust
// shapes/mod.rs
pub mod circle;      // публичный подмодуль
pub mod square;
```

🖼️
```
   crate (main.rs / lib.rs)
   ├── mod math      → math.rs
   ├── mod utils     → utils.rs
   └── mod shapes    → shapes/
                       ├── mod circle
                       └── mod square
   обращение: crate::shapes::circle::area()
```

## ⭐ Видимость: pub и приватность по умолчанию

В Rust **всё приватно по умолчанию** — это огромный плюс для инкапсуляции:

```rust
mod bank {
    pub struct Account {         // pub — структура видна
        pub owner: String,       // pub — поле видно
        balance: f64,            // приватное — только внутри bank
    }

    impl Account {
        pub fn new(owner: String) -> Self {   // pub-конструктор
            Account { owner, balance: 0.0 }
        }
        pub fn balance(&self) -> f64 {         // pub-метод доступа
            self.balance
        }
        fn validate(&self) {}    // приватный помощник
    }
}
```

| Видимость | Кто видит |
|-----------|-----------|
| (по умолчанию) | только текущий модуль и потомки |
| `pub` | все |
| `pub(crate)` | только этот крейт |
| `pub(super)` | родительский модуль |

💡 `balance` приватно — снаружи нельзя залезть напрямую, только через `pub fn balance()`.
Это инкапсуляция «из коробки», без усилий. Сравни с opaque-типами C / pImpl C++ / `_` Python —
в Rust приватность встроена и проверяется компилятором.

## ⭐ use — импорт имён

```rust
use std::collections::HashMap;   // теперь можно писать HashMap вместо полного пути
use crate::math::add;            // импорт своей функции
use std::io::{self, Write};      // несколько из одного модуля

// в коде:
let map: HashMap<String, i32> = HashMap::new();
```

## ⭐ Workspaces — много крейтов в одном проекте

Для больших проектов из нескольких библиотек — **workspace**:

```
myworkspace/
├── Cargo.toml          ← workspace-манифест
├── core/               ← крейт-библиотека
│   ├── Cargo.toml
│   └── src/lib.rs
├── cli/                ← крейт-программа, зависит от core
│   ├── Cargo.toml
│   └── src/main.rs
└── server/             ← ещё крейт
```
```toml
# корневой Cargo.toml
[workspace]
members = ["core", "cli", "server"]
```

💡 Workspace позволяет разбить большой проект на крейты, которые компилируются вместе и
делят зависимости. Аналог многомодульной сборки CMake в C++, но проще.

## 📖 Зависимости и crates.io

```toml
# Cargo.toml
[dependencies]
serde = { version = "1", features = ["derive"] }
rand = "0.8"
```
```bash
cargo add reqwest        # добавить зависимость командой
```

Библиотеки берутся с **crates.io** (как PyPI/npm). Cargo сам скачивает и собирает их.
