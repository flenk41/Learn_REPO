# 02 · Первая программа и cargo 🛠️

> 🎯 **Цель блока:** написать первую программу на Rust и освоить **cargo** — инструмент,
> который собирает, запускает и управляет зависимостями.

---

## ⭐ Cargo — твой главный инструмент

В Rust почти всё делается через **cargo**. Создадим первый проект:

```powershell
cargo new hello
cd hello
```

Cargo создаст структуру:
```
hello/
├── Cargo.toml        ← описание проекта и зависимостей
├── src/
│   └── main.rs       ← твой код
└── .gitignore        ← cargo сразу настраивает git!
```

`src/main.rs` уже содержит готовую программу:
```rust
fn main() {
    println!("Hello, world!");
}
```

### Собрать и запустить
```powershell
cargo run
```
Cargo скомпилирует и сразу запустит:
```
   Compiling hello v0.1.0
    Finished dev [unoptimized + debuginfo]
     Running `target\debug\hello.exe`
Hello, world!
```
🎉 Твоя первая программа на Rust!

---

## 📖 Разбираем программу

```rust
fn main() {
    println!("Hello, world!");
}
```

| Элемент | Что значит |
|---------|-----------|
| `fn main()` | главная функция — с неё начинается программа |
| `println!` | макрос вывода (восклицательный знак `!` = это макрос, не функция) |
| `"..."` | строковый литерал |
| `;` | конец инструкции |

🆚 Сравни:
```
   C:      printf("Hello\n");
   C++:    std::cout << "Hello\n";
   Python: print("Hello")
   Rust:   println!("Hello");        // ! означает макрос
```

> 💡 `println!` — это **макрос**, а не функция (отсюда `!`). Макросы в Rust мощные:
> `println!` проверяет форматную строку на этапе компиляции. Пока просто запомни `!`.

## ⭐ Команды cargo

| Команда | Что делает |
|---------|-----------|
| `cargo new имя` | создать новый проект |
| `cargo run` | собрать и запустить |
| `cargo build` | только собрать (в `target/debug/`) |
| `cargo build --release` | собрать с оптимизацией (быстрая версия) |
| `cargo check` | быстро проверить на ошибки **без** сборки бинаря |
| `cargo test` | запустить тесты |
| `cargo fmt` | отформатировать код |
| `cargo clippy` | линтер (находит проблемы и плохой стиль) |

💡 `cargo check` — твой друг во время написания: он проверяет код (включая borrow checker)
очень быстро, не тратя время на генерацию исполняемого файла.

## 📖 Форматирование вывода

```rust
fn main() {
    let name = "Гена";
    let age = 30;

    println!("Привет, {}!", name);              // {} — подстановка
    println!("{} — {} лет", name, age);
    println!("Через год: {}", age + 1);
    println!("{name} напрямую");                // можно имя прямо в {} (новые версии)
    println!("Число: {:.2}", 3.14159);          // 3.14 — форматирование
}
```

🖼️ Как работает `{}`:
```
   println!("Привет, {}!", name);
                     └┬┘    └─ подставится сюда
                  плейсхолдер
```

🆚 Похоже на f-строки Python и `<<` C++, но через макрос с проверкой при компиляции.

## 📖 Зависимости через Cargo.toml

Чтобы подключить чужую библиотеку (crate) — добавь её в `Cargo.toml`:

```toml
[dependencies]
rand = "0.8"          # библиотека случайных чисел
```

И `cargo run` сам её скачает и подключит. Библиотеки берутся с **crates.io** — это как
PyPI для Python или npm. Подробнее — когда дойдём до пет-проекта.

---

## 🧪 Эксперимент: cargo check vs build

```powershell
cargo check     # быстро — только проверка (включая borrow checker!)
cargo build     # медленнее — генерирует исполняемый файл
```

Заметь, насколько `check` быстрее. Во время разработки используй его, чтобы ловить ошибки.

## ✅ Задачи

1. **Свой проект.** Создай `cargo new visitka`, выведи имя, город, любимый язык.
2. **Форматирование.** Выведи число с 2 знаками после запятой, число в разных форматах.
3. **Несколько значений.** Выведи имя и возраст в одной строке через `{}`.
4. **cargo check.** Намеренно сделай ошибку (убери `;`), запусти `cargo check`, прочитай
   подробное сообщение компилятора.
5. **release.** Собери проект через `cargo build --release`, найди бинарь в
   `target/release/`.
