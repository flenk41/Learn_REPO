# 00 · Что такое Rust и его модель памяти 🖼️

> 🎯 **Цель блока:** понять, что такое Rust и какую главную проблему он решает. Это задаёт
> смысл всему курсу — и особенно ядру про память.

---

## 📖 Что такое Rust

Rust создан в Mozilla (первый стабильный релиз — 2015) как **системный язык**, который
сочетает несочетаемое: скорость C/C++ **и** безопасность памяти. Сегодня на нём пишут
браузеры, операционные системы, веб-серверы, блокчейны, инструменты разработки. Несколько
лет подряд он — «самый любимый язык» по опросам разработчиков.

Ключевые черты:
- 🚀 **скорость** как у C/C++ (компилируется в машинный код, нет сборщика мусора);
- 🛡️ **безопасность памяти**, доказанная компилятором;
- 🧵 **бесстрашная многопоточность** — гонки данных невозможны;
- 📦 отличные инструменты (cargo) и экосистема.

---

## ⭐ Главная проблема, которую решает Rust

Вспомни три предыдущих языка:

```
   C:      утечки, висячие указатели, переполнения буфера — в РАНТАЙМЕ (краш/взлом)
   C++:    те же опасности, RAII помогает, но ошибиться всё ещё легко
   Python: безопасно, но медленно и без контроля (сборщик мусора)
```

🎯 **Rust ловит ошибки памяти на этапе КОМПИЛЯЦИИ.** Если код, нарушающий правила работы с
памятью, не скомпилируется — баг не доедет до пользователя. При этом нет сборщика мусора:
скорость остаётся как у C.

```rust
let s1 = String::from("привет");
let s2 = s1;              // владение ПЕРЕМЕЩЕНО в s2
println!("{}", s1);      // ❌ ОШИБКА КОМПИЛЯЦИИ: s1 больше не владеет данными
                         //    компилятор не даст использовать перемещённое значение
```

💡 В C++ похожий код «использование после move» **скомпилировался бы** и, может, упал в
рантайме. В Rust компилятор **останавливает тебя сразу**, с понятным объяснением.

## 🖼️ Четыре подхода к памяти — где Rust

```
   C: вручную          C++: вручную+RAII     Python: GC          Rust: компилятор
   ┌──────────┐        ┌──────────┐          ┌──────────┐        ┌──────────────┐
   │ malloc   │        │ объекты  │          │ создал   │        │ создал       │
   │ free←сам │        │ } деструктор│       │ (GC      │        │ } владение   │
   │ забыл?   │        │   освобождает│      │  уберёт) │        │   кончилось →│
   │ утечка   │        │            │        │          │        │   освобождено│
   └──────────┘        └──────────┘          └──────────┘        └──────────────┘
   опасно              контроль+RAII         удобно, медленно    безопасно+быстро,
                                                                  проверено КОМПИЛЯТОРОМ
```

💡 Rust освобождает память так же детерминированно, как C++ (в конце владения, без GC), но
**гарантирует** отсутствие ошибок через borrow checker. Это «лучшее из всех миров»: контроль
C, безопасность Python, и проверка на этапе компиляции, которой нет ни у кого.

## 📖 Три правила владения — суть Rust

Вся модель памяти Rust строится на трёх правилах (детально — в Уровне 2):

1. **У каждого значения есть владелец** (одна переменная).
2. **Владелец может быть только один** в каждый момент.
3. **Когда владелец выходит из области видимости — значение освобождается.**

```rust
{
    let s = String::from("привет");   // s — владелец
    // ... используем s ...
}                                     // s вышел из области → память освобождена
                                      // автоматически, без free, без GC
```

🎯 Эти правила + заимствование (borrowing) + проверка компилятором = вся безопасность Rust.
Это новый способ мышления, и он — главное, чему учит этот курс.

---

## 📖 Rust компилируется и быстр

Как C и C++, Rust **компилируется** в машинный код. Поэтому:
- 🚀 скорость на уровне C++;
- нет виртуальной машины и сборщика мусора;
- готовый исполняемый файл.

```
   твой .rs ──компилятор(rustc/cargo)──► машинный код ──► программа
```

Цена — строгий компилятор и время на изучение владения. Но взамен — программы, которые
**не падают** из-за памяти и **не имеют гонок данных**.

## 🤔 Когда выбирать Rust

| Бери Rust | Когда |
|-----------|-------|
| системное ПО, ОС, драйверы | нужны скорость и безопасность |
| веб-серверы, сервисы | высокая нагрузка + надёжность |
| инструменты (CLI) | быстрые и надёжные утилиты |
| блокчейн, криптография | критична корректность |
| замена C/C++ | где ошибки памяти недопустимы |

💡 Поняв C, C++, Python и Rust, ты видишь весь спектр работы с памятью — и можешь выбрать
правильный инструмент для любой задачи.
