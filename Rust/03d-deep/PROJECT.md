# 🚀 Мини-проект · Безопасная абстракция над unsafe

> Докажи мастерство Rust: построй структуру данных, которую нельзя выразить в чистом safe Rust,
> используя локализованный unsafe, обёрнутый в полностью безопасный API. Как делает стандартная библиотека.

---

## 🎯 Что делаем

Выбери структуру/абстракцию, требующую unsafe внутри, и сделай её **безопасной снаружи** — с
доказанными инвариантами и тестами.

```
   варианты (выбери один):
   • свой Vec<T> (сырые указатели + ручная память, безопасный API push/get/...).
   • своя arena/bump-аллокатор (unsafe внутри, безопасная выдача).
   • intrusive linked list (узлы со ссылками друг на друга).
   • split-функция среза на две непересекающиеся части (&mut [T] → два &mut).
   • простой Future / async-примитив.
```

---

## 📋 Требования (на примере своего Vec)

- **Локализованный unsafe:** сырые указатели/память только внутри; снаружи — безопасный API.
- **Инварианты:** каждый unsafe-блок снабжён `// SAFETY:` — почему он корректен.
- **Безопасный API:** невозможно использовать во вред (нельзя получить висячую ссылку, double-free, UB).
- **Корректность владения:** Drop освобождает память; нет утечек/двойного освобождения.
- **Трейты:** реализуй уместные (Drop, возможно Deref, Iterator, Send/Sync — если безопасно, с обоснованием).
- **Качество:** тесты (включая краевые), прогон под Miri (детектор UB в Rust!), README с разбором инвариантов.

---

## 🌟 Бонусы

- 🥚 Generic по типу `T`; правильная обработка не-Copy типов (Drop элементов).
- 🐣 Реализуй `Iterator`/`IntoIterator` для своей структуры.
- 🦅 Сравни с std-аналогом (API, производительность).
- 🚀 Прогон под **Miri** — докажи отсутствие UB; объясни, что Miri проверяет.

---

## ✅ Критерии готовности

- [ ] Структура использует unsafe только локально, за безопасным API
- [ ] Каждый unsafe снабжён `// SAFETY:` обоснованием
- [ ] Безопасный API невозможно использовать во вред (нет UB снаружи)
- [ ] Корректное владение (Drop, нет утечек/double-free)
- [ ] Тесты + чистый прогон под Miri
- [ ] README с разбором инвариантов
