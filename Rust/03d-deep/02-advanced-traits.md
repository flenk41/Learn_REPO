## 📖 Ассоциированные типы

```rust
trait Iterator {
    type Item;                              // АССОЦИИРОВАННЫЙ тип — «выходной» тип трейта
    fn next(&mut self) -> Option<Self::Item>;
}

impl Iterator for Counter {
    type Item = u32;                        // конкретизируем при реализации
    fn next(&mut self) -> Option<u32> { ... }
}
```

```
   ассоциированный тип = тип, ПРИНАДЛЕЖАЩИЙ реализации трейта (один на impl).
   ↕ отличие от обобщённого трейта trait Foo<T>: с дженериком тип может быть РАЗНЫМ (много impl),
     с ассоциированным — ОДИН на тип (Iterator у Vec даёт ровно один Item).
   используй ассоциированный тип, когда на каждый тип — ОДНА осмысленная реализация (Iterator, Deref).
```

💡 ⭐⭐ Ассоциированные типы делают трейты выразительнее дженериков: `Iterator::Item` — это «тип,
который выдаёт ЭТОТ итератор», однозначно. Они убирают лишние параметры типов и делают код чище.
Это основа `Iterator`, `Deref`, `Add` (через `Output`) и многих трейтов stdlib.

## ⭐⭐ Статическая vs динамическая диспетчеризация

```rust
// СТАТИЧЕСКАЯ (generics, мономорфизация):
fn process<T: Draw>(item: &T) { item.draw(); }
// компилятор генерирует ОТДЕЛЬНУЮ копию process для каждого конкретного T → zero-cost, инлайнинг.

// ДИНАМИЧЕСКАЯ (trait objects, dyn):
fn process(item: &dyn Draw) { item.draw(); }
// ОДНА функция; вызов draw идёт через VTABLE (указатель на таблицу методов) в рантайме.
let shapes: Vec<Box<dyn Draw>> = vec![Box::new(Circle), Box::new(Square)];  // разнородная коллекция!
```

```
                    GENERICS (static)        dyn (dynamic)
   скорость         быстрее (инлайн, zero-cost)  чуть медленнее (vtable-вызов)
   размер кода      больше (копия на тип)     меньше (одна функция)
   гетерогенность   нет (один тип)            ДА (Vec<Box<dyn Trait>> — разные типы вместе)
   когда            горячий код, один тип     разнородные коллекции, плагины, меньше кода
```

💡 ⭐⭐ Это ключевой [trade-off Rust](../../Senior/02-decisions/08-tradeoffs.md): дженерики —
быстро (мономорфизация, как [шаблоны C++](../../Cpp/03d-deep/04-stl-internals.md)), но раздувают
код и не дают смешивать типы. `dyn Trait` — динамическая диспетчеризация через vtable (как
[виртуальные функции C++](../../Cpp/03-middle/14-inheritance-polymorphism.md)): чуть медленнее, но
позволяет гетерогенные коллекции и меньше кода. Выбирай по контексту.

## ⭐ Object safety: что можно за dyn

```
   не всякий трейт можно сделать `dyn` — он должен быть OBJECT-SAFE:
   ✅ методы НЕ возвращают Self (иначе размер неизвестен).
   ✅ методы НЕ обобщённые (vtable не может хранить бесконечно много версий).
   ✅ нет ассоциированных констант и т.п.
   если трейт нарушает → ошибка «trait is not object safe». решения: разбить трейт, использовать
   дженерики вместо dyn, или специальные приёмы.
```

💡 Понимание object safety объясняет частую ошибку «cannot be made into an object»: трейт с
обобщённым методом или возвратом `Self` нельзя за `dyn` (vtable не выразит). Тогда — дженерики.

## 📖 GAT и продвинутое (обзор)

```
   GAT (Generic Associated Types) — ассоциированные типы С ПАРАМЕТРАМИ (лайфтаймами/типами):
   trait LendingIterator { type Item<'a> where Self: 'a; fn next(&mut self) -> Option<Self::Item<'_>>; }
   позволяет, например, итератор, выдающий ССЫЛКИ на свои данные (lending iterator) — раньше невозможно.
   стабилизированы недавно; нужны для продвинутых обобщённых API.

   + супертрейты (trait A: B — A требует B), blanket impls (impl<T: X> Y for T — реализация для всех
     T, удовлетворяющих X — так работает, например, ToString для всех Display), маркерные трейты
     (Send/Sync/Sized — свойства типа без методов).
```

> 🧭 Трейты Rust — это [интерфейсы/полиморфизм из ООП](../../OOP/02-pillars/12-interfaces.md), но
> мощнее: zero-cost дженерики + опциональный dyn, без наследования.

## ⚠️ Ловушки

- ❌ Использовать дженерик-параметр трейта, где нужен ассоциированный тип (лишние параметры).
- ❌ `dyn` в горячем коде, где дженерик дал бы zero-cost (или наоборот — раздувать код мономорфизацией).
- ❌ Трейт с обобщённым методом/возвратом Self под `dyn` (не object-safe).
- ❌ Забыть `Box`/`&` для `dyn` (он unsized — нужен за указателем).
- ❌ Не знать про blanket impls (откуда «магически» взялась реализация).
