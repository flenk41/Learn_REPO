# 18 · Замыкания и итераторы

> 🎯 **Цель блока:** освоить замыкания (анонимные функции) и итераторы — выразительный,
> «функциональный» стиль обработки данных, который в Rust ещё и максимально быстрый.

---

## ⭐ Замыкания — анонимные функции

```rust
let add = |a, b| a + b;          // замыкание: |параметры| тело
println!("{}", add(2, 3));       // 5

let square = |x| x * x;
println!("{}", square(5));       // 25
```

🖼️ Синтаксис:
```
   |a, b| a + b
    └─┬─┘ └─┬─┘
   параметры  тело
```

### Захват окружения
Замыкания «захватывают» переменные из окружения (как в Python/C++):

```rust
let factor = 3;
let multiply = |x| x * factor;   // factor захвачен из окружения
println!("{}", multiply(5));     // 15

// move — забрать владение захваченным
let s = String::from("привет");
let print_it = move || println!("{}", s);   // s перемещён в замыкание
print_it();
```

💡 Замыкание = «функция + захваченное окружение». `move` передаёт владение захваченным
переменным в замыкание (нужно для потоков — Уровень 4). Это прямой аналог замыканий из
[Python-курса](../../Python/04-senior/21-decorators-closures.md).

## ⭐⭐ Итераторы — обработка коллекций

**Итератор** выдаёт элементы по одному. Главное: итераторы **ленивые** и
**композируемые** — ты строишь конвейер обработки.

```rust
let numbers = vec![1, 2, 3, 4, 5, 6];

let result: Vec<i32> = numbers
    .iter()                          // создать итератор (по ссылкам)
    .filter(|&&x| x % 2 == 0)        // оставить чётные
    .map(|&x| x * x)                 // возвести в квадрат
    .collect();                      // собрать в Vec
// result: [4, 16, 36]
```

🖼️ Конвейер:
```
   [1,2,3,4,5,6] → filter(чётные) → [2,4,6] → map(квадрат) → [4,16,36] → collect → Vec
   ленивый: ничего не считается, пока не вызван collect/sum/for
```

💡 Это как ranges в C++20 или генераторы Python — ленивый конвейер. Но в Rust он
**zero-cost**: компилятор разворачивает его в такой же быстрый код, как ручной цикл.

---

## 📖 Главные методы итераторов

```rust
let v = vec![1, 2, 3, 4, 5];

v.iter().map(|x| x * 2);                 // преобразовать каждый
v.iter().filter(|&&x| x > 2);            // отфильтровать
v.iter().sum::<i32>();                   // 15 — сумма
v.iter().count();                        // 5
v.iter().max();                          // Some(5)
v.iter().min();                          // Some(1)
v.iter().find(|&&x| x > 3);              // Some(4) — первый подходящий
v.iter().any(|&x| x > 4);                // true — есть ли
v.iter().all(|&x| x > 0);                // true — все ли
v.iter().rev();                          // в обратном порядке
v.iter().enumerate();                    // (индекс, значение)
v.iter().take(3);                        // первые 3
v.iter().skip(2);                        // пропустить 2

// свёртка (как reduce/accumulate)
let product = v.iter().fold(1, |acc, x| acc * x);   // 120
```

| Метод | Что делает |
|-------|-----------|
| `map` | преобразовать каждый |
| `filter` | оставить подходящие |
| `sum`/`product` | сумма/произведение |
| `find`/`any`/`all` | поиск/проверка |
| `fold` | свёртка (reduce) |
| `collect` | собрать в коллекцию |
| `enumerate` | добавить индексы |

---

## ⭐ Три способа создать итератор

```rust
let v = vec![1, 2, 3];

v.iter()         // по ссылкам (&T) — коллекция остаётся
v.iter_mut()     // по изменяемым ссылкам (&mut T) — можно менять
v.into_iter()    // по значениям (T) — забирает владение коллекцией
```

💡 Снова владение: `iter()` заимствует, `into_iter()` забирает. Выбирай по тому, нужна ли
коллекция после.

---

## 🧪 Примеры

```rust
// сумма квадратов чётных
let sum: i32 = (1..=10).filter(|x| x % 2 == 0).map(|x| x * x).sum();

// слова длиннее 3 букв
let words = vec!["a", "abc", "abcd", "ab"];
let long: Vec<&str> = words.into_iter().filter(|w| w.len() > 3).collect();

// перебор с индексами
for (i, val) in vec!["a", "b", "c"].iter().enumerate() {
    println!("{}: {}", i, val);
}
```
