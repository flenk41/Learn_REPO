# 17 · Коллекции

> 🎯 **Цель блока:** освоить главные коллекции Rust — `Vec`, `HashMap`, `String`. Они
> управляют памятью сами (как контейнеры C++), подчиняясь владению.

---

## ⭐ Vec — динамический массив

```rust
let mut v: Vec<i32> = Vec::new();
v.push(10);
v.push(20);
v.push(30);

let v2 = vec![1, 2, 3];          // макрос vec! для краткости

println!("{}", v[0]);            // 10 (паника при выходе за границы)
match v.get(5) {                 // безопасный доступ → Option
    Some(x) => println!("{}", x),
    None => println!("нет элемента"),
}

v.pop();                         // удалить последний → Option
println!("{}", v.len());

// перебор
for x in &v {                    // заимствуем для чтения
    println!("{}", x);
}
for x in &mut v {                // заимствуем для изменения
    *x *= 2;
}
```

🖼️ `Vec` — растущий массив в куче, который сам выделяет и освобождает память (как
`std::vector` в C++ или список Python). При выходе из области — освобождается автоматически
(владение).

> 💡 `v[5]` паникует при выходе за границы; `v.get(5)` возвращает `Option` (безопасно). В
> Rust выбор за тобой: быстро-но-паника или безопасно-через-Option.

## ⭐ HashMap — пары ключ→значение

```rust
use std::collections::HashMap;

let mut scores: HashMap<String, i32> = HashMap::new();
scores.insert(String::from("Гена"), 100);
scores.insert(String::from("Чебур"), 95);

// доступ → Option
match scores.get("Гена") {
    Some(s) => println!("{}", s),
    None => println!("нет такого"),
}

// со значением по умолчанию
let s = scores.get("Неизвестный").copied().unwrap_or(0);

// перебор
for (name, score) in &scores {
    println!("{}: {}", name, score);
}

// обновление / вставка-если-нет
*scores.entry(String::from("Гена")).or_insert(0) += 10;   // прибавить или создать
```

💡 `HashMap` — хеш-таблица (как `dict` Python / `unordered_map` C++, см.
[Python-курс](../../Python/03-middle/14-dict-set-hashing.md)). Доступ — O(1). `entry().or_insert()`
— идиома «обнови или создай».

### Подсчёт частот — классика
```rust
let text = "a b a c a b";
let mut counts: HashMap<&str, i32> = HashMap::new();
for word in text.split(' ') {
    *counts.entry(word).or_insert(0) += 1;
}
// {"a": 3, "b": 2, "c": 1}
```

## 📖 Другие коллекции

```rust
use std::collections::{HashSet, BTreeMap, VecDeque};

let mut set: HashSet<i32> = HashSet::new();   // уникальные элементы
set.insert(1);
set.insert(1);                                // дубликат игнорируется
println!("{}", set.contains(&1));             // true

// убрать дубликаты из Vec
let v = vec![1, 1, 2, 3, 3];
let unique: HashSet<i32> = v.into_iter().collect();
```

| Коллекция | Что это |
|-----------|---------|
| `Vec<T>` | динамический массив (по умолчанию) |
| `HashMap<K,V>` | хеш-таблица ключ→значение |
| `HashSet<T>` | уникальные элементы |
| `BTreeMap<K,V>` | отсортированная карта (на дереве) |
| `VecDeque<T>` | очередь с двух концов |
| `String` | строка (растущая) |

## ⚠️ Коллекции и владение

```rust
let mut v = vec![String::from("a"), String::from("b")];

let first = &v[0];               // заимствование
// v.push(String::from("c"));    // ❌ нельзя менять v, пока живёт &first (borrow checker!)
println!("{}", first);
v.push(String::from("c"));       // ✅ теперь можно

// перемещение из вектора забирает владение:
for s in v {                     // v перемещается, после цикла недоступен
    println!("{}", s);
}
```

💡 Коллекции подчиняются правилам владения и заимствования (Уровень 2). Это защищает от
бага «изменили коллекцию во время итерации» (в C++/Python — источник крашей).
