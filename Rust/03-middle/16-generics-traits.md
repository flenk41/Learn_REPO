# 16 · Обобщения и трейты

> 🎯 **Цель блока:** освоить обобщения (generics) — код для любого типа — и трейты —
> способ описать общее поведение. Это фундамент абстракций Rust.

---

## ⭐ Обобщения (generics)

Как шаблоны в C++, обобщения позволяют писать код для **любого типа**:

```rust
fn largest<T: PartialOrd>(list: &[T]) -> &T {   // <T> — параметр-тип
    let mut max = &list[0];
    for item in list {
        if item > max {
            max = item;
        }
    }
    max
}

let numbers = vec![3, 7, 2, 9, 1];
println!("{}", largest(&numbers));       // 9

let chars = vec!['a', 'z', 'm'];
println!("{}", largest(&chars));         // z
```

🖼️
```
   largest<T>(...)  → компилятор генерирует версию под каждый тип
   largest(&numbers) → largest для i32
   largest(&chars)   → largest для char
```

💡 Как в C++, обобщения раскрываются **на этапе компиляции** (мономорфизация) — нулевая
цена в рантайме. `<T: PartialOrd>` означает «T должен поддерживать сравнение `>`».

### Обобщённые структуры
```rust
struct Pair<T> {
    first: T,
    second: T,
}

let p = Pair { first: 1, second: 2 };
let ps = Pair { first: "a", second: "b" };
```

## ⭐⭐ Трейты — общее поведение

**Трейт** (trait) описывает набор методов, которые тип должен реализовать. Это как
«интерфейс» в других языках или абстрактный класс в C++.

```rust
trait Describe {
    fn describe(&self) -> String;          // метод, который реализуют типы
}

struct Dog { name: String }
struct Cat { name: String }

impl Describe for Dog {
    fn describe(&self) -> String {
        format!("Собака {}", self.name)
    }
}

impl Describe for Cat {
    fn describe(&self) -> String {
        format!("Кошка {}", self.name)
    }
}

let d = Dog { name: String::from("Рекс") };
println!("{}", d.describe());              // Собака Рекс
```

🖼️
```
   trait Describe { fn describe(); }   ← контракт «что должен уметь тип»
        ▲              ▲
   impl for Dog   impl for Cat        ← каждый тип реализует по-своему
```

💡 Трейт — это контракт: «любой тип с этим трейтом умеет `describe`». Это аналог
интерфейсов/абстрактных классов, но гибче — трейт можно реализовать даже для чужого типа.

### Методы по умолчанию
```rust
trait Greet {
    fn name(&self) -> String;
    fn hello(&self) -> String {            // реализация по умолчанию
        format!("Привет, я {}", self.name())
    }
}
```

## ⭐ Трейты как ограничения (trait bounds)

Можно требовать, чтобы обобщённый тип реализовывал трейт:

```rust
fn print_all<T: Describe>(items: &[T]) {   // T должен реализовать Describe
    for item in items {
        println!("{}", item.describe());
    }
}

// синтаксис с where (для читаемости при многих ограничениях)
fn process<T>(item: T) where T: Describe + Clone {
    // ...
}

// impl Trait — короткий синтаксис
fn make_describable() -> impl Describe {
    Dog { name: String::from("Бобик") }
}
```

💡 `<T: Describe>` гарантирует, что у `T` есть метод `describe`. Компилятор проверяет это
при компиляции — никаких «метод не найден» в рантайме.

## 📖 Стандартные трейты

Rust использует трейты повсюду:

| Трейт | Что даёт |
|-------|----------|
| `Debug` | печать `{:?}` |
| `Display` | печать `{}` (своё форматирование) |
| `Clone` | `.clone()` |
| `PartialEq` | `==` |
| `PartialOrd` | `< > <= >=` |
| `Iterator` | перебор (модуль 18) |
| `Default` | значение по умолчанию |

```rust
use std::fmt;
impl fmt::Display for Dog {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "🐕 {}", self.name)
    }
}
// теперь println!("{}", dog) работает
```
