# 03 · Переменные и изменяемость

> 🎯 **Цель блока:** понять переменные Rust и его необычную особенность — **неизменяемость
> по умолчанию**. Это первый шаг к безопасности.

---

## ⭐ Переменные неизменяемы по умолчанию

```rust
let x = 5;
println!("{}", x);   // 5
x = 6;               // ❌ ОШИБКА КОМПИЛЯЦИИ! x неизменяема
```

🆚 В C/C++/Python переменные изменяемы по умолчанию. В Rust — **наоборот**: `let x = 5`
создаёт **неизменяемую** переменную. Чтобы менять — нужно явно сказать `mut`:

```rust
let mut x = 5;       // mut = mutable (изменяемая)
x = 6;               // ✅ теперь можно
println!("{}", x);   // 6
```

💡 Почему так? Неизменяемость по умолчанию — это безопасность: компилятор знает, что
значение не поменяется, ловит случайные изменения и помогает с многопоточностью (Уровень 4).
Привыкай: помечай `mut`, **только** когда правда нужно менять.

---

## 📖 Константы

```rust
const MAX_USERS: u32 = 100;        // const ВСЕГДА неизменяема, тип обязателен
const PI: f64 = 3.14159;
```

Отличия `const` от `let`:
- тип указывать **обязательно** (`: u32`);
- значение должно быть известно при компиляции;
- пишутся ЗАГЛАВНЫМИ_БУКВАМИ;
- нельзя `mut`.

## ⭐ Затенение (shadowing)

Можно объявить переменную с тем же именем заново — это **затенение**, не изменение:

```rust
let x = 5;
let x = x + 1;       // новая переменная x (затеняет старую) = 6
let x = x * 2;       // ещё одна = 12
println!("{}", x);   // 12

// затенение даже меняет ТИП:
let spaces = "   ";          // строка
let spaces = spaces.len();   // число! (новая переменная)
```

🖼️
```
   let x = 5;        x ──► [5]
   let x = x + 1;    x ──► [6]   (новая переменная, старая [5] затенена)
```

> 💡 Разница с `mut`: `mut` меняет **значение той же** переменной (тип неизменен).
> Затенение создаёт **новую** переменную (можно сменить тип). Затенение удобно, когда
> временно преобразуешь значение.

## 📖 Вывод типа

Rust обычно сам выводит тип, но иногда нужно указать явно:

```rust
let x = 5;              // компилятор выведет i32
let y: f64 = 3.14;      // явно указали тип
let z = 5u8;            // суффикс типа (u8)
let guess: u32 = "42".parse().expect("не число");   // тип нужен для parse
```

💡 Rust статически типизирован (как C/C++), но с умным выводом типов — пишешь меньше, чем
в C++, но всё проверяется при компиляции, в отличие от Python.
