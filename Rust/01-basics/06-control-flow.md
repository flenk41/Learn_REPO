# 06 · Управление потоком и match

> 🎯 **Цель блока:** освоить условия, циклы и — главное — `match`, мощнейший инструмент
> сопоставления с образцом в Rust.

---

## 📖 Условия `if / else if / else`

```rust
let n = 7;
if n % 2 == 0 {
    println!("чётное");
} else if n % 3 == 0 {
    println!("делится на 3");
} else {
    println!("другое");
}
```

> ⚠️ Условие должно быть **именно** `bool`. В Rust нельзя `if n { ... }` для числа (как в
> C, где не-ноль = истина). Только `if n != 0 { ... }`. Это строже и безопаснее.

`if` как выражение (из прошлого модуля):
```rust
let max = if a > b { a } else { b };
```

## 🔁 Циклы

### `loop` — бесконечный цикл
```rust
let mut count = 0;
loop {
    count += 1;
    if count == 5 { break; }
}

// loop тоже может возвращать значение через break!
let result = loop {
    count += 1;
    if count == 10 { break count * 2; }   // break со значением
};
println!("{}", result);    // 20
```

### `while`
```rust
let mut n = 5;
while n > 0 {
    println!("{}", n);
    n -= 1;
}
```

### `for` — перебор (самый частый)
```rust
for i in 0..5 {            // 0, 1, 2, 3, 4 (диапазон, не включая 5)
    println!("{}", i);
}
for i in 1..=5 {           // 1, 2, 3, 4, 5 (включая 5)
    println!("{}", i);
}

let arr = [10, 20, 30];
for x in arr {             // перебор элементов (как range-for в C++ / for-in в Python)
    println!("{}", x);
}
```

🆚 `for` в Rust перебирает элементы/диапазоны (как Python), а не считает вручную (как C).

## ⭐⭐ match — суперинструмент

`match` сопоставляет значение с образцами. Мощнее `switch` из C — он **обязан покрыть все
случаи** (компилятор проверяет!) и умеет разбирать структуры.

```rust
let number = 3;
match number {
    1 => println!("один"),
    2 => println!("два"),
    3 => println!("три"),
    4 | 5 => println!("четыре или пять"),   // несколько значений
    6..=10 => println!("от шести до десяти"),  // диапазон
    _ => println!("что-то ещё"),            // _ = всё остальное (обязателен!)
}
```

🖼️
```
   match number {
       образец => действие,
       ...
       _ => действие по умолчанию    ← match должен покрыть ВСЕ случаи
   }
```

> 💡 **Главное отличие от switch:** Rust требует, чтобы `match` покрыл **все** возможные
> значения. Забыл случай — ошибка компиляции. Это ловит баги «а что если придёт другое
> значение». В Уровне 3 `match` раскроется полностью с `Option` и `enum`.

### match как выражение
```rust
let day = 3;
let name = match day {
    1 => "Пн",
    2 => "Вт",
    3 => "Ср",
    _ => "?",
};
println!("{}", name);    // Ср
```

## 📖 if let — короткий match для одного случая

```rust
let value = Some(5);
if let Some(x) = value {       // если value это Some(...) — взять x
    println!("есть значение: {}", x);
}
```

💡 `if let` — удобное сокращение, когда интересен только один образец. Подробно с `Option`
— в Уровне 3.
