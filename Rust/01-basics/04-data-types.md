# 04 · Типы данных

> 🎯 **Цель блока:** освоить базовые типы Rust. Они строгие и явные — это часть
> безопасности.

---

## 📖 Скалярные типы

### Целые числа
```rust
let a: i32 = -100;     // знаковое 32-битное (по умолчанию для целых)
let b: u32 = 100;      // беззнаковое (только ≥ 0)
let big: i64 = 10_000_000_000;
let byte: u8 = 255;
```

| Тип | Размер | Знак |
|-----|--------|------|
| `i8`/`u8` | 1 байт | ±/+ |
| `i32`/`u32` | 4 байта | ±/+ |
| `i64`/`u64` | 8 байт | ±/+ |
| `isize`/`usize` | размер указателя | для индексов/размеров |

💡 В Rust тип целого указывается явно в имени (`i32`, `u8`). `usize` используется для
индексов массивов и размеров — как `size_t` в C.

### Дробные, логические, символы
```rust
let pi: f64 = 3.14159;     // f64 по умолчанию (двойная точность)
let x: f32 = 1.5;          // одинарная точность
let ok: bool = true;       // настоящий булев тип
let letter: char = 'А';    // char — 4 байта, поддерживает Unicode (даже эмодзи!)
let emoji: char = '🦀';
```

## ⚠️ Rust строг с типами: нет неявных преобразований

```rust
let x: i32 = 5;
let y: i64 = 10;
// let z = x + y;          // ❌ ОШИБКА! нельзя складывать i32 и i64
let z = x as i64 + y;      // ✅ явное преобразование через as
```

> 💡 В C `int + long` сработает молча (с неявным преобразованием). В Rust — **ошибка**:
> преобразования всегда явные (`as`). Это предотвращает скрытые баги.

### Переполнение проверяется!
```rust
let x: u8 = 255;
let y = x + 1;             // ⚠️ в debug-сборке — ПАНИКА (краш с сообщением)
                           //    в release — заворачивается (как в C)
```
💡 В отладочной сборке Rust ловит переполнение и сообщает о нём, а не молча портит данные.

## ⭐ Кортежи (tuple)

Группировка разных типов:
```rust
let person: (String, i32, f64) = (String::from("Гена"), 30, 1.75);

// доступ по индексу
println!("{}", person.1);      // 30

// деструктуризация (распаковка)
let (name, age, height) = person;
println!("{} {} {}", name, age, height);
```

🆚 Как `std::tuple` в C++ или кортежи Python, но с проверкой типов.

---

## ⭐ Массивы (array)

Фиксированный размер, все элементы одного типа:
```rust
let arr: [i32; 5] = [1, 2, 3, 4, 5];   // [тип; размер]
let zeros = [0; 10];                    // 10 нулей
println!("{}", arr[0]);                 // 1
println!("{}", arr.len());              // 5

// ⚠️ выход за границы — ПАНИКА (а не порча памяти, как в C!)
// println!("{}", arr[10]);             // паника: index out of bounds
```

🖼️ Массив лежит в памяти подряд (как в C), но Rust **проверяет границы** при доступе:
```
   arr: [1][2][3][4][5]
   arr[10] → паника (безопасно), а не чтение чужой памяти
```

> 💡 Массив имеет фиксированный размер. Для динамического — `Vec` (Уровень 3). Чаще
> используют `Vec`, а массивы — когда размер известен заранее.
