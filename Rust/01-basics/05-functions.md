# 05 · Функции

> 🎯 **Цель блока:** освоить функции Rust и важную особенность — **выражения против
> инструкций**, которая отличает Rust от C-подобных языков.

---

## 📖 Основы

```rust
fn main() {
    greet("Гена");
    let result = square(5);
    println!("{}", result);    // 25
}

fn greet(name: &str) {                 // параметр: имя: тип
    println!("Привет, {}!", name);
}

fn square(x: i32) -> i32 {             // -> тип возвращаемого значения
    x * x                              // ← БЕЗ return и БЕЗ ; (это выражение!)
}
```

🖼️ Анатомия:
```
   fn square(x: i32) -> i32 {
       │      └──┬───┘   └─ тип результата
    имя       параметр: тип
```

> 💡 Типы параметров и результата указываются **обязательно** (в отличие от Python). Это
> часть статической проверки.

---

## ⭐ Выражения vs инструкции — ключевая особенность

В Rust **почти всё является выражением** (возвращает значение). Последнее выражение в блоке
**без точки с запятой** становится результатом:

```rust
fn square(x: i32) -> i32 {
    x * x        // выражение без ; → это возвращаемое значение
}

// эквивалент с явным return:
fn square2(x: i32) -> i32 {
    return x * x;    // тоже работает, но не идиоматично
}
```

> ⚠️ **Важно:** `x * x` (без `;`) — это возврат значения. А `x * x;` (с `;`) — это
> инструкция, которая возвращает `()` (пустоту) → ошибка типа! Точка с запятой превращает
> выражение в инструкцию.

### Блок — тоже выражение
```rust
let y = {
    let a = 3;
    a + 1            // блок возвращает 4 (без ;)
};
println!("{}", y);   // 4
```

🆚 В C/C++ блок ничего не возвращает. В Rust блок `{}` — выражение, его значение —
последняя строка без `;`. Это делает код выразительным.

## 📖 if тоже выражение

```rust
let number = 5;
let result = if number > 0 { "плюс" } else { "минус или ноль" };
println!("{}", result);     // плюс
```

🆚 Это как тернарный оператор, но обычный `if`. В Rust не нужен отдельный `? :`.

---

## 📖 Несколько возвращаемых значений — через кортеж

```rust
fn min_max(a: i32, b: i32) -> (i32, i32) {
    if a < b { (a, b) } else { (b, a) }
}

let (mn, mx) = min_max(7, 3);     // деструктуризация
println!("{} {}", mn, mx);        // 3 7
```

## ⚠️ Передача аргументов — пока по значению

```rust
fn print_len(s: String) {
    println!("{}", s.len());
}

let text = String::from("привет");
print_len(text);
// println!("{}", text);   // ❌ ОШИБКА! владение text перешло в функцию
```

> ⚠️ Это первый намёк на **владение** (Уровень 2). Когда передаёшь `String` в функцию,
> владение **перемещается** туда, и снаружи переменная больше не доступна. Чтобы этого
> избежать, используют **ссылки** (`&`) — но это уже ядро курса. Пока запомни этот момент.
