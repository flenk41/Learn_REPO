## 📖 Rust устроен иначе, чем C/C++

```
   ни заголовков, ни прямого включения текста. вместо этого:
   • МОДУЛЬ (mod) — пространство имён для группы функций/типов.
   • файл = модуль (Rust сам связывает имя модуля с файлом).
   • ВИДИМОСТЬ: по умолчанию всё ПРИВАТНО; наружу открывается через pub.
   • ПУТИ: до элемента добираешься через crate::module::item или сокращаешь через use.

   компилятор сам находит и собирает файлы по дереву модулей — никакого ручного перечисления .c/.cpp.
```

💡 ⭐ Главное отличие: в Rust ты не «включаешь файлы», а **объявляешь модули** (`mod`), и компилятор
сам подтягивает соответствующие файлы. Сборкой целиком занимается `cargo` — ты не вызываешь компилятор
с перечнем файлов руками.

## ⭐⭐ Модуль в отдельном файле (на примере)

🖼️
```
   структура проекта (cargo new myapp):
   src/
   ├── main.rs          ← корень бинарного крейта
   └── math_utils.rs    ← наш модуль

   src/math_utils.rs — МОДУЛЬ
   ─────────────────────────────────────────────
   pub fn add(a: i32, b: i32) -> i32 { a + b }   // pub — видно снаружи модуля
   pub fn mul(a: i32, b: i32) -> i32 { a * b }
   fn secret() {}                                // без pub — приватно, только внутри модуля

   src/main.rs — ИСПОЛЬЗОВАНИЕ
   ─────────────────────────────────────────────
   mod math_utils;                 // подключить файл math_utils.rs как модуль

   use math_utils::add;            // (необязательно) сократить путь к add

   fn main() {
       println!("{}", add(2, 3));            // через use — просто add
       println!("{}", math_utils::mul(2, 3)); // или полным путём
   }
```

💡 ⭐⭐ Три шага: **`mod math_utils;`** (подключить файл как модуль) → пометить нужные функции
**`pub`** (иначе снаружи не видны) → вызывать как `math_utils::add(...)` или сократить через
**`use`**. `mod имя;` говорит компилятору «есть модуль, его код в `имя.rs`».

## ⭐ pub — без него не видно

```
   приватность по умолчанию — фишка безопасности Rust: модуль прячет детали, наружу только то, что пометил pub.

   pub fn add(...)        // функция видна снаружи
   fn helper(...)         // приватна: вызвать можно ТОЛЬКО внутри своего модуля
   pub struct Point { pub x: i32, y: i32 }   // pub у поля — отдельно (x публично, y нет)

   попытка вызвать приватную функцию из другого модуля → ошибка компиляции «private».
```

💡 ⭐ Если из `main` не видно твою функцию — почти всегда забыл `pub`. Это не баг, а защита:
Rust заставляет осознанно решать, что входит в публичный интерфейс модуля, а что — внутренняя кухня.

## ⭐ Вложенные модули и папки

```
   МОДУЛЬ ВНУТРИ ФАЙЛА (inline) — для мелочи:
   mod geometry {
       pub fn area(w: f64, h: f64) -> f64 { w * h }
   }
   fn main() { println!("{}", geometry::area(2.0, 3.0)); }

   МОДУЛЬ-ПАПКА (для крупного) — src/shapes/ с файлом src/shapes/mod.rs (или src/shapes.rs):
   src/
   ├── main.rs            // mod shapes;
   └── shapes/
       ├── mod.rs         // pub mod circle;  (объявляет подмодули)
       └── circle.rs      // pub fn area(r: f64) -> f64 { ... }
   вызов: shapes::circle::area(2.0)

   пути: crate:: (от корня), self:: (текущий модуль), super:: (родительский).
```

💡 ⭐ Маленькую группу — `mod {}` прямо в файле; крупную — отдельный файл или папка с подмодулями.
`use` сокращает длинные пути: `use crate::shapes::circle::area;` → потом просто `area(...)`.

## 📖 Библиотека vs бинарь (кратко)

```
   • бинарный крейт — есть main.rs с fn main() (программа).
   • библиотечный крейт — есть lib.rs (код для переиспользования другими).
   модули (mod/pub/use) работают одинаково в обоих. внешние библиотеки (crates.io) подключают в
   Cargo.toml и используют через use имя_крейта::... — подробнее в разделе 03b.
```

## ⚠️ Ловушки

- ❌ Забыть `mod math_utils;` в main.rs → «cannot find module» (файл есть, но не подключён).
- ❌ Функция без `pub` → «function is private» при вызове из другого модуля.
- ❌ Думать, что нужно «include» как в C — в Rust подключают МОДУЛЬ (`mod`), а не текст файла.
- ❌ Путать `mod` (объявить/подключить модуль) и `use` (сократить путь к уже существующему).
- ❌ Перечислять файлы при сборке вручную — это делает `cargo` сам (`cargo run`/`cargo build`).
- ❌ Путаться в путях: `crate::` от корня, `super::` к родителю, `self::` текущий.
