# 02 · Инструменты наблюдения за ОС 🛠️

> 🎯 **Цель блока:** познакомиться с инструментами, которыми мы будем «подглядывать» за ОС весь
> трек: ps/top/htop, /proc, strace, free, dmesg, Диспетчер задач.

---

## 🛠️ Базовый набор

```
   ps / top / htop   — какие процессы запущены, сколько едят CPU/памяти
   free / vmstat     — сколько памяти занято/свободно, подкачка (swap)
   /proc (Linux)     — виртуальная «файловая система» о состоянии ядра и процессов
   strace            — все системные вызовы процесса (рентген границы user/kernel)
   lsof              — какие файлы/сокеты открыты
   dmesg             — сообщения ядра (загрузка, устройства, ошибки)
```

💡 На **Windows** аналоги: Диспетчер задач, Process Explorer/Process Monitor (Sysinternals),
`Get-Process` в PowerShell. Идеи те же — посмотреть процессы, память, активность.

---

## 🛠️ Что показывает каждый (коротко)

```bash
top            # живой список процессов: CPU%, MEM%, PID, состояние
ps aux         # снимок всех процессов разом
free -h        # память: total / used / free / swap
strace ls      # системные вызовы команды ls
cat /proc/meminfo   # подробно про память (Linux)
cat /proc/<PID>/status  # инфо о конкретном процессе
dmesg | tail   # последние сообщения ядра
```

🖼️
```
   вопрос                              инструмент
   «что сейчас грузит CPU/память?»  →  top / htop / Диспетчер задач
   «сколько свободной памяти, swap?» →  free / vmstat
   «что делает этот процесс?»        →  strace, /proc/<PID>/
   «какие файлы открыты?»            →  lsof
   «что говорит ядро?»               →  dmesg
```

---

## ⭐ /proc — окно в ядро (Linux)

`/proc` — не настоящие файлы на диске, а **интерфейс к ядру** в виде файлов. Читаешь файл —
видишь состояние системы прямо сейчас.

```
   /proc/cpuinfo            — про процессор
   /proc/meminfo            — про память
   /proc/<PID>/status       — состояние процесса PID
   /proc/<PID>/maps         — КАРТА ПАМЯТИ процесса (понадобится в уровне 2!)
   /proc/<PID>/fd/          — открытые файлы процесса
```

💡 `/proc/<PID>/maps` — особенно ценен: он покажет адресное пространство процесса (стек, кучу,
библиотеки) — это прямой мост к ядру трека (виртуальная память). Запомни его.

---

## 🛠️ Установка

```
   Linux         — почти всё уже есть; htop/strace/lsof ставятся из пакетного менеджера
   macOS         — top/ps есть; dtruss вместо strace; htop через brew
   Windows       — Диспетчер задач встроен; Process Explorer/Monitor — Sysinternals (бесплатно)
   WSL           — полноценный Linux-набор на Windows (рекомендуется для практики)
```

💡 Для практики этого трека удобнее всего **Linux или WSL** — там есть `/proc`, `strace` и
прозрачное наблюдение за ядром. На чистом Windows опирайся на Диспетчер задач и Sysinternals.

---

## ⚠️ Ловушки

- ❌ Имена/наличие команд различаются по ОС (strace ↔ dtruss, нет /proc на Windows).
- ❌ Читать снимок `ps` как «живую» картину — для динамики нужен `top`/`htop`.
- ❌ Пугаться сотен процессов в списке — это норма, ОС управляет ими всеми.

---

## 🛠️ Практика

1. Запусти `top`/`htop` (или Диспетчер задач) — найди самый «прожорливый» процесс.
2. `free -h` — сколько памяти занято и есть ли swap.
3. На Linux/WSL: `cat /proc/<PID>/maps` для любого процесса — посмотри его карту памяти
   (вернёмся к ней в уровне 2).
4. `strace`-ни простую команду и найди системные вызовы.
