# 03 · Процесс и его адресное пространство 🖼️⭐

> 🎯 **Цель блока:** понять, что такое процесс, из чего он состоит и как выглядит его память —
> мост между «программой на диске» и «работой в RAM».

---

## 📖 Программа vs процесс

```
   ПРОГРАММА — файл на диске (пассивный код, инструкции)
   ПРОЦЕСС   — программа, ЗАПУЩЕННАЯ в памяти (живой, выполняется)
```

💡 Одна программа может породить **много** процессов (открой браузер дважды — два процесса той
же программы). Процесс — это «программа + её состояние»: память, открытые файлы, регистры CPU,
позиция выполнения.

---

## ⭐ Адресное пространство процесса

У каждого процесса — **своё** адресное пространство (своя «карта памяти»). Вот как оно обычно
устроено:

🖼️
```
   высокие адреса
   ┌────────────────────┐
   │  Стек (stack)       │  ← локальные переменные, вызовы функций; растёт ВНИЗ
   │        ↓            │
   │        ...          │
   │        ↑            │
   │  Куча (heap)        │  ← динамическая память (malloc/new); растёт ВВЕРХ
   ├────────────────────┤
   │  BSS / Data         │  ← глобальные переменные
   │  Text (код)         │  ← сами инструкции программы (только чтение)
   └────────────────────┘
   низкие адреса
```

💡 ⭐ Вот тот самый стек и куча из треков C/C++/Rust — но теперь ты видишь, **где** они живут:
внутри адресного пространства процесса, которое создаёт ОС. Стек и куча растут навстречу друг
другу. Это можно увидеть вживую в `/proc/<PID>/maps` (модуль 02).

---

## ⭐ Иллюзия «своей памяти»

Важнейший момент: эти адреса — **виртуальные**. У двух процессов адрес `0x400000` указывает на
**разную** физическую память. Каждый процесс думает, что вся память — его.

```
   процесс A: адрес 0x400000  ─┐
                               ├─► ОС переводит в РАЗНУЮ физическую RAM
   процесс B: адрес 0x400000  ─┘
```

💡 Как ОС это делает — ядро уровня 2 (виртуальная память). Пока запомни: **изоляция памяти**
процессов — фундамент. Один процесс не может прочитать память другого (если ОС явно не разрешит).

---

## 📖 Что ещё входит в процесс

```
   PID            — уникальный номер процесса
   состояние       — выполняется / ждёт / готов (модуль 04)
   регистры CPU    — текущее состояние вычислений
   таблица файлов  — какие файлы/сокеты открыты
   права           — от чьего имени работает (пользователь)
   родитель/дети    — процессы образуют дерево (модуль 04)
```

💡 Всю эту информацию ядро хранит в структуре («блок управления процессом», PCB). При
переключении процессов ОС сохраняет/восстанавливает это состояние — контекстное переключение
(модуль 06).

## ⚠️ Ловушки

- ❌ Путать программу (файл) и процесс (запущенный экземпляр).
- ❌ Думать, что адреса в процессе — это реальные адреса в RAM. Они **виртуальные**.
- ❌ Считать, что процессы видят память друг друга. По умолчанию — нет, они изолированы.

---

## 🛠️ Практика

1. Запусти одну программу дважды — в `top`/Диспетчере увидишь два процесса с разными PID.
2. На Linux/WSL: `cat /proc/<PID>/maps` — найди `[stack]`, `[heap]`, области кода и библиотек.
3. Сопоставь увиденное со схемой адресного пространства выше.
