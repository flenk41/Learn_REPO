## 📖 Что такое исключение

Когда что-то идёт не так (деление на ноль, файл не найден, неверный тип), Python
**выбрасывает исключение** — программа останавливается с сообщением об ошибке:

```python
x = int("abc")        # ValueError: invalid literal for int()
y = 10 / 0            # ZeroDivisionError
z = [1, 2][5]         # IndexError
d = {}["key"]         # KeyError
```

Вместо краха эти ситуации можно **перехватить** и обработать.

## ⭐ try / except

```python
try:
    age = int(input("Возраст: "))
    print(f"Через год: {age + 1}")
except ValueError:
    print("Это не число!")        # выполнится, если int() упал
```

🖼️
```
   try:  ──► выполняется код
              │
        ошибка? ──нет──► идём дальше
              │ да
              ▼
        except: ──► обрабатываем, программа НЕ падает
```

### Перехват разных ошибок

```python
try:
    data = [1, 2, 3]
    index = int(input("Индекс: "))
    print(data[index])
except ValueError:
    print("Введи число!")
except IndexError:
    print("Нет такого индекса!")
except Exception as e:            # ловит любую (на крайний случай)
    print(f"Ошибка: {e}")
```

> ⚠️ Не лови «голый» `except:` без типа — он спрячет даже опечатки и Ctrl+C. Лови
> **конкретные** исключения, которые ожидаешь.

## 📖 Полная конструкция: try / except / else / finally

```python
try:
    f = open("data.txt")
    data = f.read()
except FileNotFoundError:
    print("Файл не найден")
else:
    print("Успех:", data)         # выполнится, если НЕ было ошибки
finally:
    print("Этот блок выполнится ВСЕГДА")   # очистка ресурсов
```

| Блок | Когда выполняется |
|------|-------------------|
| `try` | всегда (основной код) |
| `except` | если возникла ошибка |
| `else` | если ошибки НЕ было |
| `finally` | **всегда** (даже при ошибке) — для очистки |

## 📖 Выбрасывание своих исключений: `raise`

```python
def withdraw(balance, amount):
    if amount > balance:
        raise ValueError("Недостаточно средств")   # выбросить ошибку
    return balance - amount

try:
    withdraw(100, 500)
except ValueError as e:
    print(e)              # Недостаточно средств
```

## 📖 Свои классы исключений

```python
class InsufficientFunds(Exception):
    """Не хватает денег на счёте."""
    pass

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFunds(f"Не хватает {amount - balance} руб")
    return balance - amount

try:
    withdraw(100, 500)
except InsufficientFunds as e:
    print("Своя ошибка:", e)
```

💡 Свои исключения делают код понятнее: ты ловишь именно `InsufficientFunds`, а не общий
`ValueError`.

## ⭐ EAFP — питоничный стиль обработки

В Python принят подход **EAFP** («проще попросить прощения, чем разрешения»): пробуй
сделать, а ошибку обработай — вместо множества предварительных проверок.

```python
# LBYL (look before you leap) — менее питонично
if key in d and isinstance(d[key], int):
    value = d[key]

# EAFP — питонично
try:
    value = d[key]
except KeyError:
    value = 0
```

💡 EAFP часто быстрее и читаемее, особенно когда «ошибка» — редкий случай.

## ✅ Задачи

1. **Безопасный ввод числа.** Считывай число, пока пользователь не введёт корректное
   (`try/except` в цикле).
2. **Калькулятор без падений.** Обработай деление на ноль и нечисловой ввод.
3. **Безопасный доступ.** Функция, возвращающая элемент списка по индексу или значение
   по умолчанию, если индекс вне диапазона.
4. **Своё исключение.** Создай `InvalidAgeError`, выбрасывай его, если возраст < 0 или > 150.
5. **finally.** Напиши код, который открывает «ресурс» и гарантированно «закрывает» его в
   `finally`, даже если внутри ошибка.
6. **Парсер чисел.** Из списка строк `["1", "x", "3", "abc", "5"]` собери список чисел,
   пропуская нечисловые (EAFP).
7. ⭐ **Повтор при ошибке.** Функция, которая повторяет операцию N раз при исключении,
   прежде чем сдаться.
