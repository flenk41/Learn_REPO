## 📖 Измеряем размер объектов: `sys.getsizeof`

```python
import sys

sys.getsizeof(0)            # ~28 байт — даже маленькое число это объект!
sys.getsizeof("")          # ~49 байт
sys.getsizeof("a"*1000)    # ~1049
sys.getsizeof([])          # ~56
sys.getsizeof({})          # ~64
```

> ⚠️ `getsizeof` считает только **сам объект**, не вложенные! У списка `[big_obj]` он
> покажет размер списка из ссылок, а не размер `big_obj`. Для рекурсивного подсчёта нужен
> обход (или библиотека `pympler`).

🖼️ Почему Python «тяжёлый»: каждое число — полноценный объект с заголовком (тип, счётчик
ссылок, значение), а не 4 байта как `int` в C:

```
   C:      int x;      → 4 байта
   Python: x = 0       → ~28 байт (заголовок объекта + значение)
```

Это цена удобства. Поэтому для миллионов чисел используют `array`/`numpy` (модуль 24).

## ⭐ __slots__ — рекап с замером

Из модуля 16: `__slots__` убирает `__dict__` и резко сокращает память объектов.

```python
import sys

class Normal:
    def __init__(self, x, y):
        self.x, self.y = x, y

class Slotted:
    __slots__ = ("x", "y")
    def __init__(self, x, y):
        self.x, self.y = x, y

# На миллионе объектов разница в памяти — кратная.
# Замерь: создай по 1_000_000 объектов и сравни общий расход (через tracemalloc ниже).
```

## ⭐ tracemalloc — встроенный поиск утечек

`tracemalloc` показывает, **где** в коде выделяется память. Незаменим для поиска утечек.

```python
import tracemalloc

tracemalloc.start()

# ... подозрительный код ...
data = [str(i) for i in range(100000)]

current, peak = tracemalloc.get_traced_memory()
print(f"Сейчас: {current/1024/1024:.1f} МБ, пик: {peak/1024/1024:.1f} МБ")

# Топ мест, где выделена память
snapshot = tracemalloc.take_snapshot()
for stat in snapshot.statistics("lineno")[:5]:
    print(stat)

tracemalloc.stop()
```

🖼️ Вывод покажет строки кода с объёмом выделенной памяти — ищешь самые «жирные».

## ⭐ Циклические ссылки и `gc`

Из модуля 11: подсчёт ссылок не справляется с циклами, их убирает сборщик мусора.

```python
import gc

class Node:
    def __init__(self):
        self.ref = None

a = Node()
b = Node()
a.ref = b
b.ref = a            # цикл: a → b → a
del a, b             # имена удалены, но объекты ссылаются друг на друга

print(gc.collect())  # вернёт число собранных объектов > 0 (нашёл цикл)
```

> 💡 Можно посмотреть, что мешает сборке: `gc.garbage` (объекты, которые GC не смог
> собрать), `gc.get_referrers(obj)` (кто ссылается на объект).

## ⭐ weakref — слабые ссылки (не держат объект живым)

**Слабая ссылка** указывает на объект, но **не увеличивает** счётчик ссылок. Если все
обычные ссылки исчезли — объект удаляется, даже если на него есть слабая ссылка.

```python
import weakref

class Data:
    pass

obj = Data()
weak = weakref.ref(obj)     # слабая ссылка

print(weak())               # <Data object> — объект жив
del obj                     # убрали ЕДИНСТВЕННУЮ сильную ссылку
print(weak())               # None — объект удалён!
```

🖼️
```
   obj ────(сильная)────► [Data]  счётчик = 1
   weak ───(слабая)─────►        (не считается!)

   del obj → счётчик = 0 → [Data] удалён → weak() возвращает None
```

💡 Зачем: **разрывать циклы** (родитель держит детей сильно, дети ссылаются на родителя
слабо → нет цикла) и строить **кэши**, которые не мешают сборке мусора
(`weakref.WeakValueDictionary`).

## ⭐ Ловушка: склейка строк в цикле

Строки неизменяемы (Уровень 2!), поэтому `+=` в цикле создаёт **новую строку каждый раз** —
квадратичная сложность и куча мусора:

```python
# 🐢 ПЛОХО — на каждой итерации новая строка
result = ""
for word in words:
    result += word           # создаётся новый объект-строка снова и снова

# 🚀 ХОРОШО — собрать в список, склеить один раз
result = "".join(words)      # одна операция
```

💡 `"".join(...)` — питоничный и быстрый способ. Это прямое следствие неизменяемости строк.

## 📖 sys.intern — ручное интернирование

Если в программе много одинаковых строк (например, повторяющиеся ключи), `sys.intern`
заставит хранить их в **одном** экземпляре — экономия памяти:

```python
import sys
a = sys.intern("очень_частый_ключ")
b = sys.intern("очень_частый_ключ")
print(a is b)        # True — один объект в памяти
```
