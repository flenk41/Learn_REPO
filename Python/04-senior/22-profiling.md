# 22 · Профилирование памяти и скорости 🛠️⭐

> 🎯 **Цель блока:** научиться **измерять**, а не угадывать — где код медленный и где он
> ест память. Senior не оптимизирует наугад; он профилирует, находит узкое место и чинит
> именно его.

---

## 📖 Золотое правило оптимизации

> «Преждевременная оптимизация — корень всех зол» (Дональд Кнут).

Сначала **измерь**, потом оптимизируй. Узкое место почти всегда не там, где кажется.

🖼️ Рабочий процесс:
```
   1. Написал работающий код
   2. Медленно/жрёт память? → ПРОФИЛИРУЙ
   3. Нашёл узкое место (часто 1 функция из 100)
   4. Оптимизировал именно его
   5. Замерил снова — стало лучше? оставляй
```

## 🛠️ Замер времени: timeit

Для измерения маленьких кусков кода — `timeit` (запускает много раз, усредняет):

```python
import timeit

# Сравним два способа собрать строку
t1 = timeit.timeit('"".join(str(i) for i in range(100))', number=10000)
t2 = timeit.timeit('s="".join([str(i) for i in range(100)])', number=10000)
print(t1, t2)
```

В терминале:
```bash
python -m timeit "sum(range(1000))"
```

Простой замер «вручную» — `time.perf_counter`:
```python
import time
start = time.perf_counter()
# ... код ...
print(f"{time.perf_counter() - start:.4f} сек")
```

## 🛠️ Профилирование скорости: cProfile

`cProfile` показывает, **сколько времени** заняла каждая функция — главный инструмент.

```python
import cProfile

def heavy():
    total = 0
    for i in range(1_000_000):
        total += i ** 2
    return total

cProfile.run("heavy()")
```

Или из терминала для целого скрипта:
```bash
python -m cProfile -s cumulative my_program.py
```

🖼️ Вывод — таблица:
```
   ncalls  tottime  cumtime  function
   1000000  0.421    0.421    heavy (вот где время!)
   ...
```

| Колонка | Значение |
|---------|----------|
| `ncalls` | сколько раз вызвана |
| `tottime` | время в самой функции (без вложенных) |
| `cumtime` | время с вложенными вызовами |

> 💡 Сортируй по `cumtime` (общее) или `tottime` (без подвызовов), чтобы найти главного
> «пожирателя» времени. Для визуализации есть `snakeviz`.

## 🛠️ Профилирование памяти

### tracemalloc (встроенный, из модуля 20)
```python
import tracemalloc
tracemalloc.start()
data = [bytes(1000) for _ in range(10000)]
current, peak = tracemalloc.get_traced_memory()
print(f"Пик: {peak/1024/1024:.1f} МБ")
tracemalloc.stop()
```

### memory_profiler (сторонний, построчно)
```bash
pip install memory_profiler
```
```python
from memory_profiler import profile

@profile                       # покажет память по строкам
def make_data():
    a = [i for i in range(100000)]
    b = [i*2 for i in range(100000)]
    return b
```
```bash
python -m memory_profiler my_program.py
```
Вывод покажет, **сколько МБ** добавила каждая строка.

## 📖 Замер сложности — большое O на практике

```python
# Замерь время на размерах 1000, 10000, 100000 и сравни рост:
import time
for n in [1000, 10000, 100000]:
    start = time.perf_counter()
    _ = sum(i for i in range(n))
    print(n, time.perf_counter() - start)
# Линейный рост? Квадратичный? Профилирование подтверждает теорию.
```
