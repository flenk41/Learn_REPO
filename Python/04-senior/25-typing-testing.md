## ⭐ Аннотации типов (type hints)

Python динамический, но можно **подсказывать типы** — для читаемости и проверки
инструментами. Сам Python их не проверяет в рантайме (это подсказки).

```python
def greet(name: str, times: int = 1) -> str:
    return (f"Привет, {name}! " * times).strip()

age: int = 25
names: list[str] = ["Гена", "Кот"]
scores: dict[str, int] = {"Гена": 100}

from typing import Optional
def find(key: str) -> Optional[int]:    # может вернуть int или None
    return None
```

🖼️
```
   def add(a: int, b: int) -> int:
              └─типы входа─┘   └─тип результата
```

💡 Типы — это документация, которую проверяет машина. В больших проектах они спасают от
кучи ошибок и делают код понятным. IDE по ним даёт точные подсказки.

## 🛠️ mypy — проверка типов

`mypy` статически проверяет, что типы согласованы — **до запуска** программы:

```bash
pip install mypy
mypy my_program.py
```

```python
def double(x: int) -> int:
    return x * 2

double("привет")     # mypy: error — ожидался int, передан str
```

> 💡 Это похоже на компилятор C, который ловит ошибки типов заранее. Только тут опционально.

## ⭐ Тестирование: pytest

Тесты — код, который проверяет твой код. Профи **всегда** покрывают логику тестами.

**mymath.py:**
```python
def add(a, b):
    return a + b

def divide(a, b):
    if b == 0:
        raise ValueError("деление на ноль")
    return a / b
```

**test_mymath.py:**
```python
import pytest
from mymath import add, divide

def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0

def test_divide():
    assert divide(10, 2) == 5

def test_divide_by_zero():
    with pytest.raises(ValueError):     # проверяем, что бросается исключение
        divide(1, 0)
```

Запуск:
```bash
pip install pytest
pytest                    # найдёт и запустит все test_*.py
pytest -v                 # подробно
```

🖼️ Вывод:
```
   test_mymath.py ...                    [100%]
   3 passed in 0.01s
```

💡 Правило профи: написал функцию → напиши тест. Изменил код → тесты подтверждают, что
ничего не сломалось (регрессия).

## ⭐ Линтеры и форматтеры

| Инструмент | Что делает |
|-----------|-----------|
| **ruff** | сверхбыстрый линтер: находит ошибки, плохой стиль, неиспользуемые импорты |
| **black** | форматтер: приводит код к единому стилю автоматически |
| **mypy** | проверка типов |

```bash
pip install ruff black mypy

ruff check .              # найти проблемы
ruff check . --fix        # и починить, что можно
black .                   # отформатировать весь код
mypy .                    # проверить типы
```

> 💡 PEP 8 — официальный стиль Python. `black` и `ruff` следят за ним автоматически —
> тебе не нужно спорить о пробелах, инструменты решают.

## 📖 Логирование вместо print

В реальных программах вместо `print` используют `logging` — с уровнями и управлением:

```python
import logging
logging.basicConfig(level=logging.INFO)

logging.debug("детали для отладки")
logging.info("обычное событие")
logging.warning("предупреждение")
logging.error("ошибка!")
```

💡 Можно включать/выключать уровни, писать в файл — без правки кода. `print` для такого
не годится.
