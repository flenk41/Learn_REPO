## 📖 Почему Python медленный для вычислений

Из модуля 20: каждое число — полноценный объект (~28 байт, с заголовком). Список чисел —
это массив **ссылок** на разбросанные по куче объекты.

🖼️ Сравни хранение `[1, 2, 3]`:

```
   Python list:   [ссылка]→[obj 1]   [ссылка]→[obj 2]   [ссылка]→[obj 3]
                  объекты разбросаны, на каждый — заголовок, постоянные
                  проверки типа → медленно, много памяти, плохо для кэша

   numpy array:   [1][2][3]  ← числа лежат ПОДРЯД, как массив int в C
                  компактно, кэш-дружелюбно, операции на C-скорости
```

Плюс на каждую операцию Python проверяет типы, считает ссылки, работает через
интерпретатор. Для миллионов чисел это катастрофически медленно.

## ⭐ numpy — массивы как в C

`numpy` хранит числа **подряд** (как массив в C) и выполняет операции на скомпилированном
C-коде. Ускорение — в десятки-сотни раз.

```python
import numpy as np

a = np.array([1, 2, 3, 4, 5])
b = np.array([10, 20, 30, 40, 50])

print(a + b)          # [11 22 33 44 55] — поэлементно, без цикла!
print(a * 2)          # [2 4 6 8 10]
print(a.sum())        # 15
print(a.mean())       # 3.0
```

### ⭐ Векторизация: операции без циклов

```python
# 🐢 Чистый Python — цикл по миллиону объектов
result = [x ** 2 for x in range(1_000_000)]

# 🚀 numpy — векторизованная операция на C
arr = np.arange(1_000_000)
result = arr ** 2          # в десятки раз быстрее, в разы меньше памяти
```

🖼️ Векторизация = одна операция применяется ко всему массиву сразу, в C-коде, без
накладных расходов интерпретатора Python.

💡 **Правило:** если делаешь численные расчёты над большими массивами — используй numpy и
**избегай циклов**. Это основа всего научного Python (pandas, ML, обработка данных).

## 📖 Память: numpy против списка

```python
import numpy as np
import sys

py_list = list(range(1_000_000))
np_arr = np.arange(1_000_000)

# numpy хранит компактно — в разы меньше памяти
print(np_arr.nbytes)              # ~8 МБ (8 байт на число)
# список: ~28 байт на объект + ссылки = в несколько раз больше
print(np_arr.dtype)               # int64 — фиксированный тип, как в C
```

> 💡 `dtype` (тип элементов) выбираешь сам: `int8`, `int32`, `float64`. Меньший тип =
> меньше памяти. Это прямое управление памятью, как выбор `char`/`int` в C!

## 📖 Встроенные средства тоже быстрее ручных циклов

Даже без numpy: встроенные функции реализованы на C и быстрее ручных циклов.

```python
# 🐢 ручной цикл
total = 0
for x in data:
    total += x

# 🚀 встроенная (на C)
total = sum(data)

# map / filter тоже на C
squared = list(map(lambda x: x*x, data))
```

| Вместо | Используй |
|--------|-----------|
| ручной цикл суммирования | `sum()` |
| ручной поиск макс/мин | `max()`, `min()` |
| склейка строк `+=` | `"".join()` |
| ручная фильтрация | comprehension / `filter` |
| числовые массивы | `numpy` |

## 📖 Когда нужен максимум — C-расширения

Если узкое место остаётся даже после numpy, есть варианты ускорить «горячий» код:

| Инструмент | Что делает |
|-----------|-----------|
| **Cython** | компилирует Python-подобный код в C |
| **numba** | JIT-компиляция функций (`@njit`) — без переписывания |
| **ctypes / cffi** | вызов готовых C-библиотек из Python |
| **C-расширения** | модули на чистом C (тут пригодится C-курс!) |

```python
# numba — просто добавь декоратор
from numba import njit

@njit
def heavy(n):
    total = 0
    for i in range(n):
        total += i * i
    return total          # компилируется в машинный код, скорость близка к C
```

💡 Это мост к твоему C-курсу: критичные по скорости части пишут на C, а Python используют
как «клей». Так устроены numpy, pandas, TensorFlow — ядро на C, интерфейс на Python.
