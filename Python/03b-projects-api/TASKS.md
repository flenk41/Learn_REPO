# ✅ Задачи · Проекты и API (Python)

---

## 🟢 Блок A — структура проекта

1. ⭐ **Скрипт → пакет** — преврати свой проект в `src/myapp/` с модулями.
2. ⭐⭐ **__init__ как API** — настрой `from myapp import ...`, `__all__`, `__version__`.
3. ⭐⭐ **Публичное/приватное** — раздели на публичные и `_приватные` имена.
4. ⭐⭐ **pyproject.toml** + `pip install -e .`, импорт из другого места.
5. ⭐⭐⭐ **Цикл импорта** — создай, поймай, исправь реструктуризацией.

---

## 🟢 Блок B — проектирование API

6. ⭐⭐ **Класс-клиент** с публичными методами и `_приватными` помощниками.
7. ⭐⭐ **Типы + docstring** на всех публичных функциях (+ проверка `mypy`).
8. ⭐⭐ **Свои исключения** вместо возврата None/печати.
9. ⭐⭐ **dataclass-результат** вместо словаря.
10. ⭐⭐⭐ **API-ревью** — понятен ли модуль только по именам/типам/docstring.

---

## 🟢 Блок C — веб-API

11. ⭐ **GitHub-пользователь** — имя, репозитории.
12. ⭐⭐ **Поиск** репозиториев с параметрами.
13. ⭐⭐ **Обработка** 404 и ошибок сети (timeout, connection).
14. ⭐⭐ **Погода** через open-meteo (без ключа) по координатам.
15. ⭐⭐ **Ключ из окружения** для API с авторизацией.
16. ⭐⭐⭐ **Пагинация** — собрать все элементы постранично с уважением к лимитам.
17. ⭐⭐⭐ **Свой клиент** — класс-обёртка над API с типами, исключениями, сессией.

## 🧩 Где тренироваться

- [Public APIs](https://github.com/public-apis/public-apis) — каталог открытых API.
- [open-meteo](https://open-meteo.com) — погода без ключа.
- [GitHub REST API](https://docs.github.com/rest), [JSONPlaceholder](https://jsonplaceholder.typicode.com) — для тренировки.
- [httpbin.org](https://httpbin.org) — тестовый сервер для запросов.
