## 🧭 Язык и проектирование — две стороны ООП

В этом разделе — **как писать ООП на Python** (синтаксис: `@property`, `super`, `ABC`,
dunder). А **как проектировать** объектами язык-агностично (четыре столпа, SOLID, паттерны,
чистая архитектура) — это отдельный трек **🏛️ ООП**. Они дополняют друг друга:

| Здесь (язык) | Трек 🏛️ ООП (проектирование) |
|---|---|
| `class`, `@property`, `super()`, `ABC` | [абстракция](../../OOP/02-pillars/08-abstraction.md), [инкапсуляция](../../OOP/02-pillars/09-encapsulation-deep.md) |
| наследование vs композиция в Python | [наследование](../../OOP/02-pillars/10-inheritance.md), [композиция > наследования](../../OOP/03-design/16-composition-over-inheritance.md) |
| duck typing, полиморфизм dunder | [полиморфизм](../../OOP/02-pillars/11-polymorphism.md) |
| перегрузка операторов, протоколы | [SOLID](../../OOP/03-design/13-solid-intro.md), [паттерны](../../OOP/04-patterns/19-creational.md) |

💡 Совет: пройди этот раздел, чтобы **писать** ООП на Python, и трек 🏛️ ООП — чтобы
**проектировать** хорошо на любом языке.
