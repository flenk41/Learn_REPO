# ✅ Задачи раздела · Углублённое ООП

> Закрепляем инкапсуляцию, наследование/MRO, абстракцию/полиморфизм и магические методы.
> Решения держи в одном пакете — пригодятся для [мини-проекта](PROJECT.md).

---

## 🟢 Базовые

1. **Temperature.** `celsius` через `@property` (запрет ниже −273.15), `fahrenheit` —
   вычисляемое свойство. Read-only там, где не нужно присваивание.
2. **BankAccount v2.** Поле `_balance` инкапсулировано через свойство с проверкой; методы
   `deposit`, `withdraw` (с защитой от ухода в минус).
3. **super().** Иерархия `Employee` → `Manager` (добавляет `team`). Конструктор `Manager`
   правильно зовёт `super().__init__(...)`.
4. **Полиморфизм.** Список разных `Shape` (Circle, Square, Triangle), функция `total_area`
   без единого `isinstance`.

## 🟡 Средние

5. **ABC PaymentMethod.** Абстрактный метод `pay(amount)`, реализации `Card`, `SBP`, `Cash`.
   Проверь невозможность создать абстрактный/неполный класс.
6. **MRO-эксперимент.** Построй ромб `A→B,C→D`, переопредели метод в B и C. Предскажи результат
   `D().method()` и `D.__mro__`, потом проверь.
7. **Vector2D.** Полный набор операторов: `__add__`, `__sub__`, `__mul__`, `__eq__`, `__abs__`,
   `__repr__`. Должен складываться, сравниваться, сортироваться.
8. **Контекст-менеджер.** `Timer` (`__enter__`/`__exit__`), печатающий длительность блока —
   корректно срабатывает даже при исключении внутри `with`.

## 🔴 Со звёздочкой

9. ⭐ **Дескриптор Validated.** Переиспользуемый дескриптор с проверкой (например, «строка не
   пустая»). Примени к нескольким полям класса `User`.
10. ⭐ **Protocol vs ABC.** Реши одну задачу (например, `render(obj)`) двумя способами — через
    `ABC` и через `typing.Protocol`. Опиши в комментарии, когда что удобнее.
11. ⭐ **Свой контейнер.** Класс `OrderedSet`: хранит уникальные элементы в порядке добавления.
    Поддержи `__len__`, `__iter__`, `__contains__`, `__getitem__`, `__repr__`.
12. ⭐⭐ **Mixin-набор.** Сделай `JsonMixin` и `ComparableMixin`. Собери класс `Product`,
    подмешав оба. Убедись, что он сериализуется и сортируется.

---

## 🎯 Критерии «сделано»

- [ ] Инкапсуляция через `@property`, а не публичные поля везде
- [ ] `super().__init__(...)` со скобками и аргументами
- [ ] Полиморфизм вместо `if isinstance`
- [ ] Хотя бы один `ABC` или `Protocol`
- [ ] Хотя бы один контекст-менеджер и один перегруженный оператор
- [ ] Понимаю `__mro__` своих классов
