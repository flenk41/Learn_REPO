# 🚀 Проект уровня 2 · Свой производительный веб-сервер

> Доведи сервер до полноценного: HTTP поверх event loop с конкурентностью, отдающий статику и
> REST, устойчивый и быстрый. Это классическая senior-веха — синтез ОС, сетей, конкурентности.

> 🛠️ C/C++/Rust + epoll (Linux). С тестами, бенчмарками, защитой.

---

## 🎯 Что делаем

Собери всё уровня в один веб-сервер, который реально можно открыть в браузере и нагрузить.

```
   epoll event loop  +  пул потоков (многоядерность)  +  HTTP-парсинг  +  статика/REST  +  защита
```

---

## 📋 Требования

### Сетевой движок (модули 09-11)
- Event loop на epoll, неблокирующий I-O, держит тысячи соединений.
- Конкурентность: пул потоков или event-loop-per-core (многоядерность).
- Управление состоянием соединений (буферы, частичные read/write).
- Устойчивость: разрывы, медленные клиенты, graceful shutdown.

### HTTP (модуль 12)
- Парсинг запроса (метод, путь, заголовки, тело по Content-Length).
- Роутинг: статические файлы + несколько динамических эндпоинтов (REST/JSON).
- Корректные статусы/заголовки (200/404/400/500, Content-Length/Type), keep-alive.

### Безопасность и надёжность
- Защита от path traversal; валидация размеров (анти-DoS); 400 на кривые запросы — без падений.
- (бонус) HTTPS через библиотеку.

### Качество
- Тесты (curl/скрипты): роуты, статусы, краевые случаи, безопасность.
- Бенчмарк (ab/wrk): req/sec; сравнение моделей конкурентности.
- TSan-чистота (нет гонок), нет утечек fd/памяти.
- Сборка + README (запуск, конфиг, бенчмарки).

## 🌟 Бонусы

- 🥚 Конфиг (порт, корень статики, число потоков).
- 🐣 Логирование запросов, метрики (req/sec, активные соединения).
- 🦅 REST-API поверх твоего key-value хранилища (мост к Ур.3): GET/POST/DELETE /kv/{key}.
- 🚀 Сравни throughput с nginx на статике; объясни разрыв.
- 🚀 io_uring вместо epoll (новейший асинхронный I-O) — [см. модуль 23](../04-language/23-whats-next.md).

## ✅ Критерии готовности

- [ ] Event loop (epoll) держит много соединений в неблокирующем режиме
- [ ] Конкурентность задействует ядра без гонок (TSan чисто)
- [ ] HTTP: парсинг, роутинг, статика, REST, корректные статусы, keep-alive
- [ ] Защита: path traversal, валидация, устойчивость к кривым запросам
- [ ] Браузер открывает страницы; curl-тесты проходят
- [ ] Бенчмарк показывает разумный req/sec; нет утечек fd/памяти
- [ ] Сборка и README
