# 09 · TCP echo-сервер 🖼️⭐

> 🎯 **Проект:** построй простейший работающий сервер — принимает соединение, читает данные,
> отправляет их обратно (echo). Это твой первый рабочий сетевой код.

> 🛠️ C/C++/Rust + POSIX-сокеты (Linux/WSL/Mac). Проверка — `nc`/`telnet`/свой клиент.

---

## 📖 MVP сервера: эхо

```
   ECHO-сервер — отправляет обратно то, что получил. идеальный MVP:
   • минимален (вся суть сервера: accept→read→write), но РАБОТАЕТ.
   • легко проверить (подключился, отправил «hello», получил «hello»).
   от него нарастишь всё остальное.
```

---

## ⭐ Milestones

```
   MVP (один клиент, blocking):
   1. socket() → bind(порт) → listen(). обработай ошибки каждого.
   2. accept() — дождись клиента, получи его fd.
   3. цикл: read() из клиента → write() то же обратно. пока клиент не закрыл.
   4. close() клиента.
   проверка: nc localhost ПОРТ → набери текст → получи его обратно.

   наращивание:
   5. ОБСЛУЖИВАТЬ КЛИЕНТОВ ПО ОЧЕРЕДИ: после close — снова accept (цикл вокруг accept).
   6. КОРРЕКТНОЕ ЧТЕНИЕ: read даёт СКОЛЬКО ЕСТЬ (не «всё сообщение») — читай в цикле до нужного/EOF.
   7. ОБРАБОТКА ОШИБОК/СИГНАЛОВ: клиент резко отключился, SIGPIPE при write в закрытый сокет.
   8. SO_REUSEADDR — чтобы перезапускать сервер сразу (иначе «Address already in use»).
   готово: стабильный echo-сервер, обслуживает клиентов по очереди, не падает на разрывах.
```

---

## ⭐⭐ Ключевые уроки и подводные камни

```
   🌊 TCP — ПОТОК БАЙТОВ, не сообщения:
      read() может вернуть ЧАСТЬ сообщения или НЕСКОЛЬКО склеенных. «отправил 100 байт» ≠ «read даст 100».
      → надо читать в ЦИКЛЕ и/или знать ГРАНИЦЫ сообщений (длина/разделитель — это протокол, модуль 12).
      это фундаментальный урок сетевого программирования. (связь: [TCP](../../Network/02-transport/09-tcp.md))

   🔁 BLOCKING вызовы: accept/read БЛОКИРУЮТ поток, пока не будет данных. поэтому «один клиент за раз»
      в этой версии: пока обслуживаешь одного, другой ждёт в accept. это решит event loop (модуль 10).

   ⚠️ ОБРАБОТКА ОШИБОК — в сети ВСЁ может сломаться: клиент отключился, сеть упала, write в закрытый
      сокет (SIGPIPE). проверяй возврат КАЖДОГО syscall, обрабатывай разрывы gracefully.

   🧹 РЕСУРСЫ — close() каждого fd. утечка fd → исчерпание лимита → сервер не принимает новых.
```

💡 ⭐⭐ Главный урок: **TCP — это поток байтов, а не сообщения**. Новички ждут, что одно
`send` = одно `recv` — нет. Нужно либо читать до разделителя/EOF, либо передавать длину. Это
определяет дизайн любого протокола поверх TCP (включая твой HTTP в модуле 12).

## 📖 Как проверить

```
   ✅ запусти сервер, подключись: nc localhost ПОРТ (или telnet). отправь текст → вернулся.
   ✅ несколько клиентов по очереди: один отключился → следующий обслуживается.
   ✅ разрыв: клиент резко закрыл (Ctrl-C) → сервер не падает, продолжает работать.
   ✅ перезапуск: останови и сразу запусти → нет «Address already in use» (SO_REUSEADDR).
   ✅ большой объём: отправь много данных → всё вернулось (проверь чтение в цикле).
   ✅ fd: длительная работа с многими подключениями → нет утечки дескрипторов (lsof/ss).
```

## 🌟 Расширения

```
   🥚 свой простой клиент (socket→connect→write→read) вместо nc.
   🐣 неблокирующие сокеты (fcntl O_NONBLOCK) — подготовка к event loop.
   🦅 простой текстовый протокол (команды с разделителем \n) — мост к key-value (Ур.3).
   🚀 логирование подключений/ошибок, graceful shutdown по сигналу.
```

## ⚠️ Ловушки

- ❌ Ждать, что один read = одно сообщение (TCP — поток, читай в цикле).
- ❌ Не проверять возврат syscalls (всё может failed).
- ❌ Забыть close() → утечка fd.
- ❌ Нет SO_REUSEADDR → «Address already in use» при перезапуске.
- ❌ Падение на SIGPIPE (write в закрытый сокет) — игнорируй/обработай сигнал.
- ❌ Не обрабатывать резкий разрыв клиента.

## ✅ Задачи

1. **MVP echo.** socket→bind→listen→accept→read/write→close. Проверь через nc.
2. **Цикл клиентов.** Обслуживай по очереди (accept в цикле). Перезапуск без ошибки (SO_REUSEADDR).
3. ⭐ **Корректное чтение.** Убедись, что большие/частичные данные читаются полностью (цикл read).
4. **Устойчивость.** Сервер не падает на разрывах/SIGPIPE. Проверь обрывами.
5. ⭐ **Свой клиент.** Напиши клиент вместо nc.
