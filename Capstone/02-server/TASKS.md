# ✅ Задачи уровня 2 · Свой сетевой сервер

> Строй сервер по шагам: echo → event loop → конкурентность → HTTP. Linux/WSL для epoll.
> Проверяй через nc/curl/браузер/бенчмарки.

---

## 🟢 Базовые

1. **Echo-сервер.** socket→bind→listen→accept→read/write. Проверь через nc. Обслуживай клиентов по очереди.
2. **Корректное чтение.** Убедись, что большие/частичные данные читаются полностью (TCP — поток).
3. **Устойчивость.** Сервер не падает на разрывах/SIGPIPE; перезапуск без «Address already in use».

## 🟡 Средние

4. **Event loop.** Перестрой на epoll + неблокирующий I-O. Держи много клиентов в одном потоке.
5. **Состояние соединений.** Буферы чтения/записи на каждый fd; корректные частичные read/write.
6. **HTTP MVP.** Отвечай валидным HTTP, открой в браузере. Разбери метод/путь, добавь роутинг + 404.
7. **Статика + безопасность.** Отдавай файлы с Content-Type; защити от path traversal; 400 на кривьё.

## 🔴 Со звёздочкой

8. ⭐ **Конкурентность.** Пул потоков + потокобезопасная очередь; общий ресурс под mutex/atomic.
   Прогони под TSan.
9. ⭐ **Нагрузка.** Бенчмарк (ab/wrk): req/sec на статике. Сравни blocking / event loop / +потоки.
10. ⭐ **REST + JSON.** Эндпоинты GET/POST с JSON — мостик к key-value хранилищу (Ур.3).
11. ⭐⭐ **HTTPS.** Добавь TLS через библиотеку (не свою крипту). Открой по https.
