# 🚀 Проект уровня 1 · Своя мини-библиотека контейнеров (mini-STL)

> Объедини проекты уровня в единую **библиотеку контейнеров и памяти** — твой маленький аналог
> STL. Это доказывает, что ты владеешь памятью, обобщениями и владением на уровне Senior.

> 🛠️ C++ (идеально) или C. С тестами, без утечек, с бенчмарками.

---

## 🎯 Что делаем

Собери в одну библиотеку (с единым README, сборкой, тестами) всё из уровня: контейнеры + умные
указатели + аллокаторы, которые работают вместе.

```
   mini-stl/
   ├── vector<T>            (модуль 03)
   ├── list<T> / stack / queue  (модуль 04)
   ├── hash_map<K,V>        (модуль 05)
   ├── unique_ptr / shared_ptr  (модуль 06)
   ├── arena / pool allocator   (модуль 07)
   ├── tests/               (тесты на всё)
   └── benchmarks/          (сравнение со std)
```

---

## 📋 Требования

### Контейнеры
- `vector<T>` — авторост, правило 5, итераторы (range-for работает).
- `list<T>`, `stack<T>`, `queue<T>` — на узлах/буфере.
- `hash_map<K,V>` — цепочки/адресация, ресайзинг по load factor.
- Все шаблонные, с тестами на краевые случаи.

### Память
- `unique_ptr`, `shared_ptr` (+ `weak_ptr`) — RAII, владение, подсчёт ссылок.
- `arena` и `pool` аллокаторы с выравниванием.

### Интеграция (главное!)
- Контейнеры используют твои умные указатели/аллокаторы, где уместно (например, узлы из pool).
- Всё работает вместе: `vector<shared_ptr<MyType>>`, `hash_map` с узлами из аллокатора.

### Качество
- Тесты на каждый компонент (вход → ожидание), включая краевые.
- **Чисто под ASan/Valgrind** — ноль утечек.
- Система сборки (CMake/Make), README (как собрать/запустить/тесты).

---

## 🌟 Бонусы

- 🥚 Бенчмарки vs std (vector/unordered_map/malloc) — таблица, насколько близко.
- 🐣 STL-совместимость: твои контейнеры работают с алгоритмами `<algorithm>` (через итераторы).
- 🦅 exception safety (strong guarantee) хотя бы для vector.
- 🚀 small-buffer optimization для vector/string; рандомизация хеша против hash-flooding.
