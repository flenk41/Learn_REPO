# 06 · Свой smart pointer 🖼️⭐⭐

> 🎯 **Проект:** реализуй умные указатели (`unique_ptr`, `shared_ptr`) — учит RAII, владению,
> move-семантике и подсчёту ссылок. Сердце современного C++.

> 🛠️ Язык: C++ (тема именно про него). Применяешь: [C++/RAII/smart pointers](../../Cpp/02-memory/11-smart-pointers.md),
> [⚙️ память процесса](../../ComputerScience/03-execution/16-process-layout.md).

---

## 📖 Что строим и зачем

```
   УМНЫЙ УКАЗАТЕЛЬ — обёртка над сырым указателем, которая АВТОМАТИЧЕСКИ освобождает память
   (RAII: ресурс привязан к времени жизни объекта). избавляет от ручного delete и утечек.

   • unique_ptr — ЕДИНОЛИЧНОЕ владение. нельзя копировать, можно перемещать. при разрушении — delete.
   • shared_ptr — РАЗДЕЛЯЕМОЕ владение. счётчик ссылок; delete когда счётчик дошёл до 0.

   реализовав их сам, ты ПОЙМЁШЬ RAII, move и владение глубже любого учебника.
```

🖼️
```
   RAII:  { unique_ptr<T> p(new T); ... }  → при выходе из {} деструктор p вызовет delete. АВТО.
   shared_ptr: [объект][счётчик=2] ← два shared_ptr владеют. один умер → счётчик=1. оба умерли → delete.
```

## ⭐ Milestones

```
   === unique_ptr ===
   1. ШАБЛОН класс с сырым указателем T*. конструктор берёт указатель, деструктор — delete.
   2. operator* и operator-> (разыменование как у указателя).
   3. ЗАПРЕТИ копирование (= delete копирующих) — единоличное владение.
   4. РАЗРЕШИ перемещение (move конструктор/присваивание): передать владение, обнулить источник.
   5. release()/reset()/get().

   === shared_ptr ===
   6. блок управления со СЧЁТЧИКОМ ссылок (выделить вместе с объектом или отдельно).
   7. копирование → счётчик++. разрушение → счётчик--; если 0 → delete объект и блок.
   8. move, reset, use_count.
   9. (⭐) weak_ptr — слабая ссылка, не владеет, не мешает удалению (решает циклы).
   готово: твои unique/shared работают как стандартные в программе, без утечек, тесты проходят.
```

---

## ⭐⭐ Ключевые решения и уроки

```
   🔑 RAII — главная идея: ресурс (память) привязан к ВРЕМЕНИ ЖИЗНИ объекта. деструктор освобождает
      автоматически и детерминированно (даже при исключении). это «C++-способ» не иметь утечек.

   🚫 unique_ptr НЕ КОПИРУЕТСЯ — иначе два владельца → двойное delete. но ПЕРЕМЕЩАЕТСЯ (move):
      передать владение, источник обнулить. это учит move-семантике вживую.
      (связь: [move в C++](../../Cpp/02-memory/12-move-semantics.md))

   🔢 shared_ptr — ПОДСЧЁТ ССЫЛОК: счётчик++/-- при копировании/разрушении. delete при 0.
      ⚠️ счётчик должен быть ОБЩИМ (на куче), не в каждой копии. ⚠️ потокобезопасность счётчика
         (atomic) — для многопотока.

   ♻️ ЦИКЛЫ shared_ptr — A держит B, B держит A → счётчики никогда не 0 → УТЕЧКА. решение: weak_ptr
      (не увеличивает счётчик). это важный урок о пределах подсчёта ссылок.

   ⚠️ КРАЕВЫЕ: nullptr, self-move, self-reset, исключение в конструкторе.
```

💡 ⭐⭐ Уроки бесценны: **RAII** (автоосвобождение через время жизни), **move** (передача владения
вместо копии), **подсчёт ссылок** (и его ловушка — циклы). Это ядро современного C++ и Rust
([ownership](../../Rust/02-memory/08-ownership.md) — те же идеи, но в системе типов). Поняв это,
ты пишешь без утечек естественно.

## 📖 Как проверить

```
   ✅ unique_ptr: при выходе из области — объект удалён (деструктор сработал, проверь логом/счётчиком).
   ✅ unique нельзя копировать (не компилируется), можно move (владение перешло, источник null).
   ✅ shared_ptr: копии увеличивают use_count; разрушение уменьшает; delete при 0 (ровно один раз).
   ✅ цикл shared (без weak) → утечка (ASan покажет); с weak_ptr → нет утечки.
   ✅ память: ASan/Valgrind чисто; объект удаляется РОВНО один раз.
```

## 🌟 Расширения

```
   🥚 поддержка кастомного делитера (как у std).
   🐣 make_unique/make_shared (фабрики).
   🦅 weak_ptr с lock() (получить shared, если объект жив).
   🚀 потокобезопасный счётчик (atomic) — для многопотока (мост к Ур.2).
```

## ⚠️ Ловушки

- ❌ Разрешить копирование unique_ptr → двойное delete.
- ❌ Счётчик shared не общий (копия в каждом) → неверный подсчёт.
- ❌ Циклы shared_ptr → утечка (нужен weak_ptr).
- ❌ Забыть обнулить источник при move → двойное освобождение.
- ❌ Не обработать nullptr/self-assignment.
- ❌ Непотокобезопасный счётчик в многопотоке (atomic).
