# 04 · Свой связный список / стек / очередь 🖼️⭐

> 🎯 **Проект:** реализуй связный список и на его основе стек и очередь — учит работе с указателями,
> узлами и сравнению с массивом по локальности.

> 🛠️ Применяешь: [C/указатели](../../C/02-memory/09-pointers.md), [Алгоритмы/списки](../../Algorithms/01-structures/04-linked-lists.md),
> [⚙️ локальность](../../ComputerScience/01-hardware/07-ram-hierarchy.md).

---

## 📖 Что строим

```
   СВЯЗНЫЙ СПИСОК — цепочка узлов, каждый хранит данные + указатель на следующий (и/или предыдущий).
   в отличие от массива — узлы РАЗБРОСАНЫ по куче, связаны указателями.

   узел: [ data | next ] ──► [ data | next ] ──► [ data | next ] ──► nullptr
   на его базе:
   • СТЕК (LIFO) — push/pop с одного конца.
   • ОЧЕРЕДЬ (FIFO) — enqueue с одного конца, dequeue с другого.
```

🖼️
```
   список:  head ──► [1|•]──► [2|•]──► [3|null]
   вставка в начало: O(1) (новый узел → next=head → head=новый).
   вставка/доступ по индексу: O(n) (надо пройти от head).
   ↕ сравни с массивом: доступ O(1), но вставка в середину O(n) + сдвиг.
```

## ⭐ Milestones

```
   MVP: односвязный список int — push_front, обход/печать.
   1. УЗЕЛ: структура { data; next; }. список хранит head (и tail для O(1) в конец/очередь).
   2. push_front / push_back, обход, размер.
   3. удаление (front/по значению), поиск, очистка (освободить ВСЕ узлы — частая утечка!).
   4. ШАБЛОН list<T>.
   5. СТЕК на списке: push/pop/top (или на своём vector — сравни).
   6. ОЧЕРЕДЬ на списке: enqueue/dequeue (нужен head и tail для O(1) с обоих концов).
   7. (⭐) двусвязный список (prev+next) — удаление узла за O(1) при наличии указателя на него.
   готово: список/стек/очередь работают, без утечек, тесты проходят.
```

---

## ⭐⭐ Ключевые решения и уроки

```
   🧠 ВЛАДЕНИЕ ПАМЯТЬЮ — каждый узел в куче (new/malloc). освобождай КАЖДЫЙ при удалении/в деструкторе.
      классическая утечка: удалил узел, но не его data, или забыл пройти весь список при clear.
      (C++: умные указатели упрощают — но сначала сделай вручную, чтобы понять.)

   📍 РАБОТА С УКАЗАТЕЛЯМИ — главный навык:
      • вставка/удаление = аккуратное переподключение next/prev. рисуй на бумаге!
      • краевые: пустой список, один элемент, удаление head/tail.
      • «потерянный» указатель: переподключил неправильно → утечка или crash.

   ⚡ ЛОКАЛЬНОСТЬ vs МАССИВ — важнейший урок:
      список «прыгает» по памяти (узлы разбросаны) → промахи кэша → обход в РАЗЫ медленнее массива
      при той же O(n). (связь: [кэш](../../ComputerScience/01-hardware/08-cache.md))
      → список хорош для частых вставок/удалений в середину; для обхода — массив/vector.
```

💡 ⭐⭐ Два урока: (1) **ручное владение узлами** — дисциплина освобождения, переподключения
указателей (рисуй схемы!); (2) **список медленнее массива на обходе** из-за локальности — то, что
Big-O не показывает, а железо определяет. Реализовав оба, ты выбираешь структуру осознанно.

## 📖 Как проверить

```
   ✅ push/pop с обоих концов → правильный порядок (стек LIFO, очередь FIFO).
   ✅ удаление: head, tail, середина, единственный, из пустого.
   ✅ память: ASan/Valgrind — НЕТ утечек (особенно проверь clear/деструктор).
   ✅ (бонус) бенчмарк: обход списка vs массива на 10млн — во сколько раз медленнее? объясни.
```

---

## 🌟 Расширения

```
   🥚 двусвязный + циклический список.
   🐣 итераторы (begin/end) для range-for.
   🦅 intrusive list (узел внутри объекта) — как в ядрах ОС.
   🚀 lock-free очередь (продвинутое, для многопотока) — мост к конкурентности (Ур.2).
```

---

## ⚠️ Ловушки

- ❌ Утечки: забыть освободить узлы при clear/в деструкторе.
- ❌ Неправильное переподключение указателей при вставке/удалении (рисуй!).
- ❌ Не обработать краевые: пустой, один элемент, удаление head/tail.
- ❌ Очередь без tail-указателя → enqueue O(n) вместо O(1).
- ❌ Ожидать от списка скорости массива на обходе (локальность!).
