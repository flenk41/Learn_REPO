# ✅ Задачи уровня 3 · Своё хранилище / БД

> Строй хранилище по шагам: in-memory KV → протокол → персистентность → индексы → транзакции.
> Используй свои структуры данных (Ур.1) и сервер (Ур.2).

---

## 🟢 Базовые

1. **In-memory KV.** SET/GET/DEL/EXISTS на своей хеш-таблице. Тесты на перезапись/отсутствие.
2. **Типы и TTL.** Добавь список/счётчик и истечение ключей.
3. **Протокол.** Текстовый протокол команд/ответов; парсер с framing (границы в потоке TCP).

## 🟡 Средние

4. **По сети.** Подключи хранилище к своему серверу (Ур.2); команды по сети через nc/клиент.
5. **WAL.** Журнал изменений + восстановление при старте. Хранилище переживает рестарт.
6. **fsync trade-off.** Замерь скорость при разных стратегиях fsync. Объясни долговечность↔скорость.
7. **Snapshot + компакция.** Дамп состояния, загрузка, компакция разросшегося лога.

## 🔴 Со звёздочкой

8. ⭐ **На диск + индекс.** Данные на диске (> RAM); упрощённый индекс (SSTable или B-tree).
9. ⭐ **LSM-поток.** memtable → SSTable → чтение по слоям; compaction.
10. ⭐ **Транзакции.** MULTI/EXEC атомарно; откат при ошибке; потокобезопасность (TSan).
11. ⭐⭐ **Изоляция.** Покажи аномалию без изоляции, реши блокировками; (бонус) простой MVCC.

---

## 🎯 Критерии «сделано»

- [ ] In-memory KV с типами и TTL на своих структурах
- [ ] Протокол с корректным framing, доступ по сети
- [ ] Персистентность (WAL + snapshot), переживает рестарт
- [ ] Понимаю trade-off долговечность/скорость (fsync)
- [ ] (⭐) индекс на диске (B-tree/LSM), транзакции/изоляция
