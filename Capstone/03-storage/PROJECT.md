# 🚀 Проект уровня 3 · Своя мини-БД / Redis-подобное хранилище

> Построй настоящее хранилище данных: сетевое, с командами, персистентностью и (бонус) индексами/
> транзакциями. Синтез структур данных, сервера, I-O и надёжности — серьёзная senior-работа.

> 🛠️ C/C++/Rust. Выбери фокус: Redis-подобное (in-memory + персистентность) ИЛИ мини-БД (на диске + индексы).

---

## 🎯 Что делаем

Выбери одно направление (или сделай гибрид):

**A. Redis-подобное** (in-memory, упор на скорость и типы): KV + типы данных + TTL + сетевой
протокол + персистентность (WAL/snapshot) + транзакции.

**B. Мини-БД** (на диске, упор на хранение больше RAM): storage engine (B-tree/LSM) + индексы +
WAL + транзакции + простой запрос по ключу/диапазону.

```
   структуры данных (Ур.1) + сервер (Ур.2) + персистентность + индексы/транзакции = твоя БД
```

---

## 📋 Требования (направление A — Redis-подобное)

- **Ядро:** KV на своей хеш-таблице; типы (строки, списки, множества, счётчики); TTL.
- **Сеть:** доступ через свой TCP-сервер (Ур.2) + текстовый протокол с framing.
- **Персистентность:** WAL (с проигрыванием) + snapshot + компакция; переживает рестарт.
- **Конкурентность:** много клиентов без гонок (TSan чисто).
- **Транзакции (бонус):** MULTI/EXEC атомарно.

## 📋 Требования (направление B — мини-БД)

- **Storage engine:** данные на диске; индекс (упрощённый B-tree или LSM/SSTable).
- **Операции:** put/get/delete/range по ключу; работает на данных > RAM.
- **Надёжность:** WAL + восстановление; (бонус) транзакции с изоляцией.
- **Эффективность:** минимум дисковых обращений; кэш страниц.

---

## 🌟 Бонусы

- 🥚 CLI-клиент и/или REST-доступ (через свой HTTP-сервер).
- 🐣 Бенчмарк (SET/GET throughput, latency) — сравни с настоящим Redis/SQLite.
- 🦅 Pub/sub (A) или диапазонные запросы/вторичные индексы (B).
- 🚀 Репликация (master→replica проигрывает WAL) — основы распределённости.
- 🚀 Уровни изоляции / MVCC.

---

## ✅ Критерии готовности

- [ ] Сетевое хранилище с командами через свой протокол
- [ ] Несколько типов/операций на своих структурах данных
- [ ] Персистентность: переживает рестарт (WAL/snapshot или storage engine)
- [ ] Понятен и применён trade-off долговечность/скорость (fsync)
- [ ] Конкурентный доступ без гонок (TSan)
- [ ] (направление B) индекс на диске, данные > RAM
- [ ] Тесты, бенчмарк, сборка, README
