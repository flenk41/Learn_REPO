# 20 · Вычислитель (evaluator) 🖼️⭐⭐

> 🎯 **Проект:** заставь язык РАБОТАТЬ — обходи AST и вычисляй его. Финал базового интерпретатора:
> твой язык реально исполняет программы.

> 🛠️ Применяешь: рекурсию (обход дерева), структуры данных (окружение переменных), предыдущие модули.

---

## 📖 Вычисление = обход дерева

```
   ВЫЧИСЛИТЕЛЬ (tree-walking interpreter) РЕКУРСИВНО обходит AST и считает значение каждого узла:
   • число → само значение.
   • бинарная операция → вычислить left, вычислить right, применить оператор.
   • переменная → взять значение из ОКРУЖЕНИЯ.
   • if → вычислить условие → выполнить нужную ветку.
   рекурсия по дереву = вычисление программы. просто и красиво.
```

🖼️
```
   eval(  [+]      )  →  eval([2]) + eval([*])  →  2 + (eval([3])*eval([4]))  →  2 + 12  →  14
          /   \
        [2]   [*]
             /   \
           [3]   [4]
   обход снизу вверх (post-order): сначала дети, потом операция над ними.
```

💡 ⭐⭐ Это завершает конвейер: текст→лексер→токены→парсер→AST→**eval→результат**. Твой язык
работает! Tree-walking интерпретатор (обход AST) — самый простой способ исполнения; так начинали
многие языки. Применение рекурсии по дереву в чистейшем виде.

## ⭐ Окружение: переменные и области видимости

```
   ОКРУЖЕНИЕ (environment) — хранилище переменных: имя → значение (твоя хеш-таблица из Ур.1!).
   • let x = 5  →  записать x=5 в окружение.
   • использование x  →  прочитать из окружения (нет → ошибка «undefined variable x»).

   ОБЛАСТИ ВИДИМОСТИ (scopes):
   • блок { } / функция создаёт ВЛОЖЕННОЕ окружение.
   • поиск переменной: в текущем окружении → если нет, в родительском → ... (цепочка окружений).
   • это объясняет, как работают локальные/глобальные переменные в любом языке!
```

💡 ⭐ Реализовав окружения, ты понимаешь области видимости изнутри — почему локальная «перекрывает»
глобальную, что такое замыкание (функция + её окружение). Это тот же [стек вызовов/области из ⚙️ CS](../../ComputerScience/03-execution/17-call-stack.md),
но на уровне семантики языка.

## ⭐⭐ Milestones

```
   MVP: вычислять арифметические выражения (калькулятор) — обход AST.
   1. eval(node): switch по типу узла → число / бинарная операция (рекурсивно left/right).
   2. ПЕРЕМЕННЫЕ: окружение (map имя→значение). let записывает, использование читает.
   3. УПРАВЛЕНИЕ: if/else (вычислить условие, выбрать ветку), while (цикл с проверкой).
   4. ОБЛАСТИ: вложенные окружения для блоков; цепочка поиска переменных.
   5. ФУНКЦИИ: объявление (имя+параметры+тело), вызов (новое окружение, привязать аргументы, eval тело, вернуть).
      (⭐) рекурсивные функции, замыкания.
   6. ВСТРОЕННЫЕ: print, базовые операции; типы (числа, строки, булевы).
   7. ОШИБКИ ВРЕМЕНИ ВЫПОЛНЕНИЯ: деление на ноль, неопределённая переменная, неверные типы → понятно.
   8. REPL: цикл «ввёл выражение → вычислил → показал» (как python).
   готово: твой язык исполняет программы с переменными, условиями, циклами, функциями.
```

## 📖 Tree-walking vs компиляция (мост к модулю 21)

```
   ОБХОД AST (этот модуль) — просто, но МЕДЛЕННО (рекурсия по дереву на каждое исполнение).
   для скорости языки КОМПИЛИРУЮТ AST в БАЙТКОД и исполняют на виртуальной машине (модуль 21) или
   в машинный код (как [⚙️ CS компиляция](../../ComputerScience/02-toolchain/11-c-to-asm.md)).
   но для понимания и многих задач tree-walking достаточно (так работает, например, Ruby исторически).
```

## ⚠️ Ловушки

- ❌ Не обрабатывать ошибки времени выполнения (деление на ноль, undefined var) → крэш вместо сообщения.
- ❌ Игнорировать области видимости (всё в одном окружении) → переменные «протекают».
- ❌ Функции без нового окружения → параметры портят внешние переменные.
- ❌ Забыть про типы (складывать строку и число?) → решить поведение явно.
- ❌ Не сделать REPL — труднее играть с языком.
- ❌ Гнаться за скоростью на tree-walking (для скорости — байткод, модуль 21).

## ✅ Задачи

1. **Калькулятор.** eval AST арифметики. "2+3*(4-1)" → 11. Сделай REPL.
2. **Переменные.** Окружение; let и использование. Ошибка на undefined.
3. **Управление.** if/else и while работают. Напиши программу (например, факториал циклом).
4. ⭐ **Функции.** Объявление и вызов с новым окружением. Рекурсия (факториал рекурсивно).
5. ⭐ **Ошибки + типы.** Деление на ноль, неверные типы → понятные runtime-ошибки.
