# 🚀 Финальный проект · Свой язык программирования

> Венец капстоунов: построй работающий интерпретатор своего мини-языка — от текста до исполнения.
> Это синтез рекурсии, структур данных, архитектуры и понимания тулчейна. Самый «вау»-проект.

> 🛠️ Любой язык реализации. Цель — рабочий язык с переменными, управлением, функциями.

---

## 🎯 Что делаем

Полный интерпретатор мини-языка: лексер → парсер → AST → вычислитель (+ бонус: байткод-VM).
Язык должен исполнять реальные программки (факториал, FizzBuzz, простые алгоритмы).

```
   исходник твоего языка ──► лексер ──► парсер/AST ──► eval ──► результат
                                                    (+ опц.: компилятор → байткод → VM)
```

---

## 📋 Требования

### Язык (минимум)
- Арифметика с приоритетами и скобками.
- Переменные (объявление + использование, области видимости).
- Управление: if/else, while (или for).
- Функции: объявление, вызов, параметры, возврат, **рекурсия**.
- Типы: числа + хотя бы строки/булевы. Встроенный print.

### Реализация (конвейер)
- **Лексер:** текст → токены, с позициями и понятными ошибками.
- **Парсер:** recursive descent → AST, правильные приоритеты, синтаксические ошибки.
- **Вычислитель:** обход AST, окружения/области, функции, runtime-ошибки.
- **REPL** + запуск из файла.

### Качество
- Тесты: набор программ → ожидаемый вывод (включая ошибки).
- Понятные сообщения об ошибках (синтаксис и рантайм) с позицией.
- README: грамматика языка, как запустить, примеры программ.

### Демо-программы (должны работать)
- Факториал (рекурсивно и циклом), FizzBuzz, числа Фибоначчи, простая сортировка.

## 🌟 Бонусы

- 🥚 Больше возможностей: массивы/списки, строки-операции, ещё встроенные функции.
- 🐣 Замыкания (функции, захватывающие окружение).
- 🦅 **Байткод-VM** (модуль 21): компиляция AST в байткод + стековая VM. Бенчмарк vs tree-walking.
- 🚀 Система типов / проверка типов перед исполнением.
- 🚀 Компиляция в нативный код (через LLVM или свой кодген) — вершина.
