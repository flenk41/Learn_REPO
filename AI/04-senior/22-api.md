## 📖 Что такое API ИИ

До сих пор ты работал с ИИ через чат (браузер). **API** (Application Programming Interface)
позволяет обращаться к ИИ **из кода** — встраивать его в программы, сайты, автоматизацию.

🖼️
```
   Чат:  человек → браузер → ИИ → ответ человеку
   API:  твоя программа → запрос к ИИ → ответ в программу → программа что-то с ним делает
```

💡 Это та же концепция API, что в [языковых треках](../../Cpp/03b-projects-api/README.md)
(раздел «Проекты и API»): ты обращаешься к чужому сервису по сети и получаешь ответ. Здесь
сервис — это ИИ.

---

## ⭐ Как устроен запрос к ИИ-API

Базовый запрос содержит:
```
- модель        (какую версию использовать)
- сообщения     (системный промпт + диалог)
- параметры     (температура, лимит токенов и т.д.)
```

Пример (псевдокод, идея одинакова для всех языков):
```python
# Python, для примера
response = ai_client.chat(
    model="...",
    messages=[
        {"role": "system", "content": "Ты помощник-переводчик."},
        {"role": "user", "content": "Переведи 'привет' на японский"},
    ],
    temperature=0.7,
)
print(response.text)
```

🖼️ Роли сообщений — это то, что ты изучал:
```
   system → системный промпт (модуль 10): постоянная роль
   user   → твои сообщения
   assistant → ответы ИИ (для контекста диалога)
```

💡 Через API ты **программно** собираешь контекст (модуль 08): сам решаешь, что положить в
`messages`. Вся история диалога — это массив сообщений, который ты передаёшь каждый раз
(ведь у модели нет памяти между запросами!).

## ⭐ Ключевые параметры

| Параметр | Что делает |
|----------|-----------|
| **temperature** | случайность: 0 = строго/предсказуемо, 1+ = креативно/разнообразно |
| **max_tokens** | лимит длины ответа |
| **model** | какая модель (быстрая/мощная) |
| **system** | системный промпт (роль) |
| **stop** | последовательности, на которых остановиться |

```
   temperature 0.0  →  факты, код, точность (детерминированно)
   temperature 0.7  →  баланс (по умолчанию)
   temperature 1.0+ →  творчество, разнообразие идей
```

💡 `temperature` — важная ручка: для фактов и кода ставь низкую, для брейншторма — высокую.
В чате ты этим не управляешь, а через API — да.

## ⭐ Зачем API: что становится возможным

```
✅ Встроить ИИ в свой сайт/приложение (чат-бот, помощник).
✅ Обработать тысячи элементов автоматически (не вручную по одному).
✅ Построить свои инструменты: RAG-системы (модуль 20), агентов (модуль 21).
✅ Интегрировать ИИ в рабочие процессы (письма, отчёты, модерация).
✅ Контролировать параметры (temperature, модель) точно.
```

💡 Чат — для людей. API — для автоматизации и продуктов. Всё, что ты изучил про промпты,
работает и через API — просто промпт теперь формирует **программа**.

---

## ⚠️ Важное про API

```
⚠️ Стоимость: API платный, считается по ТОКЕНАМ (вход + выход). Длинные
   контексты = дороже. Следи за расходом.
⚠️ Ключи: API-ключ — это секрет. Никогда не публикуй его в коде/git
   (храни в переменных окружения — как учили в языковых треках!).
⚠️ Лимиты: есть ограничения на частоту запросов (rate limits).
⚠️ Без памяти: модель не помнит прошлые запросы — историю передаёшь сам в messages.
```

> 💡 Связь с языковыми треками: работа с ИИ-API — это ровно тот навык «работы с веб-API»,
> что в разделах «Проекты и API» (reqwest в Rust, requests в Python, cpr в C++, libcurl в
> C). ИИ-API — частный случай REST-API с JSON.

## 📖 Официальные SDK

У всех провайдеров есть библиотеки для популярных языков:
```
- Python:     официальные SDK (самый популярный язык для ИИ)
- JavaScript: для веба и Node
- другие:     через HTTP-запросы напрямую (любой язык)
```

💡 Начни с Python, если хочешь экспериментировать с ИИ-API — там самая богатая экосистема
(связь с твоим Python-треком).
