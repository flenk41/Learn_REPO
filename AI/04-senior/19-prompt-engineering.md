# 19 · Промпт-инжиниринг как дисциплина ⭐

> 🎯 **Цель блока:** перейти от отдельных приёмов к системному мышлению. Промпт-инжиниринг —
> это инженерная дисциплина со своими паттернами и методологией.

---

## 📖 От интуиции к инженерии

На предыдущих уровнях ты освоил приёмы. Эксперт отличается тем, что **системно проектирует**
промпты: знает паттерны, комбинирует их осознанно, тестирует и улучшает по данным (а не «на
глаз»).

🖼️
```
   Новичок:  пишет промпт → не вышло → пишет другой наугад
   Эксперт:  анализирует задачу → выбирает паттерны → тестирует → измеряет → улучшает
```

---

## ⭐ Сводка паттернов (твой арсенал)

Всё, что ты изучил, — это паттерны. Держи их как чек-лист при проектировании промпта:

| Паттерн | Когда применять | Модуль |
|---------|-----------------|--------|
| Структура (роль/задача/контекст/формат) | всегда | 03 |
| Few-shot (примеры) | нужен стиль/формат/стабильность | 06, 14 |
| Chain-of-thought (по шагам) | сложная логика, рассуждение | 13 |
| Декомпозиция | большие задачи | 16 |
| Роли и персоны | экспертный взгляд | 15 |
| Системный промпт | постоянное поведение | 10 |
| Заземление на данные | работа с фактами, против галлюцинаций | 12, 17 |
| Самопроверка / критика | повышение качества | 07, 13 |
| Уточняющие вопросы | сбор контекста | 07, 11 |

💡 Мастерство — не в знании одного «секретного» промпта, а в умении **выбрать и
скомбинировать** нужные паттерны под конкретную задачу.

---

## ⭐ Продвинутые паттерны

### Self-consistency (несколько попыток → консенсус)
```
"Реши задачу 3 раза независимо, затем выбери ответ, который встречается чаще
всего / наиболее обоснован."
```
💡 Для важных задач несколько независимых решений надёжнее одного.

### Step-back (сначала общий принцип)
```
"Прежде чем решать конкретную задачу, сформулируй общий принцип/подход. Потом
примени его."
```
💡 «Отступить на шаг» к общей идее улучшает решение конкретики.

### Rubric (явные критерии оценки)
```
"Оцени текст по критериям: ясность (1-10), убедительность (1-10), грамотность
(1-10). По каждому — обоснование. Затем улучши слабейший."
```

### Meta-prompting (ИИ улучшает промпт)
```
"Вот моя задача и текущий промпт. Предложи улучшенную версию промпта и объясни,
что изменил и почему."
```

## ⭐ Структурирование сложных промптов

Для серьёзных промптов используй чёткую разметку (как код):

```
# КОНТЕКСТ
[фон, данные]

# ЗАДАЧА
[что сделать]

# ТРЕБОВАНИЯ
- [требование 1]
- [требование 2]

# ПРИМЕРЫ
[few-shot]

# ФОРМАТ ВЫВОДА
[структура ответа]

# ВАЖНО
[критичные правила, что нельзя]
```

💡 Чёткая разметка (заголовки, секции) помогает модели не смешивать части. Это «инженерный»
промпт, который легко читать, тестировать и поддерживать.

---

## 📖 Промпт как код: версионирование и поддержка

В профессиональной работе промпты:
- **хранят** (как код в репозитории);
- **версионируют** (помечают изменения);
- **тестируют** (на наборе случаев — модуль 23);
- **переиспользуют** (шаблоны, библиотеки).

💡 Если промпт критичен для работы/продукта — относись к нему как к коду: документируй,
тестируй, улучшай итеративно. Это и есть дисциплина.

## ✅ Задачи

1. **Паттерн-аудит** — возьми сложную задачу, осознанно выбери и перечисли, какие паттерны
   применишь и почему.
2. **Self-consistency** — реши важную задачу 3 раза, сравни, выбери лучшее.
3. **Step-back** — реши задачу, начав с общего принципа.
4. **Rubric** — оцени и улучши текст по явным критериям.
5. **Meta-prompting** — попроси ИИ улучшить твой промпт, сравни результаты.
6. ⭐ **Инженерный промпт** — собери сложный промпт с полной разметкой (секции) под реальную
   задачу.

3. Что такое self-consistency и step-back?
4. Зачем чёткая разметка в сложных промптах?
5. Почему промпт стоит воспринимать «как код»?
