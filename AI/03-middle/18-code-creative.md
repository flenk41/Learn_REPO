## 💻 Часть 1: ИИ для кода

> 💡 Связь с остальным курсом: если ты проходил треки C/C++/Python/Rust — ИИ станет твоим
> напарником в программировании. Но он ошибается, поэтому **код всегда проверяй и тестируй**.

### Как просить код
```
"Напиши функцию на Python, которая [что делает].
Вход: [описание]. Выход: [описание].
Добавь комментарии и обработку ошибок.
Приведи пример использования."
```

💡 Указывай: **язык, задачу, вход/выход, требования** (стиль, обработка ошибок, комментарии).
Чем конкретнее — тем меньше доработок.

### Отладка
```
"Вот код и ошибка. Объясни причину и исправь:
КОД: [код]
ОШИБКА: [текст ошибки]"

"Этот код работает неправильно: ожидаю X, получаю Y. Найди баг."
```

### Объяснение и обучение
```
"Объясни, что делает этот код, строка за строкой."
"Перепиши этот код чище и объясни улучшения."
"Какие здесь есть потенциальные проблемы (производительность, безопасность)?"
```

### ⚠️ Правила работы с кодом от ИИ
```
⚠️ ВСЕГДА тестируй код — ИИ ошибается и «галлюцинирует» функции/библиотеки.
⚠️ Не вставляй секреты/пароли в промпт.
⚠️ Понимай код, прежде чем использовать — не копируй вслепую.
⚠️ Проверяй, существуют ли упомянутые библиотеки и методы.
```

---

## 📖 Приёмы для кода

```
"Используй только стандартную библиотеку, без внешних зависимостей."
"Напиши с тестами (unit-тесты)."
"Объясни решение, прежде чем писать код."     ← chain-of-thought для кода
"Предложи 2 подхода и сравни их."
"Отрефактори этот код по принципам чистого кода."
```

💡 Декомпозиция (модуль 16) отлично работает для кода: «сначала спроектируй структуру, потом
реализуй по частям».

## 🎨 Часть 2: ИИ для творчества

### Генерация идей
```
"Придумай 20 идей для [чего]. От безопасных до самых смелых."
"Используй технику мозгового штурма: не фильтруй, давай количество."
"Скрести две несвязанные концепции: [A] и [B] — что получится?"
```

💡 Для творчества проси **количество и разнообразие**, потом выбирай. Не жди один идеальный
вариант сразу.

### Тексты и истории
```
"Напиши короткий рассказ. Жанр: [жанр]. Герой: [кто]. Конфликт: [какой].
Тон: [настроение]. Объём: [сколько]."

"Продолжи эту историю в том же стиле: [начало]"
"Перепиши этот абзац в стиле [автор/настроение]."
```

### Управление стилем
```
"Сделай текст более живым: добавь метафоры и конкретные образы."
"Тот же смысл, но с юмором."
"Пиши короткими рублеными фразами для напряжения."
```

### Преодоление «творческого блока»
```
"Я застрял с [проект]. Задай мне провокационные вопросы, чтобы расшевелить идеи."
"Предложи 5 неожиданных поворотов для моего сюжета."
```

## ⭐ Творчество: итерации и комбинирование

💡 Творческая работа с ИИ — это диалог (модуль 07): генерируй много → выбирай лучшее →
комбинируй → дорабатывай. ИИ — партнёр по брейнштормингу, а не автомат «нажми и получи
шедевр». Финальный вкус и отбор — за тобой.

> ⚠️ Помни про авторство и оригинальность: ИИ-тексты стоит дорабатывать и пропускать через
> себя, а не публиковать «как есть». Этику разберём в Senior-блоке.

## 🧪 Эксперименты

1. **Код:** попроси функцию, протестируй её — найди, работает ли (часто есть мелкие баги).
2. **Идеи:** попроси 20 идей на тему, оцени, как растёт смелость к концу списка.
3. **Стиль:** перепиши один абзац в 3 разных стилях.

---

## ✅ Задачи

1. **Код по спецификации** — закажи функцию с чёткими требованиями, протестируй.
2. **Отладка** — дай ИИ сломанный код с ошибкой, проверь, найдёт ли причину.
3. **Объяснение кода** — попроси объяснить незнакомый код построчно.
4. **20 идей** — мозговой штурм на свою тему, отбери лучшие 3.
5. **Рассказ по параметрам** — закажи историю с заданными жанром/героем/конфликтом.
6. **Стилизация** — перепиши текст в 3 стилях.
7. ⭐ **Творческий проект** — доведи творческую работу (текст/идею/концепцию) до результата
   через итерации с ИИ.
