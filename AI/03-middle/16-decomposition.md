# 16 · Декомпозиция задач 🖼️⭐

> 🎯 **Цель блока:** научиться разбивать большую задачу на части, с которыми ИИ
> справляется надёжно. Это ключевой навык для серьёзной работы.

---

## 📖 Почему большие задачи проваливаются

Если дать ИИ огромную задачу одним промптом, он часто:
- упускает детали;
- делает поверхностно;
- путается в сложной логике;
- упирается в лимит длины ответа.

```
❌ "Напиши мне полный бизнес-план кофейни на 20 страниц со всеми расчётами,
   маркетингом, юридическими аспектами и финансовой моделью"
   → получишь поверхностную воду по всем пунктам
```

🎯 Решение — **декомпозиция**: разбить на подзадачи и решать по очереди. Прямая параллель с
программированием, где большую программу делят на функции и модули.

---

## ⭐ Принцип: одна задача — один фокус

```
✅ Вместо одного огромного промпта:
   1. "Опиши концепцию кофейни и целевую аудиторию"
   2. "Теперь составь меню и ценообразование"
   3. "Теперь маркетинговый план на первые 3 месяца"
   4. "Теперь примерная финансовая модель"
   5. "Собери всё в единый документ"
```

🖼️
```
   Большая задача
   ├── подзадача 1  →  качественный результат
   ├── подзадача 2  →  качественный результат
   ├── подзадача 3  →  качественный результат
   └── сборка       →  целостный итог
```

💡 Каждая подзадача получает полное «внимание» ИИ → выше качество. А контекст (модуль 11)
связывает части в целое.

---

## ⭐ Как разбивать: попроси ИИ помочь

Не знаешь, как разбить? Спроси у ИИ:

```
"Я хочу [большая задача]. Разбей её на логические подзадачи, которые мы будем
решать по очереди. Пока не выполняй — только составь план."
```

💡 ИИ сам предложит декомпозицию. Ты корректируешь план, затем идёте по шагам. Это
объединяет декомпозицию с паттерном «план → исполнение» (модуль 11).

---

## ⭐ Типы декомпозиции

### По этапам (последовательно)
```
исследование → структура → черновик → редактура → финал
```

### По частям (параллельно)
```
статья = введение + раздел 1 + раздел 2 + ... + заключение
```

### По аспектам
```
анализ решения = финансы + риски + сроки + ресурсы + альтернативы
```

### По уровням детализации
```
сначала общая картина → потом детали каждого блока
```

---

## ⭐ Декомпозиция сложного рассуждения

Для аналитических задач разбивай **сам процесс мышления**:

```
"Помоги решить, какой ноутбук купить. Действуем по шагам:
1. Сначала задай мне вопросы о моих потребностях и бюджете.
2. На основе ответов составь критерии выбора.
3. Предложи 3 варианта с обоснованием.
4. Помоги сравнить их по критериям.
5. Дай финальную рекомендацию."
```

💡 Ты задаёшь **структуру рассуждения**, а ИИ наполняет её. Сочетание декомпозиции +
chain-of-thought (модуль 13) даёт самые качественные результаты на сложных задачах.

---

## 📖 Когда декомпозировать

| Декомпозируй | Можно одним промптом |
|--------------|----------------------|
| большой результат (документ, план) | короткий ответ |
| много аспектов/частей | одна простая задача |
| сложная многоступенчатая логика | прямой вопрос |
| важно высокое качество каждой части | черновик «на скорую руку» |

---

## 🧪 Эксперимент: монолит vs декомпозиция

1. Дай ИИ большую задачу одним промптом.
2. Реши ту же задачу декомпозицией (план → части → сборка).
3. Сравни глубину и качество. Декомпозиция почти всегда выигрывает на сложном.

---

## ✅ Задачи

1. **Разбей задачу** — возьми большую задачу, попроси ИИ предложить декомпозицию.
2. **Монолит vs части** — выполни эксперимент выше.
3. **По этапам** — проведи задачу через исследование → черновик → финал.
4. **Структура рассуждения** — реши решение-выбор по заданным тобой шагам.
5. **Сборка** — реши задачу по частям, затем собери в единый целостный результат.
6. ⭐ **Большой проект** — возьми реальную крупную задачу (план, документ, исследование) и
   проведи её полной декомпозицией.
