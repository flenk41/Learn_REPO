# 17 · Работа с данными и документами 🛠️

> 🎯 **Цель блока:** научиться использовать ИИ для анализа текстов, документов и данных —
> суммирование, извлечение, преобразование. Это одна из самых полезных способностей ИИ.

---

## 📖 ИИ как обработчик информации

ИИ отлично работает с текстом, который ты ему **даёшь**: суммирует, извлекает, переводит,
структурирует, анализирует. Главное — правильно подать данные (вспомни заземление из модуля
12 — это снижает галлюцинации).

---

## ⭐ Суммирование

```
"Сделай краткое резюме этого текста в 5 пунктах:
\"\"\"
[длинный текст]
\"\"\""

"Выдели 3 главные мысли из статьи."
"Перескажи этот документ для занятого руководителя — только суть, 100 слов."
"Сделай TL;DR одним предложением."
```

💡 Уровни суммирования: одно предложение → абзац → ключевые пункты → подробное резюме.
Выбирай под задачу.

---

## ⭐ Извлечение данных

Вытащи из текста структурированную информацию:

```
"Извлеки из письма: имя отправителя, дату, главную просьбу, срок.
Выдай в виде списка."

"Найди в этом тексте все упоминания цен и собери их в таблицу:
товар | цена."

"Извлеки все email-адреса и имена из текста в формате JSON."
```

🖼️
```
   неструктурированный текст  →  ИИ  →  структурированные данные (таблица/JSON/список)
```

💡 Это превращает «кашу» текста в данные, с которыми удобно работать. Сочетай с few-shot
(модуль 14) для точного формата.

---

## ⭐ Преобразование и переформатирование

```
"Преврати эти заметки в структурированный план."
"Перепиши этот формальный текст простым языком."
"Преобразуй список в таблицу с колонками."
"Сделай из этого черновика готовое письмо."
"Переведи технический текст на язык, понятный клиенту."
```

---

## ⭐ Анализ текста

```
"Какой тон у этого письма? Не звучит ли оно грубо?"
"Найди логические противоречия в этом тексте."
"Какие аргументы за и против приводятся в статье?"
"Оцени этот текст: сильные и слабые стороны."
"Какие вопросы остаются без ответа после прочтения?"
```

---

## 📖 Работа с длинными документами

> ⚠️ Помни про контекстное окно (модуль 08): очень длинный документ может не влезть. Приёмы:

```
1. Разбей документ на части, обрабатывай по разделам (декомпозиция, модуль 16).
2. Сначала суммируй каждую часть, потом работай с резюме.
3. Используй ИИ с большим контекстом или функцией загрузки файлов.
```

💡 Многие ИИ умеют **принимать файлы** (PDF, документы) — это удобнее, чем вставлять текст.
Но принцип тот же: ИИ работает с тем, что попало в контекст.

---

## ⚠️ Заземляй на данные

```
"Отвечай ТОЛЬКО на основе этого документа. Если информации нет — скажи об этом."
```

> ⚠️ Без этой инструкции ИИ может «дополнить» ответ из общих знаний (и нагаллюцинировать).
> При работе с конкретными данными всегда заземляй (модуль 12).

---

## 🧪 Эксперимент: обработай реальный текст

Возьми реальную статью/письмо/документ и проведи через все операции:
1. Суммируй в 3 пункта.
2. Извлеки ключевые данные в таблицу.
3. Проанализируй тон.
4. Переформатируй во что-то полезное.

---

## ✅ Задачи

1. **Многоуровневое резюме** — суммируй текст на 3 уровнях (1 предложение / абзац / пункты).
2. **Извлечение в таблицу** — вытащи структурированные данные из неструктурированного текста.
3. **JSON-извлечение** — извлеки данные строго в JSON (для дальнейшей обработки).
4. **Преобразование** — преврати заметки/черновик в готовый структурированный документ.
5. **Анализ** — проанализируй тон и аргументы реального текста.
6. **Заземление** — обработай документ с инструкцией «только по тексту», проверь, что не
   выдумывает.
7. ⭐ **Длинный документ** — обработай большой текст по частям с финальной сборкой.
