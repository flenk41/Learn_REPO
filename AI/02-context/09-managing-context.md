# 09 · Управление контекстом 🖼️⭐

> 🎯 **Цель блока:** научиться **управлять** памятью ИИ — что положить в контекст, как
> структурировать, когда начать заново. Это превращает понимание из прошлого модуля в навык.

---

## ⭐ Принцип 1: всё нужное — в контексте

ИИ учитывает только то, что есть в текущем диалоге. Значит, **вся необходимая информация
должна быть подана явно**.

```
❌ "Проверь договор на риски"
   (какой договор? ИИ его не видит)

✅ "Проверь этот договор на риски для арендатора:
   [полный текст договора]"
```

💡 Не рассчитывай, что ИИ «знает» твой документ, проект или ситуацию. Дай ему всё прямо в
промпте.

---

## ⭐ Принцип 2: структурируй контекст

ИИ читает контекст лучше, когда он **организован**. Используй разделы, метки, разделители:

```
ЗАДАЧА: составь план статьи.

ТЕМА: польза утренней зарядки.

АУДИТОРИЯ: занятые офисные работники 30–45 лет.

ТРЕБОВАНИЯ:
- 5 разделов
- практичный тон
- каждый раздел с подзаголовком

ДАННЫЕ (исследования, которые надо учесть):
"""
[вставка материалов]
"""
```

🖼️
```
   Каша из текста         →  ИИ путается, что главное
   Структура с метками    →  ИИ чётко видит задачу, данные, требования
```

💡 Метки заглавными (`ЗАДАЧА:`, `ДАННЫЕ:`) и разделители (`"""`) помогают модели не смешать
инструкцию с данными. Особенно важно для длинных промптов.

---

## ⭐ Принцип 3: порядок имеет значение

Размести информацию логично: **сначала контекст и инструкции, в конце — конкретный вопрос**.

```
[роль и общий контекст]
[данные / материалы]
[конкретная задача в конце]  ← то, что ИИ должен сделать прямо сейчас
```

💡 Финальную инструкцию модель «видит» свежее всего. Длинные данные — выше, чёткий запрос —
внизу.

---

## ⭐ Принцип 4: освежай контекст в долгих диалогах

В длинном чате важное может «выпасть» (модуль 08). Решения:

```
"Напомню ключевое: мы пишем книгу про космос для детей 8 лет,
тон весёлый. Теперь продолжи главу про Марс."
```

💡 Периодически **резюмируй** важное своими словами или проси ИИ это сделать:
```
"Подведи итог нашего обсуждения в 5 пунктах, чтобы зафиксировать договорённости."
```
Затем этот итог можно перенести в новый чат как стартовый контекст.

---

## ⭐ Принцип 5: знай, когда начать новый чат

| Оставайся в чате | Начни новый чат |
|------------------|-----------------|
| продолжаешь ту же задачу | переключаешься на другую тему |
| нужна предыдущая информация | прошлый контекст мешает/путает |
| дорабатываешь результат | хочешь «чистый» взгляд |
| | чат стал очень длинным и «тормозит» |

> 💡 Если ИИ «застрял» на неверном подходе и упорно его повторяет — иногда проще начать
> новый чат с улучшенным промптом, чем переубеждать. Контекст с ошибками тянет назад.

---

## 📖 Приём: «перенос контекста» между чатами

Поскольку между чатами памяти нет, переноси нужное вручную:

```
1. В конце чата: "Собери всё важное из нашего обсуждения в виде краткого брифа."
2. Скопируй бриф.
3. В новом чате вставь его как стартовый контекст: "Вот контекст проекта: [бриф].
   Теперь помоги с..."
```

💡 Это «сохранение и загрузка» памяти вручную — прямой аналог сериализации данных в файл из
программирования.

---

## 🧪 Эксперимент: структура vs каша

1. Дай ИИ сложную задачу одним сплошным абзацем (задача + данные + требования вперемешку).
2. Дай ту же задачу со структурой (метки `ЗАДАЧА/ДАННЫЕ/ТРЕБОВАНИЯ`, разделители).
3. Сравни качество и точность ответов.

---

## ✅ Задачи

1. **Структурируй промпт.** Возьми сложную задачу, оформи с метками и разделителями.
2. **Каша vs структура** — выполни эксперимент выше.
3. **Резюме диалога.** В длинном чате попроси ИИ подвести итог договорённостей.
4. **Перенос контекста.** Собери бриф в одном чате, перенеси в новый, продолжи работу.
5. **Решение о новом чате.** Опиши 3 ситуации: когда остаться в чате, когда начать новый.
6. ⭐ **Длинный документ.** Если документ не влезает — придумай, как подать его частями (по
   разделам), сохраняя связность.

---

## ❓ Проверь себя

1. Почему вся нужная информация должна быть в контексте?
2. Как структура промпта помогает ИИ?
3. Куда лучше ставить конкретный вопрос — в начало или конец?
4. Как «освежать» контекст в долгом диалоге?
5. Когда лучше начать новый чат?
6. Как перенести контекст между чатами?
