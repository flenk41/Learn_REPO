# 06 · Разведка и сканирование 🖼️⭐

> 🎯 **Цель блока:** понять, как атакующий «прощупывает» систему перед атакой, — чтобы видеть свою
> поверхность атаки и сокращать её.

> ⚠️ Сканирование чужих систем без разрешения — незаконно ([модуль 02](../00-foundations/02-law-authorization.md)).
> Практикуй ТОЛЬКО на своих/лабораторных целях или в рамках авторизованного пентеста/scope.

---

## 📖 Разведка — первый этап атаки (и защиты)

```
   прежде чем атаковать, атакующий узнаёт о цели как можно больше:
   • какие сервисы/порты открыты, какие версии ПО (известные уязвимости?)
   • какие технологии (сервер, фреймворк, CMS)
   • точки входа: эндпоинты, формы, поддомены
   • публичная информация (OSINT — см. трек соц. инженерии).

   защитник делает ТО ЖЕ САМОЕ на своей системе, чтобы увидеть, что видит атакующий, и закрыть лишнее.
```

💡 ⭐ Разведка на себе = инвентаризация поверхности атаки. «Что торчит наружу? Какие версии? Что
можно убрать?» — это защитное применение наступательного навыка.

---

## ⭐ Виды разведки

```
   ПАССИВНАЯ (не трогая цель напрямую): публичные данные, DNS, поисковики, утечки, заголовки.
   АКТИВНАЯ (взаимодействие с целью): сканирование портов/сервисов, перебор директорий, фингерпринтинг.
            ⚠️ активная по чужому = уже несанкционированное действие. Только своё/scope.
```

```
   что обычно выясняют (на СВОЕЙ системе — для защиты):
   • открытые порты и сервисы (ss/nmap) → лишние закрыть
   • версии ПО → обновить уязвимые
   • технологии (заголовки ответа, отпечатки) → не раскрывать лишнего
   • директории/эндпоинты → убрать забытые/тестовые
```

## ⭐⭐ Сканирование портов: nmap (в лабе)

```
   nmap — стандартный инструмент инвентаризации сети. На СВОЕЙ лабе:
   nmap 192.168.56.101              — какие порты открыты на учебной мишени
   nmap -sV 192.168.56.101          — + версии сервисов (где известные уязвимости?)
   nmap -p- 192.168.56.101          — все порты

   ЗАЩИТНЫЙ вывод: каждый открытый порт — точка входа. Закрой ненужные, обнови сервисы,
   спрячь версии. То, чего не видно/нет, нельзя атаковать.
```

🖼️
```
   скан своей мишени:
   PORT     STATE  SERVICE   VERSION
   22/tcp   open   ssh       OpenSSH 7.2     ← обновить? ограничить доступ?
   80/tcp   open   http      Apache 2.4.7    ← старая версия — уязвимости!
   3306/tcp open   mysql     MySQL 5.5       ← БД торчит наружу?! закрыть от интернета!
   ↓ защитник видит проблемы и чинит: закрыть 3306, обновить, ограничить 22.
```

💡 ⭐⭐ Главная защитная мысль: **сканируй себя раньше, чем это сделают другие**. Интернет
постоянно сканируется ботами; то, что открыто и уязвимо, найдут за минуты. Регулярная
само-инвентаризация (порты, версии, эндпоинты) + закрытие лишнего = огромная часть защиты.

## 📖 Веб-разведка и инструменты

```
   для веб-приложений (на своих/в scope):
   • заголовки ответа — какой сервер/фреймворк раскрывается (убрать лишнее: Server, X-Powered-By).
   • robots.txt, sitemap — что раскрыто.
   • перебор директорий — найти забытые/тестовые эндпоинты (и убрать их).
   • прокси типа Burp Suite / OWASP ZAP — смотреть и анализировать HTTP-трафик своего приложения.

   ЗАЩИТА: не раскрывай версии/технологии в заголовках и ошибках; убирай тестовые/отладочные
   эндпоинты из прода; закрывай служебное.
```

---

## ⚠️ Ловушки

- ❌ Сканировать чужие адреса «просто посмотреть» — это уже за гранью закона.
- ❌ Случайно сканировать «всю подсеть», задев чужие/реальные хосты (проверь цель и изоляцию лабы).
- ❌ Раскрывать версии ПО/технологий наружу (дарит атакующему список известных уязвимостей).
- ❌ Оставлять открытыми ненужные порты (БД, админки) в интернет.
- ❌ Забывать про забытые тестовые/отладочные эндпоинты — частая точка входа.
- ❌ Считать «меня не найдут» — боты сканируют весь интернет постоянно.

## ✅ Упражнения (только своё/лаба)

1. **Скан себя.** В лабе просканируй свою учебную мишень (`nmap -sV`). Какие порты/версии? Что бы
   ты закрыл/обновил?
2. **Свои порты.** На своей машине посмотри открытые порты (`ss -tulpn`). Есть лишнее, торчащее наружу?
3. **Заголовки.** Посмотри заголовки ответа своего веб-приложения. Раскрывает ли оно сервер/
   фреймворк/версию? Как убрать?
4. **Поверхность.** Составь список всех точек входа своего приложения. Что можно закрыть/убрать?

---

## ❓ Проверь себя

1. Зачем нужна разведка атакующему и как это применяет защитник?
2. Чем пассивная разведка отличается от активной (и где граница закона)?
3. Почему «сканируй себя раньше других» — защитная стратегия?
4. Что раскрывать наружу не стоит и почему?
