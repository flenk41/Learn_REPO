# 🚀 Проект уровня 2 · «Найди и почини» + методичка защиты

> Самый практичный проект трека: пройди намеренно уязвимое приложение, найди дыры (поняв атаку),
> и для каждой **напиши защиту** — собрав личную методичку «уязвимо → безопасно».

> ⚠️ Работай ТОЛЬКО с учебными мишенями (OWASP Juice Shop, DVWA, WebGoat) или своим кодом. Никаких
> чужих систем ([модуль 02](../00-foundations/02-law-authorization.md)).

---

## 🎯 Что делаем

Две связанные части: (A) пройти уязвимое приложение и задокументировать находки с защитой; (B)
взять **свой** небольшой проект (или написать мини-приложение) и сделать его безопасным по Top-10.

```
   учебная мишень: атака → понимание → защита   +   свой код: аудит → исправление по Top-10
```

---

## 📋 Часть A · Разбор уязвимого приложения

Возьми OWASP Juice Shop (или DVWA/WebGoat) — они созданы для легального обучения.

Для **каждой** найденной уязвимости (минимум 5, разных классов) задокументируй:
```
   • КЛАСС (по OWASP Top-10): что это.
   • МЕХАНИКА: как работает (коротко, для понимания — не пошаговый «эксплойт-гайд»).
   • ПОЧЕМУ ВОЗМОЖНА: какая ошибка в коде/конфиге.
   • ЗАЩИТА: как это исправить в коде (конкретно) + сквозной принцип, который вернул бы защиту.
```

## 📋 Часть B · Сделай свой код безопасным

Возьми свой проект (веб-приложение/API) — или напиши маленькое с формой логина, списком записей и
API. Проведи аудит и исправь по разобранным классам:
```
   ✅ инъекции   → параметризованные запросы везде
   ✅ XSS        → экранирование вывода, нет raw-вставки недоверенного, CSP
   ✅ доступ     → серверная проверка прав/владения, deny by default, против IDOR
   ✅ CSRF/SSRF  → токены+SameSite; allow-list для серверных запросов
   ✅ конфиг     → нет дефолтов/debug, заголовки безопасности, секреты вне кода
   ✅ auth       → безопасные сессии, лимиты, правильное хранение паролей (предвестник модуля 17)
```

## 📋 Часть C · Методичка «уязвимо → безопасно»

Собери короткий справочник: для каждого класса — парный сниппет (плохой/хороший код) + одно
предложение «почему фикс работает». Это твой рабочий артефакт и шпаргалка для команды.

## 🌟 Бонусы

- 🥚 Подключи свой **сканер из Уровня 1** к своему проекту — ловит ли он что-то?
- 🐣 Добавь автотесты безопасности (проверь, что IDOR/инъекция больше не проходят после фикса).
- 🦅 Сделай before/after: уязвимая версия эндпоинта и защищённая, с пояснением.
- 🚀 Прогони свой защищённый код через OWASP ZAP/линтер безопасности (в лабе) — что находит?

## ✅ Критерии готовности

- [ ] 5+ уязвимостей разных классов разобраны (механика + защита) на учебной мишени
- [ ] Свой код проверен и исправлен по основным классам Top-10
- [ ] Параметризация, экранирование, серверная авторизация — внедрены
- [ ] CSRF/SSRF/конфигурация/секреты закрыты
- [ ] Готова методичка «уязвимо → безопасно» с парными сниппетами
- [ ] Каждый фикс объяснён через сквозной принцип

🎉 Готово? Ты умеешь находить дыры (понимая атаку) и закрывать их. Дальше — писать код безопасным
**с самого начала**: [защитное программирование](../03-defensive-code/15-secure-coding.md).
