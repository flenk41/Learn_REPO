# 03 · Лаборатория: легальная среда для практики 🖼️⭐

> 🎯 **Цель блока:** собрать **легальную** среду, где можно безопасно практиковать и атаку, и
> защиту — без риска навредить кому-то и нарушить закон.

> ⚠️ Вся практика трека — ТОЛЬКО в такой среде или на авторизованных целях ([модуль 02](02-law-authorization.md)).

---

## 📖 Зачем своя лаборатория

```
   практиковаться на чужих системах — НЕЛЬЗЯ (преступление, модуль 02).
   практиковаться на проде — ОПАСНО (сломаешь рабочее).
   → нужна ИЗОЛИРОВАННАЯ среда, которую тебе разрешено ломать как угодно: своя.
   там ты атакуешь и защищаешь сколько хочешь, ничего не нарушая и никому не вредя.
```

💡 ⭐ Лаборатория — это твой «полигон»: всё, что ты делаешь внутри неё, легально, потому что это
твои собственные машины и приложения, изолированные от чужого. Это обязательное условие, чтобы
учиться наступательным техникам этично.

---

## ⭐ Из чего собрать лабу

```
   🖥️ ВИРТУАЛИЗАЦИЯ — изолируй всё в ВМ (VirtualBox/VMware/гипервизор), отдельная виртуальная сеть.
      • атакующая машина: дистрибутив для тестирования безопасности (Kali/Parrot) или обычный Linux
        с инструментами.
      • уязвимая мишень: намеренно дырявые учебные образы (см. ниже).
      • сеть «host-only»/изолированная — чтобы атаки НЕ выходили наружу в реальный интернет.

   🎯 НАМЕРЕННО УЯЗВИМЫЕ ПРИЛОЖЕНИЯ (созданы ДЛЯ обучения, ломать их легально):
      • DVWA (Damn Vulnerable Web App), OWASP Juice Shop, WebGoat, bWAPP — учебные веб-мишени.
      • Metasploitable, специальные «vulnhub»-образы — уязвимые ОС для тренировки.

   🚩 CTF и онлайн-полигоны (легальные, с разрешением площадки):
      • Hack The Box, TryHackMe, PicoCTF, OverTheWire, PortSwigger Web Security Academy.
      • там специально разрешено атаковать учебные цели в рамках платформы.
```

🖼️
```
   ┌─ изолированная виртуальная сеть (НЕ связана с реальным интернетом для атак) ─┐
   │   [ атакующая ВМ ]  ───►  [ уязвимая мишень: DVWA / Juice Shop ]            │
   │      инструменты            намеренно дырявое приложение для учёбы           │
   └──────────────────────────────────────────────────────────────────────────┘
   всё внутри — твоё, ломать можно. наружу — нельзя.
```

---

## ⭐⭐ Правила безопасной лабы

```
   ✅ ИЗОЛЯЦИЯ: атакующие действия — только внутри лабы, не в реальную сеть. Проверь сетевой режим ВМ.
   ✅ СНИМКИ (snapshots): делай перед экспериментами — откатишься, если «сломал» мишень.
   ✅ ОТДЕЛЬНО ОТ РАБОЧЕГО: лаба не на рабочем ноуте с реальными данными/доступами.
   ✅ НЕ ТЕСТИРУЙ «заодно» соседние/чужие адреса — только свои мишени в своей подсети.
   ✅ ВРЕДОНОСНЫЕ ОБРАЗЦЫ (если изучаешь) — только в полностью изолированной ВМ, без сети, осторожно.
   ✅ ОБНОВЛЯЙ свою хост-систему — уязвимые образы держи ВНУТРИ, не открывай их наружу.
```

💡 ⭐⭐ Золотое правило лабы: **атака не должна покинуть периметр**. Самая частая ошибка новичка —
запустить скан/эксплойт «в интернет» вместо изолированной сети. Настрой сеть ВМ как host-only/
internal и убедись, что мишени недоступны извне. Так ты тренируешься, не задевая никого.

> 🧭 Виртуализация и сети — это [ОС](../../OS/04-advanced/19-virtualization.md) и [Сети](../../Network/01-basics/03-ip-addresses.md)
> из курса. Лаба — отличная практика и для них.

## 📖 С чего начать новичку

```
   минимальный старт (без сложной инфраструктуры):
   1. PortSwigger Web Security Academy — бесплатные интерактивные лаборатории по веб-уязвимостям
      прямо в браузере (легально, для обучения).
   2. OWASP Juice Shop — запусти локально (один контейнер), ломай и чини у себя.
   3. TryHackMe / PicoCTF — управляемые задания «для начинающих».
   → не нужно сразу строить целый виртуальный полигон; начни с управляемых легальных площадок.
```

---

## ⚠️ Ловушки

- ❌ Запустить скан/атаку «в интернет» вместо изолированной сети — можно задеть чужое (и это незаконно).
- ❌ Тренироваться на реальном чужом сайте «он же простой» — преступление (модуль 02).
- ❌ Держать лабу на рабочей машине с реальными данными/VPN в прод.
- ❌ Качать «уязвимые/хакерские» образы откуда попало — могут быть реально вредоносными.
- ❌ Забыть снимки → сломал мишень и потерял прогресс.
- ❌ Открыть уязвимую мишень наружу (её взломают реальные боты — и это уже твоя проблема).

---

## ✅ Упражнения

1. **Собери минимум.** Запусти одну учебную мишень (Juice Shop локально или PortSwigger Academy в
   браузере). Убедись, что она доступна только тебе.
2. **Изоляция.** Если делаешь ВМ — настрой host-only/internal сеть и проверь, что мишень
   недоступна из реального интернета.
3. **Снимок.** Сделай snapshot мишени до экспериментов и потренируйся откатываться.
4. **План практики.** Выбери площадку (TryHackMe/PicoCTF/Academy) и пройди одно вводное задание.

---

## ❓ Проверь себя

1. Почему нельзя практиковаться на чужих/реальных системах?
2. Из чего состоит лаборатория (атакующая ВМ, мишени, изоляция)?
3. Что такое намеренно уязвимые приложения и зачем они?
4. Почему «атака не должна покинуть периметр»?
