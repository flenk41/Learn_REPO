# 18 · Криптография для разработчика 🖼️⭐

> 🎯 **Цель блока:** научиться **правильно применять** криптографию в коде — не теорию алгоритмов,
> а практику: что брать, как использовать, какие ошибки не совершать.

---

## 📖 Твоя работа — применять, а не изобретать

```
   разработчику почти никогда не нужно ПИСАТЬ криптоалгоритмы — нужно ПРАВИЛЬНО ПРИМЕНЯТЬ готовые.
   90% реальных крипто-уязвимостей — это ошибки ПРИМЕНЕНИЯ (слабый алгоритм, плохая случайность,
   переиспользование, хардкод ключей), а не взлом самого шифра.
   → твоя цель: знать, ЧТО брать и КАК использовать, и где грабли.
```

💡 ⭐ Повторим золотое правило (модуль 08): **не изобретай криптографию**. Бери проверенные
библиотеки (стандартные крипто-модули языка, libsodium и т.п.) и стандартные алгоритмы.
Самодельное = почти гарантированная дыра.

---

## ⭐ Что для чего (практическая шпаргалка)

```
   ЗАДАЧА                         → ЧТО БРАТЬ
   хранить пароли                 → bcrypt / argon2 / scrypt (модуль 17), НЕ обычный хеш
   проверить целостность          → SHA-256+ (не MD5/SHA1)
   шифровать данные «на диске»     → AES-GCM (даёт и шифрование, и контроль целостности)
   защитить канал/трафик           → TLS (HTTPS), не своё (модуль про TLS в Сетях)
   подпись/проверка подлинности    → HMAC (симм.) или асимметричные подписи
   токены/ключи/соли (случайность) → КРИПТОСТОЙКИЙ ГСЧ (secrets/CSPRNG), НЕ обычный random
   обмен ключами / «войти через»    → готовые протоколы (OAuth/OIDC), не свой
```

🖼️
```
   ❌ свой «шифр»: XOR с фиксированным ключом / base64 «для секретности»
   ✅ AES-GCM из проверенной библиотеки + случайный ключ + уникальный nonce
   разница: первое ломается за минуты, второе — отраслевой стандарт.
```

## ⭐⭐ Частые ошибки применения (и как правильно)

```python
# ❌ обычный random для секрета — ПРЕДСКАЗУЕМ
import random
token = str(random.randint(0, 999999))      # угадывается!

# ✅ криптостойкий генератор
import secrets
token = secrets.token_urlsafe(32)           # непредсказуемый токен
```

```
   ОШИБКИ ПРИМЕНЕНИЯ:
   ❌ обычный random() для токенов/паролей/солей → ✅ криптостойкий ГСЧ (secrets/CSPRNG).
   ❌ хардкод ключей в коде/репозитории         → ✅ из окружения/менеджера секретов (модуль 19).
   ❌ переиспользование nonce/IV                 → ✅ уникальный на каждое шифрование.
   ❌ шифрование без проверки целостности        → ✅ AEAD (AES-GCM) — шифрует И защищает от подмены.
   ❌ MD5/SHA1, ECB-режим, устаревшее            → ✅ современные алгоритмы/режимы.
   ❌ base64/«обфускация» вместо шифрования       → ✅ настоящее шифрование (base64 НЕ защищает).
   ❌ хранить/передавать ключи рядом с данными    → ✅ управление ключами отдельно.
```

💡 ⭐⭐ Самые частые грабли — **случайность** и **ключи**. Используй криптостойкий ГСЧ для всего
секретного (`secrets` в Python и аналоги), и никогда не держи ключи в коде. Эти две привычки
закрывают большую часть реальных крипто-ошибок.

---

## 📖 Кодирование ≠ шифрование ≠ хеширование

```
   КОДИРОВАНИЕ (base64, hex) — представление данных, НЕ защита. Любой раскодирует. Не для секретов.
   ШИФРОВАНИЕ — обратимо С КЛЮЧОМ. Для конфиденциальности (данные на диске/в передаче).
   ХЕШИРОВАНИЕ — необратимо. Для целостности и (со спец-алгоритмами) хранения паролей.
   ⚠️ частая ошибка новичка: «закодировал base64 = спрятал». Нет — это видно всем.
```

## ⚠️ Ловушки

- ❌ Писать свою криптографию/«шифр».
- ❌ Обычный `random` для секретов (предсказуем).
- ❌ Хардкод ключей в коде/репозитории.
- ❌ Переиспользование nonce/IV; устаревшие алгоритмы/режимы (MD5, ECB).
- ❌ Путать base64 (кодирование) с шифрованием.
- ❌ Шифровать без контроля целостности (подмена шифртекста).

## ✅ Упражнения

1. **Случайность.** Найди в коде генерацию токенов/паролей/солей. На обычном random? Переведи на
   криптостойкий ГСЧ (`secrets`/аналог).
2. **Ключи.** Поищи хардкод ключей/секретов в коде. Вынеси в окружение (мост к модулю 19).
3. **Шифрование данных.** Если шифруешь что-то — используешь AEAD (AES-GCM) с уникальным nonce?
   Проверь.
4. **base64-аудит.** Нет ли мест, где base64 используется «для секретности»? Замени на настоящее
   шифрование, если нужна защита.
