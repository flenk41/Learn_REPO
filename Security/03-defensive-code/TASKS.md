# ✅ Задачи уровня 3 · Защитное программирование

> Пишем безопасный код и защитные инструменты. Всё — на своих проектах/в лабе.

---

## 🟢 Базовые

1. **Принципы.** Для каждого из 8 принципов безопасного кода найди в своём проекте пример
   соблюдения или нарушения.
2. **Allow-list.** Перепиши проверку поля с deny-list на allow-list (описание допустимого) +
   безопасное значение по умолчанию.
3. **Пароли.** Проверь хранение паролей в своём проекте. Переведи на bcrypt/argon2, если не так.

## 🟡 Средние

4. **Fail securely.** Найди обработку ошибок, где при сбое «пропускается» доступ. Сделай fail closed.
5. **Случайность и ключи.** Переведи генерацию токенов на криптостойкий ГСЧ; вынеси хардкод-секреты
   в окружение.
6. **Аудит зависимостей.** Запусти `audit` пакетного менеджера, обнови критичные уязвимости,
   добавь lock-файл.
7. **Монитор логов.** Напиши инструмент, обнаруживающий перебор паролей в логе (сводка по IP).

## 🔴 Со звёздочкой

8. ⭐ **Аудитор конфигурации.** Расширь мини-сканер: права файлов + секреты в коде + ещё одна
   проверка. Каждая находка — риск + способ исправления.
9. ⭐ **Безопасная форма с нуля.** Напиши мини-приложение (регистрация/вход) правильно: валидация
   (allow-list), bcrypt/argon2, безопасные сессии, экранирование, без секретов в коде.
10. ⭐ **Проверка целостности.** Инструмент, который ведёт baseline хешей важных файлов и сообщает
    об изменениях.
11. ⭐ **Безопасность в CI.** Встрой в свой проект автозапуск сканера секретов + аудита
    зависимостей + линтера безопасности (pre-commit/CI).

---

## 🎯 Критерии «сделано»

- [ ] Пишу код по принципам безопасности (не доверяй вводу, fail securely…)
- [ ] Валидирую через allow-list на сервере
- [ ] Храню пароли через bcrypt/argon2, секреты — вне кода
- [ ] Применяю крипту правильно (криптостойкий ГСЧ, проверенные алгоритмы)
- [ ] Аудирую зависимости
- [ ] Написал хотя бы 2 рабочих защитных инструмента
