# 🚀 Проект уровня 1 · «Спаси легаси»

> Возьми реальный кусок плохого кода и доведи его до состояния, которым гордишься — применив
> весь арсенал ремесла: чистый код, именование, рефакторинг малыми шагами, осознанный техдолг.

---

## 🎯 Что делаем

Найди **легаси-функцию или модуль** (свой старый код, открытый проект, или «грязный» пример
ниже) и проведи его через полный цикл улучшения — задокументировав каждый шаг.

```
   плохой код  →  [рефакторинг микрошагами]  →  чистый код + разбор решений
```

---

## 📋 Этапы

### 1. Зафиксируй «до»
- Сохрани исходный код.
- Выпиши, что в нём плохо: имена, длина, вложенность, дублирование, магические числа, отсутствие
  тестов.

### 2. Подстели страховку
- Напиши хотя бы пару тестов на текущее **поведение** (что код делает сейчас).
- Без них рефакторинг вслепую — это не рефакторинг, а молитва.

### 3. Рефактори микрошагами
- Минимум **6 коммитов**, каждый — один приём (переименование, выделение функции, guard clause,
  магическое число → константа, устранение дублирования).
- После каждого шага — тесты зелёные.
- В сообщении коммита пиши, **какой приём** применил.

### 4. Зафиксируй «после» и разбор
- Покажи код «до/после».
- Объясни **каждое** ключевое решение: почему так лучше.
- Отметь, где сознательно **не стал** усложнять (YAGNI) — это тоже решение.

### 5. Техдолг-заметка
- Если что-то осталось несделанным — оформи как осознанный долг: что, почему отложено, когда отдать.

---

## 🧪 Пример грязного кода (если нет своего)

```python
def calc(d, t):
    r = 0
    for i in range(len(d)):
        if t == 1:
            r = r + d[i] * 0.9
        elif t == 2:
            r = r + d[i] * 0.8
        else:
            r = r + d[i]
    if r > 1000:
        r = r - r * 0.05
    return r
```
Что тут не так? Имена, магические числа (0.9/0.8/0.05/1000/1/2), `range(len())`, тип-через-число,
смешаны скидка по типу и оптовая скидка. Преврати в читаемый код с понятными именами и без магии.

## 🌟 Бонусы

- 🥚 Добавь линтер/форматтер и приведи код к нему.
- 🐣 Сделай «ревью самому себе»: пройди финальный diff и оставь 5 замечаний, как ревьюер.
- 🦅 Покажи код другу/коллеге и попроси ревью. Сравни их замечания со своими.

---

## ✅ Критерии готовности

- [ ] Есть тесты на поведение, написанные ДО рефакторинга
- [ ] 6+ коммитов микрошагами, каждый с названием приёма, тесты зелёные
- [ ] Код «после» читается без комментариев-«что»
- [ ] Каждое решение объяснено; где-то осознанно применён YAGNI
- [ ] Оставшийся долг оформлен явно
