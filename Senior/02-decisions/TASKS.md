# ✅ Задачи уровня 2 · Решения и компромиссы

> Это ядро трека. Задачи — это **разборы решений** на реальных или приближенных к реальности
> ситуациях. Цель — натренировать мышление «варианты → критерии → контекст → выбор».

---

## 🟢 Базовые

1. **Разбор размена.** Возьми 3 своих прошлых технических решения. Для каждого назови, что ты
   разменял (что выиграл / что потерял). Осознавал ли тогда?
2. **Три варианта.** Для текущей задачи придумай минимум 3 подхода. Таблица плюсов/минусов.
   Выбери и обоснуй контекстом.
3. **Постфактум-оценка.** Сравни оценку и факт по 3 задачам. Во сколько ошибся и почему?

## 🟡 Средние

4. **Контекст-флип.** Возьми «монолит vs микросервисы» (или SQL vs NoSQL, REST vs gRPC). Опиши
   два разных контекста, в которых верны противоположные варианты.
5. **Декомпозиция-оценка.** Большую задачу разбей на куски ≤1 дня, оцени диапазоном с допущениями.
   Сравни с «интуицией целиком».
6. **Build vs Buy.** Для конкретной потребности (auth / email / поиск / графики) реши: писать или
   взять готовое? Учти скрытые цены зависимости.
7. **Напиши ADR.** Оформи реальное решение из проекта как ADR: контекст, решение, **2 отвергнутые
   альтернативы с причинами**, последствия.

## 🔴 Со звёздочкой

8. ⭐ **Матрица рисков.** Для проекта построй карту топ-5 рисков (вероятность × ущерб) и выбери
   ход для каждого (снизить/избежать/принять/переложить).
9. ⭐ **Двери.** Раздели 5 текущих решений на «в одну сторону» (необратимые) и «в две». По каждому —
   как это меняет, сколько времени стоит тратить.
10. ⭐ **Защита решения.** Возьми спорное решение и подготовь его защиту: альтернативы, критерии,
    почему именно так для этого контекста. Представь, что тебя оспаривает скептик.
11. ⭐⭐ **Развенчай серебряную пулю.** Возьми технологию/паттерн, который «все хвалят». Найди
    реальный контекст, где он — плохой выбор. Аргументируй.

---

## 🎯 Критерии «сделано»

- [ ] Вижу цену (что теряю) у любого решения
- [ ] Всегда рассматриваю 2–3 варианта по критериям и контексту
- [ ] Даю честные оценки диапазоном с допущениями
- [ ] Знаю, когда не писать код (buy / не делать)
- [ ] Фиксирую важные решения как ADR с альтернативами
- [ ] Действую при неопределённости, управляя рисками
