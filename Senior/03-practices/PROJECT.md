# 🚀 Проект уровня 3 · «Въезжаю в чужой проект»

> Самый реалистичный senior-сценарий: тебя кинули в **незнакомый большой проект** и дали задачу.
> Этот проект тренирует весь арсенал практик — чтение кода, отладку, проектирование под
> изменения, документацию — на настоящем чужом коде.

---

## 🎯 Что делаем

Возьми **реальный открытый проект** среднего размера (не свой!), в котором ещё не разбирался, и
проведи полный цикл: пойми → найди → измени/почини → задокументируй.

```
   незнакомый репозиторий → карта → задача → изменение под страховкой → разбор
```

Как выбрать проект: популярная библиотека на твоём языке, CLI-утилита, небольшой веб-сервис.
Размер — чтобы не прочитать за час, но и не утонуть (тысячи, не миллионы строк).

---

## 📋 Этапы

### 1. Карта (чтение «сверху вниз»)
- Точки входа, крупные модули, как связаны. Нарисуй схему (метро-карта).
- Где запускается? Где тесты? Есть ли ADR/доки?
- ⏱️ Ограничь себя 30–45 минутами — не ныряй в детали.

### 2. Один сценарий end-to-end
- Выбери один пользовательский путь (команда CLI, эндпоинт, функция API).
- Проследи его от входа до результата отладчиком/поиском. Опиши поток.

### 3. Поставь себе задачу
- Найди настоящий issue в трекере проекта **или** придумай маленькое осмысленное улучшение/фикс.
- Сформулируй: какую проблему решаешь? (модуль 16) Какие крайние случаи?

### 4. Измени под страховкой
- Есть тесты? Используй их. Нет на нужном месте? Напиши характеризующий тест сначала.
- Сделай изменение **микрошагами** (рефакторинг-стиль), оставаясь «зелёным».
- Если что-то странное — выясни через `git blame`/тесты, прежде чем менять.

### 5. Разбор и документация
- Опиши: как въезжал, где было трудно, что помогло (тесты? история? схема?).
- Если решал баг — мини post-mortem: корневая причина, как нашёл, как предотвратить.
- Предложи одно улучшение документации проекта (что помогло бы новичку, как тебе).

---

## 🌟 Бонусы

- 🥚 Реально оформи Pull Request в проект (если фикс полезный) — пройди настоящее ревью.
- 🐣 Напиши «onboarding-заметку» для воображаемого нового разработчика этого проекта.
- 🦅 Найди место с сильным зацеплением и опиши, как бы спроектировал его под изменения.
- 🚀 Профайль горячий путь проекта — есть ли неожиданное узкое место?

---

## ✅ Критерии готовности

- [ ] Есть верхнеуровневая карта проекта (схема), составленная без чтения всего подряд
- [ ] Прослежен один сценарий end-to-end
- [ ] Сформулирована настоящая проблема/задача (не «поменяю что-нибудь»)
- [ ] Изменение сделано микрошагами под страховкой тестов
- [ ] Есть разбор: как въезжал, что помогло, корневая причина (если баг)
- [ ] Предложено улучшение документации
