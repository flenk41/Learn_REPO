# 00 · Зачем связывать языки 🖼️

> 🎯 **Цель блока:** понять, зачем вообще использовать несколько языков в одном проекте и
> где это применяют. Это задаёт смысл всему треку.

---

## 📖 Один язык — не всегда лучший выбор

У каждого языка свои сильные стороны (ты видел это во всех треках):

| Язык | Силён в | Слаб в |
|------|---------|--------|
| **C** | скорость, контроль, железо | скорость разработки, безопасность |
| **C++** | скорость + абстракции | сложность |
| **Rust** | скорость + безопасность памяти | порог входа, время компиляции |
| **Python** | быстрая разработка, экосистема | скорость выполнения |
| **JavaScript** | веб, UI, повсеместность | тяжёлые вычисления |

🎯 **Идея полиглотного программирования:** взять **сильные стороны каждого** и соединить.
Не выбирать «или-или», а использовать «и-и».

---

## ⭐ Классический паттерн: быстрое ядро + удобная оболочка

🖼️
```
   ┌─────────────────────────────────────────┐
   │  Python / JavaScript  (удобный интерфейс) │  ← быстро писать, гибко
   ├─────────────────────────────────────────┤
   │  C / C++ / Rust       (быстрое ядро)      │  ← скорость, тяжёлые вычисления
   └─────────────────────────────────────────┘
```

💡 Большинство твоих любимых инструментов устроены **именно так**:
- **NumPy, PyTorch, TensorFlow** — Python снаружи, C/C++/CUDA внутри (математика быстрая).
- **Браузеры** — JS-движок на C++, веб-страницы на JS.
- **Node.js** — JS поверх C++ (движок V8).
- **многие Python-библиотеки** — обёртки над C-кодом (cryptography, lxml, pillow).
- **VS Code, Discord** — UI на JS/Electron, тяжёлое — на нативном коде.

## 📖 Реальные причины связывать языки

```
1. ⚡ Скорость:      переписать узкое место на C/Rust (Python в 50× медленнее).
2. ♻️ Переиспользование: уже есть проверенная C-библиотека — зачем переписывать?
3. 🌍 Экосистема:    нужна библиотека, которая есть только в другом языке.
4. 🧩 Постепенная миграция: переписывать старый проект по частям (C → Rust).
5. 🖥️ Платформа:     железо/ОС требуют C; UI требует JS; логика удобна на Python.
6. 👥 Команда:       разные части пишут специалисты по разным языкам.
```

---

## 🖼️ Пример из жизни: ускорение Python

```
   Проблема: Python-функция обрабатывает изображение 10 секунд — слишком медленно.

   Решение:
   1. Профилируешь, находишь узкое место (тяжёлый цикл).
   2. Переписываешь ТОЛЬКО его на C (или Rust).
   3. Вызываешь C-функцию из Python через FFI.
   4. Результат: 10 секунд → 0.1 секунды, при этом 95% кода остаётся на Python.
```

💡 Не нужно переписывать **весь** проект. Переписывают **критичные 5%**, а остальное
остаётся удобным. Это и есть мощь интеграции.

## 📖 Что мы изучим в треке

```
🔗 FFI       — прямой вызов C-функций из других языков (основа всего).
📦 Биндинги  — готовые «мосты»: PyO3 (Rust↔Python), N-API (C↔JS)…
🌐 WebAssembly — запуск C/C++/Rust в браузере и не только.
🪆 Встраивание — встроить интерпретатор (Python/Lua) внутрь C/C++ приложения.
🔌 Клей      — связь через процессы, IPC, REST/gRPC (когда FFI избыточен).
🖥️ Драйверы  — как код говорит с операционной системой и железом.
```

## 📖 Цена интеграции

> ⚠️ Связывать языки — мощно, но не бесплатно:
> - **сложность сборки** (нужно собрать оба языка, согласовать);
> - **отладка через границу** труднее (где упало — в Python или в C?);
> - **ошибки памяти** на стыке (главная тема Уровня 2);
> - **накладные расходы** на пересечение границы (для частых мелких вызовов).
>
> Поэтому интегрируют **осознанно**: там, где выигрыш (скорость, переиспользование)
> оправдывает усложнение. Иногда проще оставить один язык.
