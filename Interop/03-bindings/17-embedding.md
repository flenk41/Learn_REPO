# 17 · Встраивание интерпретатора 🖼️

> 🎯 **Цель блока:** понять обратное направление интеграции — встроить интерпретатор
> (Python/Lua/JS) **внутрь** нативного приложения, чтобы пользователи могли его скриптовать.

---

## 📖 Embedding — наоборот, чем FFI

До сих пор мы вызывали быстрый код (C/Rust) из удобного (Python). **Встраивание** —
наоборот: нативное приложение (C/C++) **содержит** внутри интерпретатор скриптового языка.

🖼️
```
   FFI:        Python  ──вызывает──►  C   (удобное зовёт быстрое)
   Embedding:  ┌─ C/C++ приложение ─┐
               │  встроенный Python  │  ←  приложение зовёт скрипты
               └─────────────────────┘
```

💡 Зачем? Чтобы пользователи (или ты сам) могли **расширять и настраивать** приложение
скриптами **без перекомпиляции**.

---

## ⭐ Где встраивание применяют

```
   🎮 Игры:        Lua/Python для логики уровней, поведения, модов (World of Warcraft, Roblox).
   🎨 Творческое ПО: Blender (Python внутри), GIMP (Script-Fu/Python), Photoshop.
   📝 Редакторы:    Neovim (Lua), Emacs (Lisp) — настройка и плагины на скриптах.
   🗄️ Базы данных:  PostgreSQL (PL/Python, PL/Perl) — функции на скриптах.
   ⚙️ Приложения:   правила, формулы, автоматизация, задаваемые пользователем.
```

💡 Общая идея: **ядро на быстром компилируемом языке + поведение на гибком скриптовом**.
Пользователь меняет скрипты — приложение ведёт себя иначе, не пересобираясь.

---

## ⭐ Пример: встроить Python в C

Python предоставляет C API для встраивания:

```c
#include <Python.h>

int main() {
    Py_Initialize();                          // запустить интерпретатор Python

    // выполнить Python-код прямо из C
    PyRun_SimpleString("print('Привет из встроенного Python!')");
    PyRun_SimpleString("result = sum([x**2 for x in range(10)])");
    PyRun_SimpleString("print(result)");

    Py_Finalize();                            // остановить интерпретатор
    return 0;
}
```
```bash
gcc embed.c -o embed $(python3-config --cflags --ldflags --embed)
```

🖼️
```
   C-приложение  ──Py_Initialize──►  [ внутри живёт Python-интерпретатор ]
                 ──PyRun_String──►   выполняет Python-код
                 обменивается данными через Python C API
```

💡 Теперь C-программа может загружать и выполнять Python-скрипты, передавать им данные и
получать результаты. Пользователь пишет логику на Python, ядро — на C.

## ⭐ Lua — король встраивания

**Lua** специально создан для встраивания: крошечный, быстрый, простой C API. Поэтому он
доминирует в играх.

```c
#include <lua.h>
#include <lauxlib.h>
#include <lualib.h>

int main() {
    lua_State* L = luaL_newstate();           // создать Lua-машину
    luaL_openlibs(L);
    luaL_dostring(L, "print('Привет из Lua!')"); // выполнить Lua-скрипт
    lua_close(L);
    return 0;
}
```

💡 Lua-машина занимает ~100 КБ, легко встраивается в любое C/C++ приложение. Игровые движки
дают игрокам Lua для модов именно поэтому. JS тоже встраивают (движки V8, QuickJS).

---

## 📖 Двусторонний обмен

При встраивании нужно в обе стороны:
```
   Приложение → скрипт:  передать данные (состояние игры, параметры).
   Скрипт → приложение:  вызвать функции приложения (создать объект, изменить мир).
```

> 💡 Для этого приложение **регистрирует свои функции** в интерпретаторе — скрипт может их
> вызывать. Это та же граница памяти (Уровень 2): данные между нативным кодом и скриптом
> переупаковываются, нужно следить за владением и временем жизни.

---

## 📖 Встраивание vs расширение — две стороны

```
   Расширение (extending):  скрипт ГЛАВНЫЙ, зовёт нативные модули (Python + C-расширение).
   Встраивание (embedding): нативный код ГЛАВНЫЙ, зовёт скрипты (игра + Lua).
```

💡 Часто используют оба сразу: приложение встраивает интерпретатор И регистрирует в нём
нативные функции для скриптов.

## ✅ Задачи

1. **Встрой Python.** Напиши C-программу, которая через `Py_Initialize` выполняет
   Python-код.
2. **Передай данные.** Из C задай переменную в Python, выполни вычисление, прочитай
   результат.
3. **Встрой Lua** (по желанию) — выполни Lua-скрипт из C.
4. **Загрузка скрипта из файла.** Сделай так, чтобы приложение выполняло скрипт из внешнего
   файла (пользователь может его менять).
5. ⭐ **Скриптуемое мини-приложение.** Сделай C/C++ программу, поведение которой задаётся
   внешним скриптом (Python/Lua), с регистрацией хотя бы одной нативной функции.
