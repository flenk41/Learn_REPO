# 🚀 Пет-проект уровня 3 — Полиглотное приложение

> 🎯 **Цель:** построить реальное приложение из **двух или более языков**, выбрав правильный
> способ связи. Это демонстрирует главный навык трека: соединять языки осознанно.

---

## 📋 Что делаем

Приложение, где разные части написаны на разных языках по их сильным сторонам. Идеи:

| Проект | Связка | Способ |
|--------|--------|--------|
| **Ускоренный анализатор данных** | Python (интерфейс) + Rust (вычисления) | PyO3 |
| **Веб-приложение с быстрым ядром** | JS (фронт) + Rust/Python (бэк) | REST + WASM |
| **Игра с моддингом** | C++/Rust (движок) + Lua/Python (логика) | встраивание |
| **CLI с нативным ускорением** | Python (CLI) + C/Rust (горячий путь) | FFI / PyO3 |
| **Микросервисы** | 2+ сервиса на разных языках | REST/gRPC |
| **Браузерный инструмент** | JS (UI) + Rust (обработка) | WASM |

---

## ✅ Требования (минимум)

- [ ] Минимум **два языка**, каждый — по своей сильной стороне.
- [ ] **Осознанный выбор способа связи** (FFI / биндинг / WASM / встраивание / REST) с
      обоснованием, почему именно он.
- [ ] Корректная работа с **границей** (память/данные, если FFI; контракт, если сеть).
- [ ] **Обработка ошибок** через границу.
- [ ] Приложение решает **реальную** задачу (не «hello world»).
- [ ] **README**: какие языки, почему, как собрать и запустить, как они связаны.

---

## 🪜 Пошаговый план

1. **Выбери задачу** и реши, какие части на каком языке (где нужна скорость? где удобство?
   где UI?).
2. **Выбери способ связи** (Уровень 0–3): тесный FFI или свободный REST? Обоснуй.
3. **Реализуй части** на каждом языке.
4. **Свяжи их** выбранным способом, отладь границу.
5. **Обработай ошибки** на стыке.
6. **Документируй** архитектуру и связи.

---

## 🧱 Пример архитектуры: «Быстрый обработчик логов»

```
   ┌─ Python (CLI + интерфейс) ─┐
   │  argparse, вывод, отчёты    │   ← удобство, гибкость
   └────────────┬────────────────┘
                │ PyO3 (тесная связь)
   ┌────────────▼────────────────┐
   │ Rust (парсинг + агрегация)   │   ← скорость + безопасность памяти
   │  обрабатывает гигабайты лога │
   └──────────────────────────────┘

   Обоснование: парсинг больших файлов — узкое место (Rust в 50× быстрее Python),
   но CLI и отчёты удобнее на Python. PyO3 даёт тесную безопасную связь.
```

---

## 💪 Усложнения (для «отлично»)

- ➕ Три языка (например JS-фронт + Python-API + Rust-ядро).
- ➕ Замеры: докажи выигрыш от интеграции (скорость до/после).
- ➕ Несколько способов связи в одном проекте (FFI для ядра + REST для внешнего сервиса).
- ➕ Полное управление памятью на границе (безопасная обёртка из Уровня 2).
- ➕ Сборка одной командой (скрипт/Makefile/CI, собирающий все языки).
- ➕ Тесты для каждой части и для интеграции.
