# 13 · C + Rust 🖼️⭐

> 🎯 **Цель блока:** научиться связывать C и Rust в обе стороны. Это популярнейшая связка:
> Rust постепенно заменяет C, сохраняя совместимость с огромной C-экосистемой.

---

## 📖 Зачем связывать C и Rust

```
   - использовать готовые C-библиотеки из Rust (их миллионы);
   - постепенно переписывать C-проект на Rust по функции (как делают в Linux, Firefox);
   - дать Rust-код в виде C-совместимой библиотеки для любого языка.
```

💡 Rust **специально** спроектирован для совместимости с C — у него есть `extern "C"`,
сырые указатели, и инструменты автогенерации мостов.

---

## ⭐ Rust вызывает C

Объявляешь C-функции в `extern` блоке и зовёшь их в `unsafe`:

```rust
// объявляем C-функции (из libc или своей библиотеки)
extern "C" {
    fn abs(input: i32) -> i32;
    fn sqrt(x: f64) -> f64;
}

fn main() {
    unsafe {
        println!("{}", abs(-5));    // 5 — вызвали C
        println!("{}", sqrt(16.0)); // 4
    }
}
```

🖼️
```
   Rust код  ──extern "C"──►  C-функция (через C-ABI)
   вызов в unsafe (Rust не может проверить безопасность чужого C)
```

> ⚠️ Вызовы C — **всегда `unsafe`** (модуль про [unsafe в Rust](../../Rust/04-senior/22-unsafe-ffi.md)):
> Rust не контролирует C, ответственность на тебе. Оборачивай в безопасный Rust-API.

---

## ⭐ Rust отдаёт функции для C

```rust
#[no_mangle]                              // не искажать имя (модуль 01)
pub extern "C" fn rust_add(a: i32, b: i32) -> i32 {
    a + b
}
```

Собираешь Rust как C-совместимую библиотеку (в `Cargo.toml`):
```toml
[lib]
crate-type = ["cdylib"]    # динамическая библиотека с C-ABI (.so/.dll)
# или "staticlib" для статической
```

Теперь `rust_add` можно вызвать из C, Python (ctypes!), любого языка — она выглядит как
обычная C-функция.

💡 Это значит: написал безопасное быстрое ядро на Rust → отдал как C-библиотеку → используют
все. Лучшее из миров: безопасность Rust + универсальность C-ABI.

---

## ⭐ Автогенерация мостов: bindgen и cbindgen

Писать `extern` блоки вручную для больших библиотек — мучение. Есть инструменты:

| Инструмент | Что делает |
|-----------|-----------|
| **bindgen** | C-заголовок (`.h`) → Rust-объявления (C → Rust) |
| **cbindgen** | Rust-код → C-заголовок (`.h`) (Rust → C) |

🖼️
```
   bindgen:   library.h  ──►  автоматические extern-объявления в Rust
   cbindgen:  rust_lib.rs ──►  автоматический library.h для C-пользователей
```

```bash
# bindgen: сгенерировать Rust-биндинги к C-библиотеке
bindgen library.h -o bindings.rs
```

💡 Эти инструменты экономят часы и убирают ошибки ручного описания сигнатур/структур
(вспомни, как легко ошибиться с раскладкой — Уровень 2). Для серьёзных проектов — обязательны.

## 📖 Передача данных C ↔ Rust

Те же правила, что в Уровне 2 (раскладка, владение, строки):

```rust
use std::os::raw::c_char;
use std::ffi::{CString, CStr};

// Rust-строка → C-строка
let rust_str = "привет";
let c_string = CString::new(rust_str).unwrap();   // добавляет \0
let ptr = c_string.as_ptr();                       // *const c_char для C

// C-строка → Rust-строка
unsafe {
    let c_str = CStr::from_ptr(ptr);
    let rust_str = c_str.to_str().unwrap();
}
```

💡 Rust даёт типы `CString`/`CStr` специально для границы с C — они управляют `\0` и
владением безопасно. `#[repr(C)]` на структуре заставляет Rust использовать C-раскладку:

```rust
#[repr(C)]                  // C-совместимая раскладка (как C, а не Rust-оптимизированная)
struct Point { x: i32, y: i32 }
```

> ⚠️ Без `#[repr(C)]` Rust может переупорядочить поля структуры (он оптимизирует раскладку).
> Для границы с C **всегда** ставь `#[repr(C)]` — иначе раскладка разойдётся.

## ✅ Задачи

1. **Rust зовёт C.** Вызови `abs`/`sqrt` из libc в Rust через `extern "C"`.
2. **Rust для C.** Напиши `#[no_mangle] extern "C"` функцию, собери как `cdylib`, вызови из
   Python через ctypes.
3. **CString.** Передай строку Rust → C и обратно через `CString`/`CStr`.
4. **#[repr(C)].** Создай C-совместимую структуру, передай между Rust и C.
5. **bindgen** (по желанию) — сгенерируй биндинги к простому C-заголовку.
6. ⭐ **Безопасная обёртка.** Оберни `unsafe` вызовы C в безопасный Rust-API (как учили в
   Rust-треке).
