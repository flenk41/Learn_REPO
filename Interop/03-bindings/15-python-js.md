# 15 · Python + JavaScript 🖼️

> 🎯 **Цель блока:** разобраться, как связать Python и JavaScript. Это особая пара — они
> живут в разных мирах (бэкенд vs браузер), поэтому способы связи другие.

---

## 📖 Особенность: Python и JS редко в одном процессе

В отличие от C/Rust, Python и JS обычно **не** вызывают друг друга напрямую через FFI —
они работают в разных средах (Python на сервере, JS в браузере/Node). Поэтому связь чаще
через **сеть, процессы или WebAssembly**.

🖼️
```
   Типичная архитектура:
   ┌─ Браузер (JS) ─┐   HTTP/WebSocket   ┌─ Сервер (Python) ─┐
   │  интерфейс     │ ◄────────────────► │  логика, данные    │
   └─────────────────┘                    └────────────────────┘
```

---

## ⭐ Способ 1: REST/HTTP API (самый частый)

Python — бэкенд с API, JS — фронтенд, который его зовёт. Это связь из разделов «Проекты и
API» языковых треков!

```python
# Python (Flask/FastAPI) — сервер
from fastapi import FastAPI
app = FastAPI()

@app.get("/square/{n}")
def square(n: int):
    return {"result": n * n}
```
```javascript
// JavaScript — клиент
const res = await fetch("http://localhost:8000/square/5");
const data = await res.json();
console.log(data.result);   // 25
```

💡 Это **самый распространённый** способ связать Python и JS: Python отдаёт данные/логику
через HTTP, JS их потребляет. Связка слабая (разные процессы), но гибкая и масштабируемая.

---

## ⭐ Способ 2: Node.js вызывает нативный код (N-API)

Если нужен быстрый нативный код в Node (JS-рантайм), используют **N-API** — FFI для Node:

```
   Node.js (JS)  ──N-API──►  C/C++/Rust нативный модуль
```

- **node-gyp / node-addon-api** — C/C++ аддоны для Node.
- **neon** / **napi-rs** — писать Node-аддоны на **Rust**.

💡 Это аналог PyO3, но для JS. `import addon from './native'` — и зовёшь быстрый нативный
код. Так ускоряют Node-приложения (как Python ускоряют через C/Rust).

---

## ⭐ Способ 3: Pyodide — Python в браузере (через WASM)

**Pyodide** — это CPython, скомпилированный в WebAssembly. Позволяет запускать **Python
прямо в браузере** и связывать его с JS:

```javascript
// JS загружает Pyodide и выполняет Python в браузере
let pyodide = await loadPyodide();
let result = pyodide.runPython(`
    sum([x**2 for x in range(10)])
`);
console.log(result);   // 285 — Python выполнился в браузере!
```

💡 Pyodide даёт двустороннюю связь JS ↔ Python **в браузере**: JS может звать Python-функции
и наоборот, делиться данными. Используется для интерактивных учебников, дата-приложений без
сервера. Это WebAssembly в действии (модуль 16).

---

## ⭐ Способ 4: запуск как подпроцесс

Простейший способ — Python запускает JS-скрипт (или наоборот) как отдельный процесс:

```python
import subprocess, json

result = subprocess.run(
    ["node", "script.js", "arg"],
    capture_output=True, text=True
)
data = json.loads(result.stdout)   # JS вернул JSON через stdout
```

💡 Грубо, но работает для разовых задач: один язык запускает программу на другом, общение
через ввод-вывод (часто JSON). Это связь «через процессы» (модуль 18).

## 📋 Как выбрать способ Python ↔ JS

```
   Веб-приложение (фронт + бэк)        →  REST/HTTP API (способ 1)
   Реальное время (чат, игры)         →  WebSocket
   Быстрый нативный код в Node        →  N-API (C/C++/Rust)
   Python в браузере / без сервера    →  Pyodide (WASM)
   Разовый запуск скрипта             →  подпроцесс
```

💡 В большинстве реальных проектов Python↔JS — это **REST API** между сервером и
браузером. FFI здесь редкость (разные миры). Это нормально: «свободная связь» (Уровень 0)
тут уместнее «тесной».
