# 14 · Rust + Python 🖼️⭐

> 🎯 **Цель блока:** связать Rust и Python через PyO3 — современный и удобный способ писать
> быстрые Python-модули на Rust. Это горячий тренд: безопасность Rust + удобство Python.

---

## 📖 Почему Rust + Python — отличная пара

```
   Python: удобство, экосистема, скорость разработки
   Rust:   скорость + БЕЗОПАСНОСТЬ памяти (в отличие от C!)
```

💡 Раньше быстрые Python-модули писали на C (опасно) или C++ (сложно). Rust даёт **ту же
скорость без ошибок памяти** — а PyO3 делает связку удобной. Поэтому многие современные
Python-библиотеки переписывают на Rust (polars, pydantic-core, ruff, cryptography частично).

---

## ⭐ PyO3 — мост Rust ↔ Python

**PyO3** — библиотека Rust, которая превращает Rust-функции в Python-функции почти без
усилий. Никакого ручного ctypes — PyO3 берёт на себя преобразование типов и памяти.

```rust
use pyo3::prelude::*;

// обычная Rust-функция, помеченная для Python
#[pyfunction]
fn sum_of_squares(numbers: Vec<f64>) -> f64 {
    numbers.iter().map(|x| x * x).sum()
}

// модуль Python
#[pymodule]
fn fast_math(m: &Bound<'_, PyModule>) -> PyResult<()> {
    m.add_function(wrap_pyfunction!(sum_of_squares, m)?)?;
    Ok(())
}
```

🖼️
```
   Rust функция (Vec<f64> → f64)
        │ PyO3 автоматически конвертирует:
        │ Python list ↔ Rust Vec, всё управление памятью корректно
        ▼
   import fast_math; fast_math.sum_of_squares([1,2,3])   # работает из Python!
```

💡 Заметь: ты пишешь **обычный Rust** с `Vec<f64>`, а PyO3 сам превращает Python-список в
Rust-вектор и обратно — безопасно, с правильной памятью. Никаких сырых указателей.

## ⭐ maturin — сборка одной командой

**maturin** собирает Rust-проект в готовый Python-пакет:

```bash
pip install maturin
maturin develop        # собрать и установить в текущее окружение (для разработки)
maturin build          # собрать пакет (.whl) для распространения
```

Затем в Python:
```python
import fast_math
print(fast_math.sum_of_squares([1.0, 2.0, 3.0]))   # 14.0
```

💡 `maturin develop` — и твой Rust-код доступен как Python-модуль. Цикл разработки почти как
с чистым Python, но скорость — Rust.

## ⭐ Классы и более сложные типы

PyO3 умеет отдавать Rust-структуры как Python-классы:

```rust
use pyo3::prelude::*;

#[pyclass]
struct Counter {
    count: i64,
}

#[pymethods]
impl Counter {
    #[new]
    fn new() -> Self { Counter { count: 0 } }

    fn increment(&mut self) { self.count += 1; }

    fn get(&self) -> i64 { self.count }
}
```
```python
c = Counter()
c.increment()
c.increment()
print(c.get())    # 2 — это Rust-объект, но используется как Python-класс!
```

💡 PyO3 связывает время жизни Rust-объекта с Python-объектом автоматически (RAII-обёртка из
[модуля 09](../02-boundary/09-memory-ownership.md), но бесплатно). Память управляется
правильно без твоих усилий.

## 📖 Почему это безопаснее C-подхода

| | **Python + C (ctypes)** | **Python + Rust (PyO3)** |
|--|------------------------|--------------------------|
| Ошибки памяти в ядре | возможны (C небезопасен) | предотвращены (Rust) |
| Преобразование типов | вручную (argtypes) | автоматически (PyO3) |
| Управление памятью на границе | вручную | автоматически |
| Сборка | вручную (gcc + ctypes) | одна команда (maturin) |

💡 PyO3 убирает **оба** класса проблем: и ошибки памяти в ядре (Rust безопасен), и ошибки на
границе (PyO3 управляет ею). Поэтому это рекомендуемый современный способ ускорять Python.
