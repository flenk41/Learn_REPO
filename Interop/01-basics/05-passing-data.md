# 05 · Передача данных через границу 🖼️⭐

> 🎯 **Цель блока:** разобраться, как данные разных типов пересекают границу языков. Здесь
> начинаются тонкости памяти — ядро трека.

---

## 📖 Главная сложность: языки представляют данные ПО-РАЗНОМУ

Python-список ≠ C-массив. Python-строка ≠ C-строка. Python-объект `int` ≠ C `int`. Чтобы
передать данные, их нужно **преобразовать** в формат, понятный обоим — обычно в C-формат.

🖼️
```
   Python int (объект ~28 байт)  ──ctypes──►  C int (4 байта)   ← переупаковка на границе
   Python list (массив ссылок)   ──ctypes──►  C array (подряд)  ← другое расположение!
   Python str (Unicode-объект)   ──ctypes──►  C char* + длина   ← кодировка, нуль-терминатор
```

💡 Граница языков — это место, где данные **переупаковываются**. Понимание расположения в
памяти (вся первая часть курса!) здесь окупается.

---

## ⭐ Простые типы — легко

Числа передаются прямо (с правильным `argtypes`):
```python
lib.add.argtypes = [ctypes.c_int, ctypes.c_int]   # int → int
lib.scale.argtypes = [ctypes.c_double]            # float → double
```
💡 Скалярные типы (числа, char, bool) — самый простой случай: одно значение, копируется по
значению.

---

## ⭐ Массивы — указатель + длина

В C массив = непрерывный блок + размер отдельно. Создаём C-массив из Python-данных:

```python
# чтение
data = [1.5, 2.5, 3.5]
arr = (ctypes.c_double * len(data))(*data)   # C-массив
lib.process.argtypes = [ctypes.POINTER(ctypes.c_double), ctypes.c_size_t]
lib.process(arr, len(data))

# C может ИЗМЕНИТЬ массив на месте (он же передан по указателю):
lib.double_all(arr, len(arr))
result = list(arr)   # читаем изменённые данные обратно
```

🖼️
```
   arr ──► [1.5][2.5][3.5]   ← C видит этот блок по указателю,
                               может читать И писать в него
```

> ⚠️ C пишет в **тот же** блок памяти. Это быстро (нет копий), но опасно: C может выйти за
> границы массива (помнишь buffer overflow из [C-курса](../../C/02-memory/10-arrays-strings.md))
> и испортить чужую память. Всегда передавай правильную длину!

## ⭐ Строки — особый случай

Строки сложнее всего (целый [модуль 10](../02-boundary/10-strings.md) посвящён им):

```python
# Python str → C char* (нужно закодировать в байты!)
text = "привет"
c_str = text.encode("utf-8")          # str → bytes
lib.print_str.argtypes = [ctypes.c_char_p]
lib.print_str(c_str)

# C char* → Python str
lib.get_message.restype = ctypes.c_char_p
result = lib.get_message().decode("utf-8")   # bytes → str
```

> ⚠️ Три ловушки строк на границе:
> 1. **Кодировка** — Python-строка это Unicode, C-строка это байты. Нужно `encode`/`decode`.
> 2. **Нуль-терминатор** — C-строка кончается `\0`, Python-строка хранит длину.
> 3. **Владение** — кто освобождает память строки? (модуль 06, 09)

## ⭐ Структуры — описать раскладку

Чтобы передать C-структуру, опиши её раскладку в Python:

```c
// C
typedef struct {
    int x;
    int y;
} Point;
double distance(Point* p1, Point* p2);
```
```python
# Python — точная копия раскладки!
class Point(ctypes.Structure):
    _fields_ = [("x", ctypes.c_int),
                ("y", ctypes.c_int)]

lib.distance.argtypes = [ctypes.POINTER(Point), ctypes.POINTER(Point)]
lib.distance.restype = ctypes.c_double

p1 = Point(0, 0)
p2 = Point(3, 4)
print(lib.distance(ctypes.byref(p1), ctypes.byref(p2)))   # 5.0
```

> ⚠️ Поля и их порядок должны **точно** совпадать с C, включая выравнивание (padding,
> [C-курс модуль 20](../../C/04-senior/20-alignment-ub.md)). Ошибся в раскладке — прочитаешь
> мусор. Это разберём в [модуле 11](../02-boundary/11-structs.md).

## 📋 По значению vs по указателю

```
   По значению (копия):     числа, маленькие структуры → безопасно, но копируется
   По указателю (ссылка):   массивы, большие структуры, изменяемые данные
                            → быстро (нет копий), но C получает доступ к твоей памяти
```

🖼️
```
   По значению:   Python ─копия→ C   (C работает со своей копией)
   По указателю:  Python ──ссылка──► [данные] ◄── C   (общая память — быстро, но рискованно)
```

💡 Большие данные передают по указателю (без копирования), но тогда **обе стороны делят
память** — отсюда все риски Уровня 2.
