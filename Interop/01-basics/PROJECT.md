# 🚀 Пет-проект уровня 1 — Ускоряем Python через C

> 🎯 **Цель:** взять медленную Python-функцию, переписать её узкое место на C и ускорить в
> десятки раз через FFI — сохранив удобный Python-интерфейс. Это классический реальный
> сценарий интеграции.

---

## 📋 Что делаем

Выбираешь вычислительно тяжёлую задачу, реализуешь её на чистом Python (медленно) и на C
(быстро, через FFI), оборачиваешь C в безопасный Python-класс и **доказываешь ускорение
замерами**.

Идеи задач:
- обработка изображения (фильтр, яркость) над массивом пикселей;
- числовая обработка большого массива (статистика, свёртка);
- симуляция (множество частиц, клеточный автомат);
- криптография/хеширование «вручную»;
- поиск/сортировка на больших данных.

---

## ✅ Требования (минимум)

- [ ] Реализация на **чистом Python** (эталон, медленная).
- [ ] Та же логика на **C**, собранная в библиотеку (.so/.dll).
- [ ] Вызов C из Python через **ctypes** с правильными сигнатурами.
- [ ] **Безопасная обёртка** (класс с проверками), а не сырые вызовы.
- [ ] Корректная работа с **памятью** (caller-allocated или парные make/free).
- [ ] **Замер ускорения** (`time`/`timeit`): во сколько раз быстрее.
- [ ] Проверка, что результаты Python и C **совпадают**.

---

## 🪜 Пошаговый план

1. **Выбери задачу** с тяжёлым циклом по данным.
2. **Python-версия.** Реализуй прямолинейно, замерь время.
3. **C-версия.** Перепиши горячий цикл на C, собери в библиотеку (`-O2`!).
4. **FFI.** Подключи через ctypes, объяви сигнатуры, передай данные (массив + длина).
5. **Обёртка.** Спрячь FFI за классом с проверками и понятным API.
6. **Проверка.** Убедись, что C и Python дают одинаковый результат.
7. **Замер.** Сравни время, посчитай ускорение.

## 🧱 Скелет

`fast.c`:
```c
#include <stddef.h>

// горячий цикл на C: сумма квадратов
double sum_of_squares(const double* arr, size_t n) {
    double total = 0.0;
    for (size_t i = 0; i < n; i++) {
        total += arr[i] * arr[i];
    }
    return total;
}
```
```bash
gcc -shared -fPIC -O2 fast.c -o fast.so
```

`accelerated.py`:
```python
import ctypes, time

class FastMath:
    def __init__(self, lib="./fast.so"):
        self._lib = ctypes.CDLL(lib)
        self._lib.sum_of_squares.argtypes = [
            ctypes.POINTER(ctypes.c_double), ctypes.c_size_t]
        self._lib.sum_of_squares.restype = ctypes.c_double

    def sum_of_squares(self, data):
        n = len(data)
        arr = (ctypes.c_double * n)(*data)
        return self._lib.sum_of_squares(arr, n)

def python_version(data):
    return sum(x * x for x in data)

# --- проверка и замер ---
data = [float(i) for i in range(1_000_000)]

t = time.perf_counter()
r1 = python_version(data)
t_py = time.perf_counter() - t

fast = FastMath()
t = time.perf_counter()
r2 = fast.sum_of_squares(data)
t_c = time.perf_counter() - t

assert abs(r1 - r2) < 1e-3, "результаты расходятся!"
print(f"Python: {t_py:.3f}с,  C: {t_c:.3f}с,  ускорение: {t_py/t_c:.1f}x")
```

> 💡 Заметь: преобразование списка в C-массив тоже занимает время. Для честного замера
> учитывай это — и подумай, как его уменьшить (см. усложнения).

## 💪 Усложнения (для «отлично»)

- ➕ Передавай данные через `numpy`-массив (нулевое копирование — `numpy.ctypeslib`).
- ➕ Реализуй C-версию с многопоточностью (OpenMP), сравни.
- ➕ Сделай вторую реализацию на **Rust** (превью Уровня 3), сравни с C.
- ➕ Обработай краевые случаи (пустой массив, огромные числа).
- ➕ Оформи как устанавливаемый Python-пакет с собранной библиотекой.
