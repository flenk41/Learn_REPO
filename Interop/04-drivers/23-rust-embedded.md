# 23 · Драйверы на Rust и embedded 🖼️⭐

> 🎯 **Цель блока:** узнать, как Rust приходит в системное программирование — драйверы ядра
> и микроконтроллеры. Безопасность памяти там, где ошибка раньше валила всю систему.

---

## 📖 Почему Rust в драйверах — революция

Драйверы традиционно пишут на C, и **большинство уязвимостей и крахов ядра — это ошибки
памяти** (use-after-free, buffer overflow). Rust устраняет их **на этапе компиляции**
(borrow checker из [Rust-трека](../../Rust/02-memory/10-borrow-checker.md)), сохраняя
скорость C.

🖼️
```
   C-драйвер:    баг памяти  →  kernel panic / уязвимость  (ловится в рантайме, поздно)
   Rust-драйвер: баг памяти  →  НЕ КОМПИЛИРУЕТСЯ           (поймано до запуска)
```

💡 В ядре, где ошибка памяти = крах всей системы или дыра в безопасности, гарантии Rust
особенно ценны. Поэтому индустрия активно внедряет Rust в системный код.

---

## ⭐ Rust for Linux

С 2022 года ядро Linux **официально** поддерживает написание драйверов на Rust:

```
   - Можно писать модули ядра на Rust наряду с C.
   - Rust-обёртки над C-API ядра обеспечивают безопасность.
   - Драйверы на Rust уже появляются (NVMe, сетевые, GPU-части).
```

💡 Это историческое событие: впервые за десятилетия в ядро Linux добавили второй язык —
именно ради безопасности памяти. Microsoft, Google, Amazon тоже переписывают системный код
на Rust по той же причине.

## ⭐ Embedded Rust — микроконтроллеры

**Embedded** (встраиваемые системы) — это крошечные компьютеры в устройствах: датчики,
умные лампочки, автомобили, дроны. Часто без ОС вообще («bare metal»).

🖼️
```
   Микроконтроллер (например ARM Cortex-M):
   ┌─────────────────────────────────┐
   │  твой Rust/C код работает НАПРЯМУЮ│  ← нет ОС, нет защит, ты управляешь всем
   │  на железе (bare metal)           │
   └─────────────────────────────────┘
```

Rust отлично подходит для embedded:
```rust
#![no_std]                  // без стандартной библиотеки (её негде взять на МК)
#![no_main]                 // своя точка входа

// управление пином (включить светодиод) — безопасно, на скорости C
led.set_high();
```

💡 `#![no_std]` — Rust без стандартной библиотеки (на микроконтроллере нет ОС, файлов,
кучи по умолчанию). Это как kernel-программирование: голое железо, но с безопасностью
Rust. Экосистема embedded Rust (крейты `embedded-hal`, `cortex-m`) — одна из лучших.

## ⭐ HAL — абстракция над железом

**HAL** (Hardware Abstraction Layer) — слой, скрывающий различия конкретных чипов за единым
интерфейсом (как драйвер скрывает устройство):

```rust
// один и тот же код работает на разных микроконтроллерах через HAL
use embedded_hal::digital::OutputPin;

fn blink<P: OutputPin>(led: &mut P) {
    led.set_high().ok();
    // задержка
    led.set_low().ok();
}
```

💡 HAL — это снова **интеграция через общий интерфейс** (как C-ABI между языками, как
драйвер между ОС и железом). Код, написанный против HAL, переносится между чипами. Тема
«единого интерфейса» проходит через весь трек.

---

## 📖 MMIO в embedded — те же регистры

На микроконтроллере управление железом — это запись в регистры (MMIO, модуль 22), но без
ОС:

```rust
// записать в регистр периферии (через безопасную обёртку PAC/HAL)
peripherals.GPIOA.odr.write(|w| w.odr5().set_bit());   // зажечь пин 5
```

💡 Те же memory-mapped регистры, что в драйверах ядра, но Rust-обёртки (PAC — Peripheral
Access Crate) делают доступ **типобезопасным**: нельзя случайно записать в неверный регистр
или бит. Снова: Rust приносит безопасность в работу с сырой памятью железа.

```
   - Системное ПО: драйверы (Rust for Linux), ОС на Rust (Redox).

   - Автомобили, дроны, робототехника.
   - Криптография и безопасность (где ошибки недопустимы).
```
