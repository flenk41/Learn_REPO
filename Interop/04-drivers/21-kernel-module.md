# 21 · Модуль ядра Linux 🛠️⭐

> 🎯 **Цель блока:** написать свой первый модуль ядра Linux — кусочек кода, работающий
> внутри ядра. Это «hello world» драйверописания.

> ⚠️ Делай на **Linux или WSL2** (с настоящим ядром). На обычном Windows kernel-модули
> Linux не собрать. Лучше — в виртуалке/контейнере, чтобы не рисковать основной системой.

---

## 📖 Что такое модуль ядра

**Модуль ядра** (kernel module, `.ko`) — это код, который можно **загрузить в работающее
ядро** и выгрузить, не перезагружая систему. Большинство драйверов Linux — это модули.

🖼️
```
   insmod my_module.ko  →  код ЗАГРУЖАЕТСЯ в ядро, начинает работать с полными правами
   rmmod my_module      →  код ВЫГРУЖАЕТСЯ из ядра
```

💡 Модули позволяют добавлять/убирать драйверы на лету. `lsmod` показывает загруженные.

---

## ⭐ Минимальный модуль: «Hello, kernel»

`hello.c`:
```c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/kernel.h>

MODULE_LICENSE("GPL");                 // лицензия (обязательно)
MODULE_AUTHOR("Твоё имя");
MODULE_DESCRIPTION("Первый модуль ядра");

// вызывается при загрузке модуля
static int __init hello_init(void) {
    printk(KERN_INFO "Привет, ядро! Модуль загружен.\n");
    return 0;                          // 0 = успех
}

// вызывается при выгрузке модуля
static void __exit hello_exit(void) {
    printk(KERN_INFO "Пока, ядро! Модуль выгружен.\n");
}

module_init(hello_init);               // зарегистрировать функцию загрузки
module_exit(hello_exit);               // и выгрузки
```

🖼️
```
   insmod hello.ko  →  hello_init()  выполняется в ядре (печатает в лог ядра)
   rmmod hello      →  hello_exit()  выполняется
```

> ⚠️ Заметь: нет `main`, нет `printf` — это **не обычная программа**. Внутри ядра другой
> мир: `printk` вместо `printf` (вывод в лог ядра), `__init`/`__exit` — специальные метки,
> `module_init/exit` регистрируют точки входа.

## 🛠️ Сборка и загрузка

`Makefile`:
```makefile
obj-m += hello.o
all:
	make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules
clean:
	make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean
```

```bash
make                          # собрать hello.ko
sudo insmod hello.ko          # загрузить в ядро
sudo dmesg | tail             # увидеть "Привет, ядро!" в логе ядра
sudo rmmod hello              # выгрузить
sudo dmesg | tail             # увидеть "Пока, ядро!"
```

> 🛠️ Нужны заголовки ядра: `sudo apt install build-essential linux-headers-$(uname -r)`.

💡 `dmesg` показывает лог ядра, куда пишет `printk`. Это твой главный способ «видеть», что
делает модуль (в ядре нет обычного stdout).

## ⚠️ Почему kernel-программирование опасно

```
   ⚠️ Нет защиты памяти: ошибка указателя → kernel panic (крах ВСЕЙ системы).
   ⚠️ Нет стандартной библиотеки: ни printf, ни malloc (есть kmalloc), ни float по умолчанию.
   ⚠️ Утечка памяти в ядре остаётся до перезагрузки.
   ⚠️ Бесконечный цикл может подвесить систему.
   ⚠️ Отладчик подключить сложнее (система-то может упасть).
```

> ⚠️ В user space ошибка = краш **одной программы** (segfault, можно перезапустить). В
> kernel space ошибка = краш **всей системы**. Поэтому kernel-код требует предельной
> аккуратности с памятью — здесь всё, что ты учил про указатели и владение, критично
> вдвойне. Тестируй в виртуалке!

## 📖 От «hello» к настоящему драйверу

Реальный символьный драйвер добавляет:
```
   - регистрацию устройства в /dev (создаётся файл, например /dev/mydevice);
   - функции file_operations: open, read, write, release;
   - обработку данных между userspace и ядром (copy_to_user / copy_from_user);
   - возможно работу с настоящим железом (регистры, прерывания — модуль 22).
```

💡 Пользователь потом работает с устройством как с **файлом**: `cat /dev/mydevice`,
`echo "x" > /dev/mydevice`. Драйвер обрабатывает эти операции. Это разберём в
[следующем модуле](22-devices.md). Здесь главное — ты увидел, что код может жить **в ядре**.

## ✅ Задачи

> На Linux/WSL2/виртуалке.

1. **Hello-модуль.** Собери и загрузи модуль из урока, найди его сообщения в `dmesg`.
2. **Параметр модуля.** Добавь параметр (`module_param`), передай значение при загрузке
   (`insmod hello.ko myparam=5`).
3. **Свои сообщения.** Выведи из init информацию (например имя пользователя, версию ядра).
4. **lsmod.** Найди свой модуль в списке `lsmod`, посмотри `modinfo hello.ko`.
5. ⭐ **Символьное устройство.** По гайду сделай простой символьный драйвер, создающий
   `/dev/mydevice`, который при чтении отдаёт строку.
