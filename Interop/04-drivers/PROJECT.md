# 🚀 Финальные пет-проекты — системный полиглот-инженер

> 🎯 **Цель:** выбери серьёзный проект на стыке языков и/или системного программирования.
> Это вершина всего курса — здесь сходится всё: память, языки, граница, железо.

> ⚠️ Системные проекты — на Linux/виртуалке.

---

## 🏆 Проект 1 — Ускоренная библиотека с многоязычным ядром (рекомендую)

Полезная библиотека: удобный API на Python/JS + быстрое безопасное ядро на Rust/C.

**Что внутри:**
- 🧩 Ядро на Rust (быстро + безопасно) через **PyO3** или **WASM** (модули 14, 16).
- 🛡️ Чистая граница памяти (Уровень 2): без утечек, безопасная обёртка.
- 📊 Замеры: докажи ускорение vs чистый Python/JS.
- ✅ Тесты, README, сборка одной командой.

**Прокачивает:** FFI/биндинги, границу памяти, дизайн API. Как настоящие библиотеки.

## 🏆 Проект 2 — Userspace-драйвер устройства

Драйвер реального или виртуального устройства **в userspace** (безопаснее ядра).

**Что внутри:**
- 🔌 Работа с устройством через `/dev`, `/sys`, **libusb** или UIO (модуль 20).
- 🧠 Аккуратная работа с памятью и данными устройства.
- 🌐 Можно на C, Rust **или даже Python** (libusb).

**Идеи:** драйвер USB-гаджета, чтение датчиков, управление LED-лентой, работа с
последовательным портом.

**Прокачивает:** системное программирование без риска ронять ядро.

## 🏆 Проект 3 — Модуль ядра Linux

Настоящий kernel-драйвер (в виртуалке!).

**Что внутри:**
- 🖥️ Символьное устройство `/dev/...` с read/write (модули 21–22).
- 🔄 `copy_to/from_user`, обработка ошибок, освобождение ресурсов.
- 🛡️ KASAN-чистота, security-проверки данных из userspace (модуль 24).
- ➕ Бонус: версия на **Rust for Linux**.

**Идеи:** виртуальное устройство (шифратор потока, счётчик, кольцевой буфер), простой драйвер
GPIO на Raspberry Pi.

**Прокачивает:** kernel-программирование, дисциплину памяти на максимальном уровне.

## 🏆 Проект 4 — Embedded-проект (микроконтроллер)

Программа для микроконтроллера на Rust/C (если есть плата или симулятор).

**Что внутри:**
- ⚙️ Bare-metal код (`#![no_std]`), работа с регистрами (MMIO, модуль 23).
- 🔆 Управление периферией: светодиоды, кнопки, датчики, дисплей.
- 🛡️ Безопасный доступ к железу через HAL.

**Идеи:** метеостанция, умная подсветка, простой робот, игра на дисплее.

**Прокачивает:** embedded, работу с железом, Rust на bare metal.

## 🏆 Проект 5 — Полиглотная система

Распределённое приложение из нескольких языков, связанных правильными способами.

**Что внутри:**
- 🧩 3+ языка по сильным сторонам (например JS-фронт + Python-API + Rust-ядро).
- 🔗 Разные способы связи: FFI/PyO3 для ядра, REST/gRPC для сервисов (модуль 18).
- 📦 Продуманная архитектура и сборка.

## 🎓 Что дальше — и итог всего курса

Ты прошёл невероятный путь. Собери картину целиком:

```
   C       →  ручная память (фундамент)
   C++     →  память + RAII (безопасность через инструменты)
   Python  →  автоматическая память (GC)
   Rust    →  безопасность без GC (borrow checker)
   ИИ      →  контекст как память модели
   Interop →  память НА ГРАНИЦЕ языков + память железа (драйверы)
```

🎯 **Сквозная тема всего курса — память на всех уровнях:** от байтов в C до контекста ИИ,
от выделения в куче до регистров устройства, от одного языка до границы между ними. Ты
понимаешь, **как всё устроено внизу** — и это делает тебя сильным инженером в любой области.

Куда двигаться:
- **Системное:** ОС, драйверы, embedded, Rust for Linux.
- **Интеграция:** библиотеки с нативным ядром, WASM-проекты, микросервисы.
- **Специализация:** выбери домен (геймдев, ML-инфраструктура, embedded, высоконагруженные
  системы) — фундамент памяти пригодится везде.

> прошёл всю вертикаль. 🎉🚀
